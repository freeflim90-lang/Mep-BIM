# LUA BIM LABS AX BIM 선도기업 성장 비전

## 비전 문장

LUA BIM LABS는 AX(AI Transformation)를 BIM 실무에 적용하여 설계, 시공, 품질검토, 납품, 교육, 자동화 전반을 지능화하고 BIM 업계에서 선도적인 기술 기업으로 성장한다.

## 핵심 정체성

LUA BIM LABS는 단순 BIM 모델링 조직이 아니라 BIM 지식, 프로젝트 데이터, 자동화 도구, AI 에이전트를 연결하는 AX 기반 BIM 기술 조직이다.

## 선도 기업의 의미

선도 기업은 큰 회사라는 뜻만이 아니다. LUA BIM LABS 기준에서 선도 기업은 다음 조건을 갖춘 조직이다.

- BIM 실무 기준을 문서와 데이터로 구조화한다.
- 반복 검토와 문서화 업무를 자동화한다.
- Revit/Navisworks/IFC/IDS 기반 모델 품질 검증 체계를 보유한다.
- 설계와 시공 사이의 정보 단절을 줄인다.
- 내부 지식이 Obsidian/RAG/QA 데이터셋으로 축적되어 계속 재사용된다.
- 직원 교육, 고객 응대, 상품 개발이 같은 지식망 위에서 성장한다.
- Model Quality Auditor, Revit Add-in, Telegram/Revit Assistant 같은 제품형 결과물을 만든다.

## AX 적용 원칙

### 1. BIM 전문성이 먼저다

AI는 BIM 전문성을 대체하지 않는다. 공종 지식, 설계 기준, 시공성, 법규, 납품 기준을 먼저 구조화하고 AI는 이를 빠르게 찾고 적용하고 검증하는 도구로 사용한다.

### 2. 모든 반복 업무는 데이터화 후보다

반복 질문, 반복 검토, 반복 보고, 반복 산출은 FAQ, 체크리스트, JSON 룰, 스크립트, Add-in, 에이전트 기능 후보로 승격한다.

### 3. 공식 출처와 현장 경험을 함께 쓴다

법규/수치/버전 정보는 공식 출처를 우선하고, 현장 경험은 실무 적용 팁으로 분리한다. 출처와 기준일 없는 지식은 고객 답변에 단정적으로 쓰지 않는다.

### 4. 지식은 상품으로 이어져야 한다

수집된 지식은 교육자료, 표준문서, 품질검토 룰, Add-in 기능, 고객 서비스 패키지로 전환될 수 있어야 한다.

## 성장 실행 축

| 축 | 목표 | 대표 산출물 |
|---|---|---|
| 지식 자산화 | 공종/기준/프로젝트 경험을 검색 가능한 지식으로 축적 | Obsidian, QA 노트, 지식 그래프 |
| 표준화 | 반복 기준을 문서와 체크리스트로 전환 | BEP, BIM 지침, 품질 체크리스트 |
| 자동화 | 반복 업무를 스크립트/Add-in으로 전환 | Revit Add-in, Navisworks 자동화 |
| AI 에이전트화 | 질문, 검토, 보고를 맥락 기반으로 지원 | Telegram/Revit Assistant, RAG |
| 상품화 | 내부 도구를 고객 가치 제품으로 전환 | Model Quality Auditor, Store 제품 |
| 교육/확산 | 팀원이 같은 기준으로 성장 | 연차별 커리큘럼, 실무 교육 |

## 내부 자동 성장 루프

LUA BIM LABS는 지식이 쌓이는 데서 멈추지 않고, 매일 내부 성장 후보를 자동 분류한다.

| 성장 루프 | 역할 | 산출물 |
|---|---|---|
| 시간별 신호 감지 | AX/BIM/AI/스마트건설 신호 수집 | `AX_시간별_신호모니터링` |
| 일일 지식 큐레이션 | 공식 지식베이스와 Obsidian 반영 | 일일 지식 업데이트 |
| 일일 내부 성장 펄스 | 제품화/자동화/교육/품질룰/전략/지식공백 후보 분류 | `내부성장루프`, `AX_INTERNAL_GROWTH_BACKLOG` |
| 주간 전략 승격 | 반복 신호를 전략/제품/교육/자동화 후보로 승격 | `AX_전략승격리뷰` |
| 코드 개발 보고 | 코드 개발 초안/변경 요약을 Gmail로 발송 | `send_code_development_gmail.py`, Gmail 개발 보고 |

이 루프의 목적은 회사가 사람의 기억에만 의존하지 않고, 내부 지식과 반복 신호를 바탕으로 스스로 다음 학습 과제, 자동화 과제, 제품화 과제를 발견하도록 만드는 것이다.

코드 개발 보고는 Gmail SMTP 설정이 있는 경우 자동 발송한다. Gmail 앱 비밀번호가 없으면 개발 흐름을 막지 않고 로컬 로그에 설정 필요 상태를 남긴다.

## 1차 전략 제품

Model Quality Auditor는 LUA BIM LABS의 AX 방향을 보여주는 첫 번째 핵심 제품이다.

- BIM 모델 품질 검토 지식의 데이터화
- Revit/Navisworks 실무 자동화
- 고객 보고서와 품질 근거 자동 생성
- Store 제품화와 서비스형 진단 패키지 연결
- 내부 교육과 QA 데이터셋으로 지속 개선

## 전략 구호

`BIM을 지식화하고, 지식을 자동화하고, 자동화를 AX 제품으로 만든다.`
