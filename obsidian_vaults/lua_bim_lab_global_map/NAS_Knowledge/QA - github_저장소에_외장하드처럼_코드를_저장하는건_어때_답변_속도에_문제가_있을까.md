# QA - github 저장소에 외장하드처럼 코드를 저장하는건 어때? 답변 속도에 문제가 있을까?

## 질문
> **카테고리:** Claude대화 | **날짜:** 2026-05-23
> **출처:** Claude Code 세션

github 저장소에 외장하드처럼 코드를 저장하는건 어때? 답변 속도에 문제가 있을까?

## 답변

아이디어는 좋은데 구조를 나눠서 생각해야 합니다.

**방법에 따라 속도 영향이 달라집니다:**

**방법 A: 질문 올 때마다 GitHub API로 검색** → 느림
- 각 API 호출마다 100~500ms 추가
- Telegram 답변에서 체감 가능한 지연 발생
- GitHub Search API는 분당 30회 제한 (토큰 있어도)

**방법 B: GitHub에 저장해두고 주기적으로 `git pull`** → 현실적
- 평소엔 로컬에서 검색 (현재와 동일한 속도)
- 자정에 `git pull`로 최신 지식 동기화
- SSD 용량 걱정 없이 GitHub가 원본 보관소 역할

현재 구조에서 실용적인 방향은 **방법 B**입니다:

```
GitHub 저장소 (원본 보관)
    ↓ git pull (하루 1회)
로컬 data/ 폴더 (검색용 캐시)
    ↓ search_local_knowledge()
텔레그램 답변 (빠름)
```

단, SSD 50% 채우기가 목표라면 GitHub에 저장해도 로컬에 clone해야 하므로 결국 같은 용량이 필요합니다. SSD 절약 효과는 없습니다.

**요약**: 속도 문제 없이 GitHub를 쓰려면 로컬 캐시 + 주기적 동기화 구조가 맞고, 질문 시 실시간 GitHub 조회는 피해야 합니다.
