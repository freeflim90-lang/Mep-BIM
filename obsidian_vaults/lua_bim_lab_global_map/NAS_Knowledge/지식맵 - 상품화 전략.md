# 지식맵 — 상품화 전략

LUA BIM LABS의 Autodesk Store 제품 출시 전략. BIM Command Center 기준.

---

## 1. 제품 포트폴리오

| 제품군 | 현황 | 스토어 |
|---|---|---|
| BIM Command Center For Revit | 개발 중 (1순위) | Autodesk App Store |
| Navisworks Auto SearchSet | 2순위 후보 | Autodesk App Store |
| Navisworks Data Heatmap | 3순위 (낮은 준비도) | 추후 |

---

## 2. 가격 정책

| 플랜 | 가격 |
|---|---|
| Individual | USD 19/월 또는 USD 190/년 |
| Team 5-Pack | 개별 × 4.5 (약 USD 86/월) |
| Enterprise | 별도 협의 |

- 포지션: 단일 소형 플러그인보다 높고, 대형 전문 솔루션보다 낮음
- 초기 전략: 로컬 Revit 생산성 대시보드 우선 판매 → 검증 후 MEP 기능 추가

---

## 3. 제품 로드맵

### MVP (0~3개월)
- 1개 핵심 기능 (Model Health Dashboard)
- Autodesk Store 제출
- 첫 유료 고객 10명

### v1.1 (3~6개월)
- 피드백 반영 개선
- 기능 1개 추가 (Workset Dashboard)
- 리뷰 20개↑

### v2.0 (6~12개월)
- Pro 기능 추가 (MEP 특화 모듈)
- Team Plan 도입
- 2번째 Add-in (Navisworks Auto SearchSet) 출시 준비

---

## 4. 심사 기준 (Store 제출)

**심사 거절 위험 항목 → 제외**
- 불안정 기능 (크래시 발생)
- 문서 불일치 (매니페스트 ↔ 실제 기능)
- 개인정보 무단 수집
- Revit 성능 저하 (로딩 시간 2초↑)
- 외부 서버 의존 (인터넷 연결 필수 기능)

**심사 통과 기준**
- 지원 버전 명시 (Revit 2024~2026 실기 테스트)
- 설치/제거 클린 (레지스트리 잔여물 없음)
- 다른 Add-in과 충돌 없음
- 에러 발생 시 사용자 친화적 메시지

## 5. 기능 우선순위 결정 공식

```
점수 = (고객 요청 빈도) × (개발 공수 역수) × (스토어 경쟁 강도 역수)
```

| 기능 | 고객요청 | 공수역수 | 경쟁역수 | 점수 |
|---|---|---|---|---|
| Model Health Dashboard | 높음 | 중간 | 높음 | 최상 |
| Schedule Excel Export | 높음 | 높음 | 낮음 | 상 |
| Workset Dashboard | 중간 | 높음 | 높음 | 상 |
| MEP Clash Heatmap | 높음 | 낮음 | 중간 | 중 |
