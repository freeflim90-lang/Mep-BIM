# QA - 개별적으로 사용이 필요한 경우가 있어서 버튼의 배치가 워크플로수 순서로 하는건 좋으나 너무

## 질문
> **카테고리:** Claude대화 | **날짜:** 2026-05-31
> **출처:** Claude Code 세션

개별적으로 사용이 필요한 경우가 있어서 버튼의 배치가 워크플로수 순서로 하는건 좋으나 너무 통합을 해버리다보면 버튼을 보고 기능에 대한 직관성이 떯어지는 현상이 있어

## 답변

맞습니다. 핵심 원칙은 이렇게 됩니다:

**패널 = 작업 단계 (언제 쓰는가)**
**버튼 = 동사+목적어 (정확히 뭘 하는가)**

---

### 문제 사례 vs 개선안

| 나쁜 예 (과통합) | 좋은 예 (직관적 개별) |
|---|---|
| `MEP Tools` → 뭘 하는 건지 모름 | `배관 분할` / `MEP 길이 계산` → 각각 명확 |
| `Model Health Pro` → 범위 불명확 | `모델 감사` / `경고 격리` / `CAD 잔재 삭제` → 각각 독립 |
| `Clash Center` → 무슨 기능? | `간섭 마커 배치` / `간섭 그룹화` / `책임자 배정` |

---

### 제안 — 워크플로우 순서 + 개별 버튼 리본 구조

```
BIM Command Center 탭
│
├── [준비]  ─────────────────────────────────────────────
│   설정 프로필 관리  |  단위 설정  |  패밀리 가져오기  |  패밀리 내보내기
│
├── [뷰·도면]  ──────────────────────────────────────────
│   태그 정렬  |  선 정리  |  뷰 템플릿 복사  |  시트 복제  |  일괄 인쇄
│
├── [모델 처리]  ────────────────────────────────────────
│   타입 일괄 정의  |  번호 재배정  |  룸 마감 배정  |  다중 재료 태그
│
├── [선택·검사]  ────────────────────────────────────────
│   Smart 선택  |  Workset 검사  |  링크 상태  |  경고 격리
│
├── [QA·감사]  ──────────────────────────────────────────
│   모델 감사  |  선 정리  |  IFC 납품 검증
│
├── [데이터]  ───────────────────────────────────────────
│   일람표 Excel 저장  |  일람표 Excel 동기화  |  MEP 길이 계산
│
└── [간섭·조정]  ────────────────────────────────────────
    간섭 마커 배치  |  Section Box 열기  |  간섭 그룹화  |  책임자 배정
```

---

### 버튼 네이밍 원칙

```
동사(무엇을) + 목적어(어디에)

✅ 배관 분할          ← 명확
✅ 일람표 Excel 저장  ← 명확
✅ IFC 납품 검증      ← 명확

❌ MEP Tools         ← 동사 없음, 범위 불명확
❌ Model Health Pro  ← "Pro"가 뭘 추가한 건지 모름
❌ Clash Center      ← Center가 의미 없음
```

---

이 구조대로 `commands.json`과 `RibbonCommandRegistry.cs`를 수정할까요?
각 버튼이 개별로 유지되면서 패널 순서만 워크플로우 순으로 재배치하는 방향입니다.
