# QA - 현재 구조 아주 맘에 들어 지식 수집 업데이트 됨에 따라서 점차 옵시디언도 성장해야되

## 질문
> **카테고리:** Claude대화 | **날짜:** 2026-05-28
> **출처:** Claude Code 세션

현재 구조 아주 맘에 들어 지식 수집 업데이트 됨에 따라서 점차 옵시디언도 성장해야되

## 답변

현재 파이프라인이 이미 그렇게 설계되어 있다. 매일 자정 실행 순서:

```
KB 77개 파일 성장 (4엔진 검색 + Ollama)
    ↓
mqa_obsidian_tools.py graph    → MQA 볼트 그래프 재생성
    ↓
build_global_obsidian_map.py   → 전역 지식맵 재생성 (HTML 포함)
```

오늘도 정상 동작 확인: 76개 파일 업데이트 → 그래프 08:47에 자동 갱신됨.

---

**지금 한 가지 부족한 부분이 있다.** KB 섹션에 `[[연결 파일명]]` 위키링크가 없어서, 지식이 늘어도 그래프의 **연결선은 안 늘어난다.** 지금은 키워드 유사도로만 엣지를 만든다.

KB가 성장할수록 관련 파일끼리 위키링크도 자동으로 심어주면 그래프 연결이 함께 성장한다. `auto_enrich_knowledge_base.py`의 섹션 생성 프롬프트에 관련 KB 파일명을 Obsidian 링크로 포함하도록 추가할 수 있다.

예시:
```
## 소방기계 최신 기술 동향 (2026-05-28)
...내용...
- 관련: [[소방전기]], [[공조덕트]], [[건축]]
```

이렇게 되면 날이 갈수록 KB 파일들 사이에 실제 위키링크 엣지가 쌓여서 그래프가 진짜로 성장한다. 추가할까?
