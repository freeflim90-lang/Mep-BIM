# 지식맵 — 엑셀 자동화

LUA BIM LABS의 엑셀 자동화 전체 지식을 하나의 맵으로 정리한다.
원본 데이터 소스: `data/knowledge_base/엑셀자동화.md`

---

## 1. 핵심 원칙

- **구현 언어 기본값**: Python 표준 `csv` + `openpyxl`
- **VBA/매크로**: 전면 금지 (보안 리스크 + 크로스플랫폼 비호환)
- **주 구현 담당**: Qwen_Coder_8B (로컬 전용, DeepSeek API 사용 금지)
- **결과물 전달**: Telegram으로 질문자 직접 회신 (Mac 로컬 저장 금지)
- **개발 전 선행 조건**: 목적·입력·출력·시트 구조를 plan으로 확정

---

## 2. 라이브러리 선택 기준

| 우선순위 | 라이브러리 | 사용 조건 |
|---|---|---|
| 1순위 | `csv` + `openpyxl` | 기본값. BIM 리포트, 관리팀 업무 전반 |
| 2순위 | `pandas` + `openpyxl` | 집계(pivot, groupby)가 필요한 경우에만 |
| 3순위 | C# OpenXML SDK | Revit Add-in 내부에서 직접 생성 시 한정 |

> `pandas`로 xlsx 저장 시 반드시 `engine='openpyxl'` 명시

## 3. BIM 특화 출력 표준

### 간섭 리포트 (Navisworks → Excel)
```
번호 | 공종A | 공종B | 레벨 | 구역 | 간섭거리(mm) | 상태 | 담당자 | 조치기한 | 비고
```
- 상태값: 신규 / 검토중 / 조치완료 / 보류 / 제외
- 우선순위: 긴급(>50mm 관통) / 보통 / 낮음
- AutoFilter: 공종, 레벨, 상태, 담당자

### 물량 산출 (Revit Schedule → Excel)
```
패밀리 | 유형 | 레벨 | 수량 | 단위 | 길이/면적/체적 | 비고
```
- Schedule Export: visible field order 보존 (BCC P1 기준)
- 빈 행 자동 제거, 합계 행 수식 삽입

### MEP 스케줄 동기화 (Revit Schedule Excel Sync, P2)
- UniqueId 보존 필수 (재임포트 매핑 기준)
- 읽기 전용 파라미터 자동 제외
- Phase 1: CSV 내보내기 → 검증 → 임포트 파이프라인

## 4. 관리팀 자동화 템플릿

| 업무 | 주요 기능 | 출력 형식 |
|---|---|---|
| 근태 취합 | 출퇴근 집계, 지각/조퇴 표시 | 필터 테이블 .xlsx |
| 비용 정산 | 항목별 합계, 월별 피벗, 초과 강조 | 다중 시트 .xlsx |
| 구매 요청 | 품목/수량/단가/합계, 승인 상태 | 표준 테이블 .xlsx |
| 라이선스 관리 | 만료일 D-30 빨간 강조 | 조건부 서식 .xlsx |
| 월간 보고서 | 매출/비용 요약, 전월 대비 증감 | 차트 포함 .xlsx |

**보안 규칙**: 개인정보·급여·계약금액 → `***` 마스킹

## 5. 공통 코드 패턴

### 필터 테이블 생성 (기본형)
```python
from openpyxl import Workbook
from openpyxl.worksheet.table import Table, TableStyleInfo
from openpyxl.utils import get_column_letter

wb = Workbook()
ws = wb.active
headers = ["공종", "레벨", "간섭수", "상태", "담당자"]
ws.append(headers)
for row in data:
    ws.append(row)
last_col = get_column_letter(len(headers))
table = Table(displayName="리포트", ref=f"A1:{last_col}{ws.max_row}")
table.tableStyleInfo = TableStyleInfo(name="TableStyleMedium9", showRowStripes=True)
ws.add_table(table)
wb.save("output.xlsx")
```

### 인코딩 안전 CSV 읽기
```python
def read_csv_safe(path):
    for enc in ("utf-8-sig", "cp949", "utf-8"):
        try:
            with open(path, newline="", encoding=enc) as f:
                return list(csv.DictReader(f))
        except (UnicodeDecodeError, LookupError):
            continue
    raise ValueError("인코딩 감지 실패")
```

### 대용량 write_only 모드 (5만 행 이상)
```python
wb = Workbook(write_only=True)
ws = wb.create_sheet()
ws.append(headers)
for chunk in data_generator():
    ws.append(chunk)
wb.save("large.xlsx")
# 주의: write_only 모드는 셀 서식 불가
```

## 6. 파일 명명 규칙

```
YYYY-MM-DD_[업무유형]_[상세]_[버전].xlsx
```

| 버전 | 의미 |
|---|---|
| v1 | 초안 |
| v2 | 검토 반영 |
| FINAL | 최종 납품본 |

예시: `2026-05-21_간섭리포트_기계전기_v1.xlsx`

## 7. 납품 전 품질 체크리스트

- [ ] UTF-8-sig 인코딩 저장 (한글 깨짐 방지)
- [ ] 헤더 행 고정(`freeze_panes`) 적용
- [ ] AutoFilter 설정
- [ ] 빈 행/열 없음
- [ ] 숫자 컬럼 Number 서식 (문자 저장 숫자 없음)
- [ ] 날짜 컬럼 YYYY-MM-DD 형식
- [ ] 시트명 31자 이하
- [ ] 파일명에 날짜 포함
- [ ] 개인정보 마스킹 확인 (관리팀)
- [ ] 5MB 초과 시 시트 분리 또는 CSV 병행 제공

## 8. 요청 처리 흐름 (Telegram 경유)

```
팀원 Telegram 요청
    ↓ 키워드 감지 (엑셀/xlsx/근태/정산/비용)
요청 ID 채번 (MGMT-EXCEL-YYYYMMDD-NNN)
    ↓
Qwen_Coder_8B — Python/openpyxl 초안 작성
    ↓
QA 테스터 — 샘플 데이터 smoke test
    ↓
Telegram으로 질문자 직접 회신
(Mac 로컬 저장 금지 / DeepSeek API 금지)
```
