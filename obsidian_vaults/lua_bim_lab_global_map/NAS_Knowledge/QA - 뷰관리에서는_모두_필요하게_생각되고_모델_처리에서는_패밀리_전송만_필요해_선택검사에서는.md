### 🔧 수정 개발 필요

**[링크 상태]** — 3가지 독립 버튼으로 분리
- `링크 일괄 로드` — 미로드 상태 링크 전체 로드
- `링크 일괄 재로드` — 로드된 링크 전체 갱신
- `링크 일괄 언로드` — 선택 링크 언로드

**[QA·감사 = 납품 정리]** — 리포트 제거 + 삭제 기능 추가
각 항목마다 **수량 표시 + 삭제 버튼** 구조로:
- 미배치 뷰 N개 → 삭제
- CAD Import N개 → 삭제
- 미사용 뷰 템플릿 N개 → 삭제
- 미배치 룸 N개 → 삭제
- IFC 납품 검증 (룰 체크)

**[인쇄·출력]** — 인쇄 셋팅 기능 추가
- `일괄 인쇄` (기존)
- `인쇄 설정 관리` — 프린터·용지·여백·색상 프리셋 저장/불러오기

**[N-01 간섭 책임자 배정]** — 룰 방식 변경
- 기존: 파일명 키워드 → 공종 분류
- 변경: **간섭 해소 우선순위 룰** (예: MEP는 구조를 피한다, 건축은 구조를 피한다)
  ```
  구조 > 건축 > 기계 > 전기 > 소방 (우선순위 설정)
  → 우선순위 낮은 공종 = 간섭 해소 담당
  → 해당 공종 담당자 이름 자동 적용
  ```
- Excel 보고서 제거, 결과는 Navisworks 화면 내 표시만

**[N-02 간섭 그룹화]** — 좌표 기반으로 변경
- 기존: 이름·상태 패턴
- 변경: **클래시 중심 좌표(XYZ) 기준 반경 N미터 이내 클래시 자동 그룹화**
  - 같은 구역 클래시를 한 번에 확인·처리 가능

**[N-03 간섭 테스트 일괄 정의]** — 파일 연결 자동화 추가
- 기존: JSON 입력 후 테스트 생성
- 변경: **SearchSet/SelectionSet 목록 자동 인식** → 선택만 하면 공종A vs 공종B 셋팅 완성

### ❓ IFC 내보내기 (N-05) 설명 보충

Navisworks에 기본 IFC 내보내기 (`File > Export > IFC`)가 있는 것 맞습니다.

N-05가 추가로 하려던 것은:
1. **모델별 분리 내보내기** — 여러 모델이 합쳐진 NWD 파일에서 공종별로 각각 별도 IFC 파일로 분리 저장
2. **프로젝트 메타데이터 자동 포함** — 프로젝트 번호, 발주처, 납품 단계 등을 IFC 헤더에 자동 기입
3. **공종 필터 적용** — 특정 공종(예: 기계만)만 선택해서 내보내기

기본 기능과 차이점은 **"한 번에 공종별로 나눠서 메타데이터 포함해서 내보내기"** 입니다.

이 기능이 필요하신가요, 아니면 기본 Navisworks IFC 내보내기로 충분하신가요?
