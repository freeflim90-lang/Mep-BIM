# QA - 추가로 더 수집은 불가능한거야?

## 질문
> **카테고리:** Claude대화 | **날짜:** 2026-05-30
> **출처:** Claude Code 세션

추가로 더 수집은 불가능한거야?

## 답변

가능합니다. 현재 한계와 해결책을 설명할게요.

**현재 한계 — API 10,000개 캡**

| 카테고리 | API 한도 | 실제 총 규모 | 미수집 |
|---------|---------|------------|-------|
| 위생 | ~9,996 | 50,631개 | **40,635개** |
| 조명 | ~9,087 | 22,342개 | **13,255개** |
| 가구 | ~9,996 | 14,756개 | **4,760개** |
| 배관 | ~8,723 | 9,109개 | 386개 |

BIMobject Search API가 상위 카테고리 단위로 최대 ~10,000개까지만 페이지네이션을 허용합니다.

**해결책 — 서브카테고리 단위 드릴다운**

상위 카테고리 대신 **세부 서브카테고리별로 개별 수집**하면 캡을 우회할 수 있습니다:

```
위생 (50,631) → 서브카테고리별 분리 수집
  ├ 수도꼭지 (15,950개) → 별도 수집
  ├ 욕실소품 (6,083개)
  ├ 샤워실 (5,609개)
  ├ 세면기 (5,357개)
  └ ... 22개 서브카테고리
```

현재 315개 서브카테고리가 이미 수집되어 있어서 대부분이 10,000개 미만이라 전량 수집 가능합니다. 서브카테고리 드릴다운 + 중복 제거(제품 ID 기준) 방식으로 업그레이드할까요?
