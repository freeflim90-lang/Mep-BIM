# QA - 캡은 누가 씌운거야?

## 질문
> **카테고리:** Claude대화 | **날짜:** 2026-05-30
> **출처:** Claude Code 세션

캡은 누가 씌운거야?

## 답변

**BIMobject 서버**가 씌운 겁니다.

API 응답을 보면 `totalCount: 10000`으로 고정 반환되는데, 이건 백엔드에서 쓰는 **Elasticsearch**의 기본 설정 때문입니다.

```
Elasticsearch 기본값: max_result_window = 10,000
→ from + size > 10,000 이면 에러 또는 캡 처리
```

BIMobject가 Elasticsearch로 검색을 구현하면서 기본 설정 그대로 쓰고 있는 거라, 저희가 우회할 수 없는 서버 측 제한입니다.

**그래서 서브카테고리 드릴다운이 유일한 해법** — 상위 카테고리 한 번에 10,000개 대신, 서브카테고리를 나눠서 각각 10,000개씩 수집하면 실질적으로 훨씬 더 많이 가져올 수 있습니다. 진행할까요?
