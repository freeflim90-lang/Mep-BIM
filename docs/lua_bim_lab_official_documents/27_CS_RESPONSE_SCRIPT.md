## 3.1 모델·로그 자료 요청 보안 안내

고객 모델 원본, Revit Journal, 로그 파일, 스크린샷을 요청할 때는 다음 구조를 사용한다.

```
안녕하세요, [고객명]님.

문제 재현을 위해 아래 자료 중 필요한 최소 범위만 요청드립니다.

1. 우선 요청 자료
① Revit 버전 / Add-in 버전
② 오류 메시지 또는 스크린샷
③ 재현 단계
④ 익명화 가능한 로그 또는 Journal 파일

2. 보안 안내
고객명, 프로젝트명, 담당자 연락처, 라이선스 키, API 토큰, 결제정보는 보내지 말아 주세요.
모델 파일 원본이 꼭 필요한 경우에는 목적, 접근 담당자, 보관 기간, 파기 방법을 별도로 안내드린 뒤 승인된 경로로만 받겠습니다.

3. 다음 액션
먼저 위 정보로 내부 재현을 시도하고, 원본 모델 없이 재현이 어려운 경우 다시 안내드리겠습니다.

LUA BIM LABS | [이메일]
```

내부 기준:
- 고객 모델 원본과 민감 로그는 `KST06 보안제한`으로 분류한다.
- 원본 모델 요청 전 PM 또는 라이선스_보안관 승인을 받는다.
- Obsidian에는 원본 자료가 아니라 익명화된 증상, 원인, 해결 절차만 남긴다.
