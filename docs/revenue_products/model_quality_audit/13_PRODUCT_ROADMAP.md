## 1. 로드맵 원칙

1. 첫 버전은 설치 안정성, Store 심사 통과, 리포트 신뢰성을 우선한다.
2. 기능보다 반복 사용성과 고객지원 가능성을 우선한다.
3. MEP 특화 기능은 v1.1 이후 단계적으로 추가한다.
4. 외부 AI, 클라우드 전송, 자동 모델 수정은 충분한 보안/책임 기준이 준비된 뒤 검토한다.

---

## 2. 버전별 계획

| 버전 | 목표 | 핵심 기능 |
|---|---|---|
| v0.1 | 내부 프로토타입 | 기존 Model Health Dashboard 기능 정리 |
| v0.5 | Store 후보 빌드 | 리본, 대시보드, 기본 진단, 리포트 출력 |
| v1.0 | Store 첫 출시 | 모델 품질 스캔, 리포트, Entitlement, User Guide |
| v1.1 | MEP 특화 | MEP 파라미터, 시스템명, 주요 카테고리 체크 |
| v1.2 | 팀 운영 | 회사별 체크 템플릿, 규칙 프리셋 |
| v2.0 | Pro | 변경 비교, 납품 패키지 검토, 컨설팅 리포트 연계 |

## 3. v1.0 릴리스 범위

| 포함 | 제외 |
|---|---|
| 로컬 모델 품질진단 | 외부 AI 분석 |
