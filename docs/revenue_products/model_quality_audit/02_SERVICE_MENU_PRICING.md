## 1. 가격 정책 원칙

1. 초기 상품은 고정 패키지로 판매하여 견적 소요 시간을 줄인다.
2. 모델 수, 공종 수, 연면적, 검토 회차, 보고 수준에 따라 단계화한다.
3. 실제 금액은 시장, 고객 규모, 자료 상태, 긴급도에 따라 조정한다.
4. 본 문서의 금액은 내부 기준이며 외부 제출 시 별도 견적서로 변환한다.

---

## 2. 패키지 구성

| 구분 | Lite | Standard | Premium |
|---|---:|---:|---:|
| 목적 | 빠른 상태 점검 | 실무 검토 보고 | 의사결정/개선 실행 |
| 대상 | 단일 모델/소규모 | 통합 모델/중규모 | 복수 공종/중요 납품 |
| 기간 | 3~5영업일 | 7~10영업일 | 2~3주 |
| 모델 수 | 1개 기준 | 3개 이내 | 협의 |
| 간섭 검토 | 주요 간섭 샘플 | 주요 공종 간섭 | 우선순위/회의 지원 |
| 품질 체크 | 기본 항목 | 상세 항목 | 상세 + 개선 로드맵 |
| 보고서 | 요약 보고서 | 정식 보고서 | 보고서 + 발표자료 |
| 회의 | 없음 또는 30분 | 결과 설명 1회 | 킥오프/결과회의 각 1회 |
| 재검토 | 제외 | 선택 | 1회 포함 가능 |

---

## 3. 가격 산정 변수

| 변수 | 영향 |
|---|---|
| 모델 개수 | 파일별 검토 및 통합 시간이 증가 |
| 공종 수 | 기계/전기/소방/통신/건축/구조 포함 범위에 따라 증가 |
| 모델 상태 | 좌표 오류, 링크 누락, 패밀리 오류가 많을수록 증가 |
| 납품 형식 | RVT, IFC, NWD, PDF, XLSX 등 검증 형식 증가 |
| 긴급도 | 단기 납품 요청 시 우선 투입 비용 반영 |
| 회의 횟수 | 고객 설명, 협력사 회의, 보완 회의 증가 |
| 보안 조건 | 폐쇄망, 반출 제한, 현장 방문 필요 시 증가 |
