## 시설 규모 팩트 확정-오답 방지 (2026-06-21)

- **증상**: `건축.md` 의 '주요 공항·대형시설 연면적 DB'가 임의의 'X 면적/연면적이 얼마야' 질의를 높은 점수로 매칭해 웹검색을 억제(confident). DB에 없는 시설(성수 K-project·김포 공항·롯데타워 등)도 다른 시설(인천공항 등) 수치를 confident 로 서빙.
- **수정**: `assess_knowledge_answer_quality` 에 named-entity-not-covered 가드 추가 — 규모 팩트 마커(면적/연면적/규모/세대수/층수/높이/건폐율/용적률/...)가 있는 질의에서, 질의의 고유 시설명이 답변 내 **수치(㎡/만/세대/층)와 인접**해 등장하지 않으면 not-ready 처리(웹검색 허용). 카테고리어(공항/터미널/센터/타워...)는 식별자에서 제외해 '김포 공항'이 인천'공항' 부분일치로 오통과하지 않게 함. 면책 산문의 단순 언급(예: '표에 없는 예: …')은 수치 인접이 아니라 미커버로 판정.

- 회귀 테스트: `test_entity_magnitude_nearby_unit`, `test_uncovered_facility_magnitude_query_triggers_search`, `test_covered_facility_magnitude_query_stays_confident`.

## 사용자/팀 결정 사항 (코드 밖)

4. ~~**고객 노출 score 태그(UX)**~~ **[해결됨]**: 유료 Revit 애드인 답변이 '근거 문서'에 내부 파일 경로+relevance score(`knowledge/10_agents/06_스토어상용화팀/라이선스결제.md (score 86)`)를 노출하던 것 → 사용자 결정('친근한 이름만')으로 지식 베이스 이름만 표기(`참고한 지식 베이스: 라이선스결제·제품패키징`), 내부 경로·점수 제거.
5b. **오프-도메인 substring hijack(저우선)**: BIM 봇 범위 밖 비즈니스/법무 복합어가 공종 키워드를 substring으로 포함해 엔지니어링으로 confident-오답: '통신판매(전자상거래) 신고'→통신(telecom), '구조조정(조직) 계획'→구조(structural), '마감 기한 관리'(deadline)→건축('마감'=finish 키워드). 인-도메인 클래스(내화/방화/피난구조→건축)는 [해결]했으나, 이들은 **올바른 홈이 조직에 없고**(전자상거래/조직개편 전용 에이전트 부재) 저빈도라 강제 라우팅 부적절. weak→web이 바람직하나 한글 복합어 substring이라 깔끔한 차단이 어려움 → 해당 도메인 에이전트 신설 시 라우팅. (정상: 정전기/충전기→전기, 재건축→건축, 정보통신→통신은 substring이나 올바른 매칭.)
5. **off-topic-confident 클래스(시스템적 한계)**: 질의의 distinctive 컨텍스트어(주차장·GTX-A·Revit 버전)가 답변에 없고 일반어(기둥/간격·철근·revit)만 매칭돼 confident-오답이 되는 케이스. 시설 규모(면적 등)는 named-entity 가드로 해결했으나 일반 클래스는 (a) 콘텐츠 저작(권장)이나 (b) 의미 기반 relevance/IDF 스코어링(대규모·false-WEAK 위험)이 필요 → 현재는 백로그 콘텐츠 저작으로 개별 해소. 무차별 off-topic 가드는 정상 답변까지 WEAK 처리할 위험이 커 미도입.
