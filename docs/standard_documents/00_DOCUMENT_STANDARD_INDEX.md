## 1. 기준 문서화 목적

본 문서는 LUA BIM LABS가 BIM 프로젝트, 상용 Add-in 제품화, 고객 제안, 발주처 제출 업무에서 반복적으로 사용하는 공식 문서의 기준 체계를 정의한다.

현재 저장소에는 제품 개발 및 Autodesk Store 제출 문서가 존재하나, 외부 제출용 공식 문서의 공통 구조와 프로젝트 수행 기준 문서가 분산되어 있다. 이에 따라 제안서, 수행계획서, BEP, CDE 운영계획, 모델링 납품기준, 품질 검증, 물량/ROI, 회의/변경/리스크 관리 문서를 하나의 기준 체계로 정리한다.

## 2. 공식 문서 공통 테마

첨부 기준 제안서의 문서 톤을 다음 원칙으로 반영한다.

| 항목 | 기준 |
|---|---|
| 문서 인상 | 공식 제출 문서, 실무 수행 계획, 정량 목표 중심 |
| 표지 구조 | 프로젝트명, 문서명, 제출처, 작성 주체, 작성일을 상단에 명확히 표기 |
| 본문 구조 | 번호형 장/절 구성, 표와 불릿을 병행, 결론부에 기대효과 정리 |
| 문장 톤 | 과장된 홍보 문구보다 현장 적용 가능성과 납품 기준을 우선 |
| 표 구성 | 구분, 내용, 산출물, 책임, 주기, 기준값을 반복 사용 |
| 정량 표현 | 가능하면 기간, 정확도, 감소율, 허용오차, 검토 주기 등 수치화 |
| 보안/공식성 | 발주처, 시공사, 감리/CM, 협력사 간 책임과 권한을 명시 |

## 5. 작성 및 검토 원칙

1. 프로젝트별 사실과 회사 공통 기준을 분리한다.
2. 외부 제출 문서는 책임 주체, 제출 시기, 산출물 형식을 반드시 표기한다.
3. 수치 목표는 근거가 있는 경우에만 확정값으로 쓰고, 미확정 값은 “목표” 또는 “협의 후 확정”으로 표시한다.
4. 발주처 요구사항과 내부 수행 기준이 충돌할 경우 발주처 EIR/BEP 요구사항을 우선한다.
5. 법무, 개인정보, 보안, 비용 조건은 내부 검토 전 확정 문구로 표현하지 않는다.
