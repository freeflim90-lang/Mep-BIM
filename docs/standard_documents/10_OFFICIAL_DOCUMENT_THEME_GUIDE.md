## 1. 테마 적용 목적

본 가이드는 공식 문서가 고객 제출용 문서로 보일 수 있도록 표지, 목차, 본문, 표, 결론의 표현 방식을 통일하기 위한 기준이다. 외부 제출용 비개발 문서의 공식 명칭은 `LUA BIM LAB`을 사용한다.

첨부 제안서의 핵심 인상은 “현장 문제를 이해하고, 실행 체계와 산출물을 수치로 설명하는 공식 제안 문서”다. 따라서 모든 기준 문서는 단순 메모가 아니라 제출 가능한 문서 골격을 유지한다.

## 2. 표지 구성

표지는 다음 순서를 기본으로 한다.

```text
[프로젝트명 또는 기준명]
[문서명]
━━━━━━━━━━━━━━━━━━━━
[작성 주체]
제출처: [대상]
[작성년월]
```

기준 문서에서는 제출처 대신 문서번호, 문서상태, 작성일, 적용범위를 사용한다.

| 항목 | 작성 기준 |
|---|---|
| 프로젝트명 | 첫 화면에서 가장 크게 인식되도록 1~2줄로 분리 |
| 문서명 | “BIM 수행 제안서”, “BIM 수행계획서”, “BEP”처럼 공식명 사용 |
| 구분선 | `━━━━━━━━━━━━━━━━━━━━` 사용 |
| 작성 주체 | 회사명 또는 조직명 |
| 제출처 | 고객사/시공사/발주처 표기 |
| 작성일 | `2026년 5월` 또는 `2026-05-20` 중 문서 성격에 맞게 선택 |

## 3. 목차 및 번호 체계

본문은 번호형 장/절 구조를 사용한다.

| 수준 | 형식 | 예시 |
|---|---|---|
| 장 | `1. 제목` | `1. 제안 개요` |
| 절 | `1.1 제목` | `1.1 제안 배경 및 목적` |
| 항 | `1.1.1 제목` | `5.2.1 Scan to BIM 전략` |
| 표 제목 | 본문 문장으로 소개 | `다음은 주요 산출물 목록이다.` |

## 4. 문체 기준

| 구분 | 권장 |
|---|---|
| 문장 톤 | 공식적, 명확함, 현장 적용 중심 |
| 주어 | 외부 문서: “LUA BIM LAB은”, 내부 문서: “BIM팀은”, “본 문서는” |
| 종결 | “제시한다”, “정의한다”, “관리한다”, “지원한다” |
| 피해야 할 표현 | 근거 없는 최상급, 과도한 홍보 표현, 모호한 약속 |
| 좋은 표현 | “협의 후 확정”, “발주처 기준에 따름”, “목표 리드타임” |

## 5. 표 작성 기준

공식 문서에서는 긴 설명보다 표를 우선 사용한다.

| 표 유형 | 기본 컬럼 |
|---|---|
| 수행 범위 | 구분, 포함 범위, 제외/협의 필요 |
| 조직/역할 | 역할, 책임, 주요 산출물 |
| 일정 | 단계, 기간, 마일스톤, 산출물 |
| 산출물 | No., 산출물명, 형식, 제출 시기 |
| 리스크 | 리스크, 발생 가능성, 영향, 대응 전략 |
| 기대효과 | 구분, 기대 효과, 정량 목표 |

## 6. 정량 목표 작성 기준

정량 목표는 공식 문서의 신뢰도를 높이지만, 근거 없는 확정값은 피한다.

| 상태 | 표현 방식 |
|---|---|
| 실적 근거 있음 | “재시공 85% 감소 실적 기반” |
| 목표값 | “목표: 5영업일 이내 반영” |
| 협의 필요 | “프로젝트 착수 시 협의 후 확정” |
| 발주처 기준 우선 | “EIR/BEP 기준에 따름” |

## 7. 결론 작성 기준

결론은 다음 3단락을 기본으로 한다.

1. 프로젝트의 핵심 난이도와 BIM의 필요성을 다시 정리한다.
2. LUA BIM LAB의 수행 방식과 차별성을 산출물 중심으로 요약한다.
3. 고객에게 제공되는 실질 효과를 공기, 품질, 비용, 납품 관점에서 마무리한다.

## 8. Word/PDF 변환 시 권장 스타일

| 요소 | 권장 스타일 |
|---|---|
| 본문 글꼴 | 맑은 고딕 또는 Noto Sans CJK KR |
| 본문 크기 | 10.5~11 pt |
| 제목 1 | 16~18 pt, 굵게 |
| 제목 2 | 13~14 pt, 굵게 |
| 표 머리글 | 밝은 회색 배경, 굵게 |
| 본문 줄간격 | 1.15~1.3 |
| 페이지 여백 | 상하좌우 20~25 mm |
| 머리말 | 프로젝트명/문서명 |
| 꼬리말 | 페이지 번호, 문서번호 |
