# 대외 커뮤니케이션 기준

━━━━━━━━━━━━━━━━━━━━

문서번호: LBL-STD-018  
문서상태: 기준 초안  
작성일: 2026-05-20  
적용범위: 고객 이메일, 회의록, 공문, 질의응답, 외부 보고

## 목 차

1. 목적
2. 커뮤니케이션 원칙
3. 이메일 기준
4. 질의응답(RFI) 기준
5. 회의록 기준
6. 공문/공식 제출 기준
7. 민감 표현 기준

## 1. 목적

본 기준은 외부 커뮤니케이션에서 범위, 일정, 비용, 책임이 의도치 않게 확정되는 것을 방지하고, 고객과의 협의 내용이 추적 가능한 문서로 남도록 하기 위한 기준이다.

## 2. 커뮤니케이션 원칙

| 원칙 | 설명 |
|---|---|
| 명확성 | 요청, 답변, 기한, 담당자를 분명히 쓴다 |
| 기록성 | 중요한 결정은 회의록 또는 공식 문서로 남긴다 |
| 범위 통제 | 추가 업무는 “별도 협의”로 표현한다 |
| 보안 | 민감 자료는 승인된 채널로만 공유한다 |
| 일관성 | 문서번호, Revision, 날짜를 표기한다 |

## 3. 이메일 기준

권장 제목 형식:

```text
[프로젝트명][문서/안건][요청/제출/확인] 제목 - YYYY-MM-DD
```

이메일 본문 기본 구조:

```text
안녕하세요.

[목적]에 따라 아래 자료를 제출/요청드립니다.

1. 요청/제출 내용
2. 확인 필요 사항
3. 회신 요청 기한
4. 첨부 파일 목록

감사합니다.
```

## 4. 질의응답(RFI) 기준

| 필드 | 설명 |
|---|---|
| RFI ID | 고유번호 |
| 질문일 | YYYY-MM-DD |
| 질문자 | 소속/이름 |
| 관련 도면/모델 | 파일명, Revision |
| 질문 내용 | 확인이 필요한 사항 |
| 희망 회신일 | YYYY-MM-DD |
| 답변자 | 소속/이름 |
| 답변 내용 | 승인된 답변 |
| 후속 조치 | 모델/도면/일정 반영 여부 |

## 5. 회의록 기준

회의록은 회의 후 가능한 한 빠르게 공유하고, 이견이 있는 경우 정정 요청 기한을 명시한다. 회의록에 비용/일정/범위 변경이 포함될 경우 PM 승인을 거친다.

## 6. 공문/공식 제출 기준

| 항목 | 기준 |
|---|---|
| 문서번호 | 공식 문서번호 부여 |
| 수신/참조 | 조직명과 담당자 명확히 표기 |
| 제목 | 안건과 요청사항이 드러나게 작성 |
| 본문 | 사실, 요청, 근거, 기한 순서 |
| 첨부 | 제출대장과 파일 목록 포함 |
| 승인 | 내부 승인 후 발송 |

## 7. 민감 표현 기준

| 피해야 할 표현 | 권장 표현 |
|---|---|
| “무조건 가능합니다” | “자료 확인 후 가능 범위를 확정하겠습니다” |
| “무료로 처리하겠습니다” | “계약 범위 내 여부를 확인하겠습니다” |
| “문제 없습니다” | “현재 확인된 범위에서는 주요 이슈가 없습니다” |
| “완벽히 보장합니다” | “합의된 검수 기준에 따라 관리하겠습니다” |
| “즉시 반영합니다” | “영향 분석 후 반영 일정안을 공유하겠습니다” |
