# 협력사 온보딩 기준

━━━━━━━━━━━━━━━━━━━━

문서번호: LBL-STD-016  
문서상태: 내부/협력사 공유 기준 초안  
작성일: 2026-05-20  
적용범위: 외주 모델러, 협력 설계사, 스캐닝 업체, 개발/문서 협력사

## 목 차

1. 목적
2. 투입 전 확인
3. 보안 및 권한
4. 작업 기준
5. 산출물 제출
6. 품질 검수
7. 정산 및 종료

## 1. 목적

본 기준은 협력사가 프로젝트에 투입될 때 작업 범위, 보안, 파일 기준, 품질 기준, 제출 절차를 명확히 하여 재작업과 자료 유출 리스크를 줄이기 위한 문서다.

## 2. 투입 전 확인

| 항목 | 확인 |
|---|---|
| 업무범위와 산출물이 문서로 확정되었는가 | [ ] |
| 단가, 지급 조건, 검수 기준이 확정되었는가 | [ ] |
| NDA/보안서약 필요 여부가 확인되었는가 | [ ] |
| 사용 소프트웨어 버전이 확인되었는가 | [ ] |
| 작업 일정과 중간 제출일이 확정되었는가 | [ ] |

## 3. 보안 및 권한

| 항목 | 기준 |
|---|---|
| 계정 | 개인별 계정 사용, 공유 계정 금지 |
| 접근 범위 | 담당 업무에 필요한 최소 폴더 |
| 자료 반출 | PM 승인 없이 외부 저장소 업로드 금지 |
| 재위탁 | 사전 승인 없이 금지 |
| 종료 | 계정 권한 회수 및 자료 삭제 확인 |

## 4. 작업 기준

협력사는 프로젝트별 모델링 기준, 파일명 규칙, 파라미터 기준, CDE 운영 기준을 따라야 한다.

| 기준 문서 | 적용 내용 |
|---|---|
| 모델링 및 납품 기준 | LOD, 명명, 파라미터 |
| CDE 운영 기준 | 폴더, 버전, 업로드 위치 |
| 간섭 및 품질 검증 기준 | 검수 항목, 이슈 대응 |
| 회의/변경/리스크 기준 | 변경 통보, 회의록, 조치기한 |

## 5. 산출물 제출

| 산출물 | 제출 형식 | 제출 위치 |
|---|---|---|
| 작업 모델 | RVT/IFC/NWC | CDE WIP/Shared |
| 변경 내역 | XLSX/PDF | 변경대장 |
| 검토 의견 | PDF/DOCX | 회의자료 폴더 |
| 최종 납품 | 계약 기준 | Deliverables |

## 6. 품질 검수

| 검수 항목 | 기준 |
|---|---|
| 파일 열림 | 지정 버전에서 정상 열람 |
| 명명 규칙 | 파일/뷰/패밀리 기준 준수 |
| 좌표/레벨 | 프로젝트 기준과 일치 |
| 파라미터 | 필수 항목 입력 |
| 중복/누락 | 검토표 기준 통과 |
| 일정 | 중간/최종 제출일 준수 |

## 7. 정산 및 종료

정산은 승인된 산출물과 검수 완료 기록을 기준으로 진행한다. 업무 종료 시 협력사 접근권한을 회수하고, 고객 자료 반납 또는 삭제 확인을 기록한다.
