# 프로젝트 종료 및 레슨런 기준

━━━━━━━━━━━━━━━━━━━━

문서번호: LBL-STD-020  
문서상태: 기준 초안  
작성일: 2026-05-20  
적용범위: BIM 프로젝트 종료, 납품 완료, 내부 지식화

## 목 차

1. 목적
2. 종료 조건
3. 종료보고서 구성
4. 산출물 아카이브
5. 레슨런 회고
6. 표준문서 업데이트
7. 포트폴리오 활용 검토

## 1. 목적

본 기준은 프로젝트 종료 후 산출물을 정리하고, 수행 중 얻은 교훈을 다음 프로젝트의 기준 문서와 업무 절차에 반영하기 위한 내부 기준이다.

## 2. 종료 조건

| 조건 | 확인 |
|---|---|
| 계약상 최종 산출물이 제출되었는가 | [ ] |
| 고객 검수 또는 인수확인이 기록되었는가 | [ ] |
| 반려/보완 이슈가 모두 종료 또는 합의되었는가 | [ ] |
| 최종 파일이 Archive에 보관되었는가 | [ ] |
| 미수금 또는 정산 이슈가 확인되었는가 | [ ] |

## 3. 종료보고서 구성

| 섹션 | 내용 |
|---|---|
| 프로젝트 개요 | 기간, 고객, 범위, 공종 |
| 수행 요약 | 주요 업무, 회차, 투입 인력 |
| 산출물 목록 | 최종 제출 파일과 Revision |
| 주요 성과 | 간섭 해소, 물량, 품질, 일정 |
| 주요 이슈 | 지연, 변경, 품질, 보안 |
| 고객 피드백 | 승인 의견, 개선 요청 |
| 후속 지원 | 지원 기간, 담당자, 연락 채널 |

## 4. 산출물 아카이브

```text
99_Archive/
  01_Final_Deliverables
  02_Approval_Evidence
  03_Contracts_Quotation
  04_Meeting_Minutes
  05_Issue_Change_Risk
  06_Lessons_Learned
```

## 5. 레슨런 회고

| 질문 | 기록 |
|---|---|
| 잘 작동한 기준/절차는 무엇인가 | |
| 반복된 문제는 무엇인가 | |
| 견적 단계에서 놓친 범위는 무엇인가 | |
| 고객이 가장 가치 있게 본 산출물은 무엇인가 | |
| 다음 프로젝트에서 자동화할 업무는 무엇인가 | |
| 표준문서에 반영할 변경은 무엇인가 | |

## 6. 표준문서 업데이트

프로젝트 종료 후 다음 문서의 개정 필요성을 검토한다.

| 문서 | 검토 항목 |
|---|---|
| 제안서 기준 | 고객 반응, 차별화 문구 |
| 수행계획/BEP | 실제 승인 절차와 불일치 |
| CDE 기준 | 폴더/권한/배포 문제 |
| 모델링 기준 | 파라미터/LOD/명명 누락 |
| 품질 기준 | 검수 반려 항목 |
| 견적/SOW | 추가 업무 발생 항목 |
