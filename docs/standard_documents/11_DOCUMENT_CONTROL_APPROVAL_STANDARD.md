# 문서관리 및 승인 기준

━━━━━━━━━━━━━━━━━━━━

문서번호: LBL-STD-011  
문서상태: 기준 초안  
작성일: 2026-05-20  
적용범위: 내부 기준 문서, 외부 제출 문서, 프로젝트 산출 문서

## 목 차

1. 목적
2. 문서 등급
3. 문서번호 체계
4. Revision 기준
5. 작성-검토-승인 절차
6. 배포 및 보관 기준
7. 변경 이력 관리

## 1. 목적

본 기준은 LUA BIM LABS의 공식 문서가 작성자 개인 판단에 따라 임의 배포되지 않도록 문서번호, Revision, 승인권자, 배포 위치를 통제하기 위한 내부 기준이다.

## 2. 문서 등급

| 등급 | 설명 | 예시 | 외부 배포 |
|---|---|---|---|
| Internal | 내부 운영 기준 | 업무 절차, 검토 체크리스트 | 불가 |
| Review | 고객/협력사 검토용 | 제안 초안, BEP 초안 | 제한 가능 |
| Official | 공식 제출본 | 제안서, 수행계획서, 납품 보고서 | 가능 |
| Confidential | 비밀/보안 자료 포함 | NDA 자료, 계약조건, 보안구역 모델 | 승인 필요 |

## 3. 문서번호 체계

```text
LBL-[문서유형]-[일련번호]-R[Revision]
```

| 문서유형 | 의미 |
|---|---|
| STD | 기준 문서 |
| PRP | 제안서 |
| BEP | BIM Execution Plan |
| RPT | 보고서 |
| MIN | 회의록 |
| TRN | 제출/전달 문서 |
| CTR | 계약/범위 검토 |
| SEC | 보안/NDA |

## 4. Revision 기준

| Revision | 의미 |
|---|---|
| R00 | 최초 초안 |
| R01~Rnn | 검토 반영본 |
| F00 | 최종 제출본 |
| F01~Fnn | 최종 제출 후 승인된 개정본 |

파일명에는 문서번호, 문서명, Revision, 날짜를 포함한다.

## 5. 작성-검토-승인 절차

| 단계 | 책임 | 완료 기준 |
|---|---|---|
| 작성 | 작성자 | 초안 작성 및 자체 점검 |
| 내부 검토 | 담당 Lead/BIM PM | 범위, 수치, 표현, 첨부 확인 |
| 승인 | PM/대표/지정 승인자 | 외부 배포 가능 여부 승인 |
| 배포 | Document Controller | CDE/이메일/공문 배포 기록 |
| 보관 | Document Controller | 제출대장 및 Archive 저장 |

## 6. 배포 및 보관 기준

- 외부 제출 문서는 PDF를 기본으로 하고, 편집 가능한 DOCX/XLSX는 요청 또는 계약 조건이 있을 때만 제공한다.
- 이메일 본문으로 중요한 계약/범위 변경을 확정하지 않고, 첨부 문서 또는 회의록으로 근거를 남긴다.
- 최종 제출본은 `Published` 또는 `Archive` 폴더에 보관한다.

## 7. 변경 이력 관리

| 항목 | 작성 기준 |
|---|---|
| 변경일 | YYYY-MM-DD |
| Revision | R00/R01/F00 등 |
| 변경 내용 | 수정된 장/절과 요약 |
| 변경 사유 | 고객 의견, 내부 검토, 계약 변경 등 |
| 작성/승인 | 이름 또는 역할 |
