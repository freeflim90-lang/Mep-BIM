## 1. 운영 목적

BIM 회의와 변경/리스크 관리는 모델 작성 결과를 현장 의사결정으로 연결하기 위한 절차다. 모든 결정 사항은 회의록, 변경대장, 이슈대장, 리스크대장 중 하나에 기록되어 추적 가능해야 한다.

## 2. 회의 체계

| 회의명 | 주기 | 목적 | 참석자 |
|---|---|---|---|
| 킥오프 회의 | 착수 시 | 범위, 일정, 자료, 책임 확인 | 전체 핵심 참여자 |
| 주간 BIM 회의 | 주 1회 | 이슈, 간섭, 변경 현황 공유 | BIM팀, 시공사, 공종 담당 |
| 간섭 검토 회의 | 주간/격주 | 주요 Clash 조치 결정 | BIM팀, 설계사, 협력사 |
| 변경 검토 회의 | 변경 발생 시 | 영향 분석 및 승인 | 시공사, 설계사, BIM PM |
| 월간 보고 회의 | 월 1회 | 성과, 리스크, 납품 현황 보고 | 발주처/CM/시공사 |

## 3. 회의록 기준

회의록 필수 항목:

| 항목 | 내용 |
|---|---|
| 회의명/회차 | 공식 회의명 및 회차 |
| 일시/장소 | YYYY-MM-DD HH:MM |
| 참석자 | 소속, 이름, 역할 |
| 주요 논의 | 안건별 요약 |
| 결정 사항 | 승인/보류/반려 여부 |
| Action Item | 담당자, 기한, 결과물 |
| 첨부 | 모델, 도면, 보고서 링크 |

중요 판단 안건은 다음 5칸 구조로 기록한다.

| 칸 | 기록 기준 |
|---|---|
| 결론 | 회의에서 합의한 판단 또는 보류 사유 |
| 근거 | 도면/모델 Revision, BEP/EIR, 법규/기준, RFI, 회의록 |
| 적용 범위 | 공종, 구역, 단계, 계약 범위 |
| 주의 | 미확정 기준, 발주처/설계자/전문가 확인 필요 여부 |
| 다음 액션 | 담당자, 기한, 반영 문서/모델 |

## 4. 변경 관리 절차

| 단계 | 설명 | 산출물 |
|---|---|---|
| 접수 | 변경 요청/지시 수령 | 변경 접수대장 |
| 분류 | 설계/시공/발주처/현장 이슈 구분 | 변경 ID |
| 영향 분석 | 모델, 물량, 일정, 간섭 영향 검토 | 영향 분석표 |
| 승인 | 책임 주체 승인 | 승인 기록 |
| 반영 | BIM 모델 및 산출물 업데이트 | 변경 모델 |
| 배포 | CDE Published/Shared 반영 | 배포 기록 |

목표 리드타임은 프로젝트 규모와 변경 범위에 따라 착수 시 확정한다.

## 5. 이슈 관리 절차

| 상태 | 의미 |
|---|---|
| Open | 신규 등록 |
| In Progress | 담당자 조치 중 |
| Resolved | 조치 완료, 검토 대기 |
| Closed | 검토 및 승인 완료 |
| Deferred | 승인된 보류 |

이슈는 책임자와 조치기한이 없는 상태로 장기간 유지하지 않는다.

## 6. 리스크 관리 절차

| 리스크 | 발생 가능성 | 영향 | 대응 전략 |
|---|---|---|---|
| 자료 지연 | 중간 | 착수 지연 | 자료 요청 목록과 기한 명시 |
| 도면 불일치 | 높음 | 모델 오류 | 현장 조사 및 질의 관리 |
| 변경 누락 | 중간 | 시공/물량 오류 | 변경대장과 모델 Revision 연계 |
| 승인 지연 | 중간 | 배포 지연 | 승인권자와 검토기한 사전 합의 |
| 보안 자료 오배포 | 낮음 | 계약/보안 리스크 | CDE 권한 및 배포 승인 적용 |

## 7. 대장 필드 기준

### 7.1 변경대장

| 필드 | 설명 |
|---|---|
| Change ID | 변경 고유번호 |
| 접수일 | YYYY-MM-DD |
| 요청자 | 소속/이름 |
| 변경 사유 | 설계변경/현장조건/발주처 지시 등 |
| 영향 범위 | 공종, 구역, 모델, 도면 |
| 물량 영향 | 증가/감소/없음/검토 중 |
| 일정 영향 | 있음/없음/검토 중 |
| 승인 상태 | 접수/검토/승인/반려/완료 |
| 판단 근거 | 도면/모델 Revision, 기준, 회의록, RFI |
| 적용 범위 | 공종, 구역, 단계, 계약 범위 |
| 주의 사항 | 미확정 기준, 추가 확인 필요 사항 |

### 7.2 리스크대장

| 필드 | 설명 |
|---|---|
| Risk ID | 리스크 고유번호 |
| 리스크명 | 간단한 제목 |
| 원인 | 발생 원인 |
| 영향 | 비용/일정/품질/안전/보안 |
| 가능성 | High/Medium/Low |
| 영향도 | High/Medium/Low |
| 대응 전략 | 회피/완화/전가/수용 |
| 담당자 | 관리 책임자 |
| 상태 | Open/Monitoring/Closed |
