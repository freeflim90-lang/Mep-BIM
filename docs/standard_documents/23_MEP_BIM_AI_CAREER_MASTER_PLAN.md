## 1. 교육 철학

본 마스터 플랜은 LUA BIM LABS 구성원이 MEP BIM을 코어 전문성으로 삼고, 설계·시공 지식을 바탕으로 실무 판단력을 갖추며, AI와 개발 능력을 생산성 향상 도구로 활용하도록 설계한다.

AI 시대의 BIM 인재는 단순 모델러도, 단순 개발자도 아니다. 좋은 MEP BIM 인재는 설계 의도, 시공 가능성, 현장 제약, 데이터 구조를 이해하고 반복 업무를 AI와 자동화로 줄일 수 있어야 한다.

## 3. 1~20년차 성장 단계

| 단계 | 연차 | 역할 정의 | 핵심 목표 |

| Principal | 17~20년차 | Principal / 교육·전략 리더 | 표준화, 자동화 제품화, 조직 교육 체계 설계 |

## 6. AI 및 개발 역량 커리큘럼

AI와 개발 교육은 MEP BIM 판단력을 대체하지 않는다. 목적은 반복 작업을 줄이고 검토 품질을 높이는 것이다.

| 단계 | AI 활용 | 개발/자동화 | 금지/주의 |
|---|---|---|---|
| 1~2년차 | 문서 요약, 체크리스트 생성, 용어 학습 | Excel 함수, CSV 정리, 파일명 자동화 | AI 답변을 설계 기준으로 확정 금지 |
| 3~5년차 | 간섭 이슈 분류, 보고서 초안, RFI 초안 | Python 기초, pandas/openpyxl, Dynamo 기초 | 모델 변경 자동화는 검토자 승인 필요 |
| 6~8년차 | 회의록 요약, 변경 영향 분석 보조 | Revit API 개념, pyRevit/Dynamo, QA 스크립트 | API로 모델 변경 시 Transaction/백업 필수 |
| 9~12년차 | BEP/SOW 초안, 리스크 분석, 교육자료 생성 | 사내 도구 기획, Add-in 요구사항, 테스트 설계 | 자동화 결과 검증 없는 납품 금지 |
| 13년차 이상 | 조직 지식화, 표준 개정, 제품화 후보 발굴 | 제품 로드맵, API 검토, 데이터 파이프라인 | AI/자동화 도입 비용과 유지관리성 검토 |

## 8. 실습 프로젝트 체계

| 단계 | 실습 프로젝트 | 목적 |
|---|---|---|
| 입문 | 소형 기계실 MEP 모델링 | 계통, 장비, 배관/덕트 기본 이해 |

## 9. 평가 및 승급 기준

| 평가 영역 | 기준 |
|---|---|
| 기술 정확도 | 모델, 파라미터, 좌표, 납품 형식 오류율 |
| 설계/시공 이해 | 설계 의도와 현장 제약을 설명할 수 있는가 |
| 데이터 활용 | 모델에서 물량, 이슈, 변경 정보를 추출할 수 있는가 |
| AI 활용 | 초안 생성 후 검증/수정/근거 확인을 수행하는가 |
| 개발 기초 | 반복 업무를 스크립트/자동화 요구사항으로 전환하는가 |
| 커뮤니케이션 | 회의록, RFI, 보고서가 의사결정에 쓰이는가 |
| 리더십 | 후배 교육, 협력사 검수, 고객 조율을 수행하는가 |

## 10. 교육 운영 방식

| 운영 항목 | 기준 |
|---|---|
