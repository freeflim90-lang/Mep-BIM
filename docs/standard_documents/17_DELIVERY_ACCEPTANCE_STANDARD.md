# 납품 및 인수확인 기준

작성일: 2026-05-20  
적용범위: 단계 납품, 준공 납품, 보고서 제출, 고객 인수확인

## 목 차

1. 목적
2. 납품 패키지 구성
3. 제출대장 기준
4. 사전 검수
5. 고객 검수 및 인수
6. 반려 및 보완
7. 최종 보관

## 1. 목적

본 기준은 BIM 모델과 문서 산출물이 누락 없이 제출되고, 고객의 인수확인 또는 보완 요청이 추적 가능하게 관리되도록 정의한다.

## 2. 납품 패키지 구성

```text
Deliverables/
  00_Transmittal
  01_Models
  02_IFC_Navisworks
  03_Drawings
  04_Reports
  05_Quantity
  06_COBie_FM
  07_QA_Evidence
  99_Archive
```

## 3. 제출대장 기준

| 필드 | 설명 |
|---|---|
| No. | 제출 순번 |
| 문서번호 | 공식 문서번호 |
| 파일명 | 실제 제출 파일명 |
| Revision | R/F 번호 |
| 산출물명 | 모델, 보고서, 도면 등 |
| 형식 | RVT, IFC, PDF, XLSX 등 |
| 제출일 | YYYY-MM-DD |
| 제출자 | 담당자 |
| 수신자 | 고객/감리/CM |
| 상태 | 제출/검토/승인/반려/보완 |

## 4. 사전 검수

| 검수 항목 | 확인 |
|---|---|
| 제출대장과 실제 파일 목록이 일치하는가 | [ ] |
| 파일이 정상적으로 열리는가 | [ ] |
| 모델 좌표와 링크가 정상인가 | [ ] |
| PDF/XLSX 문서의 표지, 날짜, Revision이 맞는가 | [ ] |
| 보안/비공개 자료가 잘못 포함되지 않았는가 | [ ] |
| 고객 지정 형식과 폴더 구조를 준수했는가 | [ ] |

## 5. 고객 검수 및 인수

| 단계 | 내용 |
|---|---|
| 제출 | 공식 저장소 또는 이메일로 제출 |
| 접수 확인 | 고객 수신 확인 기록 |
| 검토 | 고객/감리/CM 검토 |
| 보완 | 반려 또는 의견 사항 반영 |
| 인수 | 승인 또는 인수확인 기록 |

## 6. 반려 및 보완

반려 의견은 구두로만 처리하지 않고 반려 사유, 대상 파일, 조치 담당자, 보완 기한을 기록한다.

| 필드 | 설명 |
|---|---|
| Reject ID | 반려 고유번호 |
| 대상 산출물 | 파일명/문서번호 |
| 반려 사유 | 누락, 형식 오류, 품질 문제 등 |
| 조치 담당 | 담당자 |
| 보완 기한 | YYYY-MM-DD |
| 재제출 Revision | R/F 번호 |

## 7. 최종 보관

최종 승인본은 CDE `Published`와 `Archive`에 보관한다. 고객 인수확인 메일, 공문, 회의록, 제출대장은 납품 패키지와 함께 보관한다.
