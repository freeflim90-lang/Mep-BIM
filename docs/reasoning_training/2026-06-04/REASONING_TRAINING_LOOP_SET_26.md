# LUA BIM LABS 추론 훈련 루프 세트 26

- 생성일: 2026-06-04
- 목적: 감사 개선 티켓을 실행 우선순위 큐로 배열하고, 오늘 처리할 일과 보류할 일을 구분한다.
- 훈련 초점: `개선 티켓 → 영향도 → 선행조건 → 큐 위치 → 오늘 실행`

## 우선순위 큐 코드

- `PRI01 즉시 차단`: 보안, 개인정보, 계약, 고객 신뢰에 직접 위험이 있어 먼저 막는다.
- `PRI02 납품 영향`: 현재 납품, 검수, 고객 응답에 영향을 주는 티켓을 먼저 처리한다.
- `PRI03 기준 병목`: 여러 문서와 자동화가 의존하는 공통 기준을 먼저 정한다.
- `PRI04 재검증 예약`: 공식 출처 확인이 필요하지만 즉시 확정할 수 없어 Watch와 다음 확인일을 둔다.
- `PRI05 링크 회복`: 내용은 유효하지만 Obsidian 그래프에서 고립되어 탐색성이 낮은 지식을 연결한다.
- `PRI06 중복 압축`: 반복 문서가 많아 검색 품질을 떨어뜨릴 때 대표 문서 중심으로 정리한다.
- `PRI07 승격 후보`: 교육, SOP, 상품, 자동화 후보로 가치가 있으나 파일럿이나 검토가 필요한 항목이다.
- `PRI08 백로그 유지`: 당장 영향은 낮지만 추후 참고 가치가 있어 보류 큐에 둔다.

## 1. 보안 위험 자료와 뉴스 중복 정리가 동시에 들어옴

### 큐 상황
- 티켓 A: 고객명과 계약 금액이 포함된 외부 자료 차단
- 티켓 B: 뉴스 트렌드 반복 섹션 공통 기준 분리

### 위험한 판단
문서가 지저분하니 뉴스 중복부터 정리한다.

### 우선순위 판단
티켓 A를 `PRI01 즉시 차단`으로 먼저 처리한다. 보안 위험은 정리 미관보다 앞선다.

### 오늘 실행
원문 링크를 제거하고 보안검토 기록만 남긴다. 뉴스 중복은 기준 분리 티켓으로 보류한다.

## 2. Autodesk 정책 후보와 납품 검수 QA가 동시에 들어옴

### 큐 상황
- 티켓 A: Autodesk Add-in Store 정책 후보 공식 출처 재검증
- 티켓 B: 다음 주 고객 납품 검수 QA 체크리스트 링크 보강

### 위험한 판단
Autodesk 정책이 더 커 보이니 먼저 처리한다.

### 우선순위 판단
티켓 B를 `PRI02 납품 영향`으로 먼저 처리한다. Autodesk 정책은 확정 출처 확인 전까지 `PRI04 재검증 예약`으로 둔다.

### 오늘 실행
납품 검수 QA 체크리스트를 `BIM_납품검수`, `QA_테스터`, `고객지원운영`과 연결하고, Autodesk 항목은 다음 확인일을 기록한다.

## 3. 반복 Telegram QA와 혼합 언어 지식큐레이터 섹션이 동시에 들어옴

### 큐 상황
- 티켓 A: 인천공항 면적 반복 QA 대표 FAQ 병합
- 티켓 B: 지식큐레이터 혼합 언어 자동 수집 섹션 재작성

### 위험한 판단
읽기 불편한 혼합 언어 문장부터 고친다.

### 우선순위 판단
두 티켓 모두 중요하지만, 반복 QA가 검색 신뢰도를 직접 낮추면 `PRI06 중복 압축`으로 먼저 처리한다. 혼합 언어 섹션은 `PRI03 기준 병목`인지 확인한다.

### 오늘 실행
대표 FAQ를 지정하고 반복 QA를 링크한다. 지식큐레이터 섹션은 내부 체크리스트 재작성 티켓으로 남긴다.

## 4. 공통 기준 노트와 날짜별 로그 정리가 동시에 필요함

### 큐 상황
뉴스 트렌드 심화 자동 반영 문서가 매일 반복 기준을 삽입하고 있다.

### 위험한 판단
날짜별 문서를 하나씩 줄인다.

### 우선순위 판단
공통 기준이 없으면 날짜별 로그를 정리해도 반복이 재발한다. 먼저 `PRI03 기준 병목`으로 공통 기준 노트를 만든다.

### 오늘 실행
공통 기준 노트 1개를 만들고, 오늘 이후 날짜별 로그는 해당 기준으로 링크한다.

## 5. Obsidian 링크가 없는 좋은 SOP가 발견됨

### 큐 상황
내용은 쓸 만하지만 관련 MOC, 지식업데이트, 역할 문서와 연결되지 않은 SOP가 있다.

### 위험한 판단
내용이 괜찮으니 유지한다.

### 우선순위 판단
탐색되지 않는 지식은 실제 운영에서 사라진다. `PRI05 링크 회복`으로 처리한다.

### 오늘 실행
관련 지식 2개 이상, MOC 1개, 다음 검토일을 추가한다.

## 6. 납품 검수 서비스 지식이 상품 후보로도 보임

### 큐 상황
납품 검수 체크리스트가 교육자료, 서비스 패키지, 자동화 후보로 동시에 쓰일 가능성이 있다.

### 위험한 판단
당장 상품화한다.

### 우선순위 판단
고객 판매 전 파일럿과 범위 정의가 필요하다. `PRI07 승격 후보`로 두고 파일럿 조건을 붙인다.

### 오늘 실행
서비스 정의, 교육자료, 자동화 후보를 나누고 `needs-pilot` 상태와 파일럿 기준을 기록한다.

## 7. 공식 출처가 필요한 수치 업데이트가 급해 보임

### 큐 상황
BIM 시장 규모 수치가 오래됐지만 공식·전문 출처가 아직 없다.

### 위험한 판단
대략 최신 수치로 바꾼다.

### 우선순위 판단
수치와 정책은 틀리면 신뢰를 크게 잃는다. `PRI04 재검증 예약`으로 두고 확정 반영은 보류한다.

### 오늘 실행
현재 문서에는 Watch 표시, 필요한 출처 유형, 다음 확인일만 기록한다.

## 8. 오래된 참고 문서가 많아 보임

### 큐 상황
현재 업무와 연결은 약하지만 과거 의사결정 맥락을 담은 문서가 많다.

### 위험한 판단
전부 삭제한다.

### 우선순위 판단
즉시 위험이나 납품 영향이 없으면 `PRI08 백로그 유지`로 둔다. 다만 검색 노이즈가 크면 아카이브 후보로 표시한다.

### 오늘 실행
삭제하지 않고 참고 기록, 아카이브 후보, 다음 월간 리뷰 대상으로 분류한다.

## 오늘의 사고 원칙

1. 보안과 고객 신뢰 위험은 문서 정리보다 앞선다.
2. 납품에 직접 영향이 있는 지식은 장기 연구보다 먼저 처리한다.
3. 반복 문제는 개별 문서보다 공통 기준을 먼저 고쳐야 재발이 줄어든다.
4. 출처가 약한 수치와 정책은 급해 보여도 Watch와 재검증 예약으로 둔다.
5. 우선순위 큐는 일을 많이 하는 장치가 아니라 오늘 하지 않을 일을 명확히 하는 장치다.
