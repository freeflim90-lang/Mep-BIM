# LUA BIM LABS 추론 훈련 루프 세트 27

- 생성일: 2026-06-04
- 목적: 지식 개선 우선순위 큐를 주간 실행 계획과 리뷰 지표로 전환한다.
- 훈련 초점: `우선순위 큐 → 주간 목표 → 실행 슬롯 → 리뷰 지표 → 다음 큐`

## 주간 실행 계획 코드

- `WKP01 보안 먼저`: 주간 첫 슬롯은 보안·개인정보·고객 신뢰 위험을 해소한다.
- `WKP02 납품 연결`: 납품, 검수, 고객 응답과 연결된 지식 개선을 주간 핵심 슬롯에 둔다.
- `WKP03 기준 고정`: 반복을 줄이는 공통 기준 문서를 주간 기준 슬롯으로 처리한다.
- `WKP04 재검증 예약`: 공식 출처 확인이 필요한 항목은 확인일과 담당 역할을 배정한다.
- `WKP05 링크 지표`: Obsidian 링크 보강은 관련 링크 수와 고립 문서 감소로 측정한다.
- `WKP06 중복 감소`: 반복 QA와 중복 기준은 대표 문서 지정 수로 측정한다.
- `WKP07 승격 실험`: 상품, 교육, 자동화 후보는 파일럿 조건과 실험 결과로 측정한다.
- `WKP08 보류 사유`: 이번 주 하지 않는 항목은 보류 사유와 다음 조건을 남긴다.

## 1. 주간 첫 슬롯 배정

### 큐 입력
`PRI01 즉시 차단` 2건, `PRI05 링크 회복` 5건, `PRI07 승격 후보` 3건이 있다.

### 위험한 계획
빠르게 끝나는 링크 회복 5건부터 처리한다.

### 주간 판단
주간 첫 슬롯은 `WKP01 보안 먼저`로 배정한다. 빠르게 끝나는 일보다 위험을 줄이는 일이 먼저다.

### 리뷰 지표
- 원문 링크 제거 건수
- 보안검토 기록 작성 건수
- 민감정보 잔존 여부

## 2. 납품 검수 지식 개선 슬롯

### 큐 입력
다음 주 고객 납품 검수와 연결된 QA 체크리스트 링크 보강 티켓이 있다.

### 위험한 계획
월간 리뷰 때 한꺼번에 본다.

### 주간 판단
납품 직전 지식은 `WKP02 납품 연결`로 이번 주 핵심 슬롯에 둔다.

### 리뷰 지표
- 납품 검수 체크리스트 링크 수
- 관련 SOP 연결 여부
- 고객 응답 템플릿 반영 여부

## 3. 뉴스 트렌드 반복 기준 고정

### 큐 입력
날짜별 뉴스 트렌드 운영 기준이 반복되어 지식업데이트 문서가 길어진다.

### 위험한 계획
이번 주 로그만 짧게 줄인다.

### 주간 판단
공통 기준이 없으면 반복이 재발한다. `WKP03 기준 고정`으로 공통 기준 노트 생성부터 처리한다.

### 리뷰 지표
- 공통 기준 노트 생성 여부
- 날짜별 로그의 기준 노트 링크 여부
- 반복 상세 기준 신규 삽입 감소

## 4. Autodesk 정책 재검증 예약

### 큐 입력
Autodesk 정책 후보가 공식 출처 없이 Watch 상태다.

### 위험한 계획
바쁘면 다음 주로 넘긴다.

### 주간 판단
확정 반영은 보류하되 `WKP04 재검증 예약`으로 확인일과 담당 역할을 배정한다.

### 리뷰 지표
- 공식 출처 확인 여부
- 미확인 사유 기록 여부
- 다음 확인일 기록 여부

## 5. Obsidian 링크 보강 측정

### 큐 입력
좋은 SOP가 있지만 MOC와 관련 지식에 연결되지 않았다.

### 위험한 계획
링크를 몇 개 추가하고 완료로 본다.

### 주간 판단
`WKP05 링크 지표`로 관련 링크 수와 고립 문서 감소를 함께 본다.

### 리뷰 지표
- 관련 위키링크 2개 이상
- MOC 연결 1개 이상
- 다음 검토일 기록

## 6. 반복 QA 중복 감소 측정

### 큐 입력
같은 질문의 Telegram QA가 여러 문서로 반복된다.

### 위험한 계획
중복 문서를 삭제한다.

### 주간 판단
`WKP06 중복 감소`로 대표 문서 지정과 반복 문서 링크 전환을 측정한다.

### 리뷰 지표
- 대표 FAQ 지정 수
- 반복 QA 링크 전환 수
- needs-review 수치 분리 여부

## 7. 납품 검수 상품 후보 실험

### 큐 입력
납품 검수 지식이 서비스 상품, 교육자료, 자동화 후보로 보인다.

### 위험한 계획
바로 상품 문구를 만든다.

### 주간 판단
`WKP07 승격 실험`으로 파일럿 조건을 먼저 만든다.

### 리뷰 지표
- 서비스 범위 초안
- 교육자료 초안
- 자동화 후보 목록
- needs-pilot 또는 파일럿 결과 기록

## 8. 이번 주 하지 않을 항목 정리

### 큐 입력
오래된 참고 문서와 낮은 영향도 백로그가 많다.

### 위험한 계획
보류만 하고 이유를 남기지 않는다.

### 주간 판단
`WKP08 보류 사유`로 보류 조건을 남겨야 다음 큐가 흔들리지 않는다.

### 리뷰 지표
- 보류 사유 기록
- 다음 처리 조건 기록
- 월간 리뷰 후보 분류

## 주간 실행 계획 템플릿

```text
주간 목표:
큐 입력:
이번 주 실행:
이번 주 보류:
보류 사유:
리뷰 지표:
완료 증빙:
다음 큐:
```

## 오늘의 사고 원칙

1. 주간 계획은 많은 일을 담는 목록이 아니라 위험과 납품 영향부터 정렬하는 장치다.
2. 보류 항목은 실패가 아니라 조건을 명시한 다음 큐다.
3. 링크 보강은 링크 수가 아니라 고립 지식 감소로 검토한다.
4. 중복 감소는 삭제보다 대표 문서 지정과 링크 전환으로 시작한다.
5. 주간 리뷰는 노력량보다 위험 감소, 납품 지원, 반복 감소를 측정한다.
