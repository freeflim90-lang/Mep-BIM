# LUA BIM LABS 추론 훈련 루프 세트 24

- 생성일: 2026-06-04
- 목적: Obsidian과 지식 베이스에 반영된 지식을 정기 감사하고, 중복·노후·출처 약함·링크 부족·보안 위험을 보정한다.
- 훈련 초점: `반영된 지식 → 품질 신호 → 감사 질문 → 보정 판정 → 후속 처리`

## 지식 감사 코드

- `AUD01 유지`: 현재 기준으로 충분히 정확하고 연결도 좋다.
- `AUD02 링크 보강`: 내용은 유효하지만 관련 위키링크나 역링크가 부족하다.
- `AUD03 병합`: 같은 주제의 중복 문서/섹션이 있어 통합이 필요하다.
- `AUD04 재검증`: 수치, 정책, 법규, 버전 정보가 오래됐거나 출처가 약하다.
- `AUD05 아카이브`: 현재 업무 목적성과 낮거나 오래되어 참고 기록으로 이동한다.
- `AUD06 보안 차단`: 민감정보, 고객명, 계약, 내부 경로가 남아 있어 즉시 차단한다.
- `AUD07 구조 재작성`: 원문 복사나 산만한 수집 형태라 내부 기준 문서로 재작성한다.
- `AUD08 승격 재분류`: 교육자료, QA, 상품 후보 등 더 적합한 위치로 옮긴다.

## 1. 같은 질문의 Telegram QA가 여러 개 존재

### 품질 신호
인천공항 터미널 면적 질문이 여러 날짜의 QA 문서로 반복 등록되어 있다.

### 위험한 처리
반복 질문이니 모두 FAQ로 둔다.

### 감사 질문
- 각 문서가 같은 질문을 다루는가?
- 답변 출처와 수치가 일치하는가?
- 확정 답변이 있는 문서와 needs-review 문서가 섞여 있는가?

### 보정 판정
AUD03 병합, AUD04 재검증

### 후속 처리
공식 출처가 확인된 답변을 기준 문서로 두고, 중복 QA는 기준 문서로 링크한다. 출처 미확정 답변은 needs-review로 유지한다.

### Obsidian 처리
`Team_Telegram_QA` 관련 문서에서 대표 FAQ 1개를 기준 링크로 지정하고, 나머지는 `관련 반복 질문`으로 연결한다.

## 2. 자동 수집 요약이 외국어와 한국어가 섞여 있음

### 품질 신호
지식큐레이터 문서 일부 섹션에 중국어 문장과 한국어 설명이 섞여 있어 현업 재사용성이 낮다.

### 위험한 처리
내용이 있어 보이므로 그대로 유지한다.

### 감사 질문
- 현업자가 읽고 실행 기준으로 쓸 수 있는가?
- 출처 검증 기준과 운영 기준이 명확히 분리됐는가?
- 원문 요약인지 내부 기준인지 구분되는가?

### 보정 판정
AUD07 구조 재작성

### 후속 처리
혼합 언어 요약은 원문 후보로 두고, 내부 운영 기준은 한국어 체크리스트로 재작성한다.

### Obsidian 처리
`지식큐레이터`에는 최종 운영 기준만 남기고, 자동 수집 원문성 문장은 `needs-review` 또는 아카이브 후보로 분리한다.

## 3. 뉴스 트렌드가 매일 같은 운영 기준으로 반복됨

### 품질 신호
뉴스 트렌드 심화 자동 반영 운영 기준이 날짜별로 반복되어 지식업데이트 문서가 길어진다.

### 위험한 처리
매일 기록이므로 모두 동일한 상세 기준을 유지한다.

### 감사 질문
- 반복 섹션이 매일 실제로 다른 내용을 담고 있는가?
- 공통 운영 기준과 날짜별 실행 기록을 분리할 수 있는가?
- Obsidian 그래프에서 중복 노이즈를 만들고 있지 않은가?

### 보정 판정
AUD03 병합, AUD02 링크 보강

### 후속 처리
공통 운영 기준은 하나로 유지하고, 날짜별 실행 기록은 짧은 로그와 링크로 남긴다.

### Obsidian 처리
`뉴스 트렌드 심화 자동 반영 운영 기준` 기준 노트를 만들고, 날짜별 문서는 해당 기준 노트로 링크한다.

## 4. Autodesk 정책 변경 후보가 공식 출처 없이 남아 있음

### 품질 신호
Autodesk Add-in Store 심사 기준 변경 후보가 외부 기사나 커뮤니티 요약만으로 기록되어 있다.

### 위험한 처리
업무에 중요하므로 일단 확정 기준으로 반영한다.

### 감사 질문
- Autodesk 공식 문서 또는 릴리스 노트가 있는가?
- 적용 버전과 날짜가 확인됐는가?
- 내부 배포 SOP에 영향을 주는지 확인됐는가?

### 보정 판정
AUD04 재검증

### 후속 처리
공식 출처 확인 전까지 Watch 상태로 두고, `스토어심사`와 `Revit_Addin`에는 확정 기준이 아니라 변경 후보로 표시한다.

### Obsidian 처리
공식 출처 확인 전에는 `needs-review` 태그를 유지하고, 관련 MOC에는 Watch 후보로만 연결한다.

## 5. 외부 Dynamo 팁이 내부 자동화 후보와 섞임

### 품질 신호
외부 블로그의 Dynamo 팁과 내부 테스트된 자동화 절차가 같은 문서에 섞여 있다.

### 위험한 처리
모두 Dynamo 지식으로 두고 사용한다.

### 감사 질문
- 어떤 내용이 외부 참고이고 어떤 내용이 내부 검증 절차인가?
- 저작권 위험이 있는 원문 복사가 있는가?
- 실제 Revit 템플릿에서 테스트됐는가?

### 보정 판정
AUD07 구조 재작성, AUD08 승격 재분류

### 후속 처리
외부 팁은 참고 후보로 분리하고, 내부 검증된 절차만 표준 자동화 문서로 승격한다.

### Obsidian 처리
`Dynamo` 문서에는 내부 절차만 남기고, 외부 참고는 출처와 검토 상태를 명시한 후보 노트로 연결한다.

## 6. 현장 오류 교훈이 프로젝트 회고에만 갇힘

### 품질 신호
고객 승인 대기 시간이 일정표에 빠졌다는 교훈이 프로젝트 회고에만 남아 있다.

### 위험한 처리
회고 문서에 기록됐으니 완료로 본다.

### 감사 질문
- 견적 산정 기준과 일정표 템플릿에 반영됐는가?
- 고객 안내 문구와 승인 요청 SOP로 연결됐는가?
- 다음 프로젝트에서 재사용 가능한가?

### 보정 판정
AUD08 승격 재분류, AUD02 링크 보강

### 후속 처리
프로젝트 회고에서 운영 SOP, 견적 기준, 고객 안내 문구로 승격한다.

### Obsidian 처리
`프로젝트분석`, `BIM_프로젝트_견적산정`, `고객지원운영`을 상호 링크한다.

## 7. 상품화 후보와 교육자료 후보가 구분되지 않음

### 품질 신호
납품 검수 서비스 관련 지식이 교육자료, 서비스 상품, 자동화 후보에 동시에 흩어져 있다.

### 위험한 처리
관련 지식이 많으니 그대로 둔다.

### 감사 질문
- 고객에게 판매할 서비스 범위는 무엇인가?
- 교육자료로 배포할 설명은 무엇인가?
- 자동화 가능한 반복 항목은 무엇인가?

### 보정 판정
AUD08 승격 재분류, AUD03 병합

### 후속 처리
서비스 정의, 교육자료, 자동화 후보를 별도 섹션으로 나누고 공통 출처와 파일럿 결과를 연결한다.

### Obsidian 처리
`BIM_납품검수`, `제품패키징`, `교육컨설팅`, `Revit_Addin`을 각각 역할별로 링크한다.

## 8. 보안 위험 자료가 유용한 템플릿처럼 보임

### 품질 신호
계약서, 비용, 고객명이 포함된 외부 자료에서 유용한 보고서 구조가 발견됐다.

### 위험한 처리
문서 구조가 좋아 보이므로 익명화 없이 내부 템플릿에 반영한다.

### 감사 질문
- 고객명, 계약, 비용, 내부 경로가 남아 있는가?
- 원문을 그대로 복사했는가?
- 구조만 추출해 완전히 재작성할 수 있는가?

### 보정 판정
AUD06 보안 차단, AUD07 구조 재작성

### 후속 처리
원문 반영을 차단하고, 보안 검토 후 구조만 추출해 내부 표준 템플릿으로 재작성한다.

### Obsidian 처리
민감정보 포함 원문은 링크하지 않는다. `라이선스_보안관`, `법무조항검토`, `지식큐레이터`에 보안검토 기록만 남긴다.

## 오늘의 사고 원칙

1. 지식은 많이 쌓이는 것보다 다시 찾고 믿고 쓸 수 있는 상태가 중요하다.
2. 중복 문서는 삭제보다 기준 문서와 반복 질문 링크로 먼저 정리한다.
3. 오래된 정책, 수치, 법규, 버전 정보는 자동으로 재검증 후보가 된다.
4. Obsidian 링크가 없는 지식은 그래프 안에서 고립되므로 반영 완료가 아니다.
5. 보안 위험이 있는 자료는 유용성보다 차단과 익명화가 먼저다.
