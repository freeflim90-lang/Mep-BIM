# LUA BIM LABS 추론 훈련 루프 세트 11

- 생성일: 2026-06-04
- 목적: 결과만 보이는 상황에서 가능한 원인 후보를 좁히고, 확인 순서를 설계한다.
- 훈련 초점: `결과 → 원인 후보 → 배제 질문 → 확인 순서 → 1차 조치`

## 역추론 코드

- `B01 데이터 변화`: 측정 기준, 범위, 표본, 로그가 바뀌었다.
- `B02 작업 범위 변화`: 모델링 물량, 구역, 공종, 단계가 바뀌었다.
- `B03 프로세스 변화`: 검수, 승인, 협업, 회의 흐름이 바뀌었다.
- `B04 도구/환경 변화`: 템플릿, 애드인, 링크, 좌표, 버전이 바뀌었다.
- `B05 사람/조직 변화`: 담당자, 외주사, 숙련도, 커뮤니케이션이 바뀌었다.
- `B06 고객/계약 변화`: 요구사항, 납품 기준, 우선순위가 바뀌었다.

## 1. 품질 점수가 갑자기 하락

### 결과
모델 품질 점수가 지난주 89점에서 이번 주 62점으로 떨어졌다.

### 원인 후보
- 검사 규칙이 강화됐다.
- 검사 대상 구역이 확대됐다.
- 특정 공종 모델이 새로 링크됐다.
- 템플릿 또는 파라미터 기준이 변경됐다.
- 실제 오류가 급증했다.

### 배제 질문
- 검사 규칙과 대상 카테고리는 지난주와 같은가?
- 신규 링크 모델이나 신규 구역이 포함됐는가?
- 오류가 특정 공종/구역/패밀리에 집중되는가?

### 확인 순서
1. 검사 규칙과 범위 변경 여부
2. 신규 모델/링크/구역 포함 여부
3. 오류 유형별 분포
4. 실제 샘플 수동 확인

### 1차 조치
점수 하락을 품질 악화로 단정하지 않고, 규칙/범위 변화와 실제 오류를 분리한다.

## 2. 일정 지연이 반복됨

### 결과
최근 3개 프로젝트 모두 납품이 3~5일 지연됐다.

### 원인 후보
- 실제 작업량 산정이 낮다.
- 고객 승인 대기 시간이 빠져 있다.
- 외주 재작업이 반복된다.
- 내부 검수가 납품 직전에 몰린다.
- 입력자료가 늦게 확정된다.

### 배제 질문
- 지연 시간 중 실제 작업과 대기 시간 비율은 얼마인가?
- 지연이 특정 단계에서 반복되는가?
- 견적 산정 때 검수/승인/재작업 시간이 포함됐는가?

### 확인 순서
1. 프로젝트별 지연 시간 분해
2. 대기/승인/재작업 시간 분리
3. 견적 산정 기준과 실제 투입 비교
4. 반복 병목 단계 확인

### 1차 조치
인력 추가보다 병목 원인 분해를 먼저 한다.

## 3. 애드인 사용률 감소

### 결과
지난달 70%였던 애드인 사용률이 이번 달 35%로 떨어졌다.

### 원인 후보
- 새 버전에서 오류가 늘었다.
- 프로젝트 유형이 바뀌어 기능 적합도가 낮아졌다.
- 사용자 교육이 끊겼다.
- 실행 시간이 길어졌다.
- 결과를 믿지 못하는 사례가 생겼다.

### 배제 질문
- 버전 업데이트 이후 감소했는가?
- 특정 팀이나 프로젝트에서만 감소했는가?
- 실패 로그나 수동 수정 사례가 늘었는가?

### 확인 순서
1. 버전/배포일과 사용률 변화 비교
2. 팀/프로젝트별 사용률 분리
3. 실패 로그와 사용자 불만 확인
4. 실행 시간과 결과 수정 시간 측정

### 1차 조치
기능 추가 전 신뢰/성능/적합도 문제를 분리한다.

## 4. CS 재문의 증가

### 결과
1차 응답 시간은 빨라졌지만 재문의율이 증가했다.

### 원인 후보
- 답변이 빠르지만 불충분하다.
- 고객 상황 확인 질문이 빠졌다.
- 표준 답변이 문의 유형과 맞지 않는다.
- 실제 해결 권한이 없는 답변을 했다.
- 기술 문제와 일정 문제가 섞였다.

### 배제 질문
- 재문의는 어떤 문의 유형에서 늘었는가?
- 재문의 내용이 추가 정보 요청인가, 불만인가, 같은 문제 반복인가?
- 답변에 다음 액션과 책임자가 있었는가?

### 확인 순서
1. 재문의 유형 분류
2. 원 답변과 재문의 내용 비교
3. 표준 답변 적용 여부 확인
4. 해결 시간과 만족 표현 비교

### 1차 조치
응답 속도 지표보다 해결 품질과 재문의 이유를 먼저 본다.

## 5. 외주 재제출 증가

### 결과
외주 모델 재제출 요청이 한 달 사이 두 배로 늘었다.

### 원인 후보
- 외주사 품질이 낮아졌다.
- 착수 기준 전달이 불명확했다.
- 내부 검수 기준이 강화됐다.
- 프로젝트 난이도가 높아졌다.
- 외주 납기가 과하게 짧았다.

### 배제 질문
- 재제출 사유가 반복되는가?
- 검수 기준이 바뀌었는가?
- 외주사별 차이가 있는가?
- 착수자료에 기준이 명시되어 있었는가?

### 확인 순서
1. 재제출 사유 유형화
2. 외주사별/프로젝트별 분리
3. 검수 기준 변경 여부 확인
4. 착수자료와 결과물 비교

### 1차 조치
외주사 탓으로 단정하지 않고 착수자료, 검수 기준, 난이도를 함께 본다.

## 6. 고객 클레임 감소

### 결과
고객 클레임 수가 40% 줄었다.

### 원인 후보
- 실제 품질이 개선됐다.
- 납품 건수가 줄었다.
- 클레임 기록 채널이 바뀌었다.
- 고객이 공식 클레임 대신 비공식 불만을 말한다.
- 고객 기대치가 낮아졌다.

### 배제 질문
- 납품 건수 대비 클레임 비율도 줄었는가?
- 문의/불만 기록 채널은 동일한가?
- 재작업률과 내부 수정 건수도 줄었는가?

### 확인 순서
1. 납품 건수 대비 클레임율
2. 채널별 기록 비교
3. 재작업률/내부 수정 건수 비교
4. 클레임 심각도 변화 확인

### 1차 조치
클레임 감소를 품질 향상으로 바로 단정하지 않는다.

## 7. 견적 대비 실제 투입 초과

### 결과
커스텀 자동화 프로젝트가 견적 대비 60% 더 많은 시간이 들었다.

### 원인 후보
- 요구사항이 불명확했다.
- 예외 케이스가 많았다.
- 고객 승인 지연이 있었다.
- 유지보수/수정 요청이 개발 범위에 섞였다.
- 기술 부채로 구현 비용이 커졌다.

### 배제 질문
- 초과 시간은 개발, 커뮤니케이션, 재작업, 테스트 중 어디서 발생했는가?
- 변경 요청이 기록되어 있는가?
- 최초 견적에 예외 처리와 테스트가 포함됐는가?

### 확인 순서
1. 초과 시간 유형 분해
2. 변경 요청 이력 확인
3. 요구사항 정의 문서 수준 평가
4. 기술 스파이크 필요 여부 검토

### 1차 조치
다음 견적에 요구사항 정의, 예외 처리, 테스트, 유지보수 범위를 분리한다.

## 8. 교육 후 만족도는 높지만 오류는 그대로

### 결과
교육 만족도는 4.8/5점이지만 관련 오류는 거의 줄지 않았다.

### 원인 후보
- 교육은 이해됐지만 작업 중 적용되지 않았다.
- 교육 내용과 오류 유형이 맞지 않았다.
- 템플릿/도구가 오류를 유발한다.
- 교육 직후 측정 기간이 짧다.
- 참석자와 실제 작업자가 다르다.

### 배제 질문
- 교육 내용과 측정 오류가 직접 연결되는가?
- 참석자가 실제 작업을 수행했는가?
- 작업 중 참고할 체크리스트가 있었는가?

### 확인 순서
1. 교육 내용과 오류 유형 매핑
2. 참석자/작업자 비교
3. 작업 중 참고자료 여부 확인
4. 오류 발생 순간 관찰

### 1차 조치
만족도보다 오류 유형 변화로 교육 효과를 판단한다.

## 오늘의 사고 원칙

1. 결과가 바뀌면 먼저 측정 기준과 범위가 바뀌었는지 본다.
2. 일정 지연은 작업량보다 대기, 승인, 재작업이 원인일 때가 많다.
3. 사용률 감소는 기능 부족보다 신뢰, 적합도, 배포 문제가 먼저일 수 있다.
4. 만족도와 효과는 다르다.
5. 원인 후보를 많이 적는 것보다 배제 순서를 정하는 것이 더 중요하다.
