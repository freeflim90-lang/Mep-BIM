# LUA BIM LABS 추론 훈련 루프 세트 23

- 생성일: 2026-06-04
- 목적: 지식 수집 결과를 검증하고, 업데이트/보류/승격/Obsidian 연결까지 판단한다.
- 훈련 초점: `수집 신호 → 출처 검증 → 중복/보안 확인 → 승격 위치 → Obsidian 반영`

## 지식 처리 코드

- `KB01 needs-review`: 자동 수집됐지만 확정 전 검토가 필요하다.
- `KB02 보류`: 관련성, 신뢰도, 증거가 부족해 당장 반영하지 않는다.
- `KB03 병합`: 기존 지식과 중복되므로 새 문서가 아니라 기존 섹션에 병합한다.
- `KB04 승격`: 출처와 활용성이 충분해 표준 지식으로 반영한다.
- `KB05 QA/체크리스트`: 오류 방지나 검수 항목으로 전환한다.
- `KB06 교육자료`: 반복 질문이나 개념 설명으로 재사용한다.
- `KB07 상품/자동화 후보`: 제품 기능, 서비스 패키지, 자동화 룰로 검토한다.
- `KB08 보안검토`: 개인정보, 고객명, 계약, 내부 경로, 계정/토큰 위험이 있다.

## 1. Telegram 반복 질문 수집

### 수집 신호
팀원이 "인천공항 제1터미널 면적이 얼마야"를 반복 질문했고, 자동 수집 결과에는 관련성이 낮은 Revit 강좌와 BIM 기사도 섞여 있다.

### 위험한 처리
검색 결과를 모두 FAQ에 넣는다.

### 검증 질문
- 질문의 실제 답변에 필요한 공식 출처가 있는가?
- 자동 수집 결과 중 질문과 직접 관련 없는 BIM 일반 자료가 섞였는가?
- 면적 수치가 최신 시설 기준인지 확인됐는가?

### 판정
KB01 needs-review. 공식 출처가 확인될 때까지 확정 FAQ로 승격하지 않는다.

### Obsidian 반영
`지식업데이트`에는 지식 공백 후보로 유지하고, 확정 전 `needs-review` 태그를 유지한다. 공식 면적 출처 확인 후 `Team_Telegram_QA` FAQ 후보로 이동한다.

## 2. Autodesk 공식 문서 변경 감지

### 수집 신호
Autodesk Revit API 또는 Add-in Store 정책 변경 신호가 일일 브리핑에 잡혔다.

### 위험한 처리
기사 요약만 보고 Revit_Addin 지식 베이스를 바로 수정한다.

### 검증 질문
- Autodesk 공식 문서 또는 릴리스 노트가 확인됐는가?
- 변경 날짜와 적용 버전이 명확한가?
- 기존 애드인 배포/심사 기준에 영향을 주는가?

### 판정
공식 문서 확인 전 KB01, 공식 확인 후 KB04 또는 KB05.

### Obsidian 반영
공식 확인 전에는 `스토어심사`와 `Revit_Addin`에 Watch 후보로 연결한다. 공식 확인 후 변경 기준, 적용 버전, 영향받는 내부 SOP를 기록한다.

## 3. 외부 블로그의 Dynamo 자동화 팁

### 수집 신호
외부 블로그에서 Dynamo로 파라미터를 일괄 정리하는 방법이 소개됐다.

### 위험한 처리
코드와 노드 흐름을 그대로 복사해 내부 자동화 자료로 넣는다.

### 검증 질문
- 출처가 신뢰 가능한가?
- 저작권 또는 원문 복제 위험이 있는가?
- 내부 프로젝트 템플릿과 호환되는가?
- 실제 테스트 결과가 있는가?

### 판정
테스트 전 KB02 보류. 구조만 참고해 내부 절차로 재작성 가능하면 KB06 또는 KB07 후보.

### Obsidian 반영
원문 복사 없이 `Dynamo` 지식 베이스에 참고 후보로 기록한다. 테스트 완료 후 내부 노드 흐름과 실패 조건을 자체 문서로 작성한다.

## 4. 현장 오류 반복 사례 수집

### 수집 신호
최근 3개 프로젝트에서 IFC 속성 누락이 반복됐다.

### 위험한 처리
IFC 교육자료에만 추가한다.

### 검증 질문
- 누락 속성이 발주처 공통인지 특정 발주처 기준인지 구분됐는가?
- 납품 반려로 이어졌는가?
- 자동 점검 가능한 항목과 수동 확인 항목이 분리됐는가?

### 판정
KB05 QA/체크리스트. 치명도와 반복성이 높으므로 납품 전 검수 항목으로 승격한다.

### Obsidian 반영
`IFC_OpenBIM`, `BIM_납품검수`, `EIRBEP_심사원`에 연결하고, 발주처별 확장 체크리스트 후보로 기록한다.

## 5. 고객 문의에서 서비스 상품화 신호 발견

### 수집 신호
고객들이 "납품 전 검수 리포트가 필요하다"고 반복 문의한다.

### 위험한 처리
바로 SaaS 기능 후보로 등록한다.

### 검증 질문
- 고객이 도구를 원하는가, 책임 있는 리포트를 원하는가?
- 유료 파일럿 수락 의사가 있는가?
- 반복 검수 항목이 정량 규칙으로 분리되는가?

### 판정
1차는 KB07 상품/자동화 후보 중 `서비스 후보`. SaaS는 반복 항목 확인 후 별도 후보로 분리한다.

### Obsidian 반영
`BIM_납품검수`, `제품패키징`, `고객지원운영`에 서비스 후보로 연결한다. 자동화 가능 항목은 파일럿 3건 후 분리한다.

## 6. 뉴스 브리핑의 BIM 시장 트렌드

### 수집 신호
일일 산업 브리핑에서 OpenBIM/IFC/IDS 관련 뉴스가 반복 노출됐다.

### 위험한 처리
뉴스 제목만으로 전략 문서에 확정 트렌드로 기록한다.

### 검증 질문
- 공식 표준 단체, 발주처, 주요 벤더 출처가 있는가?
- 반복 노출이 같은 보도자료 재배포인지, 독립 신호인지 확인됐는가?
- LUA BIM LABS의 상품/교육/QA와 연결되는가?

### 판정
독립 출처가 부족하면 KB02 보류 또는 Watch. 공식·전문 출처가 확인되면 KB04 승격.

### Obsidian 반영
Watch 상태는 `산업동향_데일리브리핑`에 유지한다. 승격 시 `IFC_OpenBIM`, `건설시장_트렌드`, `AX_전략승격리뷰`에 위키링크를 연결한다.

## 7. 내부 프로젝트 종료 후 교훈 수집

### 수집 신호
프로젝트 종료 회고에서 "고객 승인 대기 시간이 일정표에 빠졌다"는 교훈이 나왔다.

### 위험한 처리
프로젝트 회고에만 남긴다.

### 검증 질문
- 다른 프로젝트에서도 반복되는가?
- 견적 산정과 일정표 템플릿에 반영해야 하는가?
- 고객 안내 문구와 승인 요청 SOP가 필요한가?

### 판정
KB04 승격 및 KB05 체크리스트 후보. 반복 가능성이 높으므로 운영 지식으로 전환한다.

### Obsidian 반영
`프로젝트분석`, `BIM_프로젝트_견적산정`, `고객지원운영`에 연결한다. 일정표 템플릿에는 승인 대기 항목을 추가한다.

## 8. 민감정보가 포함된 자료 후보

### 수집 신호
외장하드 스캔에서 프로젝트 계약서, 고객명, 비용 정보가 포함된 BIM 자료가 발견됐다.

### 위험한 처리
유용해 보이는 문서 구조를 바로 지식 베이스에 반영한다.

### 검증 질문
- 개인정보, 고객명, 계약, 비용, 내부 경로가 포함됐는가?
- 원문을 열람하거나 복사하지 않고 메타데이터로만 후보 분류했는가?
- 익명화 후 구조만 추출 가능한가?

### 판정
KB08 보안검토. 승인 전 지식 베이스 반영 금지.

### Obsidian 반영
원문 반영 금지. `지식업데이트`에는 보안검토 후보로만 기록하고, `라이선스_보안관`, `법무조항검토`, `지식큐레이터`에 연결한다.

## 오늘의 사고 원칙

1. 자동 수집 결과는 확정 지식이 아니라 검토 후보로 시작한다.
2. 공식·전문 출처가 확인되지 않은 수치와 정책은 확정 FAQ로 승격하지 않는다.
3. 원문 복사는 지식화가 아니다. 구조, 기준, 판단 흐름을 내부 문서로 재작성한다.
4. Obsidian 반영은 문서 추가뿐 아니라 관련 지식 2개 이상과 연결될 때 완성된다.
5. 보안 위험이 있는 자료는 유용해 보여도 지식화보다 차단과 익명화가 먼저다.
