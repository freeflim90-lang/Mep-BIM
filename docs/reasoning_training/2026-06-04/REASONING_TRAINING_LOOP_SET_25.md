# LUA BIM LABS 추론 훈련 루프 세트 25

- 생성일: 2026-06-04
- 목적: 지식 품질 감사 결과를 실행 가능한 개선 티켓으로 전환하고, 재완료 기준과 증빙을 명확히 한다.
- 훈련 초점: `감사 코드 → 문제 범위 → 개선 티켓 → 완료 기준 → 검증 증거`

## 개선 티켓 코드

- `FIX01 링크 보강`: 관련 위키링크, 역링크, MOC 연결을 추가한다.
- `FIX02 중복 병합`: 대표 기준 문서를 정하고 반복 문서는 링크로 정리한다.
- `FIX03 출처 재검증`: 공식·전문 출처, 날짜, 적용 버전을 확인한다.
- `FIX04 구조 재작성`: 자동 수집 원문을 내부 실행 기준으로 재작성한다.
- `FIX05 보안 정리`: 민감정보를 제거하고 보안검토 기록만 남긴다.
- `FIX06 승격 이동`: QA, 교육자료, SOP, 상품 후보 등 적합한 위치로 옮긴다.
- `FIX07 기준 분리`: 공통 운영 기준과 날짜별 로그를 분리한다.
- `FIX08 완료 증빙`: 반영 파일, 링크, 샘플 적용, 다음 검토일을 기록한다.

## 1. 반복 Telegram QA 병합 티켓

### 감사 결과
같은 질문이 여러 QA 문서에 반복되고, 일부 답변은 출처가 다르다.

### 위험한 티켓
중복 QA 정리.

### 좋은 티켓
인천공항 터미널 면적 관련 반복 QA를 대표 FAQ 1개로 병합하고, 나머지 문서는 대표 FAQ로 링크한다. 공식 출처가 없는 수치는 needs-review로 유지한다.

### 적용 코드
FIX02 중복 병합, FIX03 출처 재검증, FIX08 완료 증빙

### 완료 기준
- 대표 FAQ 1개 지정
- 반복 QA 문서 2개 이상에서 대표 FAQ 링크 확인
- 공식 출처 또는 needs-review 상태 명시
- 다음 재검증일 기록

## 2. 혼합 언어 자동 수집 섹션 재작성 티켓

### 감사 결과
지식큐레이터 문서에 중국어·한국어 혼합 요약이 남아 있어 운영 기준으로 바로 쓰기 어렵다.

### 위험한 티켓
문장 다듬기.

### 좋은 티켓
혼합 언어 자동 수집 섹션을 needs-review 후보로 표시하고, 현업용 한국어 운영 체크리스트를 별도 섹션으로 재작성한다.

### 적용 코드
FIX04 구조 재작성, FIX08 완료 증빙

### 완료 기준
- 자동 수집 원문성 문장과 내부 운영 기준 분리
- 한국어 체크리스트 5개 이상 작성
- 출처 검증 필요 여부 기록
- 관련 문서 2개 이상 링크

## 3. 뉴스 트렌드 반복 기준 분리 티켓

### 감사 결과
지식업데이트 문서에 날짜별 뉴스 트렌드 운영 기준이 거의 동일하게 반복된다.

### 위험한 티켓
중복 줄이기.

### 좋은 티켓
뉴스 트렌드 심화 자동 반영 공통 기준을 기준 노트 1개로 분리하고, 날짜별 기록은 실행 로그와 링크만 남긴다.

### 적용 코드
FIX07 기준 분리, FIX01 링크 보강, FIX08 완료 증빙

### 완료 기준
- 공통 기준 노트 생성
- 날짜별 로그가 공통 기준으로 링크
- 2026-06-04 이후 기록부터 반복 상세 기준 미삽입
- Obsidian MOC 또는 관련 지식에 연결

## 4. Autodesk 정책 후보 재검증 티켓

### 감사 결과
Autodesk Add-in Store 심사 기준 변경 후보가 공식 출처 없이 Watch 상태로 남아 있다.

### 위험한 티켓
Autodesk 정책 확인.

### 좋은 티켓
Autodesk 공식 문서 또는 릴리스 노트를 확인해 적용 날짜와 버전을 기록하고, 확인 전까지 내부 SOP에는 확정 기준이 아닌 Watch 후보로 표시한다.

### 적용 코드
FIX03 출처 재검증, FIX08 완료 증빙

### 완료 기준
- 공식 출처 링크 또는 미확인 사유 기록
- 적용 날짜와 대상 버전 기록
- 내부 배포 SOP 영향 여부 판단
- 다음 확인일 기록

## 5. 외부 Dynamo 팁 정리 티켓

### 감사 결과

### 좋은 티켓

### 적용 코드
FIX04 구조 재작성, FIX06 승격 이동, FIX08 완료 증빙

### 완료 기준
- 외부 참고와 내부 검증 절차 구분
- 원문 복사 여부 점검
- Revit 템플릿 샘플 테스트 결과 기록
- Dynamo 표준 문서와 후보 노트 상호 링크

납품 검수 문서 정리.

### 좋은 티켓
납품 검수 지식을 서비스 정의, 교육자료, 자동화 후보로 분리하고, 공통 출처와 파일럿 결과를 연결한다.

- 서비스 정의 섹션 작성
- 교육자료 섹션 작성
- 자동화 후보 섹션 작성
- 파일럿 결과 또는 needs-pilot 상태 기록

## 8. 보안 위험 자료 차단 티켓

### 감사 결과
고객명, 계약, 비용이 포함된 자료가 유용한 템플릿처럼 보인다.

### 위험한 티켓
좋은 구조만 가져오기.

### 좋은 티켓
민감정보 포함 원문은 링크와 반영을 차단하고, 보안검토 기록 후 구조만 완전히 재작성해 내부 템플릿 후보로 만든다.

### 적용 코드
FIX05 보안 정리, FIX04 구조 재작성, FIX08 완료 증빙

### 완료 기준
- 원문 링크 미삽입 확인
- 고객명, 계약, 비용, 내부 경로 제거 확인
- 보안검토 기록 작성
- 구조 재작성본과 원문 미연결 상태 확인

## 오늘의 사고 원칙

1. 감사 결과는 기록만으로 끝나지 않고 개선 티켓으로 전환되어야 한다.
2. 좋은 티켓은 무엇을 고칠지보다 완료 증거가 무엇인지 먼저 보이게 한다.
3. 중복, 출처, 보안, 링크 문제는 각각 다른 완료 기준을 가진다.
4. 재작성 티켓은 원문 보존이 아니라 내부 실행 기준 생성이 목표다.
5. 완료 기준이 없는 개선 티켓은 다음 감사 때 같은 문제로 돌아온다.
