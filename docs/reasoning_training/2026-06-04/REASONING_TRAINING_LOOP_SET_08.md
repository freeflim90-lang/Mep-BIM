# LUA BIM LABS 추론 훈련 루프 세트 08

- 생성일: 2026-06-04
- 목적: 대표/실무자 첨언을 잘 받기 위한 질문을 설계한다.
- 훈련 초점: `애매한 신호 → 나쁜 질문 → 좋은 질문 → 기대 답변 → 판정 변화`

## 질문 품질 코드

- `Q01 범위 확인`: 어디까지가 대상인지 묻는다.
- `Q02 예외 확인`: 규칙이 깨지는 상황을 묻는다.
- `Q03 우선순위 확인`: 지금 처리해야 하는지 묻는다.
- `Q04 책임 확인`: 누가 결정하고 승인하는지 묻는다.
- `Q05 증거 확인`: 어떤 데이터가 있으면 판단 가능한지 묻는다.
- `Q06 고객 언어 확인`: 고객에게 어떻게 설명할지 묻는다.

## 1. 간섭 증가 신호 첨언 질문

### 애매한 신호
간섭 건수가 갑자기 늘었다.

### 나쁜 질문
이거 자동화하면 될까요?

### 좋은 질문
최근 프로젝트에서 간섭 증가가 실제 조율 이슈로 이어진 경우는 어떤 조건이 있었나요? 특정 공종, 구역, 단계, 반복 간섭 비율 중 무엇을 먼저 봐야 하나요?

### 질문 품질 코드
Q02 예외 확인, Q05 증거 확인

### 기대 답변
핵심 구역과 반복 간섭을 먼저 봐야 하는지, 단순 물량 증가를 제외해야 하는지 확인할 수 있다.

### 판정 변화
자동화 후보에서 조건부 체크리스트 후보로 보정될 수 있다.

## 2. 뷰 템플릿 변경 첨언 질문

### 애매한 신호
납품 전 시트 표현이 깨졌다.

### 나쁜 질문
템플릿 잠그면 되나요?

### 좋은 질문
납품 전 반드시 보호해야 하는 핵심 뷰 템플릿은 어떤 기준으로 정하나요? 전체 템플릿을 관리하는 것과 납품 직전 핵심 템플릿만 관리하는 것 중 현장 부담은 어느 쪽이 현실적인가요?

### 질문 품질 코드
Q01 범위 확인, Q03 우선순위 확인

### 기대 답변
핵심 템플릿의 범위와 관리 기간을 정할 수 있다.

### 판정 변화
전면 SOP가 아니라 납품 전 제한 SOP로 보정될 수 있다.

## 3. 자동 태깅 도구 사용률 첨언 질문

### 애매한 신호
자동 태깅 도구 사용률이 낮다.

### 나쁜 질문
기능을 더 만들까요?

### 좋은 질문
사용자가 도구를 안 쓰는 가장 큰 이유가 결과를 못 믿어서인지, 실행 절차가 번거로워서인지, 실패 후 복구가 불안해서인지 구분할 수 있는 사례가 있나요?

### 질문 품질 코드
Q05 증거 확인, Q02 예외 확인

### 기대 답변
기능 추가, 로그 개선, 교육 중 무엇이 먼저인지 판단할 수 있다.

### 판정 변화
기능 개발 후보에서 로그/되돌리기/교육 보강으로 수정될 수 있다.

## 4. 납품 검수 서비스 첨언 질문

### 애매한 신호
고객이 납품 검수가 어렵다고 한다.

### 나쁜 질문
SaaS 만들면 팔릴까요?

### 좋은 질문
고객이 실제로 돈을 낼 문제는 검수 자동화 도구인가요, 납품 전 책임 있는 리포트인가요, 발주처 기준 해석인가요? 과거에 고객이 가장 두려워했던 검수 실패는 무엇이었나요?

### 질문 품질 코드
Q06 고객 언어 확인, Q05 증거 확인

### 기대 답변
제품 수요와 서비스 수요를 분리할 수 있다.

### 판정 변화
SaaS 후보에서 납품 전 검수 서비스 후보로 보정될 수 있다.

## 5. IFC 체크리스트 첨언 질문

### 애매한 신호
IFC 오류가 드물지만 반려 리스크가 크다.

### 나쁜 질문
IFC 체크리스트 만들까요?

### 좋은 질문
반려로 이어진 IFC 오류 중 발주처 공통 항목과 발주처별 항목은 어떻게 나뉘나요? 내보내기 전 자동 확인 가능한 항목과 사람이 확인해야 하는 항목은 무엇인가요?

### 질문 품질 코드
Q01 범위 확인, Q05 증거 확인

### 기대 답변
공통 체크리스트와 발주처별 체크리스트를 분리할 수 있다.

### 판정 변화
일반 체크리스트에서 발주처별 SOP로 승격될 수 있다.

## 6. 외주 모델 검수 첨언 질문

### 애매한 신호
외주 모델에서 경미 오류가 반복된다.

### 나쁜 질문
외주사 교육시키면 될까요?

### 좋은 질문
이 오류가 외주사 역량 문제인지, 착수 기준 전달 문제인지, 접수 검수 기준 문제인지 구분하려면 어떤 기록을 봐야 하나요?

### 질문 품질 코드
Q05 증거 확인, Q04 책임 확인

### 기대 답변
교육, 착수자료 개선, 접수 검수 중 우선 액션을 정할 수 있다.

### 판정 변화
외주사 탓에서 외주 착수/접수 SOP 개선으로 바뀔 수 있다.

## 7. CS 표준 답변 첨언 질문

고객이 불만을 느낀 지점이 문체인지, 상황 이해 부족인지, 다음 액션 부재인지, 책임 범위 불명확성인지 구분할 사례가 있나요?

### 질문 품질 코드

## 8. 요구사항 정의 첨언 질문

유료 요구사항 정의는 재작업을 줄였지만 영업 속도를 늦춘다.

### 좋은 질문

### 질문 품질 코드
Q01 범위 확인, Q03 우선순위 확인

## 9. 교육 효과 첨언 질문

### 애매한 신호
교육 참석자의 오류는 줄었지만 미참석자 오류는 그대로다.

### 나쁜 질문
교육을 더 많이 할까요?

### 좋은 질문
이 지식은 강의로 전달해야 하나요, 작업 중 체크리스트로 보여줘야 하나요, 템플릿/패밀리 설명에 넣어야 하나요? 현업에서 오류가 발생하는 순간은 언제인가요?

### 질문 품질 코드
Q02 예외 확인, Q03 우선순위 확인

### 기대 답변
교육자료, 온보딩, 작업 체크리스트 중 적절한 확산 방식을 고를 수 있다.

### 판정 변화
반복 교육에서 온보딩/작업 중 체크리스트로 보정될 수 있다.

## 10. 기능 요청 우선순위 첨언 질문

### 애매한 신호
고객 가치 점수가 높은 기능을 개발자가 구조 리스크 때문에 반대한다.

### 나쁜 질문
그래도 개발할까요?

### 좋은 질문
고객 문제를 더 작은 기능, 반자동 방식, 기존 구조를 유지하는 우회 방식으로 해결할 수 있나요? 구조 변경 리스크를 감수해야 할 만큼 반복성과 매출 근거가 충분한가요?

### 질문 품질 코드
Q02 예외 확인, Q05 증거 확인

### 기대 답변
기술 스파이크, 반자동 MVP, 전면 개발 중 선택할 수 있다.

### 판정 변화
즉시 개발에서 기술 스파이크 또는 반자동 MVP로 수정될 수 있다.

## 오늘의 사고 원칙

1. 좋은 질문은 찬반을 묻지 않고 판단 기준을 끌어낸다.
2. "할까요?"보다 "어떤 조건에서 해야 하나요?"가 더 좋은 질문이다.
3. 첨언 질문은 대표의 시간을 줄이고, 추론의 예외 조건을 늘려야 한다.
4. 고객 언어를 묻는 질문은 제품화와 서비스화를 가르는 데 중요하다.
5. 질문이 좋아지면 침묵도 더 잘 해석할 수 있다.
