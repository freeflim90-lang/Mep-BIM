## 후보 1 — AX 전략 신호 분류 (시공 BIM 16, AX 전략 15, OpenBIM/납품검증 14, MQA 7)

### Q. 재현 조건, 영향 범위, 임시 조치 중 무엇을 먼저?
**영향 범위 먼저.**  
"시공 BIM 신호 16건"은 수치이지 문제가 아니다. 이 신호들이 실제 로드맵에 연결되어 있는가, 아니면 축적만 되고 있는가를 확인해야 한다. AX 전략 승격 기준(반복성 3회 이상, 긴급성, 시장 규모, 내부 역량 적합성)을 통과한 신호가 몇 건인지 세는 것이 영향 범위다.

### Q. 사람이 반복 판단하는 부분 → 스크립트 후보
- "이 신호가 어느 전략 카테고리인가" 판단 → 키워드 기반 자동 태깅 스크립트 (이미 `SIGNAL_KEYWORDS` 딕셔너리가 있음 — reasoning_training_digest.py 참고)
- "이 신호가 승격 기준을 통과하는가" 판단 → 승격 기준 4가지를 점수화한 자동 스코어링 스크립트 (90점 이상 즉시 Sprint 편입)
- 주간 리뷰 보고서 생성 → 카테고리별 신호 수 + 신규 승격 후보 자동 요약

### Q. 납품 전 품질 기준
AX 전략 주간 리뷰 산출물 합격 조건:
- [ ] 모든 신호에 카테고리 태그 1개 이상
- [ ] 반복성 3회 이상 신호에 승격 스코어 계산 완료
- [ ] 90점 이상 신호는 Sprint 이슈 또는 로드맵 연결 완료
- [ ] 낙하 신호(`Drop/Watch`) 처리 근거 기록

### Q. 고객 설명 언어
**시간 절감**이 1차.  
전략 리뷰를 수동으로 하면 매주 2~3시간이다. "신호 자동 분류와 스코어링으로 주간 전략 리뷰 시간을 70% 단축한다"는 내부 효율화 메시지.

## 후보 2 — Revit Add-in 신규 모듈: MEP 자동 간섭 해소 + LOD 자동 검증 → App Store

### Q. 재현 조건, 영향 범위, 임시 조치 중 무엇을 먼저?
**재현 조건 먼저.**  
"MEP 자동 간섭 해소"는 모든 간섭을 자동으로 푼다는 의미가 아니다. 현재 프로젝트에서 수동으로 반복 처리하는 간섭 유형이 무엇인지(예: 덕트-파이프 교차, 구조체 관통)를 먼저 정의해야 자동화 범위가 결정된다.

### Q. 사람이 반복 판단하는 부분 → Add-in 후보
- "이 간섭이 자동 해소 가능한 유형인가" 판단 → 간섭 유형 분류 룰셋 (Hard/허용/자동해소 가능)
- "LOD가 기준을 충족하는가" 판단 → 파라미터 존재 여부 + 값 범위 자동 검증
- App Store 출시 전 "이 기능이 다른 버전 Revit과 호환되는가" → 버전 호환성 자동 테스트 스크립트

### Q. 납품 전 품질 기준
- [ ] LOD 300 이상: 필수 파라미터 100% 채움
- [ ] MEP 자동 해소 대상 유형: 명시된 룰셋 외 간섭 수동 검토 완료
- [ ] Hard Clash: 0건
- [ ] App Store 출시 전: Revit 2024/2025 양버전 빌드 통과

### Q. 고객 설명 언어
**품질 안정**과 **시간 절감** 둘 다 사용한다.  
LOD 검증은 품질 안정("납품 기준 자동 충족 확인"), MEP 간섭 해소는 시간 절감("간섭 처리 시간 50% 단축")으로 나눠 설명한다.

## 후보 3 — Python → Dynamo → Revit API 3단계 커리큘럼

### Q. 지금 당장 확인할 근거는 무엇인가?
두 가지:
1. **기존 Starter Plan과 중복 여부**: 현재 Starter Plan 커리큘럼이 이 3단계와 얼마나 겹치는가 — 중복이 크면 통합, 아니면 고급 과정으로 포지셔닝
2. **수강 대상 명확화**: Python 업무 자동화 단계 수강자가 실제로 Dynamo → Revit API까지 진행할 의향이 있는지 기존 수강자/문의자 데이터 확인

### Q. 사람이 반복 판단하는 부분 → 자동화 후보
- "이 수강자가 다음 단계로 올라갈 준비가 됐는가" 판단 → 단계별 완료 과제 자동 채점 + 수료 기준 충족 시 다음 단계 자동 안내
- 커리큘럼 자료 배포 → 구글 드라이브 단계별 폴더 자동 권한 부여 스크립트

### Q. 고객 설명 언어
**시간 절감**이 핵심.  
"3단계 자기주도 커리큘럼으로 기존 외부 교육 의존 없이 팀 내 Revit API 개발자를 6개월 안에 육성한다"는 프레임. 온보딩 교육 비용과 시간 모두 절감.

## 추론 결론

### 실무 첨언에서 업데이트된 판단
- AX 전략 신호 분류 자동화는 이미 `SIGNAL_KEYWORDS` 딕셔너리 기반이 있으므로, 승격 스코어링 로직 추가만으로 주간 리뷰 자동화를 크게 높일 수 있다.
- MEP 자동 간섭 해소 Add-in은 "모든 간섭 자동 해소"가 아닌 "반복 유형 자동 분류 + 해소 가능 유형만 자동 처리"로 스코프를 제한해야 현업 저항이 줄어든다.
- 3단계 커리큘럼은 Starter Plan과 범위 교통정리 후에 포지셔닝해야 내부 혼선이 없다.

### 앞으로의 사고 방식
- 전략 신호 분류 스크립트가 이미 있을 때, 새 기능 추가 전 기존 코드를 먼저 확인한다.
- 자동화 범위는 "모든 케이스 자동화"가 아닌 "반복 판단 유형만 자동화, 예외는 사람"으로 명시한다.
- 새 커리큘럼을 기획할 때 기존 상품(Starter Plan)과의 연결 또는 차별화 포인트를 먼저 정의한다.
