## 1. 목적

본 문서는 CDE, 이메일, 클라우드, 소프트웨어, 내부 문서 저장소의 계정과 접근권한을 최소 권한 원칙에 따라 관리하기 위한 기준이다.

## 2. 권한 원칙

| 원칙 | 설명 |
|---|---|
| 개인 계정 | 공유 계정 사용 금지 |
| 최소 권한 | 업무에 필요한 범위만 부여 |
| 기간 제한 | 프로젝트/업무 종료 시 회수 |
| 2FA | 가능한 모든 중요 계정에 적용 |
| 기록 | 발급/변경/회수 이력 보관 |

## 3. 계정 발급 절차

| 단계 | 내용 |
|---|---|
| 요청 | 역할, 프로젝트, 필요 권한 작성 |
| 검토 | PM/경영지원/보안 확인 |
| 발급 | 개인 계정 생성 |
| 확인 | 접속 테스트 및 보안 안내 |
| 기록 | 계정대장 업데이트 |

## 4. 퇴사/종료 시 회수

| 항목 | 조치 |
|---|---|
| 이메일/클라우드 | 비활성화 또는 권한 제거 |
| CDE | 프로젝트 접근 권한 회수 |
| 소프트웨어 | 라이선스 회수 |
| 장비 | 반납 및 상태 점검 |
| 자료 | 고객 자료 삭제/반납 확인 |
