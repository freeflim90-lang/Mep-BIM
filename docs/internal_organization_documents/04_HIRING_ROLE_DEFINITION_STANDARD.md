## 5. 이력서 분석 대시보드 내재화 기준

| 항목 | 기준 |
|---|---|
| 담당 AI | HR_인재분석관 |
| 실행 기능 | 이력서분석_AI |

| 책임 부서 | 경영지원 |
| 협업 담당 | 교육컨설팅, BIM_템플릿기획관, 라이선스_보안관, CEO |
| 기본 배치 | 로컬 전용, 외부 AI API 직접 호출 금지 |

`HR_인재분석관`은 PDF 이력서를 업로드받아 경력, 프로젝트, 역할, 기술, 학력, 해외 경험을 구조화하고 채용 검토용 대시보드와 1페이지 인재 평가 보고서 초안을 만든다.

대시보드의 기본 분석 항목은 다음으로 제한한다.

| 분석 항목 | 산출물 |
|---|---|
| 후보자 개요 | 성명, 현재 직책, 총 경력, 핵심 전문 분야 |
| 경력 타임라인 | 회사, 직책, 기간, 주요 프로젝트 |
| 프로젝트 분석 | 산업/카테고리별 프로젝트 수, 연도별 추이, 역할 분포 |
| 기술 역량 | 기술명, 숙련도, BIM/MEP 관련성 |
| 인재 평가 보고서 | S/A/B 등급 초안, 핵심 역량 점수, 주요 실적, 강점, 보완 영역, 추천 배치안 |

브라우저 프론트엔드에서 `https://api.anthropic.com/v1/messages` 등 외부 AI API를 직접 호출하지 않는다. 이력서 원본, 생년, 연락처, 학력, 이전 회사/고객명, 프로젝트명은 개인정보와 민감 경력정보로 보고 로컬 처리 또는 사내 백엔드 프록시를 통해서만 다룬다.

AI가 생성한 등급과 점수는 1차 검토 보조 자료이며 채용 확정, 급여 조건, 수습 전환 여부를 단독 결정하지 않는다. 최종 판단은 대표/담당 리드와 경영지원이 채용 기준, 면접, 과제, 레퍼런스 확인을 종합하여 확정한다.

## 5.1 Telegram 이력서 파일 처리 기준

Telegram으로 PDF 이력서를 보내는 경우 파일명 또는 캡션에 `이력서`, `resume`, `cv`, `지원자`, `채용` 중 하나를 포함해야 한다. 봇은 원본 PDF를 로컬 `data/resume_intake/telegram/` 하위에 내려받고, 설치된 로컬 추출기(`pypdf`, `PyPDF2`, `pdfplumber`, `pdftotext`) 중 가능한 도구로 텍스트를 추출한다.

로컬 추출이 성공하면 `HR_인재분석관`이 규칙 기반 1차 분석을 수행하고 Telegram으로 분석 초안을 회신한다. 로컬 추출기가 없거나 스캔본이라 OCR이 필요한 경우에는 자동 분석을 중단하고 로컬 추출기/OCR 설치를 요청한다.

DeepSeek API 보조 분석은 기본 비활성이다. 꼭 필요한 경우에만 `RESUME_DEEPSEEK_FALLBACK_ENABLED=true`, `PAID_AI_ENABLED=true`가 모두 켜져 있고, 로컬에서 텍스트 추출과 개인정보 마스킹이 끝난 텍스트에 한해 사용한다. 원본 PDF는 DeepSeek로 전송하지 않는다.
