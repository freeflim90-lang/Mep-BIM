## 1. 목적

본 문서는 재직 중인 회사의 관리팀 업무에서 Excel 자동화 요청이 발생했을 때, DeepSeek API 없이 Qwen_Coder_8B와 로컬 지식만 사용하여 자동화 초안을 처리하는 기준을 정의한다.

핵심 원칙은 `DeepSeek 금지, Qwen 허용, Mac 로컬 결과물 저장 금지, Telegram 회신`이다.

## 2. 자동 실행 대상

다음 키워드가 포함된 개발 요청은 관리팀 Excel 자동화로 분류하고 별도 승인 없이 Qwen 기반 자동화 검토를 시작한다.

| 분류 | 키워드 |
|---|---|
| Excel 파일 | 엑셀, Excel, xlsx, csv, 워크북, 시트, 피벗, openpyxl |
| 관리팀 업무 | 관리팀, 정산, 근태, 휴가, 비용, 구매, 자산, 라이선스, 월간 보고서 |
| 자동화 의도 | 자동화, 개발, 생성, 정리, 취합, 검토, 내보내기 |

## 3. 운영 흐름

| 단계 | 처리 | 담당 |
|---:|---|---|
| 1 | 팀원이 Telegram `🧩 개발`로 Excel 자동화 요청 | 팀원 |
| 2 | 시스템이 관리팀 Excel 자동화인지 판정 | 조율차장 |
| 3 | DeepSeek API 사용 금지 상태로 로컬/Qwen 파이프라인 실행 | 엑셀자동화 |
| 4 | 샘플 데이터, 출력 스키마, 검증 로그 기준으로 개발 초안 작성 | Qwen_Coder_8B |
| 5 | 개인정보/급여/계약/계정정보는 답변에서 마스킹 또는 제외 | 라이선스_보안관 |
| 6 | 최종 결과를 Telegram으로 질문자에게 회신 | 조율차장 |

## 4. 회신 항목

- 요청자
- 원 요청 내용
- 예상 입력 파일 또는 열 구조
- 예상 출력물
- 개발 필요 여부
- 개인정보/급여/계약정보/외부전송/매크로 리스크
- Python/openpyxl 기반 구현 초안 또는 처리 절차
- 검증 체크리스트
- Telegram으로 전달 가능한 최종 요약

## 5. API 및 저장 제한

- 팀원과 관리팀 요청에는 DeepSeek API를 사용하지 않는다.
- Qwen_Coder_8B와 로컬 지식 기반 개발은 허용한다.
- 최종 결과물은 Mac 컴퓨터에 파일로 저장하지 않는다.
- 결과는 Telegram 메시지로 질문자에게 전달한다.
- 실제 회사 파일을 처리해야 할 경우, 파일 원본을 저장하지 않고 익명화 샘플 구조를 기준으로 설명한다.

## 7. 실행 제한

- 실제 회사 파일을 직접 수정하거나 Mac에 저장하지 않는다.
- 개인정보, 급여, 계약, 인사평가, 계정정보는 답변에 그대로 사용하지 않는다.
- VBA/매크로 방식은 보안상 기본 제외하고, Python/openpyxl 기반 자동화를 우선한다.
- 자동화 결과는 Telegram 회신 가능한 코드, 절차, 검증표, 열 구조 중심으로 제공한다.
