## 1. 목적

본 문서는 LUA BIM LABS가 수집한 지식을 모든 조직원의 역할과 학습 경로에 맞게 배분하고, 회사의 판단과 응답이 지식 근거에 기반하도록 만드는 운영 기준을 정의한다.

목표는 지식을 많이 쌓는 것이 아니라 다음 상태를 만드는 것이다.

- 조직원이 자기 역할에 필요한 지식을 매주 학습한다.
- 답변과 판단은 출처, 적용 범위, 불확실성, 다음 확인 조건을 함께 말한다.
- 자동 수집 지식은 검증 전 확정 지식처럼 쓰지 않는다.
- 프로젝트 실패, 고객 질문, 법규 변경, 제품 아이디어가 교육과 표준으로 되돌아간다.
- Obsidian은 문서 창고가 아니라 조직 사고의 지도 역할을 한다.

## 2. 조직 지식 원칙

| 원칙 | 기준 |
|---|---|
| 근거 우선 | 법령, 공식 문서, BEP/EIR, 발주처 문서, 검증된 내부 경험 순서로 근거를 확인한다. |
| 역할 맞춤 | 같은 지식도 CEO, PM, BIM 모델러, QA, CS, 교육 담당에게 다른 학습 과제로 배분한다. |
| 확정/주의 분리 | 검증된 기준, needs-review, 자동 수집, 가설, 전략 메모를 명확히 구분한다. |
| 응답 품질 | 고객/내부 응답에는 근거, 적용 범위, 리스크, 다음 액션을 포함한다. |
| 반복 학습 | 반복 질문과 실패는 FAQ, 교육자료, QA 체크리스트, 표준문서 후보로 승격한다. |
| Obsidian 연결 | 모든 승격 지식은 MOC, 역할, 공종, 제품, 교육, 검수 문서 중 2개 이상과 연결한다. |

| 코드 | 의미 | 사용 기준 |
|---|---|---|

| `KST03 적용주의` | 공식 근거는 있으나 프로젝트 조건 확인 필요 | 법규 적용 대상, 지자체 조례, 발주처 특기시방 |
| `KST04 자동수집` | 자동 수집 또는 AI 초안 | 확정 기준으로 사용 금지 |
| `KST05 가설전략` | 전략/제품/시장 가설 | 실험 조건과 검증 지표 필요 |
| `KST06 보안제한` | 민감정보 포함 또는 공개 제한 | 외부 공개 금지, 접근권한 필요 |

## 6. 근거기반 응답 규칙

내부/고객 응답은 다음 5칸을 기준으로 작성한다.

| 칸 | 질문 |
|---|---|
| 결론 | 지금 무엇을 판단하거나 안내하는가 |
| 근거 | 어떤 문서, 법령, 기준, 프로젝트 기록에 기반하는가 |
| 적용 범위 | 이 결론이 적용되는 공종, 단계, 프로젝트 조건은 무엇인가 |
| 주의 | 공식 확인, 발주처 확인, 전문가 확인이 필요한 부분은 무엇인가 |
| 다음 액션 | 누가 무엇을 확인하거나 반영해야 하는가 |

금지:
- 자동 수집 지식을 확정 법규처럼 말하지 않는다.
- 출처가 다른 기준을 하나의 일반 기준처럼 섞지 않는다.
- 고객에게 내부 가설, 미검증 수치, 민감정보를 공개하지 않는다.
- “대체로 맞다”는 이유로 납품 판정이나 견적 책임을 확정하지 않는다.

## 6.1 AI 협업 응답 게이트

AI가 2개 이상 참여해 응답 또는 판단을 만들 때는 다음 게이트를 추가로 확인한다.

| 게이트 | 질문 |
|---|---|
| 역할 | 주관 AI, 검토 AI, 반론 AI가 분리되었는가 |
| 근거 | Evidence Pass에 KST 등급 또는 내부 선례가 있는가 |
| 반론 | Challenge Pass에 실패 조건 또는 예외 조건이 있는가 |
| 합의 | Consensus Record에 합의 상태가 있는가 |
| 실행 | Execution Handoff에 담당, 기한, 검증 방법이 있는가 |
| 지식화 | KB, QA, 교육자료, 표준문서, 백로그, 선례 중 저장 위치가 있는가 |

이 게이트를 통과하지 못한 AI 협업 결과는 확정 응답이 아니라 내부 검토 초안으로 둔다.

## 7. 주간 조직 학습 운영

| 주기 | 운영 |
|---|---|
| 매일 | 신규 지식 업데이트와 `needs-review` 항목을 확인한다. |
| 주 1회 | 역할별 학습 과제 3개 이하를 배정한다. |
| 주 1회 | 각 팀은 배운 지식이 실제 응답/검수/견적에 어떻게 반영됐는지 1건 기록한다. |
| 월 1회 | 자동 수집 지식 중 프로젝트 영향이 큰 문장 10개를 샘플 감사한다. |
| 분기 1회 | 역량 매트릭스와 역할별 학습 레인을 갱신한다. |
