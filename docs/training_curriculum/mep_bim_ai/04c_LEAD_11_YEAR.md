## 6. 팀 교육 체계 기여

11년차는 단순 교안 작성을 넘어 **팀 전체 교육 품질을 높이는 구조적 기여**를 한다.

### 오류 사례집 운영

| 단계 | 내용 |
|---|---|
| 수집 | 팀원 반복 오류, 협력사 반려 사례, 납품 지적 사항 |
| 분류 | 공종별·단계별·원인별 분류 |
| 원인 분석 | 기준 불명확·교육 부족·습관적 실수 구분 |
| 예방 기준 | 체크리스트·교안 추가 항목 도출 |
| 배포 | 월간 오류 사례 1건 이상 팀 공유 |

### BEP 템플릿 표준화

- 반복 프로젝트에서 공통으로 사용 가능한 BEP 템플릿 작성
- 프로젝트 유형별(신축/리모델링/공공/민간) 분기 구조 설계
- 팀원이 빠르게 채울 수 있는 입력 가이드 포함

### 납품 검수 표준화

- 공종·납품 형식별 검수 체크리스트 통일
- 자주 지적되는 항목 → 자동화 QA 스크립트 후보로 등록

## 8. 자동화 제품화 후보 발굴

11년차는 팀에서 반복 사용되는 Add-in 수요를 분석하여 **제품화 후보로 PM/Principal에 제안**한다.

### 제품화 판단 기준

| 항목 | 판단 기준 |
|---|---|
| 반복 빈도 | 월 10회 이상 반복되는 업무 |
| 공통성 | 1개 프로젝트만이 아닌 다수 프로젝트에 해당 |
| 표준화 가능성 | 입력·처리·출력이 일정한 패턴으로 정의 가능 |
| 절감 효과 | 자동화 시 시간·오류율 절감 규모 |
| 지원 비용 | 유지보수·버전 대응 비용 감당 가능 여부 |

### 제품화 제안서 구성

```
제품명:
문제 설명: 현재 어떤 수작업 업무가 반복되는가
대상 사용자: 팀 내부 / 외부 판매 / 특정 공종 담당자
핵심 기능: 주요 기능 3가지 이내
예상 절감: 월 __ 시간, 오류율 __ % 감소
유사 사례: 기존 Add-in이나 스크립트와의 차이
지원 범위: 내부 유지보수 / 외부 판매 여부
우선순위 의견: 즉시 개발 / 검토 필요 / 장기 과제
```

## 10. 설계/시공 복합 판단

11년차는 단순 상황 대응을 넘어, **복합 원인이 얽힌 상황에서 우선순위를 정하고 해결 경로를 제시**한다.

| 복합 상황 | 11년차 판단 접근법 |
|---|---|
| 설계 변경 + 공정 지연 동시 발생 | 변경 규모와 지연 영향 분리 분석 후 우선순위 결정 |
| 협력사 품질 저하 + 납품 임박 | 단계별 검수 강화 vs 보완 기간 확보 트레이드오프 판단 |
| 발주처 추가 요청 + 계약 범위 초과 | 유상 범위 명확화 후 PM과 협의 에스컬레이션 |
| 복수 프로젝트 납품일 충돌 | 자원 재배분, 외부 지원 요청, 일정 협의 순서 결정 |
| 법규 변경 + 기납품 모델 재작업 | 변경 적용 범위·책임·비용 판단 후 발주처 보고 |
