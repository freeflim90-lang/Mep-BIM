## 1. 연차 포지셔닝

12년차는 Lead 단계의 마지막 해이자 **PM/Consultant(13~16년차)로 전환하기 위한 준비 기간**이다.

| 이전 단계(11년차) | 12년차 전환 목표 | 다음 단계(13년차 PM) |
|---|---|---|

## 4. 부서 BIM 표준화 주도

12년차는 개별 프로젝트 기준을 넘어 **부서 전체에 적용되는 BIM 표준 체계**를 개정·배포한다.

### 표준 개정 대상

| 표준 문서 | 개정 포인트 |
|---|---|
| 모델링 표준 | LOD 정의, 파라미터, 명명 규칙, 템플릿 최신화 |
| CDE 운영 기준 | 플랫폼 변경 사항, 권한 체계, 배포 절차 반영 |
| 납품 검수 기준 | 형식별 체크리스트, 자동 QA 항목 통합 |
| 협력사 온보딩 패키지 | 표준 교육 자료, 검수 의견서 양식 일원화 |
| 오류 사례집 | 연간 사례 추가, 예방 기준 업데이트 |

### 표준 개정 절차

1. 팀원·리드 대상 현행 표준 불편·공백 조사
2. 개정안 작성 → Lead/PM 검토
3. 파일럿 프로젝트에서 2~4주 적용 시범 운영
4. 시범 결과 반영 → 최종 개정본 확정
5. 팀 배포 + 교육 세션 운영

## 5. 팀 역량 평가 체계 수립

12년차는 팀원의 성장을 객관적으로 평가하고 지원할 **역량 평가 및 포트폴리오 체계**를 설계한다.

### 연차별 평가 기준 설계

| 연차 구간 | 평가 축 | 핵심 산출물 기준 |
|---|---|---|
| 1~2년차 | 모델링 정확도, 기준 준수 | 기본 모델, 체크리스트 |
| 3~5년차 | 품질 판단, 보고서 독립 작성 | 간섭 보고서, 변경 영향표 |
| 6~8년차 | 다공종 조율, CDE 운영, 회의 주도 | 월간 보고서, 조율 이슈 대장 |
| 9~12년차 | BEP 작성, 납품 승인, 팀 운영 | BEP, 납품 검수표, 교안 |

### 포트폴리오 체계

| 항목 | 내용 |
|---|---|
| 연차별 대표 산출물 | 각 연차에서 가장 완성도 높은 산출물 1~2건 보관 |
| 자기 평가서 | 해당 연도 배운 점, 어려움, 다음 목표 작성 |
| 리드 리뷰 | 강점·개선점·다음 성장 과제 피드백 기록 |
| 승급 심사 | 포트폴리오 + 리드 리뷰 기반 다음 단계 전환 판단 |

### 로드맵 수립 기준

| 항목 | 판단 기준 |
|---|---|
| 개발 우선순위 | 절감 효과·반복 빈도·표준화 가능성 종합 순위 |
| 개발 방식 | 내부 개발 / 외부 위탁 / 오픈소스 기반 선택 |
| 유지보수 계획 | Revit 버전 업그레이드 대응, 버그 수정 담당자 |
| 외부 판매 검토 | 내부 전용 vs 외부 판매 제품 구분 |
| 개발 일정 | 분기별 기능 출시 목표, MVP(최소 기능 제품) 정의 |
