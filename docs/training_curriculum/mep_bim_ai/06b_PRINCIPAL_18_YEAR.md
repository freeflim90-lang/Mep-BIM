## 1. 연차 포지셔닝

18년차는 17년차에 초안을 만든 방법론·거버넌스·교육 체계를 **완성하고 조직 전체에 정착**시키는 해다.  
또한 반복 자동화 수요를 **외부 판매 제품**으로 전환하는 전략을 실행한다.

| 이전 단계(17년차) | 18년차 전환 목표 | 다음 단계(19년차) |
|---|---|---|
| 방법론 백서 초안 | 방법론 완성본 팀 전체 배포 | 외부 발표·업계 기여 |
| 표준 거버넌스 수립 | KPI 정규 운영·분기 보고 | 지식 자산 체계 고도화 |
| 교육 체계 개선안 | 교육 체계 정규 운영·승급 연계 | 외부 교육 사업 확대 |
| AI 정책 수립 | 제품화 전략 실행·Add-in 출시 | 신사업 기획 참여 |

## 4. 방법론 완성 및 배포

### 완성 전 최종 검토

| 검토 항목 | 검토 기준 |
|---|---|
| 현장 적용 가능성 | 실제 프로젝트에서 팀원이 따라할 수 있는가 |
| 최신성 | 최신 법규·소프트웨어·AI 트렌드 반영 여부 |
| 일관성 | 챕터 간 용어·기준 충돌 없음 |
| 가독성 | 연차별 독자가 자기 파트를 빠르게 찾을 수 있는가 |
| 검증 가능성 | 기준마다 확인 방법 명시 여부 |

### 배포 및 활용 교육

1. 방법론 완성본 CDE에 공식 등록
2. 팀 전체 배포 세션 (챕터별 핵심 설명)
3. Q&A → 보완 항목 목록화
4. 프로젝트 킥오프 시 방법론 참조 의무화
5. 6개월 후 활용 현황 점검 → 개정 후보 등록

## 5. 조직 KPI 정규 운영

### 분기 KPI 사이클

| 시기 | 활동 |
|---|---|
| 분기 초 | KPI 목표값 설정 (전분기 결과 기반 조정) |
| 분기 중 | 주간 모니터링, 이상값 조기 경보 |
| 분기 말 | 집계·분석·개선 조치 도출 |
| 분기 보고 | PM·Lead 대상 KPI 보고서 발행 |

### KPI 개선 조치 원칙

- 목표 미달 항목: 원인 분석 → 교육/프로세스/자동화 중 해결책 선택
- 목표 초과 항목: 목표값 상향 조정 또는 새로운 KPI 추가
- 지속 미달 항목: 목표 현실화 또는 근본 원인 해결 프로젝트 착수
- 분기별 개선 조치는 다음 분기 KPI에 반영

## 6. Add-in 제품화 전략 실행

### 제품화 실행 단계

| 단계 | 내용 | 기간 |
|---|---|---|
| 후보 선정 | 반복 빈도·절감 효과·표준화 가능성 최종 선정 | 1개월 |
| MVP 정의 | 핵심 기능 3가지 이내 최소 제품 범위 | 1개월 |
| 개발 착수 | 내부 개발 또는 외부 위탁 결정 후 착수 | 3~6개월 |
| QA | 4가지 시나리오 기반 테스트·버그 수정 | 1~2개월 |
| 출시 | 내부 배포 또는 외부 판매 채널 결정 | 1개월 |
| 유지관리 | Revit 버전 대응, 고객 문의, 문서 업데이트 | 지속 |

### 내부/외부 판매 구분 기준

| 기준 | 내부 전용 | 외부 판매 |
|---|---|---|
| 사용 대상 | 사내 팀만 | 다수 고객사 |
| 지원 비용 | 낮음 | 높음 (고객지원·문서화) |
| 수익 모델 | 공수 절감 | 라이선스·구독 수익 |
| 보안 고려 | 내부 데이터 | 고객 데이터 보호 추가 |
| 진입 요건 | 사내 QA | 외부 QA·법무 검토 |
