| 섹션 | 핵심 내용 | 작성 책임 |
|---|---|---|
| 프로젝트 정보 | 발주처, 규모, 발주 방식, BIM 목적 | 10년차 주관 |
| 조직 및 RACI | 발주처·시공사·설계사·BIM 팀 역할 매트릭스 | 10년차 주관 |
| BIM 목표 | LOD, 활용 목적, 정량적 성과 기준 | 10년차 주관 |
| 소프트웨어·버전 | Revit, Navisworks, BIM 360 버전 | 10년차 주관 |
| 파일 명명 규칙 | 프로젝트-공종-구역-Revision | 10년차 주관 |
| CDE 구조 | WIP/Shared/Published 폴더·권한 | 10년차 주관 |
| 모델 분할 기준 | 구역, 층, 공종별 분할 방법 | 10년차 주관 |
| 간섭 검토 계획 | 주기, 규칙, 보고 형식, 책임자 | 10년차 주관 |
| 납품 계획 | 형식·날짜·검수 기준 | 10년차 주관 |
| 변경 관리 | 접수·승인·모델 반영 절차 | 10년차 주관 |
| 보안 | 자료 반출·AI 입력·접근 권한 | 10년차 주관 |

### BEP 발주처 협의 절차

1. BEP 초안 작성 → 조직 내 검토 (PM 승인)
2. 발주처 제출 → 검토 의견 수령
3. 의견 분류 (반영/불반영/협의) → 회신 작성
4. 수정본 제출 → 최종 승인 확인
5. 승인된 BEP → 팀 배포 및 기준 공지

10년차는 팀원 2~3명의 업무를 직접 배분하고, 주간 점검을 운영한다.

### 업무 배분 원칙

| 원칙 | 내용 |
|---|---|
| 역량 기반 배분 | 담당자 연차·경험에 맞는 업무 배정 |
| 기한 이중 설정 | 중간 검토일 + 최종 제출일 모두 명시 |
| 병목 사전 파악 | 주간 현황에서 지연 징후 조기 확인 |
| 오류 교육화 | 반복 오류는 개인 지적보다 기준 보완으로 해결 |

### 주간 업무 점검 운영

| 시기 | 내용 |
|---|---|
| 매주 월요일 | 이번 주 마감 과제·병목 확인, 우선순위 조정 |
| 매주 금요일 | 완료 여부 확인, 다음 주 과제 배분 |
| 월 1회 | 개인별 성장 과제 리뷰, 교안 보완 |

### 멘토링 방법

| 대상 | 10년차 멘토링 포인트 |
|---|---|
| 1~2년차 | 기본 품질 기준, 도면-모델 일치 습관 형성 |
| 3~5년차 | 판단 근거 설명, 보고서 독립 작성 능력 |

## 11. Add-in QA 설계

10년차는 Add-in 기능 명세서를 바탕으로 **QA 시나리오를 설계하고 실제 테스트를 주관**한다.

### QA 시나리오 설계 기준

| 시나리오 유형 | 설명 | 예시 |
|---|---|---|
| 정상 케이스 | 기준 충족 요소가 올바르게 통과 | LOD 350 파라미터 완비 요소 → 통과 |
| 경계 케이스 | 기준 경계값 처리 | LOD 300/350 경계 요소 처리 |
| 오류 케이스 | 파라미터 누락·잘못된 값 입력 시 동작 | 필수 파라미터 없는 요소 → 경고 표시 |
| 회귀 케이스 | 이전 버전 버그 재발 방지 | 이전 버그 재현 데이터로 검증 |

### QA 테스트 절차

1. 기능 명세서 기반 테스트 케이스 목록 작성
2. 샘플 모델·데이터 준비 (실제 프로젝트 데이터 비식별화)
3. dry-run 실행 → 예상 결과 vs 실제 결과 비교
4. 오류 케이스 재현 및 결과 기록
5. 테스트 결과 보고서 → 개발팀 피드백
