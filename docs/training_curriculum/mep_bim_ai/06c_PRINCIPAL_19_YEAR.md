## 1. 연차 포지셔닝

19년차는 조직 내부 체계가 안정된 상태에서 **외부 영향력과 사업 다각화**로 확장하는 해다.  
지식을 자산으로 체계화하고, 교육 사업·신사업·업계 표준 기여를 통해 회사의 외부 포지션을 강화한다.

| 이전 단계(18년차) | 19년차 전환 목표 | 다음 단계(20년차) |
|---|---|---|
| 방법론 배포·KPI 운영 | 지식 자산 체계 고도화 | 장기 비전·업계 리더십 |
| 제품화 전략 착수 | 외부 교육 사업 확대 | 신사업 독립 수행 |
| 외부 발표 시작 | 업계 표준 기여 활동 | 업계 권위자 포지션 |
| 교육 체계 운영 | 조직 장기 역량 설계 | 후계자 육성 완성 |

## 4. 지식 자산 체계 고도화

### 지식 자산 유형

| 유형 | 내용 | 관리 방법 |
|---|---|---|
| 방법론 | MEP BIM+AI 수행 체계 | 연 1회 개정·버전 관리 |
| 사례집 | 완료 프로젝트 교훈·성과 | 프로젝트 종료 후 즉시 등록 |
| 교육 콘텐츠 | 연차별 교육 자료·교안 | 교육 후 업데이트 |
| 도구 라이브러리 | Add-in·스크립트·템플릿 | 버전별 관리 |
| 오류 사례집 | 반복 오류·예방 기준 | 월간 업데이트 |
| 법규 동향 | BIM 관련 법규·정책 변화 | 분기 업데이트 |

### 지식 자산 맵 구성

```
LUA BIM LABS 지식 자산 맵 v[버전]

1. 방법론 자산
   - 위치 (CDE 경로)
   - 최신 버전·개정일·승인자

2. 교육 자산
   - 연차별 교육 파일 목록

3. 도구 자산
   - Add-in·스크립트 목록·버전·담당자

4. 사례 자산
   - 프로젝트별 교훈·성과 파일

5. 법규·정책 자산
   - 최신 BIM 법규·지침 목록
```

## 5. 외부 교육 사업 확대

### 사업 모델

| 모델 | 대상 | 수익 구조 |
|---|---|---|
| 기업 교육 위탁 | 발주처·시공사·설계사 BIM 팀 | 일수 기반 교육비 |
| 온라인 강의 | BIM 실무자 개인 | 구독·단건 판매 |
| LUA BIM 인증 | BIM 역량 공인 인증 | 응시료·인증료 |
| 컨설팅 + 교육 패키지 | BIM 도입 기업 | 컨설팅+교육 번들 |

### 기업 교육 운영 절차

1. 고객 BIM 성숙도 진단 → 필요 교육 범위 파악
2. 맞춤 교육 과정 설계 → 표준 과정 커스터마이징
3. 강사 배정 (Principal/PM/Lead 역할 분담)
4. 교육 실행 → 실습 과제·평가
5. 사후 지원 → 질문 대응·교육 자료 전달

## 6. 신사업 기획 참여

### 신사업 후보 발굴 방향

| 방향 | 아이디어 예시 |
|---|---|
| BIM 데이터 서비스 | 모델 데이터 → FM 플랫폼 연계 SaaS |
| AI 자동화 서비스 | BIM 품질 자동 검수 클라우드 서비스 |
| 교육 플랫폼 | LUA BIM 아카데미 온라인 플랫폼 |
| 해외 진출 | 동남아·중동 BIM 컨설팅 서비스 |
| 데이터 표준화 | MEP 공종별 파라미터 표준 라이브러리 |

### 신사업 기획서 구성

```
신사업 기획서
작성일 / 작성자

1. 사업 배경 (문제·시장 기회)
2. 사업 개요 (무엇을 제공하는가)
3. 목표 고객 (누구에게 팔 것인가)
4. 수익 모델 (어떻게 돈을 버는가)
5. 핵심 역량 (왜 LUA BIM LABS가 잘 할 수 있는가)
6. 경쟁 분석 (대안과 비교)
7. 단계별 실행 계획 (6·12·24개월)
8. 투자 규모 및 손익 분기점
9. 리스크 및 대응
```

## 7. 업계 표준 기여

### 기여 활동 유형

| 활동 | 내용 |
|---|---|
| 협회 위원회 | BIM 표준·교육·정책 위원회 참여 |
| 표준 의견서 | 정부·협회의 BIM 기준 초안에 공식 의견 제출 |
| 공동 연구 | 학계·기관과 BIM 효과성 연구 협력 |
| 오픈소스 기여 | 팀 내 도구의 일부를 오픈소스로 공개 |
| 우수 사례 발표 | 완료 프로젝트 성과를 업계 행사에서 발표 |

### 표준 의견서 작성 방법

1. 초안 기준 검토 → 현장 적용 가능성 평가
2. 우리 프로젝트 사례 근거로 개선 의견 작성
3. 구체적 수정 제안 (현행 → 개정안 형식)
4. 공식 제출 채널로 제출 → 회신 결과 기록
