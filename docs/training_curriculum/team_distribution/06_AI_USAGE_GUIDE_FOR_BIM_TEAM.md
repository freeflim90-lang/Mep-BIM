# BIM 팀 AI 활용 가이드

문서상태: 팀 배포용 초안  
작성일: 2026-05-20

---

## 1. 기본 원칙

AI는 BIM 실무자의 판단을 대체하지 않는다. AI는 자료 정리, 초안 작성, 체크리스트 생성, 반복 업무 보조에 사용하며, 설계·시공·법규·납품 판단은 담당자가 검토하고 책임진다.

---

## 2. 허용 활용

| 활용 | 예시 |
|---|---|
| 용어 학습 | MEP 용어, 도면 기호, 장비 약어 정리 |
| 체크리스트 초안 | 모델 품질, CDE 제출, 납품 검수 항목 초안 |
| 회의록 정리 | 안건, 결정사항, Action Item 정리 |
| 보고서 문장 개선 | 이슈 설명, 리스크 문장, 조치 의견 정리 |
| Excel 보조 | 필터, 정렬, 간단한 수식, CSV 정리 |
| 교육자료 초안 | 실습 과제, 퀴즈, 회고 질문 생성 |

---

## 3. 제한 활용

| 제한 항목 | 이유 |
|---|---|
| 고객 모델/도면 원본 입력 | 보안 및 기밀 유지 |
| 법규 적합성 확정 | 전문 검토 필요 |
| 설계 오류 단정 | 설계 책임자 확인 필요 |
| 시공 가능 여부 확정 | 현장 조건과 담당자 검토 필요 |
| 자동 생성 결과 그대로 납품 | 검증 책임 문제 |

---

## 4. 권장 프롬프트 구조

| 항목 | 작성 예시 |
|---|---|
| 역할 | “MEP BIM 코디네이터 관점에서” |
| 입력 | “다음 간섭 이슈 목록을 기준으로” |
| 요청 | “우선순위, 영향, 조치 의견을 표로 정리해줘” |
| 제한 | “설계 오류로 단정하지 말고 추가 확인 필요 항목을 분리해줘” |
| 출력 | “회의 보고용 1페이지 요약으로 작성해줘” |

---

## 5. 검토 체크

AI 결과를 업무에 사용하기 전 다음을 확인한다.

| 체크 항목 | 확인 |
|---|---|
| 프로젝트 정보나 고객 기밀이 포함되지 않았는가 | □ |
| 설계/시공 판단을 AI가 단정하지 않았는가 | □ |
| 수치, 단위, 공종명이 맞는가 | □ |
| 책임자, 기한, 조치가 실제 업무와 맞는가 | □ |
| 최종 제출 전 담당자가 직접 검토했는가 | □ |
