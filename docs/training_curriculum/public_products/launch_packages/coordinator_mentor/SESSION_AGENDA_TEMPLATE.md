**Version:** 1.0 | Effective: 2026-06-01

## PRE-SESSION PREPARATION CHECKLIST

### For the Client (sent 48 hours before the session)

Send this message to the client 48 hours before the scheduled Zoom:

> "Your monthly session is in 2 days — [Date] at [Time] [Time Zone]. Here's how to get the most from our 30 minutes together. Please prepare the following before we meet:
>
> 1. **One real issue from this month.** Think of one coordination challenge, clash situation, or workflow problem you've dealt with recently. Be ready to describe it in 2–3 minutes. A screenshot or brief description in writing helps.
>
> 2. **Your top question.** What is the one thing you most want to walk away with after this session?
>
> 3. **Status update on your action items.** Look back at the action items from last month's session summary (I'll send you a reminder). Which ones did you complete? Which ones are still open?
>
> 4. **Any specific topic you want covered.** If there's a lesson or topic from this month you want to go deeper on, let me know in advance and I'll prepare for it.
>
> Zoom link: [link]
> Session duration: 30 minutes
> Please be on time — we'll start promptly."

### For the Mentor (internal checklist, complete before each session)

- [ ] Review client's CRM record (experience level, subscription month, notes from past sessions)
- [ ] Review Q&A Tracker for this client's entries from the past 30 days
- [ ] Review Issue Review submissions from this month — what were the key themes?
- [ ] Note any recurring struggles or patterns from lesson responses
- [ ] Review action items from the last session's post-session summary
- [ ] Prepare 1–2 discussion points to raise proactively if the client runs out of topics
- [ ] Confirm Zoom link is active and working
- [ ] Set a 25-minute timer to keep the session on track

## SESSION STRUCTURE

### Section 1: Opening and Check-In (0:00 – 3:00)
**Duration: 3 minutes**

**Purpose:** Establish rapport, confirm the session format, set expectations for the 30 minutes.

**Mentor script guide:**
- Greet the client by name.
- Confirm the 30-minute time box: "We have exactly 30 minutes, so I'll keep us on track."
- Quick check: "How has the program been going for you this month — anything you want to flag before we dive in?"
- Transition: "Let's start with any questions or issues you prepared."

**Notes box:**
```
Client mood / engagement level: ___________________________
Any flags or concerns raised at opening: __________________
```

---

### Section 2: Issue Review and Q&A (3:00 – 13:00)
**Duration: 10 minutes**

**Purpose:** Address the real coordination problem or question the client brings to the session. This is the highest-value part of the call.

**Protocol:**
1. Ask the client to describe their prepared issue or question (allow 2–3 minutes for them to explain).
2. Ask one clarifying question before responding.
3. Provide your analysis and recommendation (5–7 minutes).
4. End with: "Does that fully answer the question, or do you want to dig into any part of that further?"

**If the client has more than one question:** Acknowledge all of them at the start of this section, prioritize with the client ("Which is most urgent?"), and table any unresolved ones for the Telegram follow-up or next session.

**Notes box:**
```
Client's issue/question: _________________________________
Key points covered in response: _________________________
Follow-up needed: Yes / No
If yes, detail: _________________________________________
```

---

### Section 3: Lesson Debrief and Concept Check (13:00 – 20:00)
**Duration: 7 minutes**

**Purpose:** Connect recent lessons to the client's actual work. Ensure concepts are understood, not just read.

**Protocol:**
1. Reference 1–2 lessons from the past month: "We covered [topic] recently. Has that come up in any of your projects?"
2. Ask a concept-check question: "If you had to explain [key concept] to a junior on your team, how would you put it?"
3. Correct any misunderstandings without making the client feel criticized. Reframe and expand.
4. Optional: Share a tip or nuance that goes beyond the lesson content.

**Common concept-check questions to use:**
- "Walk me through how you'd prioritize a list of 200 hard clashes."
- "If a subcontractor tells you 'that's not our clash', what's your first move?"
- "What's the difference between a clash group and a clash set in Navisworks?"
- "How would you structure a coordination meeting agenda for a project with 3 active subcontractors?"

**Notes box:**
```
Lessons referenced: ____________________________________
Concept check question used: ___________________________
Client's response quality: Strong / Adequate / Needs reinforcement
Gaps identified: _______________________________________
```

---

### Section 4: Action Items and Goal Setting (20:00 – 27:00)
**Duration: 7 minutes**

**Purpose:** Close the loop on last month's action items; set new action items for the coming month.

**Protocol:**
1. Review the action items from the last session (pull from the post-session summary on file).
2. For each item: Completed / In Progress / Not Started. Note why if not completed.
3. Set 2–3 new action items for the coming month. These should be specific and measurable.
   - Example: "Before next session, I want you to run a clash detection in Navisworks on your current project and bring me the top 10 hard clashes. We'll go through your approach."
4. Confirm the client understands and agrees to the action items.

**Action Item Table (to transfer to Post-Session Summary):**

| # | Action Item | Owner | Due By |
|---|-------------|-------|--------|
| 1 | | Client | Next session |
| 2 | | Client | Next session |
| 3 | | LUA BIM LABS | [date if applicable] |

**Notes box:**
```
Previous action items reviewed: ________________________
Completion rate: ___ / ___ completed
New action items set: __________________________________
```

---

### Section 5: Close and Next Steps (27:00 – 30:00)
**Duration: 3 minutes**

**Purpose:** Summarize the session, confirm next steps, maintain momentum.

**Protocol:**
1. Brief summary: "Here's what we covered today: [2–3 bullet points verbally]."
2. Confirm next session: "I'll send the scheduling link for next month's session in the first week of [next month]. Check your Telegram."
3. Remind the client of the post-session summary: "I'll send you a written summary within 24 hours — action items, key points, and anything you can reference later."
4. Encouragement / close: Keep it genuine. Acknowledge what the client is doing well.
5. End the call on time.

**Notes box:**
```
Session ended at: _______ (should be within 2 minutes of scheduled end)
Client sentiment at close: Positive / Neutral / Frustrated / Uncertain
Follow-up required beyond standard summary: Yes / No
```

**Quality check before sending the summary:**
- [ ] Action items are specific (not vague like "practice coordination")
- [ ] Each action item has a target timeframe
- [ ] Key topics match what was actually discussed
- [ ] The encouragement line is specific to this client and this session
- [ ] No grammar errors

## RESCHEDULING POLICY

### Client-Initiated Reschedule
- Clients may reschedule with a minimum of 48 hours notice at no penalty.
- Reschedule request: client sends a message on Telegram. Staff acknowledges within 4 hours and provides the Calendly link to book a new time.
- The rescheduled session must take place within the same calendar month. If the client's availability does not allow this, the session is forfeited for that month.

### No-Show (Less Than 48 Hours Notice or No Contact)
- If the client does not attend and did not reschedule with 48+ hours notice, the session counts as used for that month.
- Send the No-Show Message (Template CM-NS01) within 2 hours of the missed session start time:
  > "Hi [Name], I was in the Zoom room for our session today at [time]. I didn't want to leave without letting you know I was there. No problem at all — things come up. Just note that this counts as your monthly session. I'll send the scheduling link again at the start of next month. If you want to talk about anything in the meantime, I'm here on Telegram."

### LUA BIM LABS-Initiated Reschedule
- If LUA BIM LABS needs to reschedule, notify the client at least 72 hours in advance.
- Offer 3 alternative time slots in the same calendar month.
- If none of the alternatives work for the client, offer a 45-minute session the following month as a makeup (the only exception to the 30-minute cap).

### Technical Issues
- If the Zoom connection fails at session start: attempt to reconnect for 5 minutes, then send a WhatsApp or Telegram message to diagnose. If the session cannot proceed, it is rescheduled at no penalty to the client, even with less than 48 hours notice.
