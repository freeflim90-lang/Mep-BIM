# 2026-W23 Knowledge Curation Weekly Review

이 문서는 2026-W23 지식 수집·업데이트·Obsidian 반영 흐름을 주간 실행 계획과 리뷰 지표로 정리한다. 리뷰의 목적은 노력량 보고가 아니라 위험 감소, 납품 지원, 반복 감소, 고립 지식 감소를 확인하는 것이다.

## 주간 목표

| 코드 | 주간 목표 | 리뷰 지표 |
|---|---|---|
| `WKP01 보안 먼저` | 보안·개인정보·고객 신뢰 위험 차단 | 원문 링크 제거, 보안검토 기록, 민감정보 잔존 0건 |
| `WKP02 납품 연결` | 납품 검수·QA·고객 응답 지식 연결 | 체크리스트 링크, SOP 연결, 고객 응답 템플릿 반영 |
| `WKP03 기준 고정` | 반복되는 뉴스·큐레이션 기준을 공통 기준으로 분리 | 공통 기준 노트, 날짜별 로그 링크, 반복 상세 기준 감소 |
| `WKP04 재검증 예약` | 정책·수치·버전 후보의 Watch 상태 관리 | 공식 출처 확인 여부, 미확인 사유, 다음 확인일 |
| `WKP05 링크 지표` | Obsidian 고립 지식 감소 | 관련 위키링크 2개 이상, MOC 연결, 다음 검토일 |
| `WKP06 중복 감소` | 반복 QA와 중복 기준 압축 | 대표 FAQ 지정, 반복 문서 링크 전환, needs-review 분리 |
| `WKP07 승격 실험` | 교육·상품·자동화 후보 파일럿 조건화 | 서비스 범위, 교육자료 초안, 자동화 후보, needs-pilot |
| `WKP08 보류 사유` | 이번 주 하지 않을 항목의 조건 명시 | 보류 사유, 다음 처리 조건, 월간 리뷰 후보 |

| 납품 검수 상품화 | 파일럿 조건과 서비스 범위가 아직 분리되지 않음 | needs-pilot 기준 작성 후 재검토 |
| 반복 QA 전체 병합 | 대표 FAQ 기준 문서 선정이 선행되어야 함 | 공식 출처가 확인된 대표 FAQ 지정 |

## 리뷰 판정

- 위험 감소: 보안검토 후보는 유용성보다 원문 링크 차단과 익명화가 먼저다.
- 납품 지원: 납품 검수 지식은 QA, 고객지원, SOP로 연결될 때 실제 가치가 생긴다.
- 반복 감소: 공통 기준이 없는 날짜별 반복 문서는 계속 쌓이므로 기준 노트가 먼저다.
- 고립 지식 감소: Obsidian 링크가 없는 지식은 반영 완료가 아니라 링크 회복 후보로 본다.
