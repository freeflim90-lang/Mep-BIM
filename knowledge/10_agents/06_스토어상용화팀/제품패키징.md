## 패키지 버전 관리 기준 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: packaging,versioning,release

버전 체계: Major.Minor.Patch.Build (예: 1.2.3.456)
Major: 호환성 변경, Minor: 기능 추가, Patch: 버그 수정, Build: CI 빌드 번호.
AssemblyVersion과 FileVersion을 일치시키고 InformationalVersion에 브랜치/커밋 정보 포함.
스토어 제출 버전은 MSI 버전과 AssemblyFileVersion이 반드시 일치해야 한다.
업그레이드 시 이전 버전 자동 제거(MajorUpgrade 설정) 또는 병렬 설치 명시적 지원 여부 결정.


## 제품 패키징 Claude Code 심화 업데이트 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: packaging,MSI,Autodesk-Store,bundle,Revit-Addin,2025

- MSI 패키지 구성 표준(2025): WiX Toolset v4.x를 사용하여 Revit Add-in MSI를 제작한다. 패키지 구성 요소: ① `{ProductName}.dll`, ② `{ProductName}.addin`, ③ 의존성 DLL, ④ 리소스 파일(아이콘, 로케일). Revit 버전별 설치 경로(`%APPDATA%\Autodesk\Revit\Addins\{VERSION}\`)를 자동 감지하여 멀티버전 동시 설치를 지원한다.
- Autodesk App Store 번들 제출 규격: 제출 패키지는 ZIP 형식이며, 루트에 `bundle.xml`(앱 메타데이터), `Contents/` 폴더에 버전별 DLL, 아이콘 파일(48×48, 96×96, 256×256 PNG)을 포함한다. 번들 내 실행 파일은 EV 코드 서명 필수이며, `AuthorId`와 `AppId`는 Autodesk Developer Portal에서 발급받은 값을 사용한다.
- 멀티버전 패키지 자동화: PowerShell 스크립트로 빌드 아티팩트(`net48/`, `net8.0/`)를 버전별 폴더로 자동 정렬하고, `bundle.xml`의 버전 정보를 `AssemblyInfo.cs`에서 읽어 자동 업데이트한다. GitHub Actions에서 태그 생성 시 이 스크립트가 자동 실행되어 배포 준비 ZIP을 생성한다.
- 설치 전/후 조건 검증: MSI 설치 전에 ① Revit 설치 여부 감지(`HKLM\SOFTWARE\Autodesk\Revit`), ② .NET Runtime 버전 확인, ③ 기존 버전 제거 여부 판단 로직을 Custom Action으로 구현한다. 설치 실패 시 사용자 친화적 오류 메시지와 지원 링크를 표시한다.
- 패키지 무결성 검증: 각 릴리스 ZIP의 SHA-256 해시를 GitHub Releases 노트에 공개하고, 다운로드 후 자동 검증하는 PowerShell 원라이너를 설치 가이드에 포함한다. `Get-FileHash -Algorithm SHA256` 명령으로 10초 이내 검증 가능하다.

### 비정형 BIM 기능 패키징 전략

비정형 기능은 별도 Add-in으로 분리 vs 기존 제품에 번들링 선택:

**별도 Add-in 분리 장점:**
- Autodesk App Store 별도 등록 → 비정형 전문 브랜드 확립
- 가격 차별화 (Freeform 티어 별도 과금)
- 스토어 검색 노출 최적화 (비정형, freeform 키워드)

**기존 제품 번들링 장점:**
- 기존 고객 업셀링 용이 (추가 설치 없이 기능 활성화)
- 유지보수 단일화

**LUA BIM LABS 권장: 분리 배포 후 번들 옵션 제공**
- `BIM_Tools_Pro.exe` (정형 기본)
- `BIM_Tools_Freeform.exe` (비정형 전용, 별도 설치)
- 앱스토어: 두 제품 별도 등록, 교차 링크

### 패키징 실패 패턴 3가지

**1. 비정형 기능을 기존 패키지에 포함 → 정형 고객 혼란**
- 원인: 고급 기능 설명이 UI에 노출 → "이게 뭔가요?" CS 증가
- 해결: 비정형 기능 UI 숨김 처리, 라이선스 활성화 시에만 노출

**2. 설치 시 Dynamo/Rhino 미감지 → 비정형 기능 미활성화**
- 해결: 설치 마법사에서 Dynamo 설치 여부 자동 감지, 미설치 시 다운로드 링크 제공

**3. 멀티버전 MSI 패키징 오류**
- 원인: Revit 2022~2025 지원 시 각 버전 DLL 경로 하드코딩
- 해결: WiX Toolset으로 조건부 설치 (RevitVersion 레지스트리 감지)

```xml
<!-- WiX: Revit 버전별 조건부 설치 -->
<Component Id="RevitAddin2024">
  <Condition>REVIT2024INSTALLED</Condition>
  <File Source="$(var.BuildDir)\2024\LuaBIM.dll" />
</Component>
```
