### CS가 갖춰야 할 BIM 프로그램 기본 이해

**왜 CS가 BIM을 깊이 알아야 하는가:**
BIM Add-in 고객은 대부분 건축·설비·구조 엔지니어다. 이들은 전문 용어로 문제를 설명한다. CS가 "공유 파라미터", "작업 집합", "뷰 템플릿", "링크 파일", "중앙 파일 동기화"의 개념을 모르면 정확한 증상 수집이 불가능하고, 고객은 전문성 없는 지원에 실망해 앱 리뷰에 낮은 점수를 남긴다. CS는 BIM 프로그램의 실제 워크플로우를 이해해야 맥락 있는 질문을 할 수 있다.

---

### Revit 사용자의 일상 워크플로우 — CS 공감 포인트

**Revit 기본 작업 흐름:**
1. 중앙 파일 열기(Open Central) → 로컬 파일 복사본 작업
2. 설계 요소 배치·편집 → 작업 집합(Workset) 체크아웃
3. 동기화(Sync with Central) → 다른 팀원 변경사항 반영
4. 납품 시 NWC/IFC/PDF 내보내기

CS 공감 포인트: "동기화 중 오류가 났다"고 하면 고객은 여러 시간 작업한 내용이 날아갈까 봐 매우 긴장 상태임을 인식하고 먼저 "백업 파일이 자동 저장돼 있으니 데이터 손실 걱정은 없다"고 안심시킨 뒤 절차를 안내한다.

**Revit 프로젝트 규모별 특성:**
- 소규모(연면적 1,000㎡ 이하): 단일 사용자, 파일 10~50MB, 대부분 오류 없음
- 중규모(5,000~30,000㎡): 2~5명 협업, 중앙 파일 100~500MB, 작업 집합 오류 빈발
- 대규모(30,000㎡↑): 10명+ 협업, 중앙 파일 1GB 이상, 링크 파일 10개 이상, 성능 이슈 다발
규모를 파악하면 고객의 문제 복잡도를 즉시 예측할 수 있다.

---

### Add-in 기능별 오류 발생 맥락 이해

**MEP(기계·전기·배관) 자동 배치 기능 오류 맥락:**
- MEP Add-in은 Revit의 MEP Connector, Space, Duct/Pipe Network 등 복잡한 요소와 연동된다.
- 고객이 "자동 배치가 엉뚱한 위치에 된다" → Space 요소가 없거나 Level이 잘못 설정됨
- "배관이 겹친다" → 기존 요소와 충돌 허용 범위 설정 확인
- "덕트 사이징이 계산되지 않는다" → 시스템(System) 타입이 지정되지 않은 경우

**간섭(Clash) 검토 기능 오류 맥락:**
- Navisworks의 Clash Detective가 연동된 경우: NWC 파일 버전과 Navisworks 버전이 일치해야 결과가 정확하다.
- "간섭이 없다고 나오는데 실제로는 겹친다" → 충돌 허용 거리(Tolerance) 설정값 확인, 또는 NWC를 오래된 버전으로 내보낸 경우
- "간섭 결과가 저장이 안 된다" → NWF 파일이 아닌 NWD로 저장해야 결과가 보존됨

**일람표(Schedule) 자동화 기능 오류 맥락:**
- "일람표 데이터가 비어 있다" → 파라미터가 Instance/Type 중 잘못된 곳에 있거나, 공유 파라미터 GUID 불일치
- "값이 0으로 나온다" → 단위 변환 오류(mm vs m, Revit 내부 단위는 feet임)
- "일부 요소만 나온다" → 필터 설정이 의도치 않게 적용된 경우

### 공감하며 정보 수집하는 CS 응대 스크립트 원칙

1. **맥락 먼저 파악**: "어떤 작업을 하다가 발생했나요? 어떤 명령을 실행하셨나요?"
2. **증상 구분**: "오류 메시지가 있나요? 있다면 정확한 문구를 복사해 주세요."
3. **환경 수집 순서**: Revit 버전 → Add-in 버전 → 오류 메시지 → 로그 파일 → (마지막) 모델 파일
4. **재현 여부**: "다른 프로젝트 파일에서도 같은 문제가 발생하나요?"
5. **긴급도 판단**: 납품 마감이 가까운지 확인해 P1/P2 우선 처리 여부 결정

BIM 엔지니어는 납품 마감 직전에 Add-in 오류를 겪는 경우가 많다. CS는 "이해합니다, 빨리 해결해 드리겠습니다"라는 공감과 함께 우선순위를 높여 처리한다.
