**AI 즉시 답변 패턴 — "Revit 물량산출 데이터를 엑셀 견적서로 자동 변환할 수 있나요?"**
```python
import openpyxl
import pandas as pd

# Revit Schedule CSV → 건설 견적서 엑셀 자동 생성
df = pd.read_csv("revit_schedule.csv")  # Revit에서 내보낸 물량 데이터

# 공종별 물량 집계
grouped = df.groupby("Category").agg(
    수량=("Quantity", "sum"),
    단가=("UnitPrice", "first")
).reset_index()
grouped["금액"] = grouped["수량"] * grouped["단가"]

# 엑셀 작성 (openpyxl)
wb = openpyxl.Workbook()
ws = wb.active
ws.title = "내역서"
headers = ["공종", "수량", "단가", "금액"]
ws.append(headers)
for _, row in grouped.iterrows():
    ws.append([row["Category"], row["수량"], row["단가"], row["금액"]])

# 합계 행 추가
ws.append(["합계", "", "", sum(grouped["금액"])])
wb.save("cost_estimate.xlsx")
```

**BIM 엑셀 자동화 주요 활용 시나리오:**
| 시나리오 | 입력 | 출력 | 자동화 도구 |
|---------|------|------|-----------|
| 물량→견적 | Revit Schedule CSV | 공종별 견적서 xlsx | openpyxl + pandas |
| 클래시→보고서 | Navisworks XML | 간섭 현황표 xlsx | pandas + openpyxl |
| COBie 변환 | Revit Schedule CSV | COBie.xlsx (FM) | openpyxl |
| 공정 S-커브 | MS Project XML | S-커브 차트 xlsx | pandas + matplotlib |
| 기성청구 | WBS 진행률 + 예산 | 기성 청구서 xlsx | openpyxl + 수식 |

**건설 실무 견적서 자동화 구조:**
- 갑지(표지): 공사명·발주처·금액 자동 입력

- 집계표: 공종별 소계 자동 합산
- 내역서: 물량산출서와 단가 자동 Join

- 대용량(5만 행 이상)은 write_only=True 모드로 메모리 절약한다.

2순위: pandas + openpyxl (엔진 지정)
- 데이터 집계(pivot_table, groupby)가 필요한 경우에 한해 허용한다.

3순위: C# OpenXML SDK
- Revit Add-in 내부에서 직접 .xlsx를 생성해야 할 때만 사용한다.
- Python 환경이 없는 빌드 파이프라인 전용이다.

금지: VBA/매크로 방식
- 보안 리스크(악성코드 배포 경로), 크로스 플랫폼 비호환, 유지보수 불가로 전면 금지한다.

### 패턴 1 — 표준 필터 테이블 생성
용도: 간섭 리포트, 물량 산출, 근태/비용 집계

```python
from openpyxl.worksheet.table import Table, TableStyleInfo

wb = Workbook()
ws = wb.active
ws.title = "리포트"
headers = ["공종", "레벨", "간섭수", "조치상태", "담당자", "우선순위"]
ws.append(headers)
for row in data_rows:
    ws.append(row)
# 테이블 범위 자동 계산
last_col = get_column_letter(len(headers))
last_row = ws.max_row
table = Table(displayName="리포트테이블", ref=f"A1:{last_col}{last_row}")
table.tableStyleInfo = TableStyleInfo(name="TableStyleMedium9", showRowStripes=True)
ws.add_table(table)
wb.save("output.xlsx")
```

### 패턴 4 — 인코딩 안전 CSV 입출력
용도: Revit/Navisworks 중간 포맷, 텔레그램 수신 파일 처리
```python
import csv, io
# 읽기: BOM 있는 CP949/UTF-8 모두 처리
def read_csv_safe(path):
    for enc in ("utf-8-sig", "cp949", "utf-8"):
        try:
            with open(path, newline="", encoding=enc) as f:
                return list(csv.DictReader(f))
        except (UnicodeDecodeError, LookupError):
            continue
    raise ValueError("인코딩 감지 실패")
```

## pyRevit·Revit API 직접 연동 Excel 자동화 (2026-06-07)

**pyRevit 스크립트로 Revit 스케줄 → Excel 직접 출력 (CSV 우회 없이):**
```python
# pyRevit 환경에서 실행 (Revit 내부)
from pyrevit import revit, DB
import openpyxl

doc = revit.doc
collector = DB.FilteredElementCollector(doc).OfClass(DB.ViewSchedule)

for schedule in collector:
    if schedule.Name == "배관 물량산출":
        table = schedule.GetTableData()
        section = table.GetSectionData(DB.SectionType.Body)
        rows = section.NumberOfRows
        cols = section.NumberOfColumns

        wb = openpyxl.Workbook()
        ws = wb.active
        ws.title = schedule.Name

        for r in range(rows):
            row_data = []
            for c in range(cols):
                cell = schedule.GetCellText(DB.SectionType.Body, r, c)
                row_data.append(cell)
            ws.append(row_data)

        wb.save(r"C:\BIM\Output\배관물량산출.xlsx")
        print(f"저장 완료: {rows}행 × {cols}열")
```

**공공 발주 표준 내역서 구조 (5단 계층):**
```
내역서.xlsx
├── 갑지(표지)     : 공사명, 발주처, 공사금액, 공기, 계약일자
├── 원가계산서      : 재료비·노무비·경비 합계 → 일반관리비(6%)·이윤(15%)·VAT

└── 단가표         : 재료비+노무비 기준 (표준품셈 or 거래실례가격)
```

**openpyxl 기반 표준 내역서 자동 생성:**
```python
from openpyxl import Workbook
from openpyxl.styles import Font, Alignment, PatternFill, Border, Side
from openpyxl.utils import get_column_letter

def create_boq(project_name: str, items: list[dict]) -> Workbook:
    wb = Workbook()

    # ── 갑지 ──
    ws_cover = wb.active
    ws_cover.title = "갑지"
    ws_cover["C3"] = project_name
    ws_cover["C4"] = "건설 BIM 공사"
    total = sum(i["금액"] for i in items)
    ws_cover["C5"] = f"{total:,.0f}원"

    # ── 집계표 ──
    ws_sum = wb.create_sheet("집계표")
    headers = ["공종", "재료비", "노무비", "경비", "계"]
    for col, h in enumerate(headers, 1):
        c = ws_sum.cell(1, col, h)
        c.font = Font(bold=True)
        c.fill = PatternFill("solid", fgColor="4472C4")
        c.font = Font(bold=True, color="FFFFFF")

    # ── 내역서 ──
    ws_bq = wb.create_sheet("내역서")
    bq_headers = ["항목번호", "품명", "규격", "단위", "수량", "단가", "재료비", "노무비", "경비", "금액"]
    for col, h in enumerate(bq_headers, 1):
        ws_bq.cell(1, col, h).font = Font(bold=True)

    for row_idx, item in enumerate(items, 2):
        ws_bq.cell(row_idx, 1, item.get("번호", ""))
        ws_bq.cell(row_idx, 2, item["품명"])
        ws_bq.cell(row_idx, 3, item.get("규격", ""))
        ws_bq.cell(row_idx, 4, item.get("단위", "식"))
        ws_bq.cell(row_idx, 5, item["수량"])
        ws_bq.cell(row_idx, 6, item["단가"])
        ws_bq.cell(row_idx, 10, f"=E{row_idx}*F{row_idx}")  # 금액 수식

    # 합계 행
    last = len(items) + 1
    ws_bq.cell(last + 1, 2, "합계").font = Font(bold=True)
    ws_bq.cell(last + 1, 10, f"=SUM(J2:J{last})")

    return wb

# 사용 예
boq_items = [
    {"번호": "1", "품명": "급수배관 DN50", "규격": "KSD 3597", "단위": "m",  "수량": 120, "단가": 15000},
    {"번호": "2", "품명": "공조기 AHU-1",  "규격": "10,000CMH",  "단위": "대", "수량": 1,   "단가": 4500000},
]
wb = create_boq("○○빌딩 BIM 공사", boq_items)
wb.save("내역서_자동생성.xlsx")
```

```

노무비 합계       : ΣB

순공사원가        : A+B+C
일반관리비        : 순공사원가 × 6% (건설 공사)
이    윤          : (노무비+경비+일반관리비) × 15% (상한 15%)
공사원가          : 순공사원가+일반관리비+이윤
부가가치세        : 공사원가 × 10%
총공사비          : 공사원가+부가가치세
```

**pandas chunksize 패턴 (Revit 물량 CSV 10만 행):**
```python
import pandas as pd

chunks = []
for chunk in pd.read_csv("revit_all_elements.csv",
                          chunksize=10000, encoding="utf-8-sig"):
    # 공종 필터 + 집계 — 청크 단위
    filtered = chunk[chunk["Category"].isin(["Pipe", "Duct", "Conduit"])]
    agg = filtered.groupby(["Category", "Size"])["Length"].sum().reset_index()
    chunks.append(agg)

result = pd.concat(chunks).groupby(["Category", "Size"])["Length"].sum()
result.to_excel("물량집계.xlsx", sheet_name="공종별집계")
```

### BIM-Excel 자동화 현장 실패 패턴 5가지

**1. 수량 산출서 파라미터 미일치 → 오차 발생**
- 원인: Revit 파라미터명(한글/영문 혼용)과 엑셀 템플릿 컬럼명 불일치
- 증상: 면적 데이터 0 또는 None으로 채워짐, 단위 미변환(㎡ vs ft²)
- 해결: 파라미터 매핑 딕셔너리를 코드에 고정하지 말고 JSON 설정 파일로 분리; Revit API 단위는 항상 내부단위(feet) → `UnitUtils.ConvertFromInternalUnits()` 변환 필수

**2. 엑셀 파일 잠금 → 자동화 실패**
- 원인: 이전 실행 중 예외로 Excel 프로세스 좀비 상태
- 증상: `PermissionError: [WinError 32]` 또는 `openpyxl.utils.exceptions.InvalidFileException`
- 해결: `xlwings`로 연 경우 반드시 `wb.close()` + `app.quit()`, `with` 컨텍스트 매니저 사용; 좀비 프로세스는 `os.system("taskkill /f /im excel.exe")` (Windows) 강제종료

**3. 대용량 모델 데이터 → 메모리 초과**
- 원인: 10만 요소 이상 Revit 데이터를 pandas DataFrame에 한 번에 로드
- 해결: chunked 처리 (`pd.read_csv(chunksize=5000)`) 또는 Revit API에서 필터링 후 추출

**4. 엑셀 수식 깨짐 → 자동 계산 비활성화**
- 원인: openpyxl로 셀에 값 쓰면 기존 수식 덮어씀, data_only=True 로드 시 수식 손실
- 해결: 수식 셀은 건드리지 않고 데이터 전용 시트에만 쓰기, 수식 참조 범위를 Named Range로 분리

**5. 다국어 파일명 인코딩 오류**
- 원인: 한글 파일명 포함 경로를 Windows와 macOS 간 교환 시 CP949 vs UTF-8 충돌
- 해결: 파일명은 항상 `pathlib.Path` 사용, 경로 하드코딩 금지; 저장 시 `wb.save(str(path))` 강제

```python
import json
from pathlib import Path
import pandas as pd
import openpyxl
from openpyxl.utils import get_column_letter

# 파라미터 매핑을 JSON 설정으로 관리
PARAM_MAP = json.loads(Path("param_map.json").read_text(encoding="utf-8"))
# {"면적": "ROOM_AREA", "높이": "WALL_HEIGHT", ...}

def revit_to_dataframe(elements: list[dict]) -> pd.DataFrame:
    rows = []
    for elem in elements:
        row = {}
        for col, param_key in PARAM_MAP.items():
            val = elem.get(param_key)
            # 내부 단위(feet²) → ㎡ 변환
            if "AREA" in param_key and val is not None:
                val = round(val * 0.0929, 3)
            row[col] = val
        rows.append(row)
    return pd.DataFrame(rows)

def write_to_template(df: pd.DataFrame, template_path: Path, output_path: Path):
    wb = openpyxl.load_workbook(template_path)
    ws = wb["데이터"]  # 데이터 전용 시트
    for r_idx, row in enumerate(df.itertuples(index=False), start=2):
        for c_idx, val in enumerate(row, start=1):
            ws.cell(row=r_idx, column=c_idx, value=val)
    wb.save(str(output_path))
    wb.close()
```

```python
import xml.etree.ElementTree as ET
import pandas as pd
import openpyxl
from openpyxl.chart import AreaChart, Reference
from openpyxl.chart.series import DataPoint
from datetime import datetime

def parse_msp_xml(xml_path: str) -> pd.DataFrame:
    """MS Project XML에서 Task 목록 파싱"""
    tree = ET.parse(xml_path)
    root = tree.getroot()
    ns = {"p": "http://schemas.microsoft.com/project"}
    tasks = []
    for task in root.findall(".//p:Task", ns):
        uid = task.findtext("p:UID", namespaces=ns)
        name = task.findtext("p:Name", namespaces=ns)
        start = task.findtext("p:Start", namespaces=ns)
        finish = task.findtext("p:Finish", namespaces=ns)
        pct = task.findtext("p:PercentComplete", namespaces=ns)
        cost = task.findtext("p:Cost", namespaces=ns)
        tasks.append({"UID": uid, "Name": name, "Start": start,
                      "Finish": finish, "Pct": float(pct or 0),
                      "Cost": float(cost or 0)})
    df = pd.DataFrame(tasks)
    df["Start"] = pd.to_datetime(df["Start"])
    df["Finish"] = pd.to_datetime(df["Finish"])
    return df

def build_s_curve(df: pd.DataFrame, output_path: str):
    # 월별 계획 금액 집계
    df["Month"] = df["Finish"].dt.to_period("M")
    monthly = df.groupby("Month")["Cost"].sum().reset_index()
    monthly["누적계획"] = monthly["Cost"].cumsum()
    monthly["누적실적"] = monthly["Cost"] * (monthly["Cost"].cumsum() / monthly["Cost"].cumsum().max())  # 예시

    wb = openpyxl.Workbook()
    ws = wb.active
    ws.title = "S커브"
    ws.append(["연월", "계획금액", "실적금액", "누적계획", "누적실적"])
    for _, row in monthly.iterrows():
        ws.append([str(row["Month"]), row["Cost"], row["Cost"]*0.95,
                   row["누적계획"], row["누적실적"]])

    # 면적 차트 삽입
    chart = AreaChart()
    chart.title = "공정 S커브"
    chart.style = 10
    chart.y_axis.title = "금액(천원)"
    chart.x_axis.title = "연월"
    data = Reference(ws, min_col=4, max_col=5, min_row=1, max_row=ws.max_row)
    chart.add_data(data, titles_from_data=True)
    ws.add_chart(chart, "G2")
    wb.save(output_path)

df = parse_msp_xml("project.xml")
build_s_curve(df, "S커브_공정표.xlsx")
```
