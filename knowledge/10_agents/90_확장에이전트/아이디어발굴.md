## 성장전략그룹 아이디어 선별 기준 (2026-05-28)
- Source: LUA BIM LABS group knowledge update 2026-05-28
- Tags: 성장전략그룹,ideation,productization,rnd-handoff

아이디어발굴은 [[성장전략그룹]]의 전방 탐색 역할이다. 후보는 흥미로운 기능이 아니라 [[R&D_개발지원그룹]]이 검증 가능한 문제 단위로 정리되어야 한다.

R&D 큐 후보 최소 정보:
- 고객 문제 1문장: 누가 어떤 반복 작업 때문에 시간을 잃는가.
- 현재 대안: 수작업, Dynamo, Excel, 외부 Add-in, 컨설팅 중 무엇으로 해결 중인가.
- MVP 기능: 첫 릴리스에서 반드시 되는 1~3개 기능.
- 제외 범위: AI 자동 수정, 전체 법규 판정, 클라우드 동기화 등 이번 버전에서 하지 않을 것.
- 샘플 데이터: RVT/IFC/Excel/스크린샷/보고서 중 최소 1개.
- 수익 가설: 유료 전환, 내부 공수 절감, 교육·컨설팅 연계 중 하나.

아이디어 상태값:
| 상태 | 의미 | 다음 담당 |
|---|---|---|
| Watch | 신호는 있으나 반복성 부족 | 성장전략그룹 |
| Candidate | 반복 pain point 확인 | 전략기획 |
| Spec Ready | 요구사항 초안 가능 | 요구사항분석 |
| POC Ready | 기술 검증 착수 가능 | 프로그램개발 |
| Release Candidate | 검증·패키징 단계 | 빌드검증/제품패키징 |

- 관련: [[성장전략그룹]] · [[R&D_개발지원그룹]] · [[요구사항분석]] · [[프로그램개발]]


## BIM AI 자동화 신기술 아이디어 AEC 혁신 Claude Code 심화 업데이트 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: 아이디어발굴,BIM,AI,자동화,Revit,AEC,혁신

- MEP 자동 라우팅(AI Routing): AI를 활용해 Revit 모델 내 덕트·배관·케이블 트레이 경로를 자동 생성하는 기능은 현재 시장에서 완성도 높은 솔루션이 없는 공백 영역이다. 기술 구현 난이도는 높지만 수요가 검증됐으며, MVP 범위를 단일 층 직선 구간 자동 배치로 한정하면 6개월 내 프로토타입 개발이 가능하다.
- BIM 모델 자동 품질검사(QA Automation): Revit 모델의 요소 누락, LOD 미달, 공종 간 간섭을 자동으로 검출하고 리포트를 생성하는 Add-in이다. Model Quality Auditor를 확장해 AI 기반 패턴 인식으로 "비정상 모델링 패턴"을 감지하는 기능을 추가하면 차별화된 고부가가치 제품이 된다.
- IFC 자동 변환 및 검증 도구: 공공 BIM 납품 시 IFC 4.3 기준 적합성 자동 검사 후 보고서를 생성하는 도구는 발주처와 수주사 모두에게 필요하다. buildingSMART의 IDS(Information Delivery Specification) 표준 지원 기능을 포함하면 국제 경쟁력도 확보된다.
- 자연어 기반 BIM 조회(NLP to BIM Query): "3층 공조기 용량 합산 알려줘"와 같은 자연어 질의를 Revit API 쿼리로 변환해 답변하는 Revit Assistant 기능이다. LLM + Revit API 연동으로 구현 가능하며, Telegram Bot과 연동하면 현장 실무자의 접근성이 크게 높아진다.
- Autodesk App Store 트렌드 분석: 2024~2025년 신규 등록 상위 제품은 ①자동화·배치 도구 ②품질 검사 도구 ③데이터 추출·리포팅 도구 순이다. 이 3개 카테고리에 집중하면 다운로드 수 상위 20% 진입 가능성이 높다.
- 관련: [[Revit_Addin]] · [[프롬프트엔지니어]] · [[최고전략CSO]]


## 아이디어발굴 마스터급 경험 지식 (2026-05-29)
- Source: claude-code-enhanced 2026-05-29
- Tags: 아이디어발굴,product-discovery,bim-market,validation,mvp,customer-interview

**BIM 고객 pain point 수집 방법론 — 실제 효과 순위:**
1. **App Store 리뷰 역분석**: 경쟁 제품의 1~3점 리뷰에서 "시간이 너무 걸린다", "버그가 있다", "이 기능이 없다"는 키워드를 추출. 이것이 고객이 돈을 낼 의향이 있는 문제의 가장 직접적인 증거
2. **현장 BIM 담당자 15분 인터뷰**: "Revit에서 매일 반복하는 수작업이 무엇인가?" → 도구 없이 손으로 하는 반복 작업 목록 확보. 이상적인 인터뷰 대상: 중견 건설사 BIM 코디네이터(결정권자 아닌 사용자)
3. **BIM 관련 카페·커뮤니티 질문 분석**: 오토캐드 카페, Revit 공식 포럼, BIM 실무 카카오톡 오픈채팅에서 같은 질문이 3번 이상 반복되면 충족되지 않은 수요
4. **내부 CS 티켓 분석**: 기존 Add-in 고객의 지원 요청에서 "이 기능도 되면 좋겠다" 피드백 추출 → 이미 우리 제품을 쓰는 고객의 추가 니즈는 낮은 CAC로 수익화 가능

**아이디어 → 개발 큐 승격 필터링 기준 (7가지 체크):**
- [ ] 같은 pain point를 3명 이상의 독립적인 사용자로부터 확인했는가?
- [ ] Revit API 또는 Navisworks API로 기술 구현이 가능한가? (실기 검증 필요 항목 표시)
- [ ] MVP를 단일 Revit 버전 + 단일 건물 유형으로 좁힐 수 있는가?
- [ ] 유사 기능이 App Store에 없거나, 있어도 평점 3점 이하인가?
- [ ] 무료 버전으로 설치 수 500건을 달성할 수 있는 충분한 수요가 있는가?
- [ ] CEO 승인 기준(단건 USD 10,000 이상 매출 가능성)을 충족하는가?
- [ ] Autodesk 심사 정책(성능 저하·개인정보·복제 금지)에 위반되지 않는가?

**아이디어 실패 패턴 — "좋은 아이디어"가 제품이 되지 못하는 이유:**
- 기술 흥미 우선: 개발자가 만들고 싶은 기능을 먼저 만들고 나중에 고객을 찾음 → 설치 수 30건에서 정체
- 시장이 너무 좁음: "BIM 컨설팅 회사 5곳만 사용할 수 있는 기능" — App Store에서 유료 전환 불가 규모
- 경쟁 제품 있는 줄 몰랐음: App Store를 키워드 검색 없이 분석 → 개발 완료 후 같은 기능 발견
- Autodesk 내재화: 개발 중 Autodesk가 동일 기능을 기본 탑재 발표 → 출시 의미 없어짐
- 관련: [[성장전략그룹]] · [[R&D_개발지원그룹]] · [[아이디어발굴]] · [[최고전략CSO]]
