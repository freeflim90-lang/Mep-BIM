## 1. AI로 해결 vs 사람 파견 판단 기준

> 아래 조건에 해당하면 AI만으로는 불가 → 사람 파견 검토

### 1.1 반드시 사람이 필요한 조건

| 조건 | 사유 |
|---|---|
| 발주처 대면 보고·협의 | 신뢰관계, 즉각 판단, 비언어 소통 |
| 현장 실측·현황 조사 | 물리적 접근 필요 |
| 복잡 구간 모델링 수정 | AI가 Revit을 직접 조작 불가 |
| 다공종 실시간 조율 회의 | 즉흥적 판단·협상 필요 |
| 발주처 요구 기준 해석 | 모호한 EIR 조항 판단 |
| 법적 서명·직인 필요 문서 | 책임 소재 확보 |
| 현장 품질 검토 (실측 대조) | 물리적 확인 |
| 민감한 클레임·이슈 조율 | 감정적 조율 필요 |

### 1.2 AI 처리 가능 범위 (사람 불필요)

| 업무 | AI 도구 |
|---|---|
| 클래시 검출 및 보고서 초안 생성 | LUA Add-in + Navisworks |
| 파라미터 입력 완성도 검증 | LUA Add-in |
| 명명 규칙 준수 검토 | LUA Add-in |
| BEP·제안서 초안 작성 | LUA BIM LABS 에이전트 |
| 규격·법규 조항 검색 및 요약 | AI 에이전트 |
| 물량 산출 보고서 초안 | Revit 일람표 + 스크립트 |
| 간섭 우선순위 분류 (1차) | AI 에이전트 |
| 회의록 요약 및 이슈 트래킹 | AI 에이전트 |

### 1.3 AI + 사람 협업 (하이브리드)

| 업무 | AI 역할 | 사람 역할 |
|---|---|---|
| 간섭 조율 | 클래시 목록·우선순위 생성 | 공종 담당자와 조율, 최종 결정 |
| EIR 충족 검토 | 항목별 체크리스트 생성 | 모호한 항목 판단, 발주처 확인 |
| 모델 수정 | 수정 범위 분석·지시 | 실제 Revit 작업 수행 |
| 발주처 보고 | 보고서 초안·데이터 정리 | 프레젠테이션·질의 응답 |

## 2. 연차별 역할 정의

### 2.1 주니어 BIM (경력 1~3년)

**가능한 업무:**
- 공종별 표준 모델링 (건축·구조·MEP 기본 요소)
- 발주처 기준에 따른 파라미터 입력
- 클래시 보고서 데이터 정리 및 수정 반영 작업
- BIM 뷰·시트 작성
- 지원 자료 준비 (회의 자료, 스크린샷)

**불가한 업무:**
- 발주처 직접 대응
- BIM 조율 회의 주도
- 복잡 구간 독립 판단
- EIR 해석 및 대응 방안 결정

**적합 투입 조건:**
- 단순 반복 모델링이 주된 업무
- 시니어·중급의 지도·감독 가능한 환경
- 발주처 직접 접촉 없는 내부 작업

---

### 2.2 중급 BIM (경력 4~7년)

**가능한 업무:**
- 공종 담당 BIM 모델 전체 관리
- 공종 내 클래시 1차 검토 및 수정
- 타 공종과 기술적 협의 (이메일·문서 기반)
- 발주처 자료 제출 및 기술 질의 응답 (문서)
- BIM 표준(명명 규칙·파라미터) 적용 주도
- 주니어 지도

**불가한 업무:**
- 전체 프로젝트 BIM 코디네이터
- 고위급 발주처 보고·협상
- 복수 공종 동시 코디네이션 독립 수행

**적합 투입 조건:**
- 단일 공종 담당으로 책임 있는 업무
- 시니어 BIM 코디네이터가 총괄하는 구조
- 중소규모 프로젝트 (연면적 10,000㎡ 이하)에서 공종 리드

---

### 2.3 시니어 BIM (경력 8~15년)

**가능한 업무:**
- BIM 코디네이터 (전체 공종 조율)
- 간섭 검토 회의 주도
- 발주처 중간 보고 및 기술 협의
- EIR 해석 및 대응 방안 결정
- BEP 작성·협의·승인
- 중급·주니어 기술 지도
- 복잡 구간 모델링 판단 및 지시

**적합 투입 조건:**
- 발주처 대면 보고 필요
- 중~대형 프로젝트 (연면적 10,000㎡ 이상)
- 3개 공종 이상 동시 조율 필요
- EIR 요구 수준이 높은 프로젝트

---

### 2.4 수석 BIM (경력 15년 이상)

**가능한 업무:**
- 프로젝트 BIM 총괄 책임
- 발주처 고위급 보고 및 협상
- 대형 프로젝트 BIM 전략 수립
- 복수 프로젝트 동시 품질 관리
- 제안서 기술 책임자

**적합 투입 조건:**
- 초대형 프로젝트 (연면적 50,000㎡ 이상)
- 다수 발주처 이해관계 조율 필요
- 신규 BIM 방법론 도입 프로젝트
- 투입 비중은 낮게 (10~30%), 핵심 의사결정 시점에 집중

## 5. 파견 기간 산정 기준

### 5.1 단계별 표준 파견 기간

| BIM 단계 | 연면적 10,000㎡ 기준 | 연면적 30,000㎡ 기준 |
|---|---|---|
| 기본설계 BIM | 1~2개월 | 2~3개월 |
| 실시설계 BIM | 3~5개월 | 5~8개월 |
| 시공 BIM (수시 대응) | 공사 기간 내 격주 방문 | 공사 기간 내 주 1회 방문 |
| 준공 BIM | 1~2개월 | 2~3개월 |

### 5.2 투입률 산정

> 투입률 = 해당 프로젝트에 집중하는 시간 비율

| 상황 | 투입률 | 비고 |
|---|---|---|
| 단일 프로젝트 전담 | 80~100% | 납기 촉박 시 |
| 복수 프로젝트 병행 (2개) | 40~60% 각 | 시니어 이상만 권장 |
| 복수 프로젝트 병행 (3개 이상) | 30% 이하 각 | 수석만 권장, 품질 리스크 |
| 시공 단계 간헐적 지원 | 10~30% | 정기 방문 기반 |

## 6. 파견 불가 / 재검토 조건

아래 조건이 확인되면 파견 전 조율차장·CEO와 재협의한다.

```
□ 요구 연차 대비 가용 인력 없음 → 외주 BIM 전문가 풀 검토
□ 투입률 합계가 1인당 120% 초과 → 인력 추가 또는 납기 조정
□ 발주처 요구 기술(특수 소프트웨어)을 보유 인력 없음 → 교육 or 파트너사 협업
□ 현장이 해외 → 출장 비용·비자·체류 기간 별도 산정
□ 프로젝트 기간 3개월 미만 + 대형 규모 → 리소스 과부하 위험 플래그
```

## 8. 팀 구성 분리 파견 판단 로직

> 전제: LUA BIM LABS는 **건축·구조팀**과 **MEP팀** 두 팀 단위로 파견한다.
> 프로젝트에 따라 두 팀 모두 파견, 한 팀만 파견, 또는 전체 AI 처리 중 하나를 선택한다.

### 8.1 팀 정의

| 팀 | 담당 공종 | 핵심 역량 |
|---|---|---|
| **건축·구조팀** | 건축, 구조, 토목, 인테리어 | 건축 설계 이해, 구조 개구부 협의, 공간 계획 |
| **MEP팀** | 공조, 위생, 소방기계, 소방전기, 전기, 통신 | 계통 설계, 간섭 조율, 법규 검토 |

### 8.2 파견 구성 선택 트리

```
[수행 범위 확인]
        │
        ├─ 건축·구조 공종만 포함?
        │       └─ YES → 건축·구조팀 단독 파견
        │
        ├─ MEP 공종만 포함?
        │       └─ YES → MEP팀 단독 파견
        │
        ├─ 건축·구조 + MEP 모두 포함?
        │       │
        │       ├─ 연면적 5,000㎡ 미만 + 공종 5개 이하?
        │       │       └─ YES → 1팀 겸임 가능 여부 검토 (아래 8.3 참조)
        │       │
        │       └─ 연면적 5,000㎡ 이상 또는 공종 6개 이상?
        │               └─ 두 팀 동시 파견 원칙
        │
        └─ BEP·보고서 작성·AI 검증만?
                └─ 파견 없음, AI + Add-in 처리
```

### 8.3 겸임 가능 판단 기준 (소규모 단독 파견)

아래 조건을 **모두** 충족할 때만 1팀이 두 영역 겸임 가능.

| 조건 | 기준 |
|---|---|
| 연면적 | 5,000㎡ 미만 |
| 공종 수 | 5개 이하 |
| BIM 단계 | 기획 또는 기본설계 (LOD 200 이하) |
| 발주처 대면 보고 | 없음 또는 분기 1회 이하 |
| 개보수 여부 | 신축 또는 기존 BIM 제공 |

> 조건 하나라도 미충족 → 전문 팀 분리 파견.

### 8.4 팀별 단독 파견 적합 케이스

#### 건축·구조팀 단독

| 케이스 | 이유 |
|---|---|
| 구조 개보수·내진 보강 (MEP 미변경) | MEP 계통 미개입 |
| 건축 인허가용 BIM (공간·외피만) | MEP LOD 불필요 |
| 증축 건축계획 검토 (MEP는 별도 팀이 처리) | 역할 분리 명확 |
| 현황 실측 BIM (As-built 건축·구조) | MEP 현황은 다른 주체 |

#### MEP팀 단독

### 비정형 건물 파견 시 전문 인력 요건

정형 건물 BIM 모델러와 달리 비정형 건물에는 추가 역량 필수:

| 역량 | 정형 건물 | 비정형 건물 |
|------|---------|-----------|
| Revit 모델링 | 필수 | 필수 |
| Dynamo 스크립트 | 선택 | **필수** (곡면 자동화) |
| Grasshopper/Rhino | 불필요 | **필수** (매스 설계 연동) |
| IFC Export 설정 | 기본 이해 | **심화** (Entity 수동 매핑) |
| 파라메트릭 설계 이해 | 불필요 | **필수** |

> 비정형 프로젝트 파견 인력은 포트폴리오에 곡면 모델 사례 최소 1건 확인 필수

### 파견 실패 패턴 4가지

**1. 역량 미확인 후 파견 → 재파견 비용 발생**
- 원인: 이력서 기재 "Revit 5년" 믿고 파견 → Dynamo/MEP 설정 불가
- 해결: 파견 전 역량 체크: 실기 테스트 (샘플 모델 완성, 시간 제한 2시간)

**2. 발주처 CDE 접근 지연 → 착수 1~2주 낭비**
- 원인: CDE 계정 신청 절차 사전 파악 없음
- 해결: 파견 2주 전에 CDE 계정 신청 완료, VPN 접속 테스트까지 확인

**3. 핵심 인력 중도 이탈 → 인수인계 미흡**
- 해결: 파견 인력이 모델 관리 담당 시 인수인계 체크리스트 필수 (모델 구조, 패밀리 위치, 워크셋 기준) + 공동 모델 관리 2인 체계

**4. 비정형 건물 파견 후 생산성 저하**
- 원인: 비정형 모델링 경험 없이 파견 → 예상 공수 3배 초과
- 해결: 비정형 프로젝트는 주니어 단독 파견 금지; 시니어 + 주니어 조편성 필수

```
□ 필수 역량 (정형 건물)
  - [ ] Revit 기본 모델링 (벽/슬라브/지붕/창호)
  - [ ] MEP 시스템 생성 및 배관/덕트 라우팅
  - [ ] Navisworks Clash Detection 실행
  - [ ] IFC 내보내기 기본 설정

□ 추가 역량 (비정형 건물)
  - [ ] Dynamo 스크립트 작성 (자유 곡면 분할)
  - [ ] Grasshopper → Revit 임포트 경험
  - [ ] IFC Entity 수동 매핑 경험
  - [ ] 포트폴리오 내 곡면 구조물 사례 확인
```

- 계약 조건에서는 프로젝트 규모와 기간에 따른 인력 투입 및 교육 비용을 명시적으로 포함해야 합니다. 또한, 작업 성과를 평가하는 KPIs(Key Performance Indicators)를 설정하여 품질 관리 체계를 구축해야 합니다.
- BIM 인력의 역량 기준으로는 3D 모델링, 4D 시뮬레이션, 프로젝트 관리(PM), 협업 도구 사용 능력 등이 필수적입니다. 또한, 지속적인 교육과 업데이트를 통해 최신 BIM 기술을 익히는 것이 중요합니다.
