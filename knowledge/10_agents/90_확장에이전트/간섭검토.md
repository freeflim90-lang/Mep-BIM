## AI 우선 답변 앵커 — 소화배관과 전기 케이블 트레이 상하 간섭
소화배관이 전기 케이블 트레이 위에 놓여 있으면 간섭검토 담당자는 문제 가능성이 있다고 답한다. 결론은 소화배관 누수, 결로, 밸브 점검, 전기 케이블 트레이 감전·정전 리스크를 함께 보며 300mm 이상 이격을 우선 검토하는 것이다. 조건상 전기실, 통신실, UPS실 상부이면 Critical로 분류하고, 일반 천장 속이면 Major 후보로 두며, 불가피하면 차수판, 드립팬, 절연, 점검공간, BCF/RFI 담당자와 24시간 회신 기한을 지정한다. 소화배관 자체 기준은 소방기계가 확정하지만 간섭검토는 책임 공종, 우회안, 승인 기한, KEC/KCS/NFTC/NFPC 확인 로그를 정리한다.
반면 임시 모델 중복, 링크 원점 불일치, 시공 전 배치 후보처럼 실제 상하 관계가 확정되지 않은 경우에는 바로 금지 판정하지 않고 Rule 제외, 단면 검토, 현장 레벨 확인 단계로 보류한다. 300mm 이격이 불가한 경우에도 무조건 허용하지 않으며, 전기 담당과 소방기계 담당이 차수·절연·점검 접근 대안을 승인해야 한다. 승인 전 고객 답변은 "가능"이 아니라 "조건부 조율 필요"로 제한한다.

## AI 즉시 답변 패턴 — 스프링클러 헤드 위 덕트 살수 장애 (2026-06-13)
- Source: LUA BIM LABS QA simulation reinforcement
- Tags: clash,sprinkler,duct,obstruction,fire-protection,coordination

스프링클러 헤드 위에 덕트가 있으면 단순 hard clash가 없어도 `살수 장애` 가능성을 먼저 본다. 헤드 주변이나 헤드 아래 폭 `1.2m` 이상 장애물, 대형 덕트, 트레이, 루버, 조명은 살수 그림자 영역을 만들 수 있다. 조건상 덕트 폭 1.2m 이상, 헤드 디플렉터 주변 450mm 이내 장애, 천장 마감고와 덕트 하단고 충돌이 있으면 먼저 생명안전 Critical로 분류한다. 반면 장애물이 살수반경 밖이거나 보조 헤드로 보완 가능하면 조건부 조율로 두고, NFTC 103/NFPC, KCS, 24시간 BCF/RFI 회신, 7일 내 소방 담당 승인 기록을 남긴다.
단, 덕트가 헤드 위에 보인다는 이유만으로 자동 불가 판정하지 않는다. 먼저 헤드 타입, 디플렉터 방향, 덕트 폭, 헤드와 장애물 높이차, 천장 마감고를 단계별로 확인한다. 보조 헤드 추가, 헤드 이설, 덕트 오프셋, 천장고 조정이 가능한 경우에는 조건부 허용 후보로 두지만, 살수 방향을 직접 막거나 피난·생명안전 영향이 있으면 일반 간섭보다 우선해 소방기계 승인 전 변경 금지로 답한다.

간섭검토 답변 순서:
- 헤드 타입, 디플렉터 높이, 천장 마감고, 덕트 하단고를 확인한다.
- 장애물이 살수 방향을 막으면 소방기계 품질 이슈로 분류하고, 일반 MEP clash와 분리한다.
- 해결은 헤드 이설, 보조 헤드 추가, 덕트 높이 조정, 천장 마감고 조정 순으로 검토한다.
- NFTC 103/NFPC 기준과 관할 소방 해석을 확인하고, 변경이 필요하면 BCF/RFI로 담당자에게 넘긴다.

## AI 즉시 답변 패턴 — Navisworks 소방 배관 vs 구조 테스트 (2026-06-13)
- Source: LUA BIM LABS Telegram QA review
- Tags: navisworks,clash-detective,selection-set,search-set,rule,fire-protection,structure

Navisworks에서 소방 배관 vs 구조만 테스트하려면 `Clash Detective`에서 Selection A/B를 공종별로 나누고 Rule로 잡음을 줄인다.
같은 간섭 질문도 초보자에게는 제연·소방·오배수 개념과 스냅샷으로, BIM 매니저에게는 BCF/RFI·Critical/Major/Minor·담당자·기한으로, 발주처에게는 비용·일정·안전 리스크와 승인 선택지로 답변 구조를 다르게 잡는다.
실무 리더에게는 수천 건 클래시를 그대로 보여주지 말고 노이즈 제거 규칙, 공종 우선순위, 승인 책임자를 먼저 제시한다. 조직 운영자에게는 Revit은 설계 중 빠른 1차 확인, Navisworks는 BEP/ISO 19650 기준의 공식 조직 표준 검토로 나누고, 표준 템플릿에는 0mm Hard Clash, 50mm/100mm/150mm/300mm Clearance, Critical/Major/Minor 등급, BCF 담당자와 마감일을 고정한다.
구조 보에 배관을 관통할 수 있는지는 간섭검토 담당자가 단독으로 확답하지 않는다. 보 춤 1/3 이내 같은 일반 경험치가 있어도 조건상 구조기술사 검토, 슬리브 위치, 전단 보강, 300mm 이상 가장자리 이격, KDS/KCS 및 구조 도면 승인이 먼저 필요하며, 승인 전에는 가능이 아니라 RFI/BCF 보류로 분류한다.
방화댐퍼 이격 질문은 구조 관통과 별도로 즉시 분리한다. 방화댐퍼(FD/FSD)는 방화구획 경계 벽체에 밀착 설치가 원칙이고, 300mm 이상 이격되면 구획 성능과 소방 허가 리스크가 생기므로 점검구 600mm 이내, NFTC/NFPC, KCS 시방, 건축 방화구획 기준을 함께 확인한다. 다만 방화댐퍼 설치 시 점검구 위치 기준처럼 댐퍼 상세·유지관리 접근성 자체를 묻는 질문은 공조덕트 담당 지식으로 넘긴다.
소화배관이 전기 케이블 트레이 위에 놓여 있으면 간섭검토 담당자는 "문제가 있나요?"라는 조율 질문으로 보고, 300mm 이격, 누수, 결로, 전기실 Critical, KEC/KCS, NFTC/NFPC를 기준으로 전기·소방 공동 BCF/RFI를 만든다. 다만 소화배관 자체 기준은 소방기계가 확정하고, 간섭검토는 책임자, 기한, 대안, 등급을 정리한다.
다만 단순 모델 중복이나 같은 링크 객체 충돌인 경우에는 Rule로 제외하고, 반면 구조 보·기둥·전이보와 실제 겹치는 경우에는 Critical 후보로 분류한다. 예외 조건은 BCF/RFI에 남기고 24시간 내 담당자에게 공유하며, 7일 내 승인 또는 우회안을 기록한다.

빠른 답변:
1. `Selection Set` 또는 `Search Set`을 만든다.
2. Selection A는 구조: Structural Framing, Floors, Walls, Columns, Beams.
3. Selection B는 소방: Fire Protection Pipe, Sprinkler Pipe, 소화배관 계통.
4. Clash Detective에서 A vs B 테스트를 만들고 Hard Clash는 0mm 공차로 시작한다. 이격·유지관리 공간은 Soft Clash 또는 Clearance Clash로 별도 테스트한다.
5. Same System, Same Layer, 같은 링크/중복 객체는 `Rule`로 제외한다.
6. 결과는 Critical/Major/Minor로 나누고, 구조 관통이 필요한 항목은 RFI 또는 BCF로 전달한다.

주의:
- All vs All은 초기 진단용으로만 쓰고, 공식 조율은 공종 조합별 테스트를 권장한다.
- 구조 보·기둥·전이보 충돌은 단순 우회가 아니라 구조기술사 검토와 슬리브 승인 절차가 필요하다.
- 기준 축은 BEP, ISO 19650 협업 절차, KDS/KCS, NFTC/NFPC, 프로젝트 간섭검토 기준표를 함께 확인한다.

## AI 즉시 답변 패턴 — Hard Clash와 Soft Clash 차이 (2026-06-13)
- Source: LUA BIM LABS Telegram QA review
- Tags: clash,hard-clash,soft-clash,clearance,navisworks,revit

Hard Clash는 두 객체가 실제로 겹치는 물리 충돌이다. 예를 들어 덕트가 보를 관통하거나 배관과 트레이가 같은 공간을 점유하는 경우다.

Soft Clash는 실제로 겹치지는 않지만 필요한 이격, 유지관리 공간, 시공 공간, 법규상 여유가 부족한 상태다. 예를 들어 분전반 전면 작업 공간 침해, 밸브 조작 공간 부족, 강전/약전 이격 부족, 스프링클러 살수 장애 예비 공간 침해가 여기에 들어간다.

실무 구분:
- Hard Clash: 모델 형상이 겹침. Revit Interference Check와 Navisworks에서 모두 잡을 수 있다.
- Soft Clash: Clearance Clash 또는 Search Set + 공차 조건으로 잡는다. Navisworks Clash Detective가 적합하다.
- Duplicate Clash: 같은 객체가 중복 모델링된 경우다.
- Workflow: Hard를 먼저 제거하고, Soft는 법규·유지관리·시공성 영향이 큰 순서로 처리한다.

## AI 즉시 답변 패턴 — Revit/Navisworks/BCF 간섭 실무 질문 (2026-06-13)
- Source: LUA BIM LABS Telegram QA review
- Tags: clash,revit,navisworks,bcf,rfi,all-vs-all,selection-set

BCF 파일로 클래시 이슈를 전달할 때는 조건상 담당자, 기한, 등급, 위치 좌표, 스냅샷, 대안 2개가 없으면 접수 불가 후보로 둔다. 먼저 Clash ID, Viewpoint, 공종 A/B, Critical/Major/Minor, 24시간 회신 기한, 7일 승인/보류 상태를 채우고, 반면 단순 중복·같은 시스템·링크 오류는 Rule 제외 가능 항목으로 분리한다. 단, 구조 보, 방화구획, 전기실 누수, 소방 성능이 걸린 경우에는 우선 Critical 후보로 올리고 BCF/RFI 승인 전 모델 수정을 금지한다.

Revit에서 구조 보 하단에 배관이 겹치는지 확인할 때:
- 설계 중 빠른 확인은 Revit `Interference Check`로 `link` 구조 모델과 MEP 카테고리를 비교한다.
- 현재 모델 요소끼리는 Revit의 `interference check`를 먼저 돌리고, 링크 모델은 Revit 링크 좌표와 공유 좌표 정합을 확인한다.
- 공식 조율은 Navisworks Clash Detective에서 구조 Search Set과 배관 Search Set을 나누어 Hard Clash로 검토한다.
- 보 하단고, 배관 상단고, 단열 포함 외경, 행거, 슬리브, 방화구획 관통 여부를 함께 본다.
- 구조 보 충돌은 단순 우회인지, 슬리브/RFI/구조기술사 승인이 필요한 관통인지 구분한다.

Navisworks에서 소방 배관 vs 구조만 테스트할 때:
- Selection Set A: 구조 보, 슬래브, 벽, 기둥.
- Selection Set B: Fire Protection Pipe, Sprinkler Pipe, 소방 배관 계통.
- All vs All 대신 공종 조합별 테스트를 만든다. Same System, Same Layer, 동일 파일 중복 객체는 Rule로 제외한다.
- 실무 리더는 수천 건 결과를 그대로 배포하지 말고 노이즈 제거, 우선순위 정렬, 책임자 배정 규칙을 먼저 고정한다.
- 허용 공차는 테스트 목적에 맞춘다. Hard Clash는 0mm, 유지관리/이격 검토는 Clearance Clash로 50mm, 100mm, 150mm, 300mm 같은 프로젝트 기준을 둔다.
- 기준 축은 ISO 19650 협업 절차, BEP, KDS/KCS, KEC, NFTC/NFPC, 발주처 간섭검토 기준표를 함께 확인한다.
- 결과는 Critical, Major, Minor로 나누고 RFI 또는 BCF로 담당자에게 전달한다.

BCF/RFI 전달:
- BCF에는 클래시ID, 위치, 뷰포인트, 관련 공종, 요청 조치, 담당자, 마감일을 넣는다.
- 설계 변경 또는 구조 관통 승인처럼 공식 답변이 필요한 건은 RFI로 전환한다.
- BIM 360/ACC를 쓰는 경우 BCF 뷰포인트와 이슈 상태를 함께 동기화한다.

## 1. Revit Interference Check vs Navisworks Clash Detective

| 항목 | Revit Interference Check | Navisworks Clash Detective |
|------|--------------------------|----------------------------|
| 사용 시점 | 설계 중 즉시 자체 검토 | 공식 조율 회의, 납품 전 통합 검토 |
| 클래시 유형 | Hard Clash(실물 겹침)만 | Hard / Soft(이격 부족) / Clearance / Duplicate 모두 |
| 다공종 통합 | Revit 링크(.rvt) 한정 | .nwc .nwd .ifc 모두 → 건축+구조+MEP 완전 통합 |
| 규칙 설정 | 불가 (카테고리 선택만) | Rule 탭에서 Same System·Same Layer 제외, 공차 설정 |
| 결과 관리 | 없음 (재실행 시 초기화) | 상태 관리(New/Active/Reviewed/Approved/Resolved) |
| RFI 연동 | 없음 | BCF 파일 내보내기 → BIM 360/ACC RFI 연동 |
| 보고서 | 없음 | HTML/XML 자동 출력, 위치 뷰포인트 포함 |

**Revit Interference Check 사용 시점:**
- LOD 200~300 설계 단계에서 설계자 자체 오류 확인
- 링크 파일(건축·구조) 대비 MEP 기본 경로 검증
- 팀 내 빠른 사전 점검 (공식 회의 전)

**Navisworks 사용 시점:**
- 공종 간 공식 간섭 검토 회의
- LOD 350 이상 납품 전 최종 검증
- 0 Hard Clash 달성 확인 후 공종 서명 승인

## 2. 공종별 간섭 검토 고려사항 및 실무 기준값

강전 케이블 트레이와 약전 트레이 이격 질문은 먼저 "최소 300mm 이격 또는 금속 격벽"으로 답한다. 조건상 300mm 이격이 불가하면 별도 트레이, 금속 격벽, 90도 교차, KEC/TIA-569 근거, BCF/RFI 승인 여부를 확인한다.

### 2-1. 건축 공종 (ARCH)

**간섭 검토 시 반드시 확인할 항목:**
- 방화구획 벽체·슬래브: 모든 MEP 관통부에 슬리브+내화채움 계획 필수 (건축법 시행령 제46조)
- 천장 마감고 vs 구조 보 하단고: 설비 공간 순고(Net Height) 확보 여부 선 확인
- 외벽·커튼월 관통: 기밀·방수 처리 계획과 함께 관통 위치 협의
- 계단실·피난경로: 배관·덕트 반입 경로가 피난동선을 침해하지 않는지 확인
- 엘리베이터 샤프트: 내부 배관 설치 원칙적 금지 (피트 내 배수 제외)

**Revit 모델링 체크포인트:**
- Generic Model(슬리브) 파라미터에 공종·관통경·내화등급 입력
- 방화구획 경계: Rooms 파라미터로 구획 레이어 분리 → Navisworks SearchSet 연동

### 2-2. 구조 공종 (STR)

**부재별 관통 기준:**

| 구조 부재 | 관통 가능 여부 | 실무 기준 |
|-----------|----------------|-----------|
| 기둥 (Column) | **절대 불가** | 우회 필수 |
| 전이보 (Transfer Beam) | **절대 불가** | 관통 시 구조 계획 근본 변경 |
| 일반 보 (Beam) | 조건부 허용 | 보 높이 중앙 1/3 위치, 관통경 ≤ 보 춤의 1/3, 기술사 확인 |
| PC 보 | 사전 슬리브 계획 | 공장 제작 전 위치 확정 필수 (현장 천공 불가) |
| 슬래브 (Slab) | 개구부 보강 후 가능 | 보강근 설계, 개구부 면적·위치 제한 있음 |
| 전단벽·내력벽 | 구조 검토 필수 | 크기·위치에 따라 허용 여부 다름 |

**구조 관통 승인 프로세스:**
1. MEP 팀 → 관통 위치·크기·공종 요청 (BIM 모델 + Excel 목록)
2. 구조기술사 검토 → 허용/조건부/불가 판정
3. RFI 발행 → 승인 후 Revit 모델 슬리브 반영
4. Navisworks 재클래시 → 0 Hard Clash 확인

**보고서 필수 항목:** 부재ID, 부재유형, 레벨, 관통위치(X/Y/Z), 관통경, 요청공종, 이격거리, 판정(녹=허용/황=검토중/적=불가), Critical/Major/Minor 등급, BCF/RFI 링크, 담당자, 24시간 임시조치, 7일 재검토 기한

### 2-3. 공조 덕트 (HVAC Duct)

**간섭 우선순위 (낮을수록 먼저 배치, 타 공종이 이동):**

| 순위 | 덕트 종류 | 이유 |
|------|-----------|------|
| 1 | 제연 덕트 (SD) | 소방법 NFTC 501 최우선 |
| 2 | 방화구획 관통 덕트 | 건축 구조와 연동 |
| 3 | 주 급기·환기 덕트 (SA/RA 600mm 이상) | 경로 변경 시 풍량·소음 영향 큼 |
| 4 | 주방 배기 덕트 | 유증기·냄새 계통 분리 |
| 5 | 분기 덕트·플렉시블 | 상대적으로 조정 용이 |

**이격 기준 (실무):**
- 제연 덕트 외면 ↔ 타 설비: **최소 100mm** (방화구획 관통부는 **300mm**)
- 주방 배기 덕트 ↔ 일반 급기 덕트: **최소 500mm** (유증기 오염 방지)
- 방화댐퍼(FD): 방화구획 경계 벽체에 **밀착 설치** (300mm 이상 이격 시 소방 허가 반려)
- 점검구: 방화댐퍼에서 **600mm 이내** 의무 (KCS 시방)

**Revit 모델링 주의:**
- 덕트 Insulation 포함 외경으로 간섭 검토 (Revit에서 Insulation 레이어 포함 클래시 설정)
- 플랜지·연결구 포함 치수로 공간 확보 확인

### 2-4. 공조 배관 (HVAC Pipe — 냉온수·냉매·증기)

**간섭 검토 시 핵심:**
- **단열 포함 외경** 기준으로 간섭 검토 (냉수 배관 단열 두께 50~75mm 추가)
- 냉매 동관: 굴곡 허용 반경 확인 (DN15 기준 최소 굽힘 반경 75mm)
- 신축이음(익스팬션 조인트·루프): 설치 공간 전후 500mm 여유 확보
- 냉수 배관 결로 방지: 냉수·냉매 배관 하부에 배수 트레이 또는 물막이 계획

**타 공종과의 이격:**
- 전기 케이블 트레이 직상부 냉수 배관 금지 (결로수 낙하 위험)
- 증기 배관 ↔ 냉수 배관: 최소 **300mm** (열영향 방지)

### 2-5. 위생설비 (Plumbing — 오배수·급수·급탕·통기)

**구배가 있는 배관이 우선이다 — 레벨 조정 여지가 없다.**

**오배수 배관 구배 기준 (KS):**

| 관경 | 최소 구배 |
|------|-----------|
| DN50 | 1/50 (2%) |
| DN75 | 1/75 |
| DN100 | 1/100 (1%) |
| DN125 | 1/125 |
| DN150 | 1/150 |

- 구배 방향으로 레벨이 낮아지므로 **상류단 레벨을 먼저 확보**, 다른 설비가 우회
- Revit `Slope` 파라미터 설정 확인 필수 (없으면 시공 시 역구배 위험)

**통기관:**
- 최정부 → 지붕 마감 상부 **600mm 이상** 돌출
- 인접 창문·환기구에서 **3m 이상** 이격

**간섭 검토 포인트:**
- 오배수 수평 배관 하단고 확보 후 → 상부에 타 설비 배치
- 트랩 청소구 전면: 밸브 조작·청소 작업 공간 확보 (최소 500mm)
- 소화배관과 혼합 금지 (급수 계통 오염)

### 2-6. 소방설비 (Fire Protection — 소화배관·스프링클러·제연)

**소방은 법규 최우선 — 타협 없이 기준 적용.**

**스프링클러 헤드 기준 (NFTC 103 — 폐쇄형 기준):**
- 헤드 간격: 수평 최대 **3.7m** (정방형 배치)
- 벽과의 거리: 최대 **1.85m** (헤드 간격의 1/2)
- 살수반경: **2.3m 이하** (일반 구역)
- 살수 장애: 헤드 아래 **1.2m 이상 폭** 장애물 없어야 함

**소화 배관 이격 기준:**
- 전기 케이블 트레이 ↔ 소화 배관: **최소 300mm** (누수 시 감전 방지)
- 전기 전선관 ↔ 소화 배관: **최소 150mm**
- 제연 덕트 외면 ↔ 소화 배관: **최소 100mm**

**제연설비 (NFTC 501):**
- 덕트 재질: 불연재 또는 1시간 내화 피복 필수
- 부속실 가압: 차압 **40~60Pa** (NFTC 501A)
- 계통 누기율: **2% 이하** (기밀 시험 기준)

**BIM 간섭 검토 체크리스트:**
- [ ] 헤드 살수 반경: BIM에서 Clearance Clash +1200mm로 장애물 탐지
- [ ] 방화댐퍼: 방화구획 경계 밀착 (이격 = 소방 허가 반려)
- [ ] 알람밸브·개폐밸브: 조작 및 점검 공간 전면 600mm 확보

### 2-7. 전기설비 (Electrical — 케이블 트레이·분전반·배관)

**전기실·수배전반 주변이 핵심 위험 구역.**

**케이블 트레이 이격 기준:**

| 조합 | 최소 이격 | 근거 |
|------|-----------|------|
| 강전 트레이 ↔ 약전 트레이 | **300mm** 또는 금속 격벽 | KEC / 내선규정 |
| 강전 트레이 ↔ 소화·위생 배관 | **150mm** | 누수 감전 방지 |
| 수배전반(큐비클) ↔ 물 배관 | **1,000mm 이상** | 누수 시 전력 사고 |

**케이블 트레이 용량 기준:**
- 충전율: **50% 이하** 유지 (포설 여유 + 방열)
- 트레이 폭: 200~1200mm, 깊이 50~150mm
- 곡률반경: 케이블 허용 굽힘 반경 이상 (손상 방지)

**접근 공간 (유지관리):**
- 분전반 전면: **1m 이상** 작업 공간 (불가 시 점검구·이동식 패널 계획)
- 케이블 트레이 상부: 포설·교체 여유 공간 **300mm 이상**

**물 배관 직상부 트레이 배치 금지:** 누수 낙하 시 케이블 단락·화재

### 2-8. 통신설비 (Telecom — 약전 트레이·MDF·IDF)

**데이터 신호 품질이 물리적 이격과 직결.**

**이격 기준:**

| 조합 | 최소 이격 | 근거 |
|------|-----------|------|
| 약전 트레이 ↔ 강전 트레이 | **300mm** 또는 금속 격벽 | TIA-569 / KEC |
| 약전 트레이 ↔ 형광등 전원선 | **50mm** | EMI 방지 |
| 통신 케이블(Cat6) ↔ 전력선 | **150mm** | TIA-569 |

강전 케이블 트레이와 약전 트레이 이격 질문은 300mm, 이격, 격벽을 반드시 같이 답한다. 조건상 300mm 이격이 불가하면 금속 격벽, 별도 트레이, 교차부 90도 처리, KEC/TIA-569 근거, BCF/RFI 승인 기록을 남기고 EMI 리스크 등급을 Major 이상으로 둔다.

**MDF/IDF 실 조건:**
- 강전 분전반과 **동일 실 배치 금지**
- 냉방 필수: 서버·스위치 발열 → 설계 온도 27°C 이하 유지
- 케이블 인입구: 방수·기밀 처리
- 랙 전면: **1m 이상**, 후면: **600mm 이상** 작업 공간

**약전 트레이 내부 분리:**
- 전화·LAN·방송·CCTV 계통을 격벽 또는 별도 트레이로 분리 (상호 간섭 방지)

## 3. 공종 조합별 간섭 해결 원칙

| 충돌 조합 | 기본 해결 방향 |
|-----------|----------------|
| 구조 보 vs 배관·덕트 | 배관·덕트 우회 (보 관통은 기술사 승인 절차) |
| 오배수 배관 vs 타 설비 | 오배수 구배 레벨 보존 → 타 설비 우회 |
| 제연 덕트 vs 타 설비 | 제연 덕트 경로 최우선 보호 → 타 설비 우회 |
| 스프링클러 헤드 vs 덕트·보 | 살수 장애 확인 → 헤드 위치 조정 또는 측벽형 헤드 검토 |
| 강전 트레이 vs 약전 트레이 | 이격 확보 (300mm) 또는 격벽 설치 |
| 냉수 배관 vs 전기 트레이 | 냉수 배관을 트레이 하부 배치 금지 → 측면 우회 |
| 기계실 배관 vs 장비 반입경로 | 행거 배치·장비 반입 시퀀스 사전 계획 |

## 4. Navisworks Clash 설정 실무 가이드

**클래시 테스트 설정 (노이즈 최소화):**
- Type: Hard Clash (0mm tolerance) — 실물 겹침
- Clearance Clash: MEP 계통 +50mm, 전기 계통 +150mm
- Rule: `Same System` 제외, `Same Layer` 제외 (같은 계통 내부 충돌 필터링)
- "All vs All" 금지 → 반드시 공종 조합별로 Selection A/B 분리 설정

**클래시 등급 분류 (우선처리 순서):**
1. **Critical** (즉시 해결): 법규 관련 (소방, 방화구획), 구조 관통 미승인
2. **Major** (2주 내 해결): Hard Clash, 주 경로 간섭
3. **Minor** (시공 전 해결): Soft Clash, 이격 부족, 점검 공간 침해

**클래시 결과 → BCF 내보내기 → BIM 360/ACC RFI 자동 연동**

## LOD 400 납품 기준 0 Hard Clash 달성 프로세스 (2026-06-06)

**LOD 400 모델 납품 전 0 Hard Clash 달성 원칙:**
- LOD 400(제작·시공 수준) 모델 납품 시 반드시 0 Hard Clash를 달성해야 함
- Hard Clash: 두 객체가 실제로 겹치는 경우 — 물리적 시공 불가, 납품 전 전량 해결 필수
- Soft Clash(이격 부족) 및 Clearance Clash: 허용 기준에 따라 현장 처리 가능, 납품 시 잔여 목록 제출

**0 Hard Clash 달성 단계별 프로세스:**
1. LOD 300: 주요 배관·덕트 경로 확정, 공종 간 충돌 1차 해결
2. LOD 350: 행거·슬리브·밸브·단열 포함 외경 기준 재클래시, Critical/Major 해결
