# External AI Disclosure Rule Draft - 2026-06-05

문서상태: CEO/법무/보안관 승인 전 초안  
관련 케이스: `cases/AITEST_20260605_006.md`  
목적: 개인정보, 고객 데이터, 프로젝트 정보가 포함된 AI 협업 세션에서 외부 AI 사용 가능 여부를 단일 기준으로 정리한다.

## 1. 임시 결론

CEO, 법무조항검토, 라이선스_보안관이 별도 승인하기 전까지 고객/프로젝트/개인정보/계약 데이터가 포함된 자료는 외부 AI에 전송하지 않는다.

| 데이터 유형 | 기본 상태 | 외부 AI 사용 |
|---|---|---|
| 고객 모델 원본 `.rvt/.ifc/.nwc` | Confidential | 금지 |
| 고객 로그, Journal, 오류 리포트 원문 | Confidential | 금지 |
| 고객명, 프로젝트명, 담당자 연락처, 계약/가격 정보 | Confidential | 금지 |
| 내부 경로, 계정, 토큰, 라이선스 키 | Confidential | 금지 |
| 공개 자료 또는 회사가 직접 작성한 일반 전략 메모 | Internal/Public | 필요 시 별도 검토 |
| 민감정보 제거가 검증된 요약본 | Internal | `REDACTED_REVIEW` 후보, 승인 전 사용 금지 |

## 2. 표준 상태값

| 상태값 | 적용 기준 | 고객/외부 전송 |
|---|---|---|
| LOCAL_ONLY | 민감 신호가 하나라도 포함됨 | 외부 AI 전송 금지 |
| REDACTED_REVIEW | 민감정보 제거 증거가 있고 보안관/법무가 검토 후보로 인정 | 승인 전 전송 금지 |
| APPROVED | 보안관, 법무, 필요 시 CEO가 목적/항목/보관/학습 여부를 승인 | 승인 범위 안에서만 가능 |
| BLOCKED | 금지 데이터, 동의 없음, 제공자 조건 미확인 | 전송 금지 |

## 3. 고객 응대 문구 초안

고객에게는 아래 원칙으로 답변한다.

> 고객 모델 원본, 프로젝트명, 담당자 정보, 계약 정보, 민감 로그는 외부 AI로 전송하지 않습니다. 오류 재현에는 필요한 최소 정보만 요청하며, 원본 모델이 필요한 경우 목적, 보관 기간, 접근 담당자, 파기 방법을 별도로 안내하고 내부 보안 승인 후 처리합니다.

외부 AI 분석 요청을 받은 경우:

> 현재 정책상 고객/프로젝트/개인정보가 포함된 원문은 외부 AI에 입력하지 않습니다. 필요한 경우 내부 Local-only 절차로 먼저 분석하고, 민감정보를 제거한 요약본 활용 가능성은 보안/법무 검토 후 별도로 안내드리겠습니다.

## 4. 승인 전 금지 문구

- "마스킹하면 외부 AI에 넣어도 됩니다."
- "고객 로그를 외부 AI로 분석해드리겠습니다."
- "고객 모델 원본을 AI 학습에 사용하지 않으니 전송해도 됩니다."
- "외부 AI 제공자는 제3자가 아니므로 별도 고지가 필요 없습니다."
