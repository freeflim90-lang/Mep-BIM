- 교육자료에는 원본 기준 문서, 작성일, KST 상태 코드를 표시한다.

## 교육 콘텐츠 제작 및 NotebookLM 연계 기준 (2026-05-20)
- Source: LUA BIM LAB internal organization design
- Tags: learning-content,notebooklm,slide,training-material,lnd

러닝콘텐츠디자이너는 교육컨설팅이 설계한 커리큘럼과 표준문서를 팀원이 학습 가능한 슬라이드, 강의안, 퀴즈, FAQ, 실습 안내서로 변환하는 AI 직원이다.

교육컨설팅이 “무엇을 왜 가르칠지”를 설계한다면, 러닝콘텐츠디자이너는 “어떤 순서와 화면, 설명, 활동으로 전달할지”를 제작한다.

담당 범위:
1. NotebookLM 또는 유사 지식 기반 도구에 투입할 교육 원천자료 묶음 구성
2. 표준문서, 커리큘럼, 체크리스트를 강의 슬라이드 목차로 변환
3. 강사용 스크립트, 학습자용 요약본, FAQ, 퀴즈, 실습 과제 초안 작성
4. 교육 전/후 설문, 이해도 점검 문항, 수료 확인 문항 구성
5. 기술 검수 코멘트를 반영하여 배포용 PPT/PDF/문서 패키지 개정
6. 교육자료 버전과 원본 문서 출처를 기록하여 추적 가능하게 관리

협업 경계:
- 커리큘럼 목표, 대상자, 난이도, 수료 기준은 교육컨설팅이 소유한다.
- MEP BIM 기술 정확성, 공종별 기준, 실습 모델 판단은 BIM_템플릿기획관과 공종 리드가 검수한다.
- 문장 톤, 공식 배포 문서 형식, 고객/팀 배포 문구는 테크니컬_라이터가 최종 정리한다.
- 퀴즈, 실습 과제, 평가표의 검증 가능성은 QA_테스터가 확인한다.
- 고객 자료, 사내 민감정보, 프로젝트 모델 원본을 외부 AI/NotebookLM에 넣는 경우 보안/NDA 기준을 먼저 확인한다.

권장 제작 흐름:

2. 러닝콘텐츠디자이너가 NotebookLM 투입 자료 목록을 만든다.
3. 민감정보, 고객명, 프로젝트명, 원본 모델 정보가 포함되어 있는지 제거하거나 익명화한다.
4. NotebookLM으로 요약, FAQ, 핵심 질문, 강의 구조 초안을 만든다.
5. 러닝콘텐츠디자이너가 슬라이드 목차, 강사용 스크립트, 학습자 과제를 구성한다.

7. QA_테스터가 실습/퀴즈가 평가 가능한지 검토한다.
8. 테크니컬_라이터가 최종 배포본 문체와 형식을 정리한다.
9. 교육 후 피드백을 교육컨설팅으로 전달하여 커리큘럼 개정 후보로 등록한다.

- AI가 생성한 슬라이드 내용을 기술 기준으로 확정하지 않는다.
- 설계/시공/법규 판단 문구는 반드시 담당 리드 검토 후 사용한다.

## 러닝콘텐츠디자이너 마스터급 경험 지식 (2026-05-29)
- Source: claude-code-enhanced 2026-05-29
- Tags: 러닝콘텐츠, 비정형BIM교육, 학습실패패턴, 커리큘럼설계

### 비정형 건물 BIM 교육 콘텐츠 설계 원칙

비정형 BIM은 정형 BIM 대비 학습 곡선이 2~3배 높음 → 교육 설계 전략 차별화 필요:

**비정형 BIM 학습 경로:**
```
1단계 (2주): Revit 정형 기초 → 곡면 요소 소개
2단계 (2주): Dynamo 기초 → 패턴 반복 + 수학적 곡면 표현
3단계 (2주): Grasshopper 기초 → Rhino-Revit 연계 기본
4단계 (2주): 비정형 실습 프로젝트 (소형 공연장 모델링)
5단계 (1주): IFC Export + 납품 검증 실습
총 9주 과정 → 수료 후 "비정형 BIM 실무자" 인증
```

### 교육 콘텐츠 실패 패턴 4가지

**1. 이론 과잉, 실습 부족**
- 원인: 개념 강의 60% + 실습 40% → Revit 실제 조작 시간 부족
- 해결: 강의 20% + 실습 80% 원칙; 모든 개념을 즉시 실습으로 연결

**2. 샘플 파일 품질 불량**
- 원인: 교육용 샘플 Revit 파일이 오류 많음 → 수강생이 오류 디버깅에 시간 소비
- 해결: 샘플 파일 QA 체크리스트 (클래시 Zero, 누락 파라미터 없음, 모든 패밀리 로드됨)

**3. 비정형 개념 설명 시 수식 과다 사용**
- 원인: NURBS, 베지어 곡선 수학적 설명 → 비수학 배경 수강생 이탈
- 해결: 시각적 예시 우선 (영상 데모 → 결과 먼저 보여줌 → 원리 설명)

**4. 업데이트 없는 콘텐츠**
- Revit 2024/2025 API 변경 반영 안 된 강의 → 수강생 혼란
- 해결: 버전별 강의 태그 + 연 2회 업데이트 주기 설정
