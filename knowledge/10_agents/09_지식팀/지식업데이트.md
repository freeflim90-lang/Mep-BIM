## 지식 큐레이션·인텔리전스 운영 기준 (2026-05-21)
- Source: `docs/internal_organization_documents/21_KNOWLEDGE_CURATION_INTELLIGENCE_CELL.md`
- Tags: knowledge,curation,intelligence,company-purpose,obsidian

매일 업데이트되는 지식 DB는 단순 수집이 아니라 회사 목적성에 맞게 선별, 연결, 승격해야 한다. 지식 큐레이션은 `MEP BIM 실무 품질`, `Model Quality Auditor 상품화`, `Autodesk Add-in Store`, `연차별 교육 커리큘럼`, `AI 기반 생산성 향상`, `외부 공개 가능성`, `보안/운영 리스크 절감`을 기준으로 정보를 정리한다.

운영 기준:
1. 일일 업데이트 리포트에서 신규/변경 문서를 확인한다.
2. 목적성 기준에 따라 유지, 승격, 보류, 아카이브 후보를 나눈다.
3. 반복 오류는 오류 오답노트와 QA 체크리스트로 승격한다.
4. 교육 재사용성이 있는 정보는 교육컨설팅과 러닝콘텐츠디자이너에게 연결한다.
5. 상품화 가능성이 있는 정보는 CSO, 제품패키징, QA에 연결한다.
6. 외부 공개 전에는 Public, Client-Shareable, Internal, Confidential 분류를 확인한다.

## 외장하드 개인/업무 자료 지식 인테이크 기준 (2026-05-21)
- Source: `docs/internal_organization_documents/22_EXTERNAL_PERSONAL_ARCHIVE_KNOWLEDGE_INTAKE_STANDARD.md`, `scripts/external_knowledge_intake.py`
- Tags: external-archive,knowledge-intake,privacy,curation

외장하드 자료는 바로 지식 DB에 흡수하지 않고, 파일 내용 미열람 메타데이터 스캔으로 후보를 먼저 선별한다. BIM 표준, MEP 매뉴얼, 품질 체크리스트, Dynamo, Revit/Navisworks Add-in, 물량 산식, 프로젝트 종료 프로세스는 우선 검토 후보가 될 수 있다.

자동 차단 대상은 NPKI, 인증서, 연락처, 계약, 비용, 개인정보, 설치파일, 크랙, 원본 프로젝트 모델/도면, 사진 원본 대량 파일이다. 승인된 자료만 요약/익명화하여 표준문서, 교육자료, 제품문서, 오류 오답노트로 승격한다.

## 지식업데이트 마스터급 경험 지식 (2026-05-29)

### 비정형 건물 관련 지식 공백 (Knowledge Gap) 우선 보완 목표

현재 KB에서 비정형 건물 관련 심화 내용이 부족한 영역:

| 지식 공백 영역 | 우선순위 | 현재 KB 위치 |
|-------------|--------|------------|
| 비정형 구조 BIM (트러스, 쉘) | 높음 | [[구조.md]] 보완 필요 |
| Grasshopper → Revit 워크플로우 | 높음 | 신규 파일 생성 필요 |
| 비정형 음향 설계 BIM (공연장) | 중간 | [[설비자동제어.md]] 연계 |
| 비정형 커튼월 BIM (패널화) | 높음 | 신규 파일 생성 필요 |
| 패시브하우스 비정형 입면 설계 | 중간 | [[패시브하우스_PHIKO.md]] 연계 |

### 지식 업데이트 실패 패턴 4가지

**1. 업데이트 없이 outdated 내용 RAG에 사용 → 잘못된 CS 답변**
- 법규 개정 후 KB 미갱신 → CS 봇이 구 기준으로 답변
- 해결: 법규 변경 감지 시 관련 KB 파일 자동 만료 태그, 인간 검토 알림

**2. 중복 지식 파일 → 혼재 답변**
- 동일 주제가 2개 파일에 다른 내용으로 존재 → RAG 혼란
- 해결: 주기적 중복 탐지 스크립트 실행

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity
import glob

docs = {f: open(f).read() for f in glob.glob("knowledge/10_agents/90_확장에이전트/*.md")}
tfidf = TfidfVectorizer()
matrix = tfidf.fit_transform(docs.values())
sim = cosine_similarity(matrix)
# sim[i][j] > 0.8 → 중복 의심 쌍
```

**3. 비정형 건물 지식 업데이트 주기 과도하게 느림**
- 비정형 시장 변화 빠름 → 반기 업데이트로는 부족
- 해결: 비정형 관련 KB 파일은 월 1회 검토 주기 적용

**4. 지식 추가는 많은데 삭제 없음 → KB 비대화**
- 해결: 연 1회 KB 다이어트 (1년 이상 미참조 파일 아카이브)
