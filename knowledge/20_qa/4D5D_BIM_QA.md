**Q: 모델 객체와 공정/WBS/원가 코드가 1:1로 맞지 않으면 어떻게 하나요?**
A: 자동 산출을 확정하지 않고 매핑 오류 후보로 표시한다. 객체 분할 기준, WBS 깊이, CBS 코드, 수량 산출 단위를 먼저 맞추고, 불일치 항목은 수동 검토 목록으로 남긴다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `4D5D_BIM` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

## 2026-06-21 4D/5D BIM 매핑 추가 Q&A
- Source: LUA BIM LABS coverage hardening
- Tags: 4d,5d,bim,wbs,cbs,schedule,cost,2026

**Q: 4D 공정 시뮬레이션에서 모델 객체가 너무 잘게 쪼개져 있으면 어떻게 하나요?**
A: 결론: 객체 수를 그대로 공정에 연결하지 말고 WBS 작업 단위와 모델 그룹 기준을 먼저 맞춘다. 1. Revit Element ID, Assembly Code, WBS 단계, 공정 기간, 층/구역, LOD 300/350 모델 분할 기준을 확인한다. 2. 조건상 객체가 100개 이상으로 과분할되어도 같은 작업·구역이면 그룹핑하고, 반면 시공 순서나 장비 투입이 다르면 별도 공정으로 분리한다. 잘못 매핑하면 일정 오류, 비용, 현장 책임 리스크가 생긴다. 24시간 내 4D 담당자가 매핑표를 기록·공유하고 7일 내 승인자, 기한, 검토 로그를 남긴다.

**Q: 5D 원가 산출이 물량표와 다르면 어느 쪽을 기준으로 보고하나요?**
A: 결론: 5D 산출값을 바로 확정하지 않고 모델 수량, 산출 단위, CBS 코드, 견적 물량표를 대조한다. 1. m2/m3/EA 단위, LOD 350, 패밀리 타입, 제외 항목, 손율 %, 단가 기준일을 확인한다. 2. 조건상 모델 누락이면 BIM 보완으로 분류하고, 반면 단가표나 CBS 매핑 오류이면 원가 데이터 수정 후보로 둔다. 다만 계약 제출 전에는 승인된 견적 물량표를 우선하고, 예외 차이는 5D 검증표로 구분한다. 오판하면 비용, 계약, 납품 책임 리스크가 커진다. 24시간 내 5D 담당자가 차이표를 요청·기록하고 7일 내 PM/견적 승인, 기한, 변경 로그를 공유한다.

## 2026-06-21 4D/5D BIM 실행통제 추가 Q&A
- Source: LUA BIM LABS coverage hardening
- Tags: 4d,5d,bim,baseline,earned-value,risk,2026

**Q: 4D 시뮬레이션 일정이 현장 실제 공정과 계속 어긋나면 무엇을 조정하나요?**
A: 결론: 모델 애니메이션을 먼저 고치지 말고 기준 일정, 실제 진도, WBS 매핑, 업데이트 주기를 함께 확인한다. 1. 담당자는 기준 공정표, 24시간 현장 진도, Revit/LOD 300/350 객체, WBS 코드, Source, Tags를 기록한다. 2. 조건상 실제 공정이 계획보다 7일 이상 차이 나면 PM에게 원인 확인을 요청하고 4D baseline 변경 여부를 검토한다. 반면 모델 그룹핑 오류라면 매핑표를 정리한다. 다만 일정 지연, 비용, 계약 책임 리스크가 있으면 예외 없이 승인 로그와 변경 결과를 공유하고 보고한다.

**Q: 5D 비용 예측을 월간 보고에 넣을 때 어떤 불확실성을 표시해야 하나요?**
A: 결론: 예측값은 확정 원가가 아니므로 수량, 단가, 변경 지시, 모델 성숙도의 불확실성을 표시해야 한다. 1. 담당자는 CBS 코드, 단가 기준일, 물가 변동률 %, LOD 300/350, 변경요청, Source와 Tags를 24시간 안에 확인한다. 2. 조건상 발주처 보고나 계약 협상에 쓰이면 승인 물량표와 차이표를 요청하고 검토한다. 반면 내부 추세 분석이면 예비값으로 구분한다. 다만 비용 초과, 산출 오류, 법무 책임, 고객 약속 리스크가 있으면 승인 담당자와 7일 검토 로그를 기록하고 결과를 보고한다.

## 2026-06-21 4D/5D 변경관리 추가 Q&A
- Source: LUA BIM LABS coverage expansion
- Tags: 4d,5d,bim,change-order,baseline,cost,2026

**Q: 설계 변경으로 공정과 원가가 동시에 바뀌면 4D/5D 기준선은 언제 갱신하나요?**
A: 결론: 변경 요청이 승인되기 전에는 기준선을 바꾸지 않고 영향 분석 후보로만 둔다. 1. 담당자는 변경 지시서, Revision, WBS/CBS 코드, 모델 객체 ID, 수량 증감 %, 단가 기준일, Source/Tags를 24시간 안에 확인한다. 2. 조건상 발주처·PM 승인과 계약 영향이 확인된 경우 baseline 갱신을 요청하고, 반면 검토 중인 설계안이면 별도 시나리오로 분리한다. 무승인 반영은 일정 오류, 비용 분쟁, 법무, 책임 리스크를 만든다. 7일 내 승인자, 담당자, 기한, 변경 로그를 공유·보고한다.
