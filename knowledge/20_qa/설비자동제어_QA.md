## 2026-06-20 BAS/BMS 포인트 누락 실무 케이스 Q&A

**Q: 장비는 모델에 있는데 제어 포인트가 누락되면?**
A: 결론: 장비 태그만으로 완료 처리하지 않고 포인트 리스트와 제어 시퀀스를 먼저 확인한다. 1. 담당자는 운전/정지, 알람, 계측, 제어 포인트, BMS/BAS 연동, Revit 장비 ID, Source, Tags를 24시간 안에 기록한다. 2. 조건상 LOD 300/350 모델에 장비는 있으나 포인트가 없으면 운영 인계 리스크와 책임 범위를 RFI/BCF로 공유한다. 반면 단순 파라미터 누락이면 모델 담당자에게 수정 요청한다. 다만 보안, 법무, 유지관리 비용 리스크가 있으면 승인 담당자와 7일 보완 로그를 남긴다.

**Q: 자동제어 포인트 명칭이 도면, 시방서, BMS 태그에서 다르면 어떻게 맞추나요?**
A: 결론: 명칭 통일은 운영 인계 기준으로 하되 원문별 차이를 추적해야 한다. 1. 담당자는 제어도면, 시방서, BMS 태그, 장비 태그, Revit 파라미터, Source, Tags를 24시간 안에 매핑한다. 2. 조건상 운전/정지, 알람, 온도, 차압, 유량 포인트가 LOD 300/350 산출물에 영향을 주면 표준 명칭안을 요청하고 공유한다. 반면 오탈자 수준이면 수정 로그로 처리한다. 다만 제어 오류, 보안, 책임 리스크가 있으면 예외 없이 승인자와 7일 검토 기한을 기록한다.

**Q: 제어 시퀀스가 불명확한 장비는 BIM 납품에서 어떻게 표시하나요?**
A: 결론: 불명확한 시퀀스는 완료가 아니라 확인 필요 상태로 표시한다. 1. 담당자는 장비 운전 조건, 인터록, 알람, 센서 위치, BMS 화면 요구, Source, Tags, KST02 기준을 확인해 24시간 안에 누락 포인트를 정리한다. 2. 조건상 팬, 펌프, AHU, 밸브 제어가 안전이나 에너지 성능에 영향을 주면 RFI/BCF로 담당자 검토를 요청한다. 반면 단순 화면 명칭은 주석으로 관리한다. 다만 운영 장애, 법무, 비용 책임 리스크가 있으면 예외 없이 승인 로그와 7일 보완 계획을 공유한다.

**Q: BMS 포인트는 있는데 실제 시운전 값이 계속 0으로 나오면 BIM 담당자는 무엇을 확인하나요?**
A: 결론: 모델 속성 오류로 단정하지 않고 센서 결선, 통신 매핑, 시운전 로그를 분리해 확인한다. 1. 담당자는 BMS 태그, 센서 주소, 제어반 I/O, Revit 장비 ID, 시운전 일시, KST02, Source/Tags를 24시간 안에 기록한다. 2. 조건상 실제 장비 운전 중에도 값이 0이면 통신 오류나 결선 리스크로 보고 제어 담당자 검토를 요청하고, 반면 미운전 상태라면 테스트 조건과 LOD 300/350 납품 한계를 구분한다. 방치하면 운영 장애, 안전, 비용, 책임 리스크가 생긴다. 7일 내 승인자, 기한, 재시험 결과, BCF/RFI 로그를 공유·보고한다.

**Q: 자동제어 보안 때문에 BMS 화면 캡처를 받을 수 없으면 납품 검증은 어떻게 하나요?**
A: 결론: 보안 자료를 무리하게 요구하지 않고 대체 증거와 승인 범위를 먼저 정한다. 1. 담당자는 보안 등급, 캡처 금지 조건, 포인트 리스트, 시험성적서, 장비 태그, Source/Tags, LOD 300/350 제출 범위를 확인한다. 2. 조건상 화면 캡처가 불가한 경우 익명화 포인트표나 입회 확인서로 대체하고, 반면 발주처가 원본 증거를 요구하면 보안 승인 절차를 요청한다. 무단 공유는 개인정보, 보안, 법무, 책임 리스크가 크다. 24시간 내 담당자가 대체 증거안을 정리·공유하고 7일 내 승인자, 기한, 검증 로그를 남긴다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `설비자동제어` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
