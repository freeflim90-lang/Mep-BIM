**Q: 시공 지침이 BIM 모델에 반영되지 않으면?**
A: 결론: 시공 지침은 객체, 속성, 검토 룰, 산출물로 바꿔야 BIM 모델에 반영된다. 1. 담당자는 지침 문장, Revit 카테고리, 파라미터, BEP 산출물, LOD 300/350, Source, Tags를 24시간 안에 매핑하고 확인 로그를 공유한다. 2. 조건상 모델에서 확인할 수 있는 간격, 레벨, 접근공간은 규칙으로 만들고, 반면 양중 순서나 작업 허가처럼 현장 절차인 항목은 체크리스트로 구분한다. 다만 안전, 법무, 일정 책임 리스크가 있으면 예외 없이 승인 담당자와 7일 검토 로그를 공유하고 오류 재발 여부를 보고한다.

**Q: 시공 지침의 작업 순서를 4D BIM에 반영할 때 무엇을 남겨야 하나요?**
A: 결론: 작업 순서는 모델 요소, 일정, 선후행 조건, 검수 증거가 함께 남아야 한다. 1. 담당자는 공정 코드, Revit 요소 ID, Navisworks Timeliner, 작업 구역, 선후행 관계, LOD 300/350, Source, Tags를 24시간 안에 기록한다. 2. 조건상 구조, 설비, 마감 간섭이나 안전 통제가 필요한 작업은 4D 시뮬레이션과 RFI/BCF 후보로 공유하고 담당자 확인을 요청한다. 반면 단순 참고 절차는 현장 체크리스트로 둔다. 다만 일정 지연, 비용, 안전 책임 리스크가 있으면 예외 없이 승인 담당자와 7일 검토 기한, 승인 로그를 남긴다.

**Q: 시공 지침이 모델 품질 기준과 충돌하면 어떤 순서로 조정하나요?**
A: 결론: 충돌은 모델 편의보다 시공 가능성과 승인 기준을 우선해 조정한다. 1. 담당자는 지침 원문, BEP, LOD 300/350 요구, Revit 모델 한계, 현장 시공 상세, Source와 Tags를 확인해 24시간 안에 충돌 목록을 만든다. 2. 조건상 모델 품질 기준이 현장 안전이나 유지관리 공간을 해치면 RFI/BCF로 설계·시공 담당자 검토를 요청한다. 반면 표현 방식 차이면 모델 주석으로 정리한다. 다만 계약, 법무, 비용 리스크가 있으면 예외 없이 승인 로그와 7일 보완 계획을 공유한다.

**Q: 시공 지침에는 임시 개구부 보양이 있는데 모델에는 표현하지 않아도 되나요?**
A: 결론: 임시 개구부 보양은 영구 모델 요소가 아니어도 안전 통제와 검수 증거로 남겨야 한다. 1. 담당자는 개구부 위치, 보양 방식, 작업 구역, 4D 일정, Revit 요소 ID, LOD 300/350, Source/Tags를 24시간 안에 확인한다. 2. 조건상 추락·화재·피난 안전에 영향을 주는 경우 모델 주석 또는 Navisworks 검토 세트로 공유하고 담당자 검토를 요청한다. 반면 단기 작업 절차이면 현장 체크리스트로 분리한다. 누락되면 안전, 일정, 법무, 책임 리스크가 생긴다. 7일 내 승인자, 기한, 현장 확인 로그와 보완 결과를 보고한다.

**Q: 시공 지침의 작업 가능 시간 제한을 4D 일정에 어떻게 반영하나요?**
A: 결론: 시간 제한은 단순 메모가 아니라 공정 캘린더와 선후행 조건으로 반영한다. 1. 담당자는 야간/주간 작업 조건, 소음 제한 dB, 장비 반입 시간, Navisworks Timeliner, 공정 코드, Source/Tags를 확인한다. 2. 조건상 민원, 안전, 발주처 운영 시간에 걸리는 경우 24시간 내 제한 조건을 일정 규칙으로 기록하고 공유한다. 반면 내부 참고 수준이면 작업 지침서 링크로 둔다. 무시하면 일정 지연, 비용, 고객 약속, 책임 리스크가 커진다. 7일 내 담당자, 승인자, 기한, 변경 로그를 남긴다.
