**Q: BIMobject 패밀리를 바로 프로젝트에 써도 되나요?**
A: 바로 쓰지 않는다. 파라미터, LOD, 커넥터, 파일 크기, 라이선스, Revit 버전, 분류 체계를 확인한다. 제조사 패밀리도 프로젝트 표준에 맞게 정리해야 한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `BIMobject_패밀리_인덱스` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

**Q: 제조사 패밀리가 최신 제품이라도 프로젝트 표준에 맞지 않으면 어떻게 하나요?**
A: 결론: 최신 제품 여부보다 Revit 버전, LOD 300/350, 파라미터, 커넥터, 라이선스, 파일 크기를 먼저 검수한다. 1. 패밀리 카테고리, Shared Parameter, IFC/COBie 매핑, 타입 수, 5MB 이상 과대 형상 여부를 확인한다. 2. 조건상 납품·수량 산출에 쓰면 프로젝트 표준으로 정리하고, 반면 렌더링 참고용이면 실무 모델과 분리한다. 기준 없이 쓰면 모델 성능, 수량 오류, 저작권, 책임 리스크가 생긴다. 24시간 내 패밀리 담당자가 샘플 모델 테스트를 기록·공유하고 7일 내 승인권자, 기한, 변경 로그를 남긴다.

**Q: BIMobject 패밀리 라이선스나 출처가 불명확하면 고객 모델에 넣어도 되나요?**
A: 결론: 출처와 사용권이 불명확하면 고객 납품 모델에 넣지 않는다. 1. Source URL, 제조사명, 다운로드일, 라이선스 문구, Revit 버전, 제품 코드, Tags를 확인한다. 2. 조건상 제조사 공식 배포이고 상업 프로젝트 사용이 허용되면 후보로 두지만, 반면 재배포 금지나 출처 불명확 파일이면 내부 임시 참고로만 분리한다. 잘못 쓰면 법무, 보안, 납품, 고객 승인 리스크가 있다. 24시간 내 담당자가 출처 증빙을 요청·기록하고 7일 내 검토자, 기한, 승인 로그를 공유한다.

**Q: BIMobject 패밀리 파일 크기가 크면 모델 성능 때문에 제외해야 하나요?**
A: 결론: 파일 크기만으로 제외하지 말고 형상 복잡도, 타입 수, 커넥터, 파라미터, 납품 용도를 함께 본다. 1. 패밀리 용량 MB, Revit 버전, LOD 300/350, Nested Family 수, MEP Connector, IFC/COBie 매핑, Source/Tags를 확인한다. 2. 조건상 제품 식별용이면 경량화 패밀리로 대체하고, 반면 제조사 상세 형상이 검수 근거인 경우 별도 링크 모델이나 타입 제한으로 관리한다. 과대 패밀리는 모델 오류, 성능 저하, 납품 반려, 법무 책임 리스크를 만든다. 24시간 내 패밀리 담당자가 샘플 로딩 시간을 기록·공유하고, 경량화 요청을 검토한 뒤 7일 내 승인권자, 기한, 경량화 로그를 보고한다.

**Q: 같은 제품 패밀리가 여러 출처에 있으면 인덱스 기준은 무엇인가요?**
A: 결론: 최신성보다 공식성, 라이선스, 프로젝트 표준 적합성, 검수 이력을 기준으로 대표 패밀리를 정한다. 1. 제조사 공식 Source URL, 다운로드일, Revit 버전, 제품 코드, 파라미터명, LOD 300/350, Tags를 비교한다. 2. 조건상 공식 배포본이 프로젝트 BEP/EIR과 맞는 경우 대표본으로 두고, 반면 커넥터나 수량 산출이 틀린 경우 수정본을 별도 승인 버전으로 관리한다. 다만 출처가 불명확하면 보류하고 제조사 증빙을 요청한다. 중복 인덱스는 수량 오류, 모델 충돌, 보안 누수, 책임 리스크를 만든다. 24시간 내 담당자가 대표/보류 목록을 기록·검토하고 7일 내 승인자, 기한, 변경 로그를 공유·보고한다.

**Q: 고객이 지정한 BIMobject 패밀리가 사내 표준 파라미터와 다르면 그대로 써야 하나요?**
A: 결론: 고객 지정이라도 납품 표준, 수량 산출, IFC/COBie 매핑에 맞게 검수한 뒤 승인 버전으로 정리한다. 1. 고객 지정 근거, Source URL, Revit 버전, Shared Parameter GUID, LOD 300/350, 제품 코드, Tags를 확인한다. 2. 조건상 형상 참고용이면 링크 또는 참고 패밀리로 분리하고, 반면 납품 모델에 포함되면 사내 표준 파라미터와 커넥터를 맞춘다. 그대로 사용하면 수량 오류, 납품 반려, 법무 책임, 고객 승인 리스크가 생긴다. 24시간 내 패밀리 담당자가 차이표를 기록·공유하고 7일 내 승인권자, 기한, 수정/승인 로그를 보고한다.
