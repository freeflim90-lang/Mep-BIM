**Q: 시방서 문구를 BIM 검토 규칙으로 바꾸려면 무엇이 필요한가요?**
A: 문구를 객체, 속성, 허용값, 검수 방법, 예외 조건으로 바꾼다. 추상 문장은 그대로 자동화하지 않고 사람 검토 항목으로 분리한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `BIM_시방서` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

## 2026-06-21 BIM 시방서 자동화/검수 추가 Q&A
- Source: LUA BIM LABS coverage hardening

**Q: 시방서 문구 중 어떤 것은 자동검사로 만들고 어떤 것은 사람 검토로 남기나요?**
A: 결론: 객체, 파라미터, 수치, 허용값으로 표현되는 문구는 자동검사로 만들고, 설계 의도나 성능 판단은 사람 검토로 남긴다. 1. Revit 카테고리, Shared Parameter, LOD 300/350, mm 단위 허용값, Navisworks Search Set 가능 여부를 확인한다. 2. 조건상 "DN100 이상", "300mm 이격"처럼 수치가 있으면 룰 후보이고, 반면 "적정한 위치", "유지관리 용이"는 체크리스트와 담당자 검토로 분리한다. KDS/KCS, BEP, 시방서 Revision을 근거로 남긴다. 자동화 오판은 납품, 법무, 품질, 책임 리스크가 있으므로 24시간 내 룰 후보를 기록·공유하고 7일 내 검토자, 기한, 승인 로그를 정리한다. 잘못된 룰은 모델 오류와 고객 승인 리스크를 만들기 때문에 배포 전 샘플 검수를 거친다.

**Q: 시방서와 모델 파라미터명이 다르면 어느 쪽을 바꿔야 하나요?**
A: 결론: 납품 기준은 시방서와 BEP를 우선하되, 모델 파라미터는 매핑표로 연결하고 임의 변경하지 않는다. 1. 시방서 용어, Revit 파라미터명, IFC 속성, COBie 필드, LOD 300/350 납품표를 확인한다. 2. 조건상 내부 템플릿 이름만 다른 경우에는 매핑표로 해결하고, 반면 발주처 제출 파일 필드와 다르면 BEP 변경 또는 파라미터 수정 후보로 분류한다. 잘못 바꾸면 수량, 검수, 책임, 일정 리스크가 생긴다. 24시간 내 BIM 시방서 담당자가 차이표를 요청·기록하고 7일 내 승인권자, 기한, RFI/BCF 로그를 공유한다.

## 2026-06-21 BIM 시방서 해석 추가 Q&A

**Q: 시방서 Revision이 바뀌었는데 기존 자동검사 룰은 그대로 써도 되나요?**
A: 결론: 그대로 쓰지 말고 변경된 문구와 기존 룰을 대조해 재검증한다. 1. Revision 날짜, 변경 조항, Revit 파라미터, IFC 속성, LOD 300/350, BEP/EIR, Source/Tags를 확인한다. 2. 조건상 문구 변경이 설명 보완인 경우 룰 유지가 가능하지만, 반면 수치·허용값·검수 방법이 바뀌면 룰 버전과 샘플 모델 검사를 갱신한다. 미검증 룰은 모델 오류, 납품 반려, 일정 지연, 법무 책임 리스크를 만든다. 24시간 내 시방서 담당자가 변경표를 기록·공유하고 7일 내 승인권자, 기한, RFI/BCF 로그를 정리한다.

**Q: 시방서 문구가 "동등 이상"처럼 애매하면 BIM 검수 기준으로 어떻게 바꾸나요?**
A: 결론: "동등 이상"을 그대로 자동검사하지 말고 성능값, 인증, 제조사 자료, 승인 절차로 분해한다. 1. 적용 자재, 성능 수치 %, KS/KDS/KCS 기준, 제출 승인서, LOD 300/350 표현 범위, Source/Tags를 확인한다. 2. 조건상 수치 기준이 있으면 파라미터 룰로 만들고, 반면 성능 판단이 필요한 경우 검토자 승인 체크리스트로 분리한다. 애매한 자동화는 품질 오류, 비용 분쟁, 납품 반려, 책임 리스크를 만든다. 24시간 내 담당자가 예외 기준을 기록·공유하고 7일 내 승인자, 기한, 검토 로그를 남긴다.

## 2026-06-21 BIM 시방서 승인흐름 추가 Q&A

**Q: 시방서 자동검사 룰이 현장 승인 예외와 충돌하면 룰을 꺼도 되나요?**
A: 결론: 룰을 바로 끄지 않고 승인 예외의 범위, 기간, 대상 객체를 제한해 예외 처리한다. 1. 담당자는 승인서, 예외 사유, 대상 Revit 객체 ID, 시방서 조항, LOD 300/350, Source/Tags를 24시간 안에 확인한다. 2. 조건상 특정 구역·기간만 승인된 예외이면 룰 예외 목록으로 기록하고, 반면 기준 자체가 바뀐 경우 시방서 Revision과 룰 버전을 갱신한다. 무단 비활성화는 납품 반려, 품질 오류, 법무, 책임 리스크를 만든다. 7일 내 승인자, 담당자, 기한, 예외 검토 로그를 공유·보고한다.
