**Q: 에너지 기준 검토에 BIM 정보가 부족하면 무엇을 요청하나요?**
A: 결론: 모델 형상만으로 에너지 성능을 확정하지 않고 인증 기준과 입력값을 분리해 요청한다. 1. 담당자는 인증 기준, 열교, 기밀, 창호 U-value, 장비 효율, 면적 산정, Source, Tags, KST02 기준을 24시간 안에 정리한다. 2. 조건상 Revit 모델, BEP, LOD 300/350 정보가 에너지 계산에 쓰이면 입력값 출처와 책임 범위를 기록하고 추가 확인을 요청한다. 반면 형상 검토만 가능하면 성능 확정 표현을 보류한다. 다만 법무, 비용, 인증 실패 리스크가 있으면 승인 담당자와 7일 검토 로그를 공유하고 보고한다.

**Q: PHIKO 기준과 발주처 에너지 요구가 다르면 어떤 순서로 검토하나요?**
A: 결론: 발주처 요구, 인증 기준, 법규 기준을 분리한 뒤 더 엄격한 책임 조건을 확인한다. 1. 담당자는 PHIKO 기준, 발주처 EIR/BEP, 창호 성능, 기밀 목표, 열교 상세, Source와 Tags를 24시간 안에 비교한다. 2. 조건상 LOD 300/350 모델에 열교·창호·면적 입력값이 부족하면 담당자에게 보완 자료를 요청하고 검토한다. 반면 발주처 요구가 참고 수준이면 적용 범위를 구분한다. 다만 인증, 비용, 일정, 법무 리스크가 있으면 예외 없이 승인 로그와 7일 재검토 결과를 보고한다.

**Q: 패시브하우스 검토 결과를 BIM 납품 문서에 넣을 때 무엇을 표시해야 하나요?**
A: 결론: 결과값뿐 아니라 입력 전제, 검토 한계, 담당자 승인 로그를 함께 표시해야 한다. 1. 담당자는 면적, 창호 성능, 장비 효율, 열교 가정, 기밀 조건, Source, Tags, 24시간 확인 로그를 문서에 남긴다. 2. 조건상 Revit/BIM360 모델과 계산서가 다르면 RFI/BCF 후보로 공유하고 7일 안에 보완 요청을 기록한다. 반면 개념 검토 단계면 예비값임을 표시한다. 다만 인증 실패, 법무, 고객 약속, 비용 책임 리스크가 있으면 승인 담당자와 결과를 보고한다.

**Q: 창호 U-value가 모델과 제조사 성적서에서 다르면 무엇을 기준으로 하나요?**
A: 결론: 모델값으로 확정하지 않고 제조사 성적서, 인증 기준, 적용 위치를 대조한다. 1. 담당자는 창호 태그, U-value W/m2K, SHGC, 프레임/유리 구분, Revit 파라미터, Source/Tags, PHIKO 기준을 24시간 안에 확인한다. 2. 조건상 성적서가 최신 승인본인 경우 모델 파라미터 수정을 요청하고, 반면 모델이 가정값이면 예비값으로 표시한다. 방치하면 에너지 계산 오류, 인증 실패, 비용, 법무 책임 리스크가 생긴다. 7일 내 승인자, 담당자, 기한, 수정 로그를 공유·보고한다.

**Q: 기밀 성능 자료가 없는데 패시브하우스 가능성을 물으면 어떻게 답하나요?**
