**Q: CS 티켓이 밀리면 접수순으로 처리하나요?**
A: 결론: 접수순만 보지 않고 SLA와 고객 피해 리스크로 우선순위를 다시 정한다. 1. 담당자는 제품 사용 불가, 결제/라이선스 오류, 데이터 손실, 보안, 공개 리뷰, 일반 문의를 구분하고 24시간 안에 티켓 로그와 Source, Tags를 확인한다. 2. 조건상 BIM360, BCF/RFI, Revit 납품 중단처럼 업무가 멈추면 일반 문의보다 먼저 공유하고 기술 담당자에게 요청한다. 반면 단순 사용 질문은 기한을 안내한다. 다만 법무 책임이나 환불 이슈가 있으면 예외로 운영 리더 승인과 7일 처리 계획을 기록한다.

**Q: 같은 고객이 여러 채널로 같은 문제를 보내면 티켓을 어떻게 묶나요?**
A: 결론: 중복 티켓은 하나의 대표 건으로 묶되 원본 Source와 응답 로그는 삭제하지 않는다. 1. 담당자는 이메일, 채팅, 전화, BIM360 코멘트, BCF/RFI 번호를 확인해 24시간 안에 대표 티켓을 지정하고 고객에게 공유한다. 2. 조건상 제품 사용 불가, 결제 오류, 데이터 손상, 보안 리스크가 있으면 접수순보다 SLA 우선순위를 높인다. 반면 단순 사용 문의라면 병합 후 다음 답변 기한을 기록한다. 다만 법무 책임이나 공개 리뷰 위험이 보이면 예외로 운영 리더 승인과 기술 담당자 검토를 요청한다.

**Q: CS 답변이 기술/법무/결제 판단까지 걸치면 누가 최종 답변하나요?**
A: 결론: 고객지원운영이 창구를 맡되 최종 판단은 영역별 담당자 승인으로 확정한다. 1. 기술 오류는 프로그램개발, Revit/Navisworks/BEP 쟁점은 BIM 담당자, 환불과 결제는 운영/재무, 책임 조항은 법무에 배정하고 7일 기한과 승인 로그를 남긴다. 2. 조건상 고객이 즉시 답을 요구하면 확인 중인 범위와 임시 조치만 공유한다. 반면 보안, 데이터 손실, 계약 위반 리스크가 있으면 예외 없이 확정 답변을 보류하고 Source, Tags, 티켓 번호를 정리해 보고한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `고객지원운영` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

**Q: 고객이 "오늘 안에 해결"을 요구하면 무조건 당일 처리 약속을 해도 되나요?**
A: 결론: 무조건 약속하지 말고 영향도, SLA, 담당자 가능 시간, 임시 우회책을 먼저 확인한다. 1. 티켓 번호, 고객 등급, 제품 사용 불가 여부, 결제/라이선스, 데이터 손상, Source/Tags, 24시간 응답 기한을 기록한다. 2. 조건상 업무 중단이나 보안 오류인 경우 긴급 처리로 올리고, 반면 단순 사용 문의인 경우 확인 범위와 다음 답변 시간을 안내한다. 과한 약속은 고객 신뢰, 비용, 일정, 법무 책임 리스크를 만든다. 7일 내 고객지원운영 담당자가 승인권자, 기한, 처리/보류 로그를 공유·보고한다.

**Q: 기술팀이 원인 분석 중인데 고객이 계속 업데이트를 요구하면 어떻게 응답하나요?**
A: 결론: 확정 원인을 만들지 말고 현재 확인 범위, 임시 조치, 다음 업데이트 시간을 짧게 공유한다. 1. 기술 담당자, 재현 여부, 로그, Revit/Navisworks/BIM360 버전, 티켓 Source/Tags, 고객 영향도를 확인한다. 2. 조건상 데이터 손실·보안·납품 중단인 경우 24시간 내 운영 리더에게 에스컬레이션하고, 반면 일반 오류인 경우 7일 내 분석 계획과 우회 방법을 안내한다. 추정 답변은 응답 오류, 고객 약속, 비용, 책임 리스크를 만든다. 담당자는 업데이트 로그와 승인자, 기한, 다음 연락 시간을 기록·공유한다.
