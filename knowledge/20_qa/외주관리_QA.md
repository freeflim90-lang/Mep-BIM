**Q: 외주 산출물이 왔는데 검수가 애매하면 무엇을 확인하나요?**
A: 결론: 결과물이 좋아 보여도 권리와 유지보수 범위가 없으면 검수 완료가 아니다. 1. 담당자는 납품물, 원본/소스, 수정권, 재사용권, 하자보수, 고객자료 삭제 확인, Source, Tags를 24시간 안에 체크하고 검수 로그를 공유한다. 2. 조건상 Revit Family, Navisworks 리포트, BEP 문서, LOD 300/350 모델처럼 재사용될 산출물은 권리와 책임 범위를 계약서와 대조한다. 반면 단순 참고 자료라도 출처 로그는 남긴다. 다만 법무, 보안, 비용, 고객 약속 리스크가 있으면 예외 없이 승인 담당자와 7일 보완 기한을 공유하고, 추가 자료를 요청하고 검토하고 정리한 뒤 오류 재발 여부와 책임 추적 결과를 보고한다.

**Q: 외주사가 납품 기한은 맞췄지만 품질 기준을 일부 놓치면 어떻게 처리하나요?**
A: 결론: 기한 준수와 품질 합격은 분리해서 판단한다. 1. 담당자는 BEP, 납품 체크리스트, Revit 모델 오류, LOD 300/350 충족률, Source, Tags, 24시간 검수 로그를 기록한다. 2. 조건상 고객 납품에 영향이 있으면 하자보수 요청과 재검수 기한을 7일 안으로 지정하고 담당자에게 공유한다. 반면 내부 참고 산출물이라면 보완 우선순위를 낮출 수 있다. 다만 법무, 비용, 일정 책임 리스크가 있으면 예외 없이 승인 로그와 고객 공지 여부를 검토한다.

**Q: 외주 작업에 고객 자료가 포함되면 종료 후 무엇을 확인해야 하나요?**
A: 결론: 산출물 검수와 별도로 고객 자료 회수·삭제 증거를 확인해야 한다. 1. 담당자는 원본 파일, 접근 계정, 다운로드 링크, 백업 여부, 삭제 확인서, Source, Tags를 24시간 안에 목록화한다. 2. 조건상 BIM360, Revit 모델, 계약 도면, 개인정보 또는 보안 자료가 포함되면 외주사 담당자에게 삭제 로그와 접근 차단 증거를 요청한다. 반면 공개 자료만 사용한 경우에도 사용 범위를 기록한다. 다만 보안, 법무, 고객 신뢰 리스크가 있으면 예외 없이 승인자와 7일 확인 기한을 공유한다.

**Q: 외주사가 수정 요청을 범위 밖이라고 하면 어떻게 판단하나요?**
A: 결론: 감정적 협상이 아니라 계약 범위, 납품 기준, 하자보수 조항으로 분리해 판단한다. 1. 담당자는 계약서, 작업 범위, BEP, 납품 체크리스트, Revit/LOD 300/350 오류 목록, Source/Tags를 24시간 안에 비교한다. 2. 조건상 합의된 품질 기준 미달이면 하자보수 요청으로 기록하고, 반면 신규 요구이면 변경 계약 후보로 구분한다. 무리한 요청이나 방치는 비용, 일정, 법무, 책임 리스크를 만든다. 7일 내 승인자, 담당자, 기한, 협의 로그와 재검수 결과를 공유·보고한다.

**Q: 외주 산출물을 내부 템플릿으로 재사용하려면 무엇을 확인해야 하나요?**
A: 결론: 재사용 전에는 권리, 출처, 보안 삭제, 유지보수 가능성을 모두 확인해야 한다. 1. 담당자는 원본 파일, 저작권/재사용권, 고객 식별 정보, Source/Tags, Revit Family Type, 테스트 로그를 확인한다. 2. 조건상 BIM 템플릿, Navisworks 룰셋, BEP 문서처럼 반복 사용될 경우 24시간 내 법무·보안 검토를 요청하고 승인 로그를 남긴다. 반면 단일 프로젝트 전용 산출물은 재사용 불가로 분류한다. 누락하면 개인정보, 법무, 브랜드, 비용 책임 리스크가 생긴다. 7일 내 승인자, 기한, 정제 결과를 공유한다.
