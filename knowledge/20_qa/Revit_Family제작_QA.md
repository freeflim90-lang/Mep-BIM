**Q: 패밀리가 예쁘지만 모델이 무거워지면 어떻게 판단하나요?**
A: 형상 디테일보다는 파라미터, 커넥터, LOD, 파일 크기, 타입 관리, 교체 가능성을 먼저 본다. 렌더링용 디테일은 실무 패밀리와 분리한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `Revit_Family제작` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

**Q: MEP 패밀리 커넥터 방향이 모델마다 다르게 동작하면 어떻게 검수하나요?**
A: 결론: 형상보다 커넥터 방향, System Classification, Flow Direction, 파라미터 매핑을 먼저 검수한다. 1. Revit 2024/2025 버전, 패밀리 카테고리, 커넥터 개수, DN/mm 단위, LOD 300/350 요구를 확인한다. 2. 조건상 장비 입출구가 반대로 전파되면 패밀리 수정 후보이고, 반면 태그만 틀리면 모델 또는 뷰 템플릿 이슈로 분리한다. 오류를 방치하면 계통명, 수량, 간섭검토, 납품 책임 리스크가 생긴다. 24시간 내 패밀리 담당자가 샘플 3개 모델에서 테스트하고 7일 내 승인자, 기한, RFI/BCF 로그를 기록해 공유한다.

**Q: 제조사 패밀리를 받아 쓰면 어떤 항목을 정리해야 하나요?**
A: 결론: 제조사 패밀리는 바로 쓰지 않고 파일 크기, 타입 수, 파라미터명, 커넥터, 재질, LOD를 프로젝트 표준에 맞춘 뒤 사용한다. 1. 5MB 이상 과도한 형상, 불필요한 2D 심볼, 중복 타입, 로컬 언어 파라미터를 확인한다. 2. 조건상 렌더링용 디테일이면 실무 패밀리와 분리하고, 반면 납품·수량 산출에 쓰면 Shared Parameter와 COBie/IFC 매핑을 맞춘다. 정리 없이 쓰면 모델 성능, 수량 오류, 보안, 책임 리스크가 있다. 24시간 내 검토자와 담당자를 지정하고 7일 내 수정본, 승인 로그, 배포 위치를 기록한다.

**Q: 패밀리 타입이 너무 많으면 모두 유지해야 하나요?**
A: 결론: 모두 유지하지 말고 실제 프로젝트 사용 타입, 제조사 옵션, 수량 산출 필요성을 기준으로 정리한다. 1. 타입 수, 파일 크기 MB, 제품 코드, Shared Parameter, LOD 300/350, COBie/IFC 매핑, Source/Tags를 확인한다. 2. 조건상 발주처 납품이나 FM 인계에 필요한 타입은 유지하고, 반면 렌더링용·미사용 옵션은 별도 라이브러리로 분리한다. 타입 과다는 모델 성능 저하, 수량 오류, 일정 지연, 책임 리스크를 만든다. 24시간 내 패밀리 담당자가 타입 정리표를 기록·공유하고 7일 내 승인권자, 기한, 배포 로그를 남긴다.

**Q: 패밀리 파라미터가 프로젝트 표준과 다르면 기존 패밀리를 고쳐야 하나요?**
A: 결론: 바로 고치지 말고 납품 기준, 수량표, 태그, IFC/COBie 매핑에 영향을 주는지 확인한다. 1. Shared Parameter GUID, 파라미터명, Instance/Type 구분, Revit 버전, LOD 300/350, BEP/EIR, Source/Tags를 대조한다. 2. 조건상 태그와 수량표에 쓰이는 경우 표준 파라미터로 수정하고, 반면 내부 제작 편의용이면 매핑표로 관리할 수 있다. 무단 수정은 태그 오류, 수량 누락, 납품 반려, 법무 책임 리스크를 만든다. 24시간 내 담당자가 차이표를 요청·기록하고 7일 내 승인자, 기한, 수정 로그를 공유한다.

**Q: 프로젝트마다 수정한 패밀리를 공용 라이브러리에 바로 올려도 되나요?**
A: 결론: 바로 올리지 않고 프로젝트 예외인지 표준 개선인지 구분한 뒤 승인 버전으로 등록한다. 1. Revit 버전, 패밀리 카테고리, Shared Parameter GUID, 타입 수, 파일 크기 MB, LOD 300/350, COBie/IFC 매핑, Source/Tags를 확인한다. 2. 조건상 여러 프로젝트에서 같은 문제가 반복되면 표준 라이브러리 후보로 두고, 반면 특정 발주처 BEP/EIR만 맞춘 수정이면 프로젝트 전용으로 분리한다. 다만 커넥터·수량·태그에 영향이 있으면 예외 없이 샘플 모델 테스트와 승인 로그를 남긴다. 무단 등록은 모델 성능 저하, 수량 오류, 납품 반려, 책임 리스크를 만든다. 24시간 내 패밀리 담당자가 변경표를 기록·공유하고 7일 내 승인권자, 기한, 라이브러리 배포 로그를 보고한다.
