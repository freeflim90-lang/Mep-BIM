## 2026-06-20 설계 시방서/시공성 충돌 실무 케이스 Q&A
- Source: `knowledge/40_curation/quality/2026-06-20_EXTENSION_AGENTS_PRACTICAL_CASE_LIBRARY.md`

**Q: 설계 시방서 기준은 맞지만 시공성이 떨어지면 어떻게 하나요?**
A: 결론: 기준을 임의로 낮추지 않고 시공성 충돌을 RFI/BCF 후보로 남긴다. 1. 담당자는 시방서 조항, 설계 의도, Revit 모델 위치, Navisworks 간섭, LOD 300/350 상세 수준, Source, Tags를 24시간 안에 확인한다. 2. 조건상 설치 공간, 점검구, 유지관리 동선, 비용·일정 리스크가 있으면 대안 상세 2개와 책임 범위를 기록해 설계 담당자에게 요청한다. 반면 단순 표현 오류라면 정정 로그로 처리한다. 다만 안전, 법무, 계약 책임이 걸리면 예외 없이 승인 담당자와 7일 검토 기한을 공유한다.

**Q: 시방서 문구와 모델 속성이 서로 다르면 무엇을 기준으로 검토하나요?**
A: 결론: 어느 한쪽을 임의 기준으로 삼지 않고 계약 우선순위와 최신 승인본을 확인한다. 1. 담당자는 시방서 버전, Revit 속성, BEP, EIR, 도면 revision, Source, Tags, 24시간 확인 로그를 비교한다. 2. 조건상 LOD 300/350 납품 기준과 충돌하면 RFI/BCF로 설계팀에 질의하고 변경 책임을 구분한다. 반면 모델 입력 오류라면 BIM 담당자에게 수정 요청을 공유한다. 다만 법무, 비용, 일정 리스크가 발생하면 예외 없이 승인자와 7일 처리 기한을 기록한다.

**Q: 고객이 시방서 기준을 완화해 달라고 하면 어떻게 답하나요?**
A: 결론: 완화 요청은 가능/불가보다 승인 조건과 리스크를 먼저 안내한다. 1. 담당자는 해당 조항, 법규 KDS/KCS, 안전 기준, 성능 수치, Revit/BEP 산출물 영향, Source와 Tags를 확인해 로그로 남긴다. 2. 조건상 비용 절감 목적이라도 품질, 유지관리, 법무 책임이 커지면 완화하지 않고 대안을 요청한다. 반면 발주처 승인과 동등 성능 근거가 있으면 검토 후보가 될 수 있다. 다만 승인 전에는 확정 답변을 금지하고 담당자, 기한, 7일 검토 결과를 공유한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `설계_시방서` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

## 2026-06-21 설계 시방서 검토 추가 Q&A
- Source: LUA BIM LABS coverage expansion
- Tags: design,specification,revit,bep,rfi,2026

**Q: 설계 시방서에 "동등 이상"이라고만 되어 있으면 모델 검수 기준은 어떻게 정하나요?**
A: 결론: "동등 이상"을 그대로 검수 기준으로 쓰지 말고 성능 수치, 인증, 승인 절차, 대체 자재 조건으로 분해한다. 1. 시방서 조항, KS/KDS/KCS, 성능 %, 제조사 자료, Revit 파라미터, LOD 300/350, Source/Tags를 확인한다. 2. 조건상 수치와 인증이 명확한 경우 모델 속성 기준으로 만들고, 반면 승인 절차가 필요한 경우 RFI/BCF로 설계자 확인을 요청한다. 모호한 기준은 품질 오류, 비용 분쟁, 납품 반려, 법무 책임 리스크를 만든다. 24시간 내 담당자가 기준표를 기록·공유하고 7일 내 승인자, 기한, 검토 로그를 남긴다.

**Q: 설계 시방서 변경이 이미 작성한 모델과 수량표를 바꾸면 어떻게 처리하나요?**
A: 결론: 단순 수정으로 처리하지 않고 변경 사유, 적용 범위, 수량 영향, 승인 필요성을 먼저 확인한다. 1. 시방서 Revision, 변경 조항, Revit 모델 요소, 수량표, BEP/EIR, LOD 300/350, Source/Tags를 비교한다. 2. 조건상 변경이 신규 구역에만 적용되는 경우 해당 범위만 수정하고, 반면 기존 납품물 전체에 영향이 있으면 변경 승인과 견적 조정을 요청한다. 무승인 반영은 비용 손실, 일정 지연, 모델 오류, 책임 리스크를 만든다. 24시간 내 설계 시방서 담당자가 영향표를 기록·공유하고 7일 내 승인권자, 기한, 변경 로그를 보고한다.
