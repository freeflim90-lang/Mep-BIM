실무 보강 (2026-06-20 답변 품질 보강):

**Q: 협력사가 공동 제안서에 LUA BIM LABS 이름을 넣자고 하면 무엇을 확인하나요?**
A: 결론: 이름 사용보다 공동 산출물, 책임 범위, 고객 접점, 수익 배분, 승인권자를 먼저 확인한다. 1. 제안서 역할, BIM 납품 범위, LOD 300/350, NDA, 개인정보, 브랜드 사용 문구, 계약 책임을 확인한다. 2. 조건상 당사가 실제 수행과 검수 책임을 갖는다면 공동 제안 후보로 두고, 반면 이름만 빌려주는 구조이면 보류한다. 잘못 승인하면 법무, 브랜드, 비용, 고객 책임 리스크가 생긴다. 24시간 내 얼라이언스 담당자가 제안서를 요청·기록하고 7일 내 CEO/법무 승인, 기한, 검토 로그를 공유한다.

**Q: 파트너가 고객 데이터나 모델 파일을 공유해 달라고 하면 바로 전달해도 되나요?**
A: 결론: 바로 전달하지 않고 NDA, 고객 동의, 데이터 범위, 보안 채널, 사용 목적을 먼저 확인한다. 1. 모델 파일, 도면, 개인정보, 계약상 재위탁 가능 여부, Source 저장 위치, 접근 로그를 확인한다. 2. 조건상 고객 승인과 NDA가 있고 최소 범위 공유라면 제한 전달을 검토하고, 반면 목적이 불명확하거나 재배포 가능성이 있으면 차단한다. 무단 공유는 개인정보, 보안, 법무, 신뢰 리스크가 크다. 24시간 내 담당자가 승인 증빙을 요청·기록하고 7일 내 승인권자, 기한, 공유 로그를 남긴다.

**Q: 파트너가 공동 수익 배분을 제안하면 기술 검토보다 계약을 먼저 봐야 하나요?**
A: 결론: 계약과 기술 검토를 함께 본다. 수익 배분은 공동 산출물, 고객 소유권, 납품 책임, 지원 비용이 명확할 때만 판단한다. 1. 매출 배분 %, 담당 업무, BIM 납품 범위, LOD 300/350, 고객 접점, NDA, Source/Tags를 확인한다. 2. 조건상 파트너가 고객 확보와 1차 지원을 맡는 경우 배분 후보가 될 수 있지만, 반면 당사가 납품 책임과 하자 대응을 모두 지면 배분율을 재검토한다. 모호한 배분은 비용 손실, 법무, 브랜드, 고객 책임 리스크를 만든다. 24시간 내 얼라이언스 담당자가 조건표를 기록·공유하고 7일 내 승인권자, 기한, 계약 검토 로그를 남긴다.

**Q: 파트너 역량이 좋아 보여도 파일 보안 체계가 약하면 협력해도 되나요?**
A: 결론: 바로 협력하지 않고 보안 수준, 접근 권한, 로그, 고객 데이터 처리 절차를 검증한다. 1. CDE/ACC 권한, 파일 공유 채널, NDA, 개인정보 처리, 보안 교육, 프로젝트 이력, Source/Tags를 확인한다. 2. 조건상 샘플 데이터와 제한 권한으로 시작할 수 있지만, 반면 고객 원본 모델 접근이나 재배포 가능성이 있으면 승인 전 차단한다. 보안이 약하면 개인정보, 법무, 납품 오류, 신뢰 리스크가 생긴다. 24시간 내 담당자가 보안 체크리스트를 요청·기록하고 7일 내 승인자, 기한, 파일 공유 로그를 보고한다.

**Q: 파트너 PoC가 분위기는 좋지만 실적 지표가 애매하면 계속 연장해도 되나요?**

PoC 기간 30일/90일, 리드 수, 전환율 %, BIM 납품 범위, LOD 300/350, NDA, 고객 피드백, Source/Tags를 확인한다. 2. 조건상 공동 고객 1곳 이상과 검증 산출물이 있으면 연장 후보로 두고, 반면 책임 경계나 수익 배분이 불명확하면 우선 종료 또는 재협상으로 분리한다. 다만 고객 데이터 공유가 포함되면 예외 없이 법무·보안 승인을 먼저 받는다. 무기한 연장은 비용 손실, 브랜드 훼손, 고객 약속 오류, 법무 책임 리스크를 만든다. 24시간 내 얼라이언스 담당자가 PoC 평가표를 기록·공유하고 7일 내 승인권자, 기한, 종료/연장 로그를 보고한다.
