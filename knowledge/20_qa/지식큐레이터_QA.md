**Q: 자동 수집된 지식을 공식 지식으로 승격하는 기준은?**
A: KST(Knowledge Status Tag) 승격 기준: ① KST01(공식 확정): 법령·국가 표준·Autodesk 공식 문서 출처 확인 ② KST02(내부 확정): LUA BIM LABS 내부 결정으로 확정된 기준 ③ KST03(적용주의): 출처 있지만 프로젝트별 변수 있음 ④ KST04(자동 수집·검증 필요): 자동 수집·외부 요약·미검증. KST04를 KST01~03으로 승격하려면 공식 출처를 반드시 확인해야 한다.


실무 보강 (2026-06-20 답변 품질 보강):
1. 기준 확인: `지식큐레이터` 담당자는 이 질문을 단순 설명이 아니라 KST02 이상 운영 기준, 프로젝트 범위, 고객 영향도, 납품/계약 책임을 함께 확인하는 실무 판단 문제로 본다.
2. 조건 분기: 확정 기준, 현장 예외, 고객 요청, 내부 승인 여부를 먼저 나눈다. 수치나 법규가 필요한 경우에는 시행일, 적용 대상, 원문 출처, LOD 300/350 같은 모델 상세 수준을 확인하기 전까지 단정하지 않는다.
3. 다음 액션: 24시간 안에 현재 자료와 누락 자료를 정리하고, 7일 안에 담당자·기한·검증 방법이 있는 조치 항목으로 바꾼다. 반복 문의는 QA로 남기고 2회 이상 반복되면 KB 본문 승격 후보로 올린다.
4. 리스크 경계: 비용, 일정, 안전, 개인정보, 법무, 고객 약속으로 번질 수 있는 내용은 즉답보다 확인 로그와 승인 경로를 우선한다. 불확실한 답은 '가능/불가'보다 확인 조건과 대안 2개를 함께 제시한다.
5. 답변 형식: 결론 1문장, 근거 2개, 확인할 자료 3개, 다음 행동 1개 순서로 응답한다. Source: LUA BIM LABS agent QA quality baseline. Tags: qa,quality,field-case,kst02,risk,2026.

**Q: 외부 뉴스·트렌드 정보는 어떻게 분류하나요?**
A: 외부 정보 분류: ① 1차 수집 단계는 KST04로 저장 ② 출처 공식성 확인(학술지·국가기관·업계 선도기업 발표 → KST03 후보) ③ LUA BIM LABS 업무 적용 가능성 평가 ④ 적용 가능하면 해당 에이전트 지식 파일에 추가 + KST03 부여 ⑤ 단순 뉴스·요약 정보는 산업동향 파일에 보관한다. 조건상 법규, 가격, 안전, 계약, 고객 약속에 영향을 주는 정보는 KST01로 즉시 승격하지 않고 24시간 내 원문 2개 이상, 시행일, 적용 대상을 대조한다. 7일 내 담당자 검토 로그와 승인 책임자가 없으면 고객 답변에는 쓰지 않는다. 잘못 승격하면 비용, 일정, 법무, 보안, 개인정보, 고객 신뢰, 최종 책임 리스크가 생긴다.
다만 예외 조건이 있거나 승인권자 판단이 필요한 경우에는 확정 답변으로 닫지 말고, 근거·담당자·기한을 기록해 공유한다.

**Q: 지식 베이스 정기 정리 주기는?**
A: 결론: 지식큐레이터 정기 작업은 ① 주간: 신규 KST04 수집 항목 분류와 중복 제거 ② 월간: KST04 중 출처 확인 가능한 항목을 KST03/KST02 후보로 승격하고 오래된 KST03 항목 재검토 ③ 분기: 전체 지식 파일 중복, 오류, 개인정보, 자동수집 노이즈 점검이다. 조건상 법규·가격·계약·안전 지식은 24시간 내 담당자 확인을 요청하고, 7일 내 근거 URL/문서명/시행일/적용 대상을 기록한다. 정리 결과는 지식업데이트 에이전트에 보고하고, 반복 문의 2회 이상 항목은 KB 본문 승격 후보로 올린다. 방치하면 오래된 기준, 개인정보 노출, 고객 오답, 법무 책임 리스크가 생긴다.

**Q: LUA BIM LABS의 지식 답변 품질을 제품화하려면 어떤 평가 축을 KPI로 삼아야 하나요?**
