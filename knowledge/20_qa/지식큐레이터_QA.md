# 지식큐레이터 Q&A 지식

텔레그램 질문·자동 수집 응답 저장소 — 기준 가이드라인(지식큐레이터.md)과 분리 운영.

## 2026-06-09 지식큐레이터 실전 Q&A
- Source: LUA BIM LABS 지식 관리 운영 기준, KST 체계
- Tags: knowledge,curation,kst,promotion,qa,2026

**Q: 자동 수집된 지식을 공식 지식으로 승격하는 기준은?**
A: KST(Knowledge Status Tag) 승격 기준: ① KST01(공식 확정): 법령·국가 표준·Autodesk 공식 문서 출처 확인 ② KST02(내부 확정): LUA BIM LABS 내부 결정으로 확정된 기준 ③ KST03(적용주의): 출처 있지만 프로젝트별 변수 있음 ④ KST04(자동 수집·검증 필요): 자동 수집·외부 요약·미검증. KST04를 KST01~03으로 승격하려면 공식 출처를 반드시 확인해야 한다.

**Q: 지식 파일에 중복 내용이 생기면 어떻게 처리하나요?**
A: ① 지식큐레이터가 중복 항목 식별 ② 더 높은 KST 등급 또는 더 최신 날짜의 항목을 기준으로 채택 ③ 오래된 항목은 삭제하지 않고 `[DEPRECATED]` 태그 또는 주석 처리 후 보관 ④ 변경 이력을 지식 파일 상단에 기록. 삭제는 지식업데이트 에이전트 확인 후 수행.

**Q: 외부 뉴스·트렌드 정보는 어떻게 분류하나요?**
A: 외부 정보 분류: ① 1차 수집 단계: KST04로 저장 ② 출처 공식성 확인 (학술지·국가기관·업계 선도기업 발표 → KST03 후보) ③ LUA BIM LABS 업무에 적용 가능성 평가 ④ 적용 가능하면 해당 에이전트 지식 파일에 추가 + KST03 부여 ⑤ 단순 뉴스·요약 정보는 산업동향 파일에 보관 (지식 파일 분리). 자동 수집 정보를 KST01로 즉시 승격하지 않는다.

**Q: 지식 베이스 정기 정리 주기는?**
A: 지식큐레이터 정기 작업: ① 주간: 신규 자동 수집 항목 분류 (KST04 → 검토 대상 이동) ② 월간: KST04 항목 중 출처 확인 가능한 것 승격 처리 + 오래된 KST03 항목 재검토 ③ 분기: 전체 지식 파일 중복/오류 점검. 정기 정리 결과는 지식업데이트 에이전트에 보고하고 내재화 루프를 갱신한다.

**Q: 새로운 에이전트(AI 역할) 추가 시 지식큐레이터 역할은?**
A: ① 신규 에이전트 역할 정의 검토 ② 기존 에이전트와 역할 중복 여부 확인 ③ 신규 에이전트용 지식 파일 초안 작성 지원 ④ QA 파일 생성 필요 여부 판단 ⑤ KNOWLEDGE_AGENTS 목록 업데이트 요청. 지식큐레이터가 신규 에이전트 온보딩 체크리스트를 AI 협업운영체계 기준에 따라 검토한다.
