## 1. 연차 포지셔닝

14년차는 PM 역할을 안정적으로 수행하면서 **복수 프로젝트를 동시에 관리**하는 능력을 갖추는 해다.  
13년차에 단일 프로젝트로 연습했던 제안·SOW·보고를 이제 **2개 이상 프로젝트에서 병행**한다.

| 이전 단계(13년차) | 14년차 전환 목표 | 다음 단계(15년차) |
|---|---|---|
| 단일 프로젝트 PM | 2개 이상 프로젝트 동시 관리 | BIM 컨설팅 독립 수행 |
| 킥오프·월간 보고 | 비용·일정·품질 통합 관리 | 고객 BIM 성숙도 진단 |
| 리스크 기초 관리 | 복합 리스크 조기 대응 | AI 거버넌스 설계 |
| 팀 리드 기초 지원 | 리드 성과 관리·성장 지원 | 컨설팅 제안 독립 작성 |

## 4. 복수 프로젝트 PM 운영

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 진행 중 전체 프로젝트 목록 |
| 단계 | 제안/계약/수행/납품/종료 |

## 6. 고객 의사결정 지원

### 의사결정 구조 파악

| 항목 | PM 파악 포인트 |
|---|---|
| 승인 권한자 | 누가 최종 결정을 내리는가 |
| 의사결정 속도 | 발주처 내부 검토 기간 예측 |
| 보고 경로 | 담당자 → 팀장 → 임원 경로 파악 |
| 민감 이슈 | 예산·일정·법규 이슈는 별도 보고 |

### 선택지 제시 방법

```
[의사결정 요청]
배경: 현재 어떤 상황이 발생했는가
영향: 결정을 늦추면 어떤 리스크가 있는가

선택지 A: [방법] — 장점 / 단점 / 비용 영향
선택지 B: [방법] — 장점 / 단점 / 비용 영향

PM 권고: [A 또는 B] — 이유 1줄
결정 기한: [날짜]
```

## 7. 계약·변경 관리 심화

### 변경 관리 프로세스

1. 변경 요청 접수 → 내용·범위·사유 확인
2. 영향 분석 → 공수·일정·품질 영향 계산
3. 계약 범위 확인 → 무상/유상 여부 판단
4. 유상 변경 → 견적 제출 → 고객 승인
5. 승인 후 SOW 개정 → 팀 공지 → 작업 반영
6. 변경 이력 → 변경 관리 대장 기록

### 계약 불이익 조항 식별

| 조항 | 위험 포인트 |
|---|---|
| 무제한 수정 | 납품 후 수정 횟수 미명시 → 유상 기준 설정 |
| 포괄적 하자 | 하자 범위 불명확 → 구체적 기준 협의 |
| 과도한 지체 | 발주처 귀책 지연도 PM 책임 → 면책 조항 추가 |
| 보안 의무 | 데이터 반출·보관 기준 불명확 → 구체화 요청 |

## 8. 팀 성과 관리

### 분기별 리드 성과 리뷰

| 항목 | 리뷰 내용 |
|---|---|
| BEP·납품 품질 | 발주처 반려율, 보완 횟수, 검수 통과율 |
| 팀 운영 | 업무 배분·주간 점검·후배 지도 기록 |
| 자동화 기여 | 기능 명세·QA 산출물 완성도 |
| 성장 과제 수행 | 지난 분기 설정 과제 달성 여부 |

### 성과 관리 원칙

- 잘한 점을 먼저 명확히 인정
- 개선 과제는 구체적 행동 기준으로 제시
- 성장 지원: 교육 기회·새로운 역할·멘토링 연결
- 기록: 모든 리뷰 내용 문서화 (승급 심사 자료)

## 10. 파트너사·협력사 관계 관리

### 협력사 성과 평가

| 평가 항목 | 기준 |
|---|---|
| 납품 품질 | 반려 횟수·보완 기간·오류 유형 |

- 계약 안정성: 사업자 등록, NDA 서명, 보안 요건 충족
