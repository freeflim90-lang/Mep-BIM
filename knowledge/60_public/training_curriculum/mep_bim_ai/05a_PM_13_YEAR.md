## 1. 연차 포지셔닝

13년차는 Lead 기술 실무에서 **PM 사업·운영 관리**로 전환하는 첫 해다.  
12년차에서 보조 역할로 경험했던 제안·SOW·고객 보고를 이제 **단독으로 주관**한다.

| 이전 단계(12년차) | 13년차 전환 목표 | 다음 단계(14년차) |
|---|---|---|
| 제안서 섹션 보조 작성 | 제안서 전체 독립 작성 | 복수 프로젝트 PM 병행 |
| SOW 초안 작성 보조 | SOW·견적 주관 및 발주처 협의 | 비용·공수 관리 심화 |
| PM 보조 수행 | 프로젝트 킥오프 및 고객 정기 보고 주관 | 고객 의사결정 지원 주도 |
| 부서 표준화 주도 | 팀 리드(9~12년차) 지원·관리 | 리드 성장 평가 체계 운영 |

## 4. 제안서 독립 작성

### 제안서 필수 구성

| 섹션 | 핵심 내용 | PM 작성 포인트 |
|---|---|---|
| 고객 문제 이해 | 발주처가 직면한 BIM 수행 문제 정의 | 고객 언어로 문제 기술 |
| 수행 전략 | 공종별 BIM 계획, CDE, 납품 전략 | 차별화 요소 명시 |
| 팀 구성 | 연차별 역할, 협력사 계획 | 이름보다 역할·경력 중심 |
| 기대효과 | 간섭 감소율, 납품 오류율, 공기 절감 수치 | 과거 레퍼런스 기반 수치 |
| 수행 일정 | 단계별 납품 마일스톤 | 계약 일정과 연동 |
| 산출물 | RVT/IFC/NWD/보고서/COBie 목록 | 발주처 요건과 매핑 |

### 제안서 작성 프로세스

1. 발주처 RFP·EIR 분석 → 핵심 요건·평가 기준 파악
2. 현장·규모·공종 특성 분석 → 리스크 사전 식별
3. 제안서 초안 작성 → Lead·Principal 내부 검토
4. 차별화 포인트 강화 → 경쟁사 대비 강점 명시
5. 최종본 제출 → 프레젠테이션 준비

## 5. SOW 및 견적 주관

### SOW 핵심 항목

| 항목 | 작성 기준 |
|---|---|
| 포함 범위 | 공종, 구역, LOD, 검토 회차, 납품 형식 명시 |
| 제외 범위 | 무상 추가 방지 — 현장 조사, 구조 계산, FM 운영 등 |
| 변경 관리 | 계약 외 요청 견적·승인 절차, 변경 단가 |
| 납품 조건 | 형식, 검수 기준, 반려·보완 횟수 제한 |
| 지체 조항 | 납품 지연 시 면책 사유, 책임 범위 |
| 하자 조항 | 납품 후 수정 범위, 유상 여부 기준 |

### 견적 산정 방법

| 구성 | 산정 기준 |
|---|---|
| 직접 공수 | 공종별·단계별 투입 인일 × 단가 |
| 소프트웨어 | 라이선스·구독료 프로젝트 안분 |
| 협력사 | 외주 견적 + 관리 비용 |
| 리스크 예비비 | 전체 직접비의 5~15% |
| 마진 | 수익성 목표에 따라 조정 |

## 6. 프로젝트 킥오프 운영

### 킥오프 준비

| 항목 | PM 준비 내용 |
|---|---|
| BEP 초안 | 킥오프 전 배포, 협의 항목 표시 |
| 조직도·RACI | 발주처·시공사·설계사·BIM팀 역할 정리 |
| 일정표 | 단계별 마일스톤, 납품일, 회의 주기 |
| CDE 접근 안내 | 계정 발급, 폴더 구조, 업로드 절차 |
| 커뮤니케이션 계획 | 보고 주기, 채널, 에스컬레이션 경로 |

### 킥오프 회의 운영

1. 프로젝트 개요 공유 (목적·범위·일정)
2. BEP 핵심 기준 협의 (LOD·명명·CDE)
3. 역할·책임 확인 (RACI 공유)
4. Q&A → 합의 사항 회의록 작성
5. 다음 마일스톤·체크인 일정 확정

## 8. 비용·공수 관리 기초

| 항목 | 관리 방법 |
|---|---|
| 투입 공수 추적 | 주간 실투입 인일 vs 계획 비교 |
| 예산 소진율 | 누적 집행 비율 vs 진행률 비교 |
| 재작업 공수 | 반려·보완 발생 시 추가 공수 기록 |
| 변경 관리 | 범위 변경 → 추가 견적 → 승인 후 반영 |
| 수익성 경보 | 예산 80% 소진 시 PM 자체 경보 발동 |

## 9. 리스크 관리

### 리스크 분류

| 유형 | 예시 |
|---|---|
| 계약 리스크 | 무상 추가 범위, 지체 책임, 하자 기간 |
| 납품 리스크 | 협력사 지연, 발주처 반려, 법규 변경 |
| 보안 리스크 | 고객 모델 유출, 외부 AI 입력 |
| 일정 리스크 | 설계 변경, 자원 부족, 의사결정 지연 |
| 품질 리스크 | 협력사 품질 저하, 모델 기준 불통일 |

### 리스크 대응 원칙

- 고영향·고가능성 리스크는 즉시 고객 보고
- 중간 리스크는 완화 계획 수립 후 월간 보고 포함
- 저위험 리스크는 등록부에 기록·모니터링

## 11. 팀 리드 지원 및 관리

PM은 Lead(9~12년차)가 기술 실무를 독립적으로 수행하도록 **방향·자원·의사결정**을 지원한다.

| 지원 항목 | PM 역할 |
|---|---|
| 방향 제시 | BEP·납품 기준 결정 시 PM 승인 제공 |
| 자원 배분 | 리드 간 인력 충돌 시 우선순위 결정 |
| 에스컬레이션 | 리드가 해결 못한 이슈 고객과 직접 협의 |
| 성과 피드백 | 분기별 리드 성과 검토·다음 성장 과제 제시 |
| 동기부여 | 성과 인정, 새로운 역할 부여, 교육 기회 제공 |
