## 1. 연차 포지셔닝

15년차는 PM 실무 수행 역량을 갖춘 상태에서 **컨설팅 역할이 본격 추가**되는 해다.  
단순 수행 PM을 넘어, 고객의 BIM 도입·운영 문제를 진단하고 해결 방향을 제시한다.

| 이전 단계(14년차) | 15년차 전환 목표 | 다음 단계(16년차) |
|---|---|---|
| 수행 프로젝트 PM | BIM 컨설팅 독립 수행 | 대형 프로젝트 PM 주도 |
| 고객 의사결정 지원 | 고객 BIM 성숙도 진단 제공 | 교육 사업 참여 |
| AI 투자 판단 | AI 거버넌스 설계 주도 | 방법론 초안 작성 참여 |
| 팀 성과 관리 | 조직 내 지식 전파·세미나 | Principal 준비 시작 |

## 4. BIM 컨설팅 독립 수행

### 컨설팅 4단계 방법론

| 단계 | 내용 | 산출물 |
|---|---|---|
| 1. 진단 | 현재 BIM 수행 현황·문제·성숙도 파악 | 현황 진단 보고서 |
| 2. 설계 | 목표 수준·개선 방향·이행 계획 수립 | BIM 개선 로드맵 |
| 3. 이행 | 표준 수립·교육·파일럿 프로젝트 지원 | 이행 지원 보고서 |
| 4. 평가 | 목표 대비 성과 측정·개선 과제 도출 | 성과 평가 보고서 |

### 컨설팅 시 흔한 고객 문제 유형

| 문제 유형 | 진단 포인트 | 해결 방향 |
|---|---|---|
| BIM 형식적 수행 | LOD·파라미터 불충분, 납품용 모델만 | 목적 기반 BIM 기준 재정립 |
| 공종 간 정보 단절 | CDE 미운영, 이메일·USB 파일 공유 | CDE 도입·운영 기준 수립 |
| 협력사 품질 불균형 | 협력사별 수준 차이, 기준 미전달 | 표준 패키지·온보딩 체계 수립 |
| 자동화 없이 반복 작업 | 보고·검수·물량 수작업 | 자동화 후보 발굴·ROI 분석 |
| BIM 성과 측정 불가 | KPI 없음, 효과 비가시화 | BIM KPI 체계 설계 |

## 5. 고객 BIM 성숙도 진단

### BIM 성숙도 레벨 정의

| 레벨 | 수준 | 특징 |
|---|---|---|
| L0 | 미도입 | 2D 도면 중심, BIM 계획 없음 |
| L1 | 형식 도입 | 요건 대응용 모델, 활용 없음 |
| L2 | 기초 수행 | 공종별 모델, 간섭 검토 수행 |
| L3 | 통합 운영 | CDE 운영, 다공종 조율, 납품 체계 |
| L4 | 데이터 활용 | 물량·FM·자동화 연계, KPI 측정 |
| L5 | 전략적 자산 | BIM → 조직 지식 자산화, 제품화 |

### 진단 항목

| 진단 축 | 세부 항목 |
|---|---|
| 도구·환경 | 소프트웨어, CDE, 버전, 라이선스 |
| 프로세스 | BEP 존재 여부, CDE 운영, 납품 절차 |
| 인력 역량 | 연차별 BIM 수행 능력 |
| 품질 체계 | QA 기준, 자동화 검수 여부 |
| 데이터 활용 | 물량·FM·시공 연계 여부 |
| 거버넌스 | 표준문서, 교육 체계, AI 사용 기준 |

## 6. AI 거버넌스 설계

| 설계 항목 | 내용 |
|---|---|
| AI 사용 허용 범위 | 초안 생성·요약·비교·자동화 보조 목적 허용 |
| AI 사용 제한 범위 | 고객 모델·도면·개인정보 외부 AI 입력 금지 |
| 검증 의무 | AI 결과는 담당자가 사실 검증 후 사용 |
| 승인 절차 | 계약·법무 문구는 PM/Legal 검토 필수 |
| 유지관리 | AI 도구 변경 시 거버넌스 업데이트 |
| 위반 처리 | 위반 사례 기록·교육 반영·재발 방지 |

### AI 거버넌스 문서 구성

```
AI 거버넌스 v[버전]
작성일 / 승인자

1. 목적과 적용 범위
2. 허용 사용 사례 목록
3. 금지 사용 사례 목록
4. 검증 의무 및 절차
5. 보안 기준 (데이터 분류별 처리 방법)
6. 위반 처리 절차
7. 개정 이력
```

| 항목 | 수행 PM | 컨설팅 리드 |
