## 5. BEP 작성 입문

9년차는 BEP 전 섹션을 단독으로 작성하기보다, **핵심 섹션 2개 이상을 선배 리드의 검토 아래 독립 작성**하는 경험을 쌓는다.

### 9년차가 집중할 BEP 섹션

| 섹션 | 핵심 내용 | 작성 시 주의사항 |
|---|---|---|
| 모델링 표준 | LOD 정의, 파라미터, 명명 규칙, 템플릿 | 프로젝트 공종별 차이 반영 |
| CDE 구조 | WIP/Shared/Published 폴더·권한 설계 | 발주처 요구 CDE 플랫폼 확인 |
| 간섭 검토 계획 | 주기, 규칙, 보고 형식, 책임자·기한 | 해결 기한과 책임자 포함 |
| 납품 계획 | 납품 형식·날짜·검수 기준 | 계약 납품 일정과 동기화 |

### BEP 작성 학습 절차

1. 기존 BEP 사례 2~3개 검토 → 섹션 구조·표현 패턴 파악
2. 담당 섹션 초안 작성 → 선배 리드 리뷰
3. 리뷰 의견 반영 → 개정 이력(Rev.01) 관리
4. 작성 과정에서 발견한 표준 공백 → 개정 후보로 등록

| 3~5년차 | 판단 근거 설명하기, 보고서 초안 작성 능력 |

### 멘토링 실천 방법

- 주 1회 15분: 담당 후배 진행 상황 확인 및 피드백

- 교안 1건 이상 작성: 반복 설명했던 내용을 문서화

## 11. Add-in 요구사항 작성

9년차는 Add-in을 개발하지 않지만, **반복 업무를 자동화 문제로 정의하고 기능 명세서를 작성**하는 능력을 키운다.

### 기능 명세서 작성 양식

```
기능명: [동사+목적어] (예: 파라미터 누락 요소 강조 표시)

문제:
  - 현재 어떤 업무를 수작업으로 하고 있는가?
  - 오류 빈도, 소요 시간은?

입력:
  - 어떤 모델, 파라미터, 기준값이 필요한가?

처리:
  - 어떤 조건으로 요소를 필터링하는가?
  - 어떤 값을 읽거나 쓰는가?

출력:
  - 화면 강조, XLSX 보고서, 로그 중 무엇인가?

예외:
  - 링크 파일 포함 여부, 파라미터 없는 요소 처리

검증 방법:
  - 샘플 모델에서 예상 결과와 실제 결과 비교
```

### Revit API 핵심 개념 (이해 수준)

| 개념 | 설명 |
|---|---|
| Transaction | 모델 변경 작업의 단위 — 변경 범위 정의에 활용 |
| FilteredElementCollector | 요소 수집 쿼리 — 검수 대상 요소 범위 지정 |
| BuiltInParameter | Revit 내장 파라미터 코드 — 파라미터 명세 작성 |
| FamilyInstance | 패밀리 배치 요소 — 장비·피팅 파라미터 검토 |
| View / Sheet | 뷰와 도면 시트 구조 — 납품 뷰 자동화 요건 |
