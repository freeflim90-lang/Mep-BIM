## 1. 연차 포지셔닝

## 4. 방법론 백서 초안 작성

### 방법론 백서 챕터 구성

| 챕터 | 내용 | 분량 기준 |
|---|---|---|
| 1. 서론 | LUA BIM LABS의 MEP BIM+AI 비전과 차별화 | 3~5페이지 |
| 2. MEP BIM 수행 체계 | 단계별 LOD·납품·CDE·검수 기준 | 15~20페이지 |
| 3. 품질 관리 체계 | QA 기준, 자동 검수, 이슈 관리 | 10~15페이지 |
| 4. AI 활용 원칙 | 보조 업무 분류, 검증, 보안 기준 | 8~12페이지 |
| 5. 자동화 로드맵 | 반복 업무 → Add-in → 제품화 체계 | 8~12페이지 |
| 6. 교육·역량 체계 | 연차별 역할·교육·평가 체계 | 10~15페이지 |
| 7. 조직 표준 운영 | 개정·승인·배포·감사 거버넌스 | 5~8페이지 |
| 8. 사례 및 성과 | 프로젝트 적용 사례, KPI 결과 | 10~15페이지 |

### 방법론 작성 원칙

- 현장 경험 기반: 이론보다 실제 적용 사례 우선
- 재사용 가능: 다양한 프로젝트 유형에 공통 적용 가능
- 연차별 독자: 각 연차 담당자가 자기 파트를 찾아 적용 가능
- 버전 관리: 연 1회 이상 개정, 이력 관리
- 검증 가능: 모든 기준에 측정·확인 방법 포함

## 5. 표준 거버넌스 체계 수립

### 표준 거버넌스 구조

| 요소 | 내용 |
|---|---|
| 표준 분류 | 모델링·CDE·납품·품질·교육·AI·보안 |
| 소유권 | 표준별 담당자(Principal/PM/Lead) 지정 |
| 개정 주기 | 정기(연 1회) + 수시(법규 변경·중대 오류) |
| 승인 권한 | 분류별 승인 권한자 명시 (Lead/PM/Principal) |
| 배포 채널 | CDE, 팀 채널, 교육 교안 동시 업데이트 |
| 감사 | 반기 1회 표준 준수 현황 점검 |
| 폐기 기준 | 적용 프로젝트 없는 표준 정리 주기 |

### 표준 운영 규정 구성

```
표준 거버넌스 규정 v[버전]
승인자:

1. 표준 분류 체계
2. 표준별 소유자 목록
3. 개정 절차 (요청→검토→승인→배포)
4. 긴급 개정 절차
5. 표준 감사 절차
6. 표준 폐기 절차
7. 개정 이력
```

## 6. 교육 로드맵 전체 점검

### 갭 분석 방법

1. 각 연차별 교육 과정 수료 산출물 현황 조사
2. 팀원 인터뷰 — 교육에서 못 배운 것, 현장에서 필요했던 것
3. 프로젝트 오류 사례 → 교육 부족 연결 항목 도출
4. 산업 트렌드 — ISO 개정, 새 소프트웨어, AI 변화 반영 필요 항목
5. 개선안 작성 → 연차별 커리큘럼 파일 개정 후보 목록

### 교육 체계 개선 방향

| 항목 | 개선 포인트 |
|---|---|
| 산출물 기준 | 더 명확한 합격 기준·샘플 추가 |
| 평가 방식 | 자기 평가 + 리드 리뷰 병행 체계화 |
| AI 교육 | 연차별 AI 활용 목표 구체화 |
| 법규 교육 | BIM 의무화 기준 연차별 이해 수준 추가 |
| 포트폴리오 | 승급 심사와 연계된 포트폴리오 기준 명확화 |
