## 1. 운영 목적

이번 주 학습 목표는 공식 기준 업데이트를 읽는 데서 끝내지 않고, 각 역할이 자기 업무에서 근거 기반으로 판단하고 답변하는 산출물을 만드는 것이다.

핵심 원칙:
- 모든 학습 과제는 `KST` 상태 코드를 확인한다.
- 자동 수집 지식(`KST04`)은 확정 답변으로 쓰지 않는다.
- 산출물은 결론, 근거, 적용 범위, 주의, 다음 액션 구조를 따른다.
- 학습 완료는 읽기 완료가 아니라 실제 응답/검수/견적/교육/개발 문서에 반영된 증빙으로 판단한다.

## 3. 역할별 학습 과제

| 역할 | 이번 주 학습 과제 | 산출물 | 완료 기준 |
|---|---|---|---|
| CEO/CSO | 공식 기준 6건을 수익화/리스크 관점으로 읽기 | 전략 판단 메모 1개 | 어떤 기준이 교육/컨설팅/품질진단 상품으로 승격될지 3개 이하로 선정 |
| COO/PM | KDS/KCS, ISO/CDE, RFI/회의 5칸 구조 적용 | 회의 안건 또는 RFI 샘플 1개 | 결론, 근거, 적용 범위, 주의, 다음 액션이 모두 있음 |
| BIM 모델러 | 공종별 기준 게이트 중 자기 공종 1개 선택 | 모델 QA 체크리스트 1개 | `KST03` 적용주의 항목과 모델 확인 필드를 분리 |
| QA_테스터 | KST 상태 코드가 QA 룰에 미치는 영향 정리 | 검수 룰 후보 3개 | 자동 수집 수치를 확정 룰로 쓰지 않는 예외 문구 포함 |
| CS/기술지원 | 고객 기준 문의에 5칸 구조로 답변 연습 | CS 답변 초안 1개 | 공식 출처와 프로젝트 적용 주의 문구 포함 |
| 개발/R&D | 기준 게이트를 Add-in/자동화 룰로 바꾸는 조건 분석 | 개발 티켓 초안 1개 | 자동 검증 가능 항목과 수동 검토 항목 분리 |
| HR/경영지원 | KST 기반 역량 매트릭스 적용 | 교육 기록 샘플 1개 | 학습 지식, 상태 코드, 적용 산출물, 다음 검토일 기록 |
| 교육컨설팅 | 이번 주 자료를 15분 교육 카드로 변환 | 교육 카드 3개 이하 | 읽기, 실습, 퀴즈, 산출물 기준 포함 |
| 지식큐레이터 | 6개 기준 업데이트의 Obsidian 연결성 확인 | 큐레이션 메모 1개 | 역할, 공종, 검수/견적/교육 링크 2개 이상 확인 |
