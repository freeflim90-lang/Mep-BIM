## 1. 교육 목표

15분 안에 팀원이 다음을 수행할 수 있게 한다.

- 공식 기준과 프로젝트 적용을 분리한다.
- 자동 수집 지식을 확정 답변으로 쓰지 않는다.
- 보안 제한 데이터를 Obsidian, 교육자료, AI 프롬프트에 넣지 않는다.
- 고객/내부 답변을 결론, 근거, 적용 범위, 주의, 다음 액션 구조로 작성한다.

## 6. 응답 실습

상황:

> 고객이 “BF 기준상 이 모델은 바로 불합격인가요? 자동검수 결과에서 문폭과 화장실 회전 공간이 걸렸습니다.”라고 물었다.

작성 템플릿:

```text
결론:
[확정/미확정 범위를 먼저 말한다]

근거:
[공식 기준 / 내부 문서 / 모델 검토 결과]

적용 범위:
[공종 / 단계 / 대상시설 여부 / 인증 단계]

주의:
[KST 코드 / 자동수집 / 보안제한 / 확인 필요 사항]

다음 액션:
[담당자 / 확인 자료 / RFI 또는 BCF 등록 여부 / 다음 검토일]
```

통과 예시:

```text
결론:
현재 자동검수 결과만으로 BF 불합격을 확정할 수 없습니다. 다만 문폭과 화장실 회전 공간은 수동 검토가 필요한 위험 후보입니다.

근거:
BF/편의시설 공식 기준 업데이트와 BIM 모델러 BF QA 체크리스트에서 자동 검토 후보와 수동 검토 필요 항목을 분리하도록 정했습니다.

적용 범위:
건축 BIM 모델의 문, 화장실, 접근성 검토 후보에 한정합니다. 대상시설 여부와 예비인증/본인증 단계는 아직 확인이 필요합니다.

주의:
공식 기준은 KST01이지만 프로젝트 적용은 KST03입니다. 자동검수 수치는 KST04로 두며 확정 판정에 사용하지 않습니다. 고객 모델 원본이나 민감 로그가 있으면 KST06으로 분류합니다.

다음 액션:
PM은 대상시설, 특기시방서, 인증 단계, 지자체 조건을 확인합니다. BIM 모델러는 BCF 후보로 등록하고, 필요 시 RFI를 준비합니다. 다음 검토일은 2026-06-11입니다.
```

## 8. 수료 기준

Pass:
- KST 코드 4개를 올바르게 분류했다.
- 5칸 응답 초안에 결론, 근거, 적용 범위, 주의, 다음 액션이 모두 있다.
- KST04와 KST06을 확정 근거로 쓰지 않았다.
- 다음 검토일을 적었다.

Revise:
- 자동 수집 수치를 확정 판정으로 썼다.
- 보안 제한 정보를 교육자료/AI/Obsidian에 넣는다고 작성했다.
- 적용 범위나 다음 액션이 빠졌다.
