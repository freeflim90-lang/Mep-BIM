## 1. 샘플 목적

이 메모는 BF/편의시설 대응 범위가 확정되지 않은 프로젝트에서 견적심사원이 가격, 일정, 책임 범위를 어떻게 분리해 기록할지 훈련하기 위한 샘플이다.

핵심 원칙:
- 공식 법령과 인증제도 존재는 `KST01 공식확인`으로 인용할 수 있다.
- 프로젝트 대상성, 의무/권장, 예비인증/본인증, 지자체 사전검사 조건은 확인 전 `KST03 적용주의`로 둔다.
- 자동 수집된 치수와 항목명은 견적 참고 후보일 뿐 납품 합격, 인증 책임, 불합격 판정 근거로 쓰지 않는다.
- BF 대응은 일반 건축 모델링 공수에 묻지 않고 별도 검토/질의/재검수 공수로 분리한다.

## 2. 견적 판단 요약

| 항목 | 현재 판단 |
|---|---|
| 결론 | BF 적용 범위가 미확정이므로 기본 견적에는 `BF 기준 미확정 리스크`를 명시하고, 대상성 검토·체크리스트·인증 대응 회의·재검수 공수를 옵션 또는 별도 항목으로 산정한다. |
| 근거 | 장애인등편의법, 시행령 별표, 시행규칙 별표, BF 인증 규칙, 한국장애인개발원 BF 인증제도 페이지가 공식 출처다. |
| 적용 범위 | 건축 BIM, BF/편의시설 검토, 공공건축 또는 발주처가 BF 인증/편의시설 검토를 요구한 프로젝트. |
| 주의 | 대상시설, 시설 용도, 규모, 공공/민간 여부, 예비인증/본인증 단계, 지자체 조건 확인 전 확정 견적 금지. |
| 다음 액션 | PM이 발주처에 BF 대상성, 인증 단계, 특기시방서, 지자체 사전검사 여부를 RFI로 확인한다. |

## 4. 별도 공수 산정 항목

| 공수 항목 | 포함 업무 | 제외 또는 주의 |
|---|---|---|
| BF 대상성 검토 | 시설 용도, 규모, 공공/민간, 발주처 요구 확인 | 법률 자문 또는 인증기관 최종 판정은 제외 |
| 편의시설 체크리스트 작성 | 접근로, 주차, 출입구, 복도, 승강기, 화장실, 안내설비, 피난설비 항목화 | 자동 치수 검토만으로 적합 판정하지 않음 |
| 도면·모델 대조 | 도면 번호, 모델 뷰, 파라미터, 체크리스트 연결 | 원설계 변경 설계 책임은 별도 |
| RFI/회의 대응 | 기준 미확정 항목 질의, 회신 정리, 회의록 반영 | 회신 지연에 따른 일정 영향은 별도 리스크 |
| 예비/본인증 대응 | 심사 문서 대조, 보완 항목 추적, 재검수 | 인증 통과 보장은 제외 |
| 지자체 사전검사 대응 | 조례/사전검사 조건 확인, 보완 이슈 기록 | 관할 해석 차이는 일정 리스크로 표시 |

## 5. 견적서 삽입 문구 샘플

```text
BF/편의시설 기준 적용 범위는 현재 대상시설 여부, 의무/권장 구분, 인증 단계, 지자체 사전검사 조건이 확정되지 않은 상태입니다.

따라서 본 견적은 일반 건축 BIM 모델링 범위를 기준으로 하며, BF 대상성 검토, 편의시설 체크리스트 작성, 인증기관/설계자 질의응답, 예비인증/본인증 심사 대응, 지자체 사전검사 보완, 재검수는 별도 공수 또는 변경 견적 대상으로 산정합니다.

자동 수집된 편의시설 치수 또는 항목명은 검토 후보이며, 공식 전문과 프로젝트 조건 확인 전 납품 합격 또는 인증 통과 책임의 근거로 사용하지 않습니다.
```

| R4 차단 | 인증 통과 책임을 BIM 모델러 또는 자동검수에 전가하는 조건 | 견적 확정 보류, 계약 특수조건 검토 요청 |

## 7. 리뷰 기준

Pass:
- `BF 기준 미확정 리스크` 문구가 견적서 또는 검토 메모에 보인다.
- 대상성, 의무/권장, 인증 단계, 지자체 조건이 확인 항목으로 분리돼 있다.
- 자동검수 후보와 수동 확인 항목을 분리했다.
- 인증 통과 책임과 BIM 납품 책임을 혼동하지 않았다.

Revise:
- BF를 일반 건축 모델링 공수에 포함 처리했다.
- 자동 수집 치수를 확정 기준으로 사용했다.
- 예비인증/본인증, 지자체 사전검사, 특기시방서를 확인하지 않았다.
- 인증 통과를 견적서에서 보장하는 표현을 썼다.
