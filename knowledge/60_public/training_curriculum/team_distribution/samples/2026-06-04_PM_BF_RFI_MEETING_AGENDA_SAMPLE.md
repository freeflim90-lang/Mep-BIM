## 1. 샘플 상황

CS가 다음 고객 문의를 PM에게 에스컬레이션했다.

> 자동검수에서 장애인 화장실 문폭과 접근로 경사 후보가 표시되었습니다. 고객은 “BF 불합격인지, 바로 수정 지시를 내려도 되는지” 확인을 요청했습니다.

PM은 이를 고객 답변으로 닫지 않고, 발주처/설계자/인증기관 확인이 필요한 RFI 또는 회의 안건으로 전환한다.

## 2. RFI/회의 안건 요약

| 항목 | 내용 |
|---|---|
| 안건명 | BF/편의시설 자동검수 후보의 프로젝트 적용 기준 확인 |
| 결론 | 자동검수 후보만으로 BF 불합격 또는 설계 변경 지시를 확정하지 않는다. 대상시설, 의무/권장, 인증 단계, 지자체 조건, 특기시방서 확인 후 조치 범위를 결정한다. |
| 근거 | BF 공식 업데이트, CS BF 응답 샘플, QA BF 자동검수 룰 후보, BIM 모델러 BF QA 체크리스트 |
| 적용 범위 | 장애인 화장실, 접근로, 출입구, 복도, 승강기, 주차, 안내설비 등 BF/편의시설 BIM 검토 항목 |
| 주의 | 공식 기준 존재는 `KST01`이나 프로젝트 적용은 `KST03`이다. 자동검수 수치와 후보 이슈는 `KST04`로 두고 확정 판정에 사용하지 않는다. |
| 다음 액션 | RFI 발행, 도면/특기시방서/인증 단계 확인, 검수표 버전 갱신, 필요 시 변경 견적 검토 |

## 3. RFI 질문 초안

```text
제목: BF/편의시설 자동검수 후보 항목의 프로젝트 적용 기준 확인 요청

안녕하세요.

LUA BIM LABS BIM 검토 과정에서 장애인 화장실 문폭, 접근로 경사, 관련 증빙 링크 항목에 자동검수 후보 이슈가 표시되었습니다.
해당 결과는 모델 내 누락 또는 보완 후보를 표시한 것이며, BF 불합격 또는 인증 통과 여부를 확정한 결과는 아닙니다.

프로젝트 적용 기준 확정을 위해 아래 사항 확인을 요청드립니다.

1. 본 프로젝트가 BF 인증 또는 장애인등 편의시설 설치 대상시설에 해당하는지 확인 부탁드립니다.
2. 적용해야 할 의무/권장 편의시설 구분과 관련 별표 기준이 발주처 문서에 명시돼 있는지 확인 부탁드립니다.
3. 현재 단계가 예비인증, 본인증, 인증연장, 미대상 중 어디에 해당하는지 확인 부탁드립니다.
4. 지자체 사전검사 또는 별도 조례/협의 조건이 있는지 확인 부탁드립니다.
5. 특기시방서, EIR/BEP, 설계도서 중 BF 기준 적용 우선순위가 어떻게 정리돼 있는지 확인 부탁드립니다.
6. 자동검수 후보 항목을 설계 변경 대상으로 볼지, 추가 검토/RFI 대상으로 둘지 확인 부탁드립니다.

회신 전까지 LUA BIM LABS는 해당 항목을 `KST03 적용주의` 및 `자동검수 후보`로 관리하고, 최종 불합격 또는 인증 판정으로 표현하지 않겠습니다.
```

## 4. 회의 안건 체크리스트

| No. | 확인 항목 | 담당 | 상태 |
|---:|---|---|---|
| 1 | BF 대상시설 여부 | PM/발주처 | 미확정 |
| 2 | 의무/권장 편의시설 구분 | 설계자/PM | 미확정 |
| 3 | 예비인증/본인증/미대상 단계 | 발주처/인증 담당 | 미확정 |
| 4 | 지자체 사전검사 조건 | PM/발주처 | 미확정 |
| 5 | 특기시방서와 공식 기준 우선순위 | PM/설계자 | 미확정 |
| 6 | 자동검수 후보의 모델 수정 여부 | BIM 모델러/QA | 보류 |
| 7 | 재검수 또는 변경 견적 필요 여부 | 견적심사원/PM | 보류 |

## 5. 일정·범위 영향

| 조건 | 일정 영향 | 범위 영향 |
|---|---|---|
| 단순 증빙 링크 누락 | 낮음 | 체크리스트/도면 링크 보완 |
| 모델 요소 누락 또는 파라미터 누락 | 보통 | BIM 모델러 보완 + QA 재검수 |
| BF 대상성 또는 인증 단계 미확정 | 높음 | PM/RFI 회신 전 판정 보류 |
| 지자체 사전검사 조건 추가 | 높음 | 추가 회의, 재검수, 변경 견적 가능 |
| 인증 통과 책임 요구 | 차단 | 계약/SOW/견적 범위 재확인 필요 |

## 6. PM 운영 원칙

- CS의 고객 응답은 1차 안내이며, PM은 확정 판단이 필요한 항목을 RFI/회의 안건으로 전환한다.
- BF 자동검수 후보는 모델 수정 목록이 아니라 확인 요청 목록으로 먼저 관리한다.
- 회신 전까지 고객에게 불합격, 인증 통과, 법규 적합 확정이라는 표현을 쓰지 않는다.
- 회신 결과가 모델 수정, 재검수, 변경 견적으로 이어지면 검수표와 견적 메모를 동시에 갱신한다.
- 회의록에는 KST 상태와 책임 주체를 남긴다.

|---|---|---|
| BF 미대상 확인 | 일반 편의시설 검토로 축소 | 검수표 메모 |
| BF 대상이나 단계 미확정 | 추가 RFI 발행 | RFI 로그 |
| 본인증 대응 필요 | 일정·재검수·회의 공수 반영 | 견적 리스크 메모 |
| 지자체 조건 추가 | 관할 조건 확인 회의 등록 | 회의록/RFI |
| 모델 수정 필요 | BIM 모델러 작업지시 + QA 재검수 | BCF/QA 리포트 |
