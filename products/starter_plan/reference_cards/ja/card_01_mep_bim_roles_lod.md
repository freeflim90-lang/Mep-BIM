## BIMの役割リファレンス

| 役割 | 主な責任 |
|---|---|
| BIMマネージャー | BIM標準を策定し、CDE（共通データ環境）を管理し、プロジェクト全体のモデル品質をレビューします |
| BIMコーディネーター | 干渉チェックと調整を実施し、課題管理表を管理し、調整会議を主導します |
| MEP BIMモデラー | 専門分野別のBIMモデル（HVAC、配管、電気など）を作成・維持します |
| BIMリーダー | BEP（BIM実行計画書）を管理し、納品マイルストーンを監視し、モデリングチームを指導します |
| MEPエンジニア | 設計意図とエンジニアリング上の判断を提供し、調整変更を承認します |
| クライアント／開発者 | EIR（発注者情報要求事項）を通じてBIM要件を設定し、 deliverables（納品物）と受入基準を定義します |

## LOD定義

| LOD | 名称 | 説明 | MEPの例 |
|---|---|---|---|
| LOD 100 | 概念 | 全体のサイズと位置が示されますが、計測はできません | 平面図に記された概略のダクトゾーン |
| LOD 200 | 概略 | おおよその形状、寸法、位置が示されます | おおよそのサイズと高さでルーティングされたダクト |
| LOD 300 | 詳細 | 正確な形状、確定した寸法、接続部が示されます | 正しいサイズ、高さ、系統割り当てが設定されたダクト |
| LOD 350 | 調整 | 支持部材を含む、調整に必要なすべての要素が示されます | ハンガーが付き、全専門分野間で干渉チェック済みのダクト |
| LOD 400 | 製作 | 製造のための完全な施工図レベルの詳細が示されます | フランジ、支持部材、施工公差が示されたダクトセクション |
| LOD 500 | 現況 | 現場状況が確認済みの記録モデルです | 実際に設置されたダクト、設計からの差異が記録されたもの |

## 主要な略語

| 略語 | 正式名称 | 実務上の意味 |
|---|---|---|
| BEP | BIM実行計画書（BIM Execution Plan） | 誰が、いつ、どのようにモデルコンテンツを納品するかを定義する、チームが作成するプロジェクト文書 |
| EIR | 発注者情報要求事項（Employer's Information Requirements） | BIMの deliverables（納品物）とプロジェクトの標準を指定する、クライアントが定義した文書 |
| CDE | 共通データ環境（Common Data Environment） | すべてのプロジェクトモデルと文書が保存・管理される共有デジタルプラットフォーム |
| IFC | Industry Foundation Classes | 異なるソフトウェアプラットフォーム間でBIMデータを共有するための、オープンで非独占的なファイル形式 |
| NWC | Navisworks Cache | Navisworksでの干渉チェックとレビューに使用する、Revitから書き出された軽量モデル |
| NWD | Navisworks Document | 共有とアーカイブのために複数のNWCモデルをバンドルした、パッケージ化されたプロジェクトファイル |
| RFI | 情報依頼書（Request for Information） | 現場状況がモデルや図面と異なる場合に発行される正式な照会 |
