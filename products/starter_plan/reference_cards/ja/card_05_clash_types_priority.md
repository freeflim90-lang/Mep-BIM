## 干渉タイプの定義

| 干渉タイプ | 定義 | 例 |
|---|---|---|
| ハード干渉 | 2つのソリッド要素が物理的に交差する — 同一空間を占有する状態 | HVACダクトが構造梁のウェブを貫通するケース |
| ソフト干渉 / クリアランス干渉 | 2つの要素が規定の最小クリアランス距離内にある — 接触はしていないが、必要なアクセススペースやメンテナンススペースを侵害する状態 | ケーブルトレイがスプリンクラー配管から50mm以内にあり、最小離隔を違反しているケース |
| 重複 | 2つの同一要素がまったく同じ位置に重なっている — 通常はモデリングまたはインポート時のエラー | 異なるモデルインポートに起因する、同一経路・同一標高の2つの配管セグメント |
| ワークフロー干渉 (4D) | 2つの作業が同一場所・同一時刻にスケジュールされている — 幾何学的な干渉ではなく、時間と空間の競合 | HVAC設置工事と構造スラブ型枠工事の両方が、同一週にレベル5で計画されているケース |

## 優先度レベルマトリックス

| 優先度 | 基準 | 必要な対応 |
|---|---|---|
| **クリティカル** | 構造工事を妨げる、スラブ完了を阻害する、または安全上重要なシステムの設置を停止させる | 影響を受けるゾーンの作業開始前に必ず解決 — 直ちにエスカレーション |
| **高** | 主要システム（主ダクト幹線、主管ルート）の再ルーティングが必要 — 複数の接続要素に影響する | 現在のコーディネーションサイクル内で解決；責任業者に期限付きで割り当て |
| **中** | 軽微な調整（標高変更、小オフセット）が必要 — 1つまたは数個の要素に影響する | ゾーンのコーディネーション図面発行前に解決 |
| **低** | 軽微なクリアランス違反で、文書化による受入が可能；または非クリティカル要素のみに影響する | 文書化してレビュー・受入、または次回可能な機会に解決 |

## 干渉グループ化ルール

コーディネーションミーティングの前に、生の干渉結果をグループ化してノイズを低減し、作業に集中します：

- **レベル別にグループ化**：フロアごとに干渉を分離し、各業種が一度に1ゾーンに対応できるようにする
- **グリッドベイ別にグループ化**：同一構造ベイ内の干渉をクラスタリングし、効率的な空間レビューを実現する
- **専門分野ペア別にグループ化**：HVAC対構造、電気対配管を分離 — 異なる業種、異なるミーティング議題項目
- **ステータスでフィルタリング**：「新規」および「アクティブ」項目のみ表示 — 「レビュー済」「承認済」「解決済」はライブ議論リストから除外
- **重複を除去**：多くの生の干渉は、両方の要素方向から報告された同一の物理的競合 — 提示前にレビューして統合する

## コーディネーションミーティング議事次第テンプレート

```
MEP コーディネーションミーティング — [ゾーン / レベル] — [日付]

1. 出席確認（5分）
   - 全出席者と代表業種を記録

2. 前回アクションのレビュー（10分）
   - 前回会議の項目の解決を確認
   - トラッカー内の「解決済」項目のステータスを更新

3. 干渉レビュー — クリティカル項目（15分）
   - Navisworks / 干渉レポートを画面共有
   - 各項目の決定：修正 / 保留 / 受入
   - 責任者と期限を割り当て

4. 干渉レビュー — 高優先度項目（15分）
   - クリティカルと同じプロセス

5. 業種から報告された新規課題（10分）
   - トラッカーに未登録の現場またはモデル上の問題

6. アクションサマリー（5分）
   - 割り当てられた全アクション、責任者、期限を復唱

7. 次回ミーティング日程を確認
```
