## 4エリア品質フレームワーク

モデル品質は4つのエリアにわたって検証されます。MEPモデルを提出する前に、20項目すべてのチェックを完了してください。

---

### エリア1: 形状

- [ ] すべての要素は、目視による推定ではなく、数値オフセットを使用して正しい高さに配置されています
- [ ] 空中に浮いている要素、スラブに埋もれている要素、またはグリッドから外れて配置されている要素はありません
- [ ] すべてのダクトと配管は、有効で確認済みのサイズ（デフォルトや概算ではない）を有しています
- [ ] 機器は、構造グリッドに対して正しい位置に配置されています
- [ ] 同じ位置に重複した要素はありません（[管理] > [警告を確認]でチェック）

### エリア2: データ / パラメータ

- [ ] すべての要素に一意のマーク値があります — 空白や重複はありません
- [ ] すべての要素が名前付きシステムに割り当てられています（「デフォルト」や無名システムはありません）
- [ ] 必要なパラメータ（サイズ、レベル、システム名、材料/仕様）が入力されています
- [ ] 機器タグが設計図面または機器スケジュールと一致しています
- [ ] ファイル名がプロジェクトBEP命名規則に従っています

### エリア3: システム接続

- [ ] すべてのダクトと配管の端部が接続されています — 未接続のコネクタはありません
- [ ] システムブラウザに、未割り当てまたは「無名」のシステムは表示されません
- [ ] 機器（AHU、ポンプ、パネル）が、関連するダクト工事または配管工事に接続されています
- [ ] 重力排水は、必要な勾配（水平ではない）でモデル化されています — 排水は通常1:50
- [ ] システムインスペクタで、各アクティブシステムにエラーがゼロであることを確認します

### エリア4: コーディネーション準備

- [ ] すべてのリンクモデルが読み込まれ、最新の状態です（リンク切れはありません）
- [ ] 共有座標が確認されています — モデルが建築ベースと整合しています
- [ ] 未使用のファミリとタイプがパージされています（[ファイル] > [未使用を削除]）
- [ ] Revitの警告が確認されています — 重大な警告は解決または文書化されています
- [ ] 引き渡し前に、ファイルが中央モデルに同期されています

## LOD自己確認テーブル

| LOD | 形状確認 | パラメータ入力 | ハンガー/サポート含む | 干渉チェック済み |
|---|---|---|---|---|
| LOD 200 | 概略 | 最小限 | なし | なし |
| LOD 300 | 正確 | コアフィールド入力済み | なし | 任意 |

## クイックリファレンス: 確認すべき一般的なエラー

| エラー | 確認場所 |
|---|---|
| 未接続コネクタ | システムブラウザ — ホストされていない要素 |
| 誤ったシステムタイプ | プロパティパネル → システム分類 |
| 空白のマーク | スケジュールビュー、マークフィールドで並べ替え |
| 警告数 | [管理]タブ → [警告を確認] |
| ファイルサイズ（大） | [ファイル] > [未使用を削除]; RVTファイルサイズを確認 |
