## 충돌 유형 정의

| 충돌 유형 | 정의 | 예시 |
|---|---|---|
| 하드 충돌 | 두 개의 솔리드 요소가 물리적으로 교차하여 동일한 공간을 점유하는 경우 | HVAC 덕트가 구조용 보 웹을 통과하는 경우 |
| 소프트/간격 충돌 | 두 요소가 정의된 최소 이격 거리 이내에 위치하여 접촉하지는 않지만 요구되는 접근 또는 유지보수 공간을 위반하는 경우 | 케이블 트레이가 스프링클러 배관과 50mm 이내에 위치하여 최소 이격 거리를 위반하는 경우 |
| 중복 | 두 개의 동일한 요소가 정확히 동일한 위치에 중첩되어 있는 경우 — 일반적으로 모델링 또는 가져오기 오류 | 서로 다른 모델 가져오기에서 동일한 경로와 높이를 가진 두 개의 파이프 세그먼트 |
| 워크플로 충돌(4D) | 두 개의 작업이 동일한 위치에서 동일한 시간에 예약된 경우 — 기하학적 충돌이 아닌 시간-공간 충돌 | HVAC 설치와 구조 슬래브 거푸집 작업이 동일한 주차에 5층에서 동시에 계획된 경우 |

## 우선순위 레벨 매트릭스

| 우선순위 | 기준 | 필수 조치 |
|---|---|---|
| **Critical** | 구조 작업을 차단하거나, 슬래브 완료를 방해하거나, 안전 필수 시스템 설치를 중단시키는 경우 | 해당 구역의 모든 작업이 진행되기 전에 반드시 해결해야 함 — 즉시 에스컬레이션 |
| **High** | 주요 시스템(메인 덕트 트렁크, 주 배관 라인)의 경로 변경이 필요하며, 여러 연결 요소에 영향을 미치는 경우 | 현재 조정 주기 내에 해결하고, 책임 트레이드에 마감 기한을 지정하여 할당 |
| **Medium** | 사소한 조정(높이 변경, 작은 오프셋)이 필요하며, 하나 또는 소수의 요소에만 영향을 미치는 경우 | 해당 구역의 조정 도면이 발행되기 전에 해결 |
| **Low** | 문서화된 승인 옵션이 있는 사소한 간격 위반이거나, 중요하지 않은 요소에만 영향을 미치는 경우 | 문서화와 함께 검토 및 승인하거나, 다음 가능한 기회에 해결 |

## 충돌 그룹화 규칙

조정 회의 전에 원시 충돌 결과를 그룹화하여 노이즈를 줄이고 작업에 집중하십시오:

- **레벨별 그룹화**: 층별로 충돌을 분리하여 각 트레이드가 한 번에 한 구역을 처리하도록 함
- **그리드 베이별 그룹화**: 동일한 구조 베이 내의 충돌을 모아 효율적인 공간 검토 수행
- **분야별 그룹화**: HVAC-구조 충돌과 전기-배관 충돌을 분리 — 서로 다른 트레이드, 서로 다른 회의 안건
- **상태별 필터링**: 신규 및 활성 항목만 표시 — 검토 완료, 승인 및 해결 항목은 실시간 토론 목록에서 제외
- **중복 제거**: 많은 원시 충돌이 두 요소 방향에서 보고된 동일한 물리적 충돌인 경우가 있음 — 제시 전에 검토 및 통합

## 조정 회의 안건 템플릿

```
MEP 조정 회의 — [구역 / 레벨] — [날짜]

1. 참석 확인 (5분)
   - 모든 참석자 및 대표 트레이드 기록

2. 이전 조치 사항 검토 (10분)
   - 지난 회의 항목 해결 확인
   - 해결 항목의 상태를 트래커에서 업데이트

3. 충돌 검토 — Critical 항목 (15분)
   - Navisworks / 충돌 보고서 화면 공유
   - 각 항목에 대한 결정: 수정 / 보류 / 승인
   - 담당자 및 마감 기한 지정

4. 충돌 검토 — High 우선순위 항목 (15분)
   - Critical과 동일한 프로세스

5. 트레이드에서 보고된 신규 이슈 (10분)
   - 아직 트래커에 등록되지 않은 현장 또는 모델 이슈

6. 조치 사항 요약 (5분)
   - 모든 지정된 조치, 담당자 및 기한 재확인

7. 다음 회의 일정 확정
```
