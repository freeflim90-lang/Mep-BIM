## 4개 영역 품질 프레임워크

모델 품질은 네 가지 영역에 걸쳐 검증됩니다. MEP 모델을 제출하기 전에 20개 항목을 모두 완료하십시오.

---

### 영역 1: 형상(Geometry)

- [ ] 모든 요소는 시각적 추정이 아닌 수치 오프셋을 사용하여 올바른 레벨에 배치되어야 함
- [ ] 요소가 공중에 떠 있거나, 슬래브에 묻혀 있거나, 그리드에서 벗어나 배치되지 않아야 함
- [ ] 모든 덕트와 배관은 유효하고 확인된 사이즈를 가져야 함 (기본값 또는 근사치 불가)
- [ ] 장비는 구조 그리드를 기준으로 올바른 위치에 배치되어야 함
- [ ] 동일한 위치에 중복 요소가 없어야 함 (관리 > 경고 검토로 확인)

### 영역 2: 데이터 / 파라미터

- [ ] 모든 요소는 고유한 Mark 값을 가져야 함 — 빈 값 또는 중복 불가
- [ ] 모든 요소는 명명된 시스템에 할당되어야 함 ("기본값" 또는 이름 없는 시스템 불가)
- [ ] 필수 파라미터가 입력되어야 함: 크기, 레벨, 시스템 이름, 재료/사양
- [ ] 장비 태그는 설계 도면 또는 장비 스케줄과 일치해야 함
- [ ] 파일 이름은 프로젝트 BEP 명명 규칙을 따라야 함

### 영역 3: 시스템 연결

- [ ] 모든 덕트 및 배관 끝단이 연결되어야 함 — 열린 커넥터 불가
- [ ] 시스템 브라우저에 할당되지 않았거나 "이름 없는" 시스템이 없어야 함
- [ ] 장비(AHU, 펌프, 패널)는 관련 덕트 또는 배관에 연결되어야 함
- [ ] 중력 배수는 필요한 구배(평탄 불가)로 모델링되어야 함 — 일반적으로 배수용 1:50
- [ ] 시스템 인스펙터에서 각 활성 시스템에 대해 오류가 0건이어야 함

### 영역 4: 간섭 조정 준비(Coordination Readiness)

- [ ] 모든 링크된 모델이 로드되어 있고 최신 상태여야 함 (깨진 링크 불가)
- [ ] 공유 좌표가 확인됨 — 모델이 건축 베이스와 정렬되어야 함
- [ ] 사용하지 않는 패밀리 및 유형이 모델에서 제거됨 (파일 > 사용하지 않는 요소 제거)
- [ ] Revit 경고 검토 완료 — 중요 경고는 해결되거나 문서화되어야 함
- [ ] 인계 전에 파일이 중앙 모델과 동기화되어야 함

## LOD 자체 점검 테이블

| LOD | 형상 확인(Geometry Confirmed) | 파라미터 입력(Parameters Populated) | 행거/서포트 포함(Hangers / Supports Included) | 간섭 검토 완료(Clash-Checked) |
|---|---|---|---|---|
| LOD 200 | 근사치 | 최소 | 아니요 | 아니요 |
| LOD 300 | 정확 | 핵심 필드 입력 | 아니요 | 선택 사항 |
| LOD 350 | 정확 | 전체 세트 필수 | 예 | 예 — 필수 |
| LOD 400 | 제작 상세 | 제조사 포함 모든 필드 | 예 (상세) | 예 |
| LOD 500 | 시공 완료 상태 | 모든 필드 + 현장 검증 | 시공 완료 상태 | 해당 없음 — 기록용 |

## 퀵 레퍼런스: 확인해야 할 일반적인 오류

| 오류 | 확인 위치 |
|---|---|
| 열린 커넥터 | 시스템 브라우저 — 호스트되지 않은 요소 |
| 잘못된 시스템 유형 | 속성 패널 → 시스템 분류 |
| 빈 Mark | 스케줄 뷰, Mark 필드 기준 정렬 |
| 경고 개수 | 관리 탭 → 경고 검토 |
| 파일 크기 (큼) | 파일 > 사용하지 않는 요소 제거; RVT 파일 크기 확인 |
