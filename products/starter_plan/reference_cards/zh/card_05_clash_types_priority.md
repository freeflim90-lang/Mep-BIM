## 碰撞类型定义

| 碰撞类型 | 定义 | 示例 |
|---|---|---|
| 硬碰撞 | 两个实体元素在物理上相互交叉——它们占据同一空间 | 暖通风管穿过结构梁腹板 |
| 软碰撞/间距碰撞 | 两个元素处于规定的最小净距范围内——它们未接触但违反了所需的检修或维护空间 | 电缆桥架与喷淋管间距小于50mm，违反最小间距要求 |
| 重复碰撞 | 两个完全相同的元素堆叠在同一位置——通常为建模或导入错误 | 来自不同模型导入的两段管道具有相同路径和标高 |
| 工序碰撞（4D） | 两项施工作业安排在同一时间同一地点——属于时间-空间冲突，而非几何冲突 | 暖通安装与结构楼板支模均计划在同一周内于5层进行 |

## 优先级等级矩阵

| 优先级 | 判定标准 | 处理要求 |
|---|---|---|
| **关键** | 阻碍结构施工、阻止楼板完成，或导致安全关键系统无法安装 | 必须在受影响区域任何作业开始前解决——立即上报 |
| **高** | 需要重新规划主要系统（主风管干线、主管道线路）——影响多个关联元素 | 在当前协调周期内解决；指派给相关专业并设定截止日期 |
| **中** | 需进行小幅调整（标高偏移、小距离错位）——影响一个或少数元素 | 在区域协调图纸发布前解决 |
| **低** | 轻微间距违规且有书面接受方案；或仅影响非关键元素 | 审核并书面接受，或在下次机会时解决 |

## 碰撞分组规则

在任何协调会议之前，对原始碰撞结果进行分组以减少干扰并聚焦重点：

- **按楼层分组**：按楼层分隔碰撞，使各专业每次集中处理一个区域
- **按网格分区分组**：将同一结构分区内的碰撞聚类，便于高效的空间审查
- **按专业配对分组**：将暖通-结构碰撞与电气-给排水碰撞分开——不同专业对应不同的会议议程项
- **按状态筛选**：仅显示“新建”和“活跃”项——从实时讨论列表中排除“已审核”、“已批准”和“已解决”项
- **去除重复项**：许多原始碰撞是同一物理冲突从两个元素方向分别报告——在提交前进行审查和合并

## 协调会议议程模板

```
机电协调会议 — [区域/楼层] — [日期]

1. 签到（5分钟）
   - 记录所有参会人员及其代表专业

2. 上次行动项回顾（10分钟）
   - 确认上次会议项已解决
   - 在跟踪表中更新“已解决”项状态

3. 碰撞审查 — 关键项（15分钟）
   - 屏幕共享 Navisworks / 碰撞报告
   - 对每项做出决定：修复 / 搁置 / 接受
   - 指定负责人和截止日期

4. 碰撞审查 — 高优先级项（15分钟）
   - 流程与关键项相同

5. 各专业提出的新问题（10分钟）
   - 任何尚未录入跟踪表的现场或模型问题

6. 行动项总结（5分钟）
   - 复述所有已分配的行动项、负责人和截止日期

7. 确认下次会议日期
```
