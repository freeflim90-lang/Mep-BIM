Day 70 - 2주차 복습 — 분전반(DB) 모델링 요약

1. 중요성
분전반(DB) 모델링은 전기 분야에서 HVAC 장비 배치와 동일한 위상을 가집니다. 즉, 모델 내 전체 배전 시스템의 기준점 역할을 합니다. 2주차에서는 DB를 물리적 객체, 데이터 컨테이너, 그리고 시스템 내 모든 회로의 출발점으로 다루었습니다. 이번 복습에서는 케이블 트레이 시스템으로 넘어가기 전에 이러한 개념들을 통합 정리합니다.

2. 핵심 개념
2주차 분전반(DB) 모델링 요약

| 일차 | 주제 | 핵심 사항 |
|---|---|---|
| 63 | DB 모델링 | 정확한 치수의 패널 패밀리 사용; 전면 작업 공간 확보 (최소 600mm) |
| 68 | 부하 스케줄 | 엔지니어링 패널 스케줄의 모든 회로를 Revit 모델의 회로와 일치 |
| 69 | 회로 추적 | 모든 부하 요소를 회로에 할당; 시스템 브라우저에서 확인 — 미연결 요소(Orphan) 없음 |

분전반(DB) BIM — 필수 규칙
- 패널 이름은 모든 문서에서 일관되어야 함: Revit 패밀리 이름, 패널 스케줄 제목, 단선도(SLD) 패널 ID가 모두 정확히 일치해야 함
- DB 형상은 실제 공급업체 치수를 반영해야 함 — 일반적인 임시 자리표시자(Placeholder) 사용 금지
- 전면 작업 공간(전압 등급에 따라 600mm~1000mm)은 구역(Zone)으로 모델링하고 다른 모든 MEP 요소로부터 이격 유지
- 모든 회로 할당에는 다음이 필요함: 패널 이름, 회로 번호, 상(Phase), 부하 분류
- 고부하 장비(AHU, 펌프, UPS)는 전용 단일 부하 회로를 가져야 함
- 예비 차단기 슬롯은 빈 슬롯으로 표현되어야 함 — 모델에서 생략 금지
- 모델 내 회로 설명은 가능한 한 엔지니어링 패널 스케줄과 단어 단위로 일치해야 함
- 모든 조정 제출 전에 시스템 브라우저 점검 실행 — 미연결 요소(Orphan) 0개

분전반(DB) BIM 워크플로우 (요약):
1. 공급업체 데이터시트 치수를 사용하여 DB 패밀리 배치
2. 패널 이름 매개변수를 단선도(SLD) 및 패널 스케줄과 일치하도록 설정
3. 600mm 전면 작업 공간 구역 모델링
4. DB에서 해당 층 또는 구역의 모든 부하로 회로 할당
5. 패널 스케줄 뷰 열기 — 엔지니어링 패널 스케줄과 교차 검증
6. 시스템 브라우저에서 미연결 요소(Orphan)가 없는지 확인

3. 실제 프로젝트 관점
조정 책임자는 전기 모델을 첫 번째 간섭 검토(Clash Run)에 제출하기 전에 모든 층의 DB를 검토합니다. 그녀가 확인하는 사항: (1) 모든 DB 이름이 단선도(SLD)와 일치하는지, (2) 모든 DB 전면 작업 공간 구역에 장애물이 없는지, (3) 패널 스케줄 뷰가 엔지니어링 스케줄과 5% 이내로 일치하는지, (4) 시스템 브라우저에 미연결 부하가 0개인지. 이 20분의 사전 제출 점검은 과거 프로젝트에서 세 번의 대규모 재작업을 방지했습니다.

4. BIM 점검
- 점검: 모델 내 모든 DB의 이름이 엔지니어링 단선도(SLD) 및 패널 스케줄과 정확히 일치합니까?
- 점검: Revit의 모든 패널 스케줄 뷰가 데이터로 채워져 있습니까? — 배치된 DB 패밀리에 대해 빈 스케줄 뷰가 없습니까?
- 점검: 시스템 브라우저에서 모든 층의 미연결 전기 요소(Orphan)가 0개로 표시됩니까?

5. 흔한 실수
DB 형상을 완성하고 작업이 끝났다고 생각하는 것 — 회로 할당을 실행하거나 패널 스케줄 뷰를 검증하지 않는 경우입니다. 모델은 3D상으로 완료된 것처럼 보이지만, 전기 인텔리전스 레이어가 구축되지 않았기 때문에 모든 데이터 품질 점검에서 실패합니다.

6. 오늘의 실천 과제
현재 프로젝트 또는 교육용 모델에서 시스템 브라우저를 열고 미연결(Orphan) 전기 요소의 개수를 확인하십시오. 그 숫자를 기록하십시오. 이것이 2주차 기준선(Baseline)입니다. 3주차 목표는 최소 하나의 전체 층에서 미연결 요소(Orphan)를 0개로 만드는 것입니다.
