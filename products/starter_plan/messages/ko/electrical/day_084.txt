Day 84 - 부하 스케줄과 모델 간 검증 — 매개변수 값 교차 확인

1. 중요성
전기 엔지니어는 설계 문서의 일부로 부하 스케줄을 작성합니다. 여기에는 각 회로의 연결 부하와 각 분전반의 총 수용 부하가 명시됩니다. BIM 모델의 분전반 스케줄이 이 문서와 일치하지 않으면, 모델을 사용하여 설계 의도를 검증할 수 없으며, 모델에서 수행된 모든 부하 계산은 부정확해집니다. 매개변수 값을 교차 확인하는 것은 모델 제출 전 최종 품질 관문입니다.

2. 핵심 개념
부하 스케줄 검증 — 확인할 사항과 방법:

| 매개변수 | 엔지니어링 문서 출처 | Revit 위치 | 허용 오차 |
|---|---|---|---|
| 분전반 명칭 | 분전반 스케줄 헤더 | Family Name 매개변수 | 완전 일치 — 오차 불허 |
| 회로 개수 | 명시된 회로 수 | 분전반 스케줄 행 개수 | 완전 일치 |
| 회로 설명 | 회로 설명 열 | Circuit Description 매개변수 | 완전 또는 동등 일치 |
| 차단기 정격 (A) | 차단기 열 | Frame Size 또는 Rating 매개변수 | 완전 일치 |
| 연결 부하 (kW 또는 VA) | 연결 부하 열 | Apparent Load 또는 Load 매개변수 | 5% 이내 |
| 상 (Phase) | 상 열 (A/B/C 또는 1Ph/3Ph) | Number of Poles 매개변수 | 완전 일치 |
| 분전반 총 부하 | 분전반 스케줄 하단 합계 | Revit 분전반 스케줄 합계 | 5% 이내 |

교차 확인 수행 방법:
1. Revit 분전반 스케줄을 스프레드시트(CSV 또는 Excel)로 내보냅니다.
2. 엔지니어링 분전반 스케줄과 나란히 배치합니다.
3. 행별로 작업하며, 허용 오차 범위 내에서 일치하지 않는 값을 모두 플래그(Flag) 처리합니다.
4. 모든 플래그를 부하 스케줄 불일치 로그(Load Schedule Discrepancy Log)에 기록합니다: 분전반 ID, 회로 번호, 매개변수, 엔지니어링 값, 모델 값, 필요 조치.
5. 로그를 모델과 함께 엔지니어 검토용으로 제출합니다.

일반적인 불일치 원인:
- 엔지니어가 분전반 스케줄을 업데이트했으나 모델러에게 통보되지 않음
- 부하 값이 수동으로 입력되어 오타가 발생함
- 설계 후반에 회로가 추가되어 모델링되었으나 엔지니어링 스케줄에는 포함되지 않음
- 회로 설명에 서로 다른 명명 규칙 사용 (예: 엔지니어링 문서의 "AHU-4" vs. Revit의 "Air Handling Unit 4")

개정 관리: 검증에 사용된 엔지니어링 분전반 스케줄의 개정 번호를 항상 기록하십시오. 단선도(SLD)와 분전반 스케줄의 개정은 동기화되어야 합니다.

3. 실제 프로젝트 사례
한 BIM 코디네이터가 Revit에서 15개의 분전반 스케줄을 모두 내보내 전기 엔지니어의 부하 스케줄 제출본(Rev C)과 대조했습니다. 그 결과 DB-B1(지하층 분전반)의 총 연결 부하가 모델에서는 42kW인 반면 엔지니어링 문서에서는 38kW로, 10%의 차이(허용 오차 초과)를 발견했습니다. 조사 결과, 4kW 용량의 가정용 온수기가 모델에는 추가되었으나 엔지니어링 스케줄에는 아직 반영되지 않은 것으로 밝혀졌습니다. 코디네이터는 해당 불일치를 플래그 처리했고, 엔지니어는 스케줄을 Rev D로 업데이트했습니다.

4. BIM 체크리스트
- 체크: Revit 분전반 스케줄을 내보내 현재 엔지니어링 분전반 스케줄 개정본과 비교했으며, 모든 불일치가 기록되었습니까?
- 체크: Revit 모델의 각 분전반 총 부하가 엔지니어링 총 부하와 5% 이내로 일치합니까?
- 체크: 부하 스케줄 불일치 로그(Load Schedule Discrepancy Log)가 비공식 이메일 스레드로 남겨지지 않고 모델과 함께 제출되었습니까?

5. 흔한 실수
회로가 전기 엔지니어 팀에 의해 할당되었기 때문에 Revit 분전반 스케줄이 정확하다고 가정하는 것입니다. 엔지니어는 Revit 모델 매개변수를 업데이트하지 않고 스케줄 문서의 부하 값을 자주 업데이트합니다. 이로 인해 모델이 간섭 검토용으로 제출될 때쯤이면 두 값 사이에 상당한 차이가 발생할 수 있습니다.

6. 오늘의 실천 과제
다섯 개의 열(분전반 ID, 회로 번호, 매개변수, 엔지니어링 값, 모델 값, 필요 조치)로 구성된 빈 부하 스케줄 불일치 로그(Load Schedule Discrepancy Log) 템플릿을 작성하십시오. 이 템플릿은 모든 프로젝트에서 재사용 가능합니다. 현재 진행 중인 프로젝트가 없더라도 템플릿을 준비해두면 필요 시 즉시 검증 프로세스를 시작할 수 있습니다.
