Day 85 - Week 5 Review — 단선도(SLD) 추적 작업 요약

1. Why This Matters
SLD 추적 워크플로우 — 도면 해석, 모델 연결, 회로에서 부하까지 추적, 부하 데이터 검증에 이르는 과정 — 은 전기 BIM 모델을 단순한 3D 형상 파일에서 검증된 데이터 모델로 전환시키는 핵심 프로세스입니다. 5주차에서는 이 프로세스의 네 단계를 모두 다루었습니다. 이 워크플로우를 완전히 숙달하는 것이, 단순히 패밀리를 배치하는 수준을 넘어 유능한 전기 BIM 실무자로 성장하는 지름길입니다.

2. Core Concept
5주차 SLD 추적 요약

| Day | Topic | Key Point |
|---|---|---|
| 81 | SLD 읽기 | SLD에서 패널 ID, 모선 정격, 보호 장치, 공급 케이블 식별 |

| 84 | 부하 스케줄 검증 | Revit 패널 스케줄 내보내기; 엔지니어링 부하 스케줄과 교차 검증 — 모든 불일치 사항 기록 |

필수 규칙 — SLD 추적
- Revit 내 패널 이름은 SLD 패널 ID와 정확히 일치해야 함 — 약어 사용 금지, 형식 변경 금지
- SLD상의 모든 패널은 해당 Revit 패널 패밀리가 배치되어야 함 — 향후 설치 예정 패널도 포함
- 고아 회로(Orphan Circuit) 제로: 모든 차단기 슬롯은 부하가 연결되거나 예비(Spare)로 표시되어야 함
- 고아 부하(Orphan Load) 제로: 모델 내 모든 전기 요소에는 회로가 할당되어야 함
- 부하 검증 허용 오차: 개별 회로 ±5%, 패널 합계 ±5%
- 부하 스케줄 불일치 로그(Load Schedule Discrepancy Log)는 필수 인도물 — 선택 문서가 아님
- SLD 개정판은 검증에 사용된 패널 스케줄 개정판과 일치해야 함 — 두 개정판 모두 기록

SLD 추적 워크플로우 (요약):
1. 최신 SLD 개정판 수령 — 판독 후 패널 등록부(Panel Register) 작성
2. 모든 SLD 패널을 Revit 패밀리와 매칭 — 누락된 패널 플래그 표시
3. 모든 SLD 공급 케이블이 모델링되었는지 확인
4. 시스템 브라우저에서 모든 회로 추적 — 고아 회로 및 고아 부하 해결
5. Revit 패널 스케줄 내보내기 — 엔지니어링 부하 스케줄과 교차 검증
6. 각 조정 제출 시 패널 등록부 + 부하 스케줄 불일치 로그 제출

3. Real Project Lens
5주차 마일스톤 검토에서 BIM 리더는 전기 모델을 5주차 프레임워크 기준으로 점검합니다. 패널 등록부: SLD 패널 22개, 모두 Revit에 배치 완료 — 통과. 공급 케이블: 모두 모델링 완료 — 통과. 시스템 브라우저: 9층에 고아 부하 3개 발견 — 불합격, 모델러에게 반송. 부하 스케줄: 허용 오차 5% 초과 불일치 2건 — 불합격, 엔지니어 확인 요청 플래그 지정. 두 문제가 모두 해결된 후 모델이 재제출됩니다. 이처럼 체계적인 접근 방식은 조정 모델 동결 전에 실제 설계 조정 이슈 두 건을 발견해냅니다.

4. BIM Check
- Check: 패널 등록부가 완전하고 최신 상태인가 — 모든 SLD 패널, 해당 Revit 패밀리 배치 상태, 공급 케이블 모델링 상태가 모두 표시되어 있는가?
- Check: 시스템 브라우저에서 모델의 모든 층에 고아 회로와 고아 부하가 0건인가?
- Check: 부하 스케줄 불일치 로그가 완전한가 — 모든 패널을 포함하고, 각 플래그 항목에 대한 해결 상태가 기록되어 있는가?

5. Common Mistake
SLD 추적을 프로젝트 초기에 한 번만 수행하면 끝나는 일회성 작업으로 간주하는 것. SLD와 패널 스케줄은 설계 과정에서 여러 번 개정됩니다 — 개정이 있을 때마다 패널 이름, 회로 수, 부하 값을 모델과 대조하여 재확인해야 합니다. 개정 후 재확인을 하지 않으면 모델과 엔지니어링 문서 간에 불일치가 발생하여 동기화가 깨집니다.
