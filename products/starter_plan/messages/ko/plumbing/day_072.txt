LUA BIM LABS Starter
Day 72 - 슬리브 요청 프로세스

1. Why This Matters (왜 중요한가)
배관이 콘크리트 슬래브나 구조 벽체를 관통할 때마다, 슬리브를 사전에 매립해야 합니다. 슬리브가 제때 요청되지 않으면, 시공사는 타설 완료된 구조체에 코어 드릴링을 해야 하며, 이는 철근을 손상시키고 슬래브 강도를 약화시키며 부적합(Non-conformance)을 유발합니다. 슬리브 요청 프로세스는 배관 BIM 팀이 관통 위치를 구조 팀에 공식적으로 전달하는 방법입니다.

2. Core Concept (핵심 개념)
슬리브 요청은 MEP 팀이 구조 팀에 발행하는 공식 문서(일반적으로 도면 또는 데이터 파일)로, 필요한 모든 슬래브 또는 벽체 개구부를 식별합니다.

슬리브 요청에 반드시 포함되어야 할 사항:
- 두 개의 고정 그리드 라인에 대한 치수가 표시된 배관 위치 평면도
- 슬리브 크기: 배관 외경(OD)에 여유 공간을 더한 값 (일반적으로 배관 OD + 양측 각 50mm, 또는 프로젝트 기준에 따름)
- 슬리브 중심선 레벨 (벽체 슬리브의 경우)
- 바닥 또는 슬래브 레벨 참조
- 배관 시스템 및 용도 (예: 오배수 입상관, DN100)
- 방화 성능 요구 사항 (내화 등급 유/무)

Revit 모델에서 슬리브 요청을 준비하는 방법:
1. 모든 수직 입상관을 올바른 크기와 시스템으로 모델링합니다.
2. 각 입상관 위치에 구조 슬래브 관통 패밀리를 생성하거나 Revit 개구(Opening) 요소를 사용합니다.
3. 관통부에 필요한 매개변수가 입력된 슬리브 심볼을 배치합니다.
4. 모든 관통부의 좌표와 크기를 나열하는 슬리브 스케줄을 생성합니다.
5. 슬리브 도면을 PDF로, 스케줄을 Excel로 내보냅니다.
6. 공식 전달서(Transmittal)와 함께 구조 BIM 팀에 발행합니다.

타이밍:
- 슬리브 요청은 구조 슬래브 철근 배근 도면이 최종 승인되기 전에 발행되어야 합니다. 일반적으로 타설일 기준 2~4주 전입니다.
- 늦은 요청은 시간과 비용 손실로 이어집니다. BIM 조정 프로그램에 슬리브 요청 마일스톤을 반드시 포함시키십시오.

3. Real Project Lens (실제 프로젝트 사례)
한 병원 프로젝트에서 배관 BIM 코디네이터는 레벨 3 타설 6주 전에 47개의 슬래브 관통부에 대한 슬리브 요청을 발행했습니다. 구조 엔지니어가 목록을 검토한 결과, 주 보강 철근과 간섭되는 위치 3곳을 발견했습니다. 콘크리트가 타설되기 전에 BIM 모델에서 배관 경로가 변경되었습니다. 같은 프로젝트에서 다른 구역의 슬리브 요청 2건은 타설 이틀 전에 도착했으며, 두 곳 모두 추가 비용을 들여 긴급 코어 드릴링을 해야 했습니다.

4. BIM Check (BIM 체크리스트)
- Check: 슬래브를 관통하는 모든 수직 배관에 대해 Revit 모델에 크기와 시스템이 입력된 해당 슬리브 요소가 있는지 확인하십시오.
- Check: 조정 도면에서 그리드 라인까지의 슬리브 치수를 직접 측정할 수 있는지 확인하십시오.
- Check: 슬리브 스케줄을 구조 도면과 상호 검증하여 주 보 또는 기둥으로부터 300mm 이내에 관통부가 없는지 확인하십시오.

5. Common Mistake (흔한 실수)
흔한 실수는 BIM 모델 좌표 없이, 기억에 의존한 대략적인 치수로 2D AutoCAD 스케치 형태로 슬리브 요청을 발행하는 것입니다. 대략적인 치수는 슬리브 위치 이탈로 이어져 코어 드릴링이나 재배치가 필요하며, 이는 모두 비용이 많이 듭니다. 항상 기억이 아닌 모델에서 치수를 추출하십시오.

6. Today's Action (오늘의 실습)
Revit에서 입상관 중 하나의 위치에 슬래브 개구부(Opening) 패밀리를 배치하십시오. 슬리브 크기(Sleeve Size), 시스템(System), 방화 등급(Fire Stop Rating) 매개변수를 입력하십시오. 그런 다음 모든 개구부를 좌표와 함께 나열하는 간단한 스케줄을 작성하십시오. 이것이 실제 슬리브 요청 패키지의 시작점입니다.

Scope note:
본 내용은 일반적인 교육 콘텐츠이며, 프로젝트 검토, 엔지니어링 검증, 코드 준수 확인 또는 시공 승인을 대체하지 않습니다.
