Day 89 - 배관 인계 데이터(FM Handover Data)

1. 왜 중요한가
건물이 시설관리(FM)팀에 인계될 때, 배관 BIM 모델은 기계 시스템 운영 및 유지보수를 위한 주요 기술 참고자료가 됩니다. 모델의 형상(geometry)은 훌륭하지만 데이터가 부실하다면, FM팀은 운영상 가치가 없는 시각적 도구만 물려받게 됩니다. FM 인계에 적합한 배관 데이터를 준비한다는 것은 건물 소유주가 실제로 건물을 운영하는 데 사용할 수 있는 모델을 제공한다는 의미이며, 이는 점차 예우가 아닌 계약상 요구사항이 되고 있습니다.

2. 핵심 개념
FM팀은 배관 BIM 데이터를 세 가지 주요 활동, 즉 계획 예방 정비, 고장 조사, 그리고 향후 개조를 위해 필요로 합니다. 각 활동에는 특정 데이터 요구사항이 있습니다.

계획 예방 정비를 위해:
- 장비 자산 데이터: 제조사, 모델, 일련번호, 설치일, 유지보수 주기
- 밸브 데이터: 태그, 유형, 서비스(용도), 평시 개폐 상태, 차단 구역(해당 밸브 폐쇄 시 격리되는 회로)
- 스트레이너 위치: 태그, 서비스(용도), 청소 주기
- 테스트 포인트 위치: 압력계 탭, 온도 센서 위치, 샘플 밸브

고장 조사를 위해:
- 모델 요소와 연결된 시스템 흐름도(P&ID)
- 배관 재질, 호칭 지름, 압력 등급 — FM팀이 시스템을 열기 전에 무엇을 다루고 있는지 파악할 수 있도록 함
- 모든 서브 회로에 대한 격리 밸브 위치 — FM팀이 전체 시스템을 배수하지 않고도 특정 구역만 격리할 수 있어야 함

향후 개조를 위해:
- 시스템 설계 파라미터: 설계 유량, 설계 압력, 운전 온도 범위
- 여유 용량 데이터: 배관이 향후 확장을 위해 설계되었는지 여부
- 원본 설계 기준 참조: 시방서 및 설계 보고서와 연계

FM 인계를 위한 주요 공통 파라미터:

| 요소 | 필수 FM 파라미터 |
|---------|----------------------|
| 펌프 | 태그, 제조사, 모델, 정격 유량, 정격 양정, 모터 동력, 설치일 |
| 밸브 | 태그, 유형, 서비스(용도), 차단 구역, 접근 필요 여부, 평시 개폐 상태 |
| 팽창탱크 | 태그, 제조사, 예비 충전 압력, 총 용량, 설치일 |
| 배관 세그먼트 | 재질, 호칭 지름, 압력 등급, 단열 사양 |
| 스트레이너 | 태그, 메쉬 크기, 서비스(용도), 청소 주기 |
| 압력계 / 온도계 | 태그, 서비스(용도), 위치 설명 |

Revit에서 FM 데이터를 준비하는 방법:
1. 모든 FM 공통 파라미터가 Revit 프로젝트 템플릿에 로드되어 있는지 확인
2. 시공 진행에 따라 조달 과정에서 장비 데이터를 점진적으로 입력
3. 인계 전에 밸브 태그를 최신 P&ID 개정판과 상호 참조하여 확인
4. 자산 등록부를 클라이언트가 요구하는 형식(COBie Excel, 속성을 포함한 IFC, 또는 CAFM 전용 내보내기)으로 내보내기
5. 내보내기 파일을 FM 플랫폼에서 검증하여 데이터가 올바르게 가져와지는지 확인

3. 실제 프로젝트 사례
대규모 지역 냉방 프로젝트에서 FM 데이터 인계는 발주처 정보 요구사항(EIR)에 따라 배관 자산당 18개의 필수 데이터 필드를 포함한 COBie 스프레드시트로 지정되었습니다. 배관 BIM팀은 시공 단계 전반에 걸쳐 데이터를 점진적으로 입력해 왔습니다. 인계 시점에 95%의 필드가 완료되었습니다. 나머지 5%의 공백(대부분 현장 설치 품목의 설치일)은 시공 완료 기록을 통해 이틀 만에 채워졌습니다. FM팀은 건물 사용 첫날부터 데이터를 사용할 수 있음을 확인했습니다. 클라이언트는 배관 BIM 데이터 품질을 인계 패키지 중 가장 강력한 요소로 언급했습니다.

4. BIM 체크리스트
- 확인: 모든 기계 장비 패밀리에 제조사, 모델, 일련번호, 설치일 파라미터가 인계 모델에 입력되어 있는지 확인
- 확인: 모든 밸브에 밸브 태그, 차단 구역, 접근 필요 여부 파라미터가 입력되고 최신 P&ID와 상호 검증되었는지 확인
- 확인: COBie 또는 FM 데이터 내보내기가 클라이언트의 FM 플랫폼에서 테스트되었는지 확인 — 단순히 내보내기만 하는 것이 아니라, 가져오기 및 표시가 올바르게 되는지 확인

5. 흔한 실수
흔한 실수는 밸브 차단 구역을 FM팀이 인계 후 모델을 보고 알아낼 것이라고 간주하는 것입니다. 차단 구역은 명시적으로 문서화되어야 하며, 이상적으로는 각 밸브의 파라미터와 별도의 밸브 스케줄로 작성되어야 합니다. 건물 고장 상황에서 압박을 받는 FM 기술자가 3D 모델을 통해 배관 회로를 추적할 것이라고 기대할 수 없습니다.

6. 오늘의 실천 과제
자신의 배관 모델에서 펌프 하나를 선택하십시오. 인계에 필요한 FM 데이터 필드를 나열하십시오. 각 필드에 대해 해당 파라미터가 Revit 패밀리에 존재하는지, 그리고 값이 입력되어 있는지 확인하십시오. 파라미터가 누락된 경우, 어떤 공통 파라미터를 추가해야 하는지 기록하십시오. 이 연습을 통해 시공용 모델과 FM 인계용 모델 간의 데이터 격차를 확인할 수 있습니다.
