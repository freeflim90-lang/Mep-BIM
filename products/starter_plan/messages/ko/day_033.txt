Day 33 - 클래시 리포트 읽기

1. 왜 중요한가
클래시 리포트는 코디네이션의 공식 산출물입니다. 프로젝트 매니저부터 하도급사까지 모든 이해관계자가 무엇을 누가 고쳐야 하는지 이해하는 데 이를 사용합니다.

2. 핵심 개념
Navisworks는 클래시 리포트를 여러 형식으로 내보낼 수 있습니다. 각 필드의 의미를 이해하면 리포트를 단순 출력물이 아니라 작업 문서로 활용할 수 있습니다.

클래시 리포트의 핵심 필드:
- 클래시 이름/ID: 회의 간 항목을 추적하는 고유 식별자(예: MEP-STRUCT-0045)
- 상태(Status): 현재 상태 — New, Active, Reviewed, Resolved
- 거리(Distance): 하드 클래시의 관통 깊이(음수 = mm 단위 겹침)
- Element 1 / Element 2: 충돌하는 두 객체 — 요소 ID, 출처 파일, 카테고리 포함
- 담당자(Assigned To): 해결 책임자
- 이미지(Image): 클래시 위치를 보여 주는 뷰포인트 스크린샷
- 발견/승인 일자: 코디네이션 추적용 타임라인 표시

3. 실무 관점
신입 BIM 코디네이터가 회의 전에 클래시 리포트 PDF를 받습니다. "Element 1 / Element 2" 열을 훑어보면 대부분의 충돌이 HVAC 덕트(기계 모델)와 구조 철골(구조 모델) 사이임을 빠르게 알 수 있고, 그중 일곱 건은 덕트가 보의 웨브 관통부를 통과할 수 있어 구조 엔지니어가 이미 "Approved"로 표시했음을 알 수 있습니다.

4. BIM 점검
- 점검: 리포트에서 클래시의 두 요소에 대한 Element ID를 식별할 수 있는가?
- 점검: 충돌하는 각 요소가 어느 모델 파일(NWC)에서 왔는지 찾을 수 있는가?
- 점검: 거리 값을 읽고 그것이 간격인지 겹침인지 이해할 수 있는가?

5. 흔한 실수
초보자는 "Reviewed"와 "Resolved"를 혼동하곤 합니다. Reviewed는 누군가 클래시를 살펴봤다는 뜻이고, Resolved는 모델에서 물리적 충돌이 실제로 제거됐다는 뜻입니다. 둘을 같게 취급하면 잘못된 종결로 이어집니다.

6. 오늘의 실습
샘플 Navisworks 클래시 리포트(Autodesk가 온라인으로 예시 제공)를 찾아 클래시 다섯 건을 찾아보세요. 각각에 대해 관련된 두 요소와 현재 상태를 적어 보세요.

범위 안내:
본 콘텐츠는 일반 교육용이며, 프로젝트 검토·엔지니어링 검증·법규 적합성 확인·시공 승인이 아닙니다.
