LUA BIM LABS Starter
Day 52 - 현장 충돌 vs. 모델 충돌

**1. 왜 중요한가**
현장에서 발생하는 모든 문제가 모델 오류인 것은 아닙니다. 반대로 모든 모델 오류가 현장 문제를 유발하는 것도 아닙니다. 이 차이를 이해하면 불필요한 책임 전가를 방지하고 신속한 문제 해결이 가능해집니다.

**2. 핵심 개념**
모델 충돌(Model Conflict)은 디지털 모델 내에서 식별된 공간적 간섭을 의미합니다. 현장 충돌(Site Conflict)은 시공 중 발생하는 실제 설치 문제를 말합니다. 이 둘은 서로 관련되어 있지만 구별되는 개념입니다.

주요 차이점:

| | 모델 충돌 | 현장 충돌 |
|---|---|---|
| **발견 위치** | Navisworks / BIM 검토 도구 | 현장 설치 중 |
| **발견 시점** | 시공 전 조정 단계 | 시공 중 |
| **근본 원인** | 모델링 오류, 설계 간섭, 또는 조정 미비 | 모델과 현실 간 차이, 공종 순서 문제, 미반영 설계 변경 |
| **해결 절차** | 조정 회의 + 모델 업데이트 | RFI + 현장 지시 + As-built 반영 |
| **비용 영향** | 낮음 (시공 전 발견 시) | 높음 (재시공, 지연) |

모델에는 나타나지 않지만 현장 충돌을 유발하는 일반적인 원인:
- 슬래브 편차 (층 레벨이 모델과 정확히 일치하지 않음)
- 구조 부재의 위치 이탈 설치
- 현장에 발행되었으나 모델에 미반영된 설계 변경
- 구버전 도면을 기준으로 작업한 협력업체

**3. 실제 프로젝트 사례**
배관 협력업체가 7층에서 현장 충돌을 제기했습니다. 구조용 기둥이 모델상의 위치보다 50mm 남쪽으로 설치되어, 배관 라인이 기둥 플랜지와 간섭을 일으킨 것입니다. 모델에서는 충돌이 발견되지 않았습니다. 조사 결과, 해당 기둥은 구조 협력업체에 의해 위치를 이탈하여 설치되었습니다. 모델은 정확했지만 현실이 달랐던 것입니다. RFI를 통해 이 편차를 문서화하고, 현장에서 소폭 배관 우회가 승인되었습니다.

**4. BIM 체크**
- 체크: 간섭 검토(Clash Detection) 테스트에서는 발견되지 않는 현장 충돌의 원인 두 가지를 설명할 수 있습니까?
- 체크: RFI가 무엇이며, 현장 충돌을 공식적으로 해결하는 방식을 설명할 수 있습니까?
- 체크: 현장 충돌로 인한 편차를 BIM 모델에 어떻게 다시 반영해야 하는지 설명할 수 있습니까?

**5. 흔한 실수**
간섭이 없는(Zero-Clash) 모델이 현장에서도 문제가 없을 것이라고 가정하는 것입니다. 간섭 검토는 모델 충돌만 제거할 뿐, 시공 공차, 설치 편차, 또는 미문서화된 설계 변경까지 고려할 수 없습니다.

**6. 오늘의 실천 과제**
두 가지 짧은 시나리오를 작성하십시오. 하나는 모델 오류로 인해 발생한 현장 충돌이고, 다른 하나는 모델이 예측할 수 없었던 현장 조건으로 인해 발생한 현장 충돌입니다. 각각의 해결 방법을 설명하십시오.
