LUA BIM LABS Starter
Day 87 - 소방 설비 파라미터 체크리스트

1. 왜 중요한가
기하학적으로 완벽한 소방 설비 모델이라도 파라미터가 비어 있거나 잘못 입력되어 있다면 완전한 BIM 인도물이라고 할 수 없습니다. 파라미터는 모델을 기계가 읽을 수 있도록 만드는 핵심 요소로, 일정표(Schedule)를 구동하고, 수계 설계 엔지니어에게 정보를 제공하며, FM(Facility Management) 인계를 지원하고, 도면을 열지 않고도 모델을 조회할 수 있도록 해줍니다. 파라미터를 올바르게 입력하는 것은 형상을 정확히 모델링하는 것만큼이나 중요합니다.

2. 핵심 개념
LOD 350 납품을 위한 소방 설비 요소의 필수 파라미터:

스프링클러 헤드:
| 파라미터 | 예시 값 | 목적 |
|---|---|---|
| Mark(마크) | FP-H-03-0047 | 해당 헤드의 고유 식별자 |
| System Name(시스템명) | FP-WET-ZONE-03 | 헤드를 해당 수계 구역에 연결 |
| Head Type(헤드 유형) | Pendant Concealed(매입형) | 일정표 및 자재 조달용 |
| Coverage Area(커버리지 면적) | 12.1 m² | 커버리지 계산 검증 |
| Operating Temperature(작동 온도) | 68°C | 일반 사무실/상업용 기준 |
| Manufacturer / Model(제조사/모델) | [설계 사양에 따름] | 자재 조달 및 승인용 |

배관:
| 파라미터 | 예시 값 | 목적 |
|---|---|---|
| System Name(시스템명) | FP-WET-ZONE-03 | 연결된 헤드와 동일 |
| Nominal Diameter(호칭경) | DN50 | 사이징 일정표와 일치해야 함 |
| Pipe Level(배관 레벨) | Branch Line(가지배관) | 계층 구분 식별 |
| Material / Specification(재질/사양) | Black Steel, SCH 10 | 자재 조달용 |
| RFI Reference(RFI 참조) | RFI-FP-047 | 설계 변경 시 |

밸브 및 특수 장비:
| 파라미터 | 예시 값 | 목적 |
|---|---|---|
| Mark(마크) | FP-V-03-001 | 고유 식별자 |
| Valve Type(밸브 유형) | Zone Control Valve(구역제어밸브) | 일정표용 |
| System Name(시스템명) | FP-WET-ZONE-03 | 구역 연결 |
| Zone(구역) | Zone 03 | FM 사용 및 시운전용 |

일반적인 파라미터 오류:
- System Name(시스템명) 미입력 → 요소가 미할당되어 구역 일정표에서 보이지 않음
- Coverage Area(커버리지 면적) 미입력 → 커버리지 계산 일정표 실행 불가
- Mark(마크) 미할당 → RFI 또는 현장 조정 시 해당 요소 참조 불가
- Nominal Diameter(호칭경) 불일치 → 일정표에는 DN40으로 표시되나 모델은 DN50인 경우

3. 실제 프로젝트 사례
레벨 6 소방 설비 모델을 발행하기 전, BIM 모델러는 System Name(시스템명) = "(none)"으로 필터링된 Revit 일정표를 실행했습니다. 그 결과, 12개의 헤드와 4개의 배관 세그먼트가 시스템에 할당되지 않은 상태로 발견되었습니다. 이 요소들은 조정 경로 변경 중에 배치되었으나 시스템에 제대로 연결되지 않았던 것입니다. 각 요소는 수정되었습니다: 헤드는 시스템에 재연결되고, 배관은 올바른 시스템명으로 할당되었습니다. 이후 일정표에는 미할당 요소가 0개로 표시되었습니다.

4. BIM 체크리스트
- 체크: 모든 스프링클러 헤드에 고유한 Mark(마크), 올바른 System Name(시스템명), 그리고 Coverage Area(커버리지 면적) 파라미터가 입력되어 있는가
- 체크: 모든 배관에 System Name(시스템명)과 Nominal Diameter(호칭경)가 소방 엔지니어의 사이징 일정표와 일치하는가
- 체크: System Name(시스템명) = "(none)"으로 필터링된 Revit 일정표를 실행했을 때 소방 설비 요소가 0개로 반환되는가 — 미할당 요소가 없어야 함

5. 흔한 실수
파라미터 입력을 모델링 마지막 단계에서 처리하는 행정 업무로 간주하는 것입니다. 요소를 배치할 때 점진적으로 파라미터를 입력하는 것이, 사후에 추가하는 것보다 항상 더 정확합니다. 두 달 전에 배치된 헤드를 추적하는 것은 어렵지만, 배치 시점에 Mark(마크)와 System Name(시스템명)이 태깅된 헤드는 즉시 식별할 수 있습니다.

6. 오늘의 실천 과제
현재 작업 중인 모델이나 샘플 모델에서 모든 스프링클러 헤드의 Revit 일정표를 실행하세요. System Name(시스템명) 또는 Coverage Area(커버리지 면적)이 비어 있는 헤드를 필터링하세요. 미완료 레코드의 개수를 세고, 이를 모두 완료하는 데 필요한 시간을 추정해 보세요. 다음 모델 발행 전까지 모든 빈 항목을 채울 기한을 설정하세요.
