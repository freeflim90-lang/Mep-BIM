Day 84 - 시공상태(As-Installed)와 설계상태(As-Designed) 검증

1. 왜 중요한가
소방 BIM 모델은 처음에 설계 상태(As-Designed)를 반영하여 작성됩니다. 그러나 시공 중에는 배관 경로가 변경되고, 헤드 위치가 옮겨지며, 현장 문제 해결을 위해 추가 부속이 설치됩니다. 이러한 변경 사항이 모델에 반영되지 않으면, 준공 모델(As-Built)은 실제와 다르며, 건축주, 소방 당국, 시설 관리자에게 실제 시공 상태와 일치하지 않는 문서가 전달됩니다.

2. 핵심 개념
설계 상태와 시공 상태 간의 차이:
- 설계 상태(As-Designed): 시공 도서 발행 시점의 모델 — 엔지니어의 설계 의도를 나타냄
- 시공 상태(As-Installed / As-Built): 현장에 실제 설치된 내용을 반영하여 업데이트된 모델
- 두 상태 간의 차이는 RFI, 현장 지시, 간섭 조정 변경을 통해 누적됨

소방 분야의 RFI 프로세스:
- RFI(Request for Information, 정보 요청): 설치자가 현장 여건이 모델과 다를 때 제기
- 일반적인 소방 RFI 사례: 현장에서 발견된 장애물로 인한 헤드 위치 변경, 분기 배관 경로 변경, 자재 수급 문제로 인한 관경 변경, 추가 부속 설치
- BIM 모델러의 역할: RFI 접수 → 승인된 변경 사항을 모델에 반영 → 모델 내 변경 내역 문서화 (개정 태그 또는 파라미터 활용)

RFI별 모델 업데이트 요구 사항:
1. 영향받은 요소(헤드, 배관, 부속, 슬리브) 이동 또는 추가
2. 변경된 배관 길이 및 직경 업데이트
3. 변경된 모든 관련 파라미터 업데이트 (Mark, System Name, 위치 변경 헤드의 Coverage Area)
4. 변경이 커버리지 또는 수리계산에 영향을 미칠 경우 — 모델에서 RFI를 종결하기 전에 소방 엔지니어의 승인 요청

준공 모델 인도 요구 사항:
- 최종 모델은 인도 전에 모든 승인된 RFI 변경 사항을 반영해야 함
- 변경된 각 요소는 해당 변경을 승인한 RFI 번호를 참조해야 함 (Comments 파라미터 활용)
- 중요 경로 변경이 발생한 경우, 소방 엔지니어는 준공 수리계산서에 서명 승인해야 함
- 소방 당국 제출용으로 (모델 기반) 준공 스프링클러 배치도가 일반적으로 요구됨

3. 실제 프로젝트 사례
레벨 5 설치 단계에서 스프링클러 시공사가 8건의 RFI를 제기했습니다: 헤드 위치 변경 5건 (현장에서 덕트 장애물 발견), 분기 배관 경로 변경 2건 (보 위치가 구조 도면과 상이), 관경 변경 1건 (DN65 미확보, DN80 대체). BIM 모델러는 각 RFI를 48시간 이내에 처리하여 모델을 업데이트하고, 변경된 각 요소에 RFI 번호를 태깅했습니다. 인도 전에 업데이트된 모델이 준공 기록 세트로 발행되었습니다.

4. BIM 체크리스트
- 확인: 승인된 모든 소방 RFI가 준공 모델 발행 전에 모델에 반영되었는가
- 확인: 위치가 변경된 헤드의 커버리지 영역이 재검토되었는가 — 500mm 이동된 헤드는 간격 위반 또는 사각지대를 유발할 수 있음
- 확인: 수정된 각 요소의 Comments 또는 Description 파라미터에 RFI 번호가 기록되었는가

5. 흔한 실수
프로젝트 관리 플랫폼에서 RFI를 종결했지만 BIM 모델은 업데이트하지 않는 경우. 문서상으로는 RFI가 해결된 것으로 표시되지만, 모델에는 여전히 이전 위치가 남아 있습니다. 준공 모델이 틀려지고 소방 당국 기록이 부정확해지며, 이는 향후 검사 시 드러날 수 있는 컴플라이언스 위험입니다.

6. 오늘의 실천 과제
현재 진행 중이거나 샘플 소방 프로젝트에서 RFI 한 건을 선정하세요. Revit 모델을 열고 해당 RFI 변경 사항이 모델에 반영되었는지 확인하십시오. 반영되었다면, RFI 번호가 요소의 파라미터에 기록되어 있는지 확인하십시오. 반영되지 않았다면, 무엇을 업데이트해야 하는지 기록하십시오.
