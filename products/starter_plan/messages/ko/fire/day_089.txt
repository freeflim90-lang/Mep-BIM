Day 89 - 시운전 및 인계 데이터 — FM이 소방 설비 BIM 모델에서 필요로 하는 정보

1. 왜 중요한가
소방 설비 모델은 시공 인계로 끝나지 않습니다. 이 모델은 건물의 생명 안전 시스템에 대한 살아있는 기록이 됩니다. 시설 관리자, 건물 유지보수 팀, 그리고 미래의 소방 엔지니어들은 이 모델을 기반으로 시스템을 유지·점검·수정하게 됩니다. 형상만 완벽하고 데이터가 비어 있는 모델을 인계하는 것은 놓친 기회이며, 건물 소유자는 지속적인 운영 비용으로 그 대가를 치르게 됩니다.

2. 핵심 개념
인계 시 FM(시설 관리)이 소방 설비 BIM에서 필요로 하는 정보:

1. 시스템 구역 맵
- 각 구역이 어느 영역을 담당하는지 — 구역 밸브 위치 및 차단 절차 포함
- 중요성: 비상 대응, 계획된 유지보수 차단, 향후 피트아웃(fit-out) 조정
- 모델 내: System Name 파라미터가 구역 맵과 일치해야 함. 모든 요소에 구역이 지정되어 있어야 함.

2. 헤드 유형 및 작동 온도 기록
- FM은 각 구역에 설치된 헤드 유형(펜던트, 업라이트, 콘실드, 사이드월) 및 온도 등급을 알아야 함
- 중요성: 교체 자재 조달 — 잘못된 온도 등급을 설치하면 규정 위반
- 모델 내: Head Type 및 Operating Temperature 파라미터 (Day 87 참조)

3. 밸브 등록부
- 모든 구역 밸브, 알람 체크 밸브, 테스트 및 드레인 어셈블리의 위치, 담당 구역, 서비스 지침 목록
- 중요성: 연간 점검, 수압 시험, 유지보수 작업을 위한 차단
- 모델 내: 각 밸브에 Mark, Valve Type, Zone, Location 파라미터가 있어야 함

4. 유지보수 접근 참고 사항
- 콘실드 헤드(점검 시 커버 플레이트 제거 필요)와 표준 펜던트 헤드 구분
- 동절기 셧다운을 위해 드레인이 필요한 브랜치 라인(건식 구역) 식별
- 모델 내: 비표준 접근이 필요한 곳에 Maintenance Note 파라미터 추가

5. 준공 수리 참조
- 최종 승인된 수리 계산서(소방 엔지니어 작성)를 준공 모델과 연계
- FM이 향후 시스템을 추가 또는 수정할 때 필요 — 처음부터 다시 시작할 수 없음
- 모델 내 연결 문서 참조 또는 인계 데이터 팩으로 제공

인계 시 LOD(일반적으로 LOD 400~500):
- 모든 RFI(현장 정보 요청) 반영 (LOD 500 요소 = 설치 상태 그대로)
- Day 87 체크리스트에 따라 모든 파라미터 입력 완료
- 밸브 등록부를 스프레드시트로 내보내거나 FM 플랫폼(예: Archibus, Maximo)에 연동

3. 실제 프로젝트 사례
한 병원 시설 관리자가 구역 맵, 밸브 등록부, 콘실드 헤드 식별 정보 없이 소방 설비 모델을 인계받았습니다. 3개월 후, 유지보수 팀이 배관 수리 중 잘못된 구역을 차단하여 알람이 발생했습니다. 구역 맵과 밸브 등록부가 포함된 완전한 FM 인계 모델이 있었다면, 모델 조회 시간 5분 이내에 이를 방지할 수 있었을 것입니다.

4. BIM 체크리스트
- 확인: 모든 소방 설비 요소에 물리적 구역 밸브와 매핑되는 System Name / Zone이 있는가 — FM이 모든 헤드를 해당 차단 밸브까지 추적 가능
- 확인: 밸브 등록부 스케줄이 모델에서 내보내져 인계 데이터 패키지에 포함되었는가
- 확인: 준공 수리 계산서가 인계 패키지에 링크 또는 참조로 포함되었는가 — 설계 계산서만 있는 것은 아님

5. 흔한 실수
밸브 등록부 없이 소방 설비 모델을 인계하고, FM이 도면에서 구역을 파악할 것이라고 가정하는 것. 도면에는 차단 절차가 표시되지 않습니다. FM 팀은 다음을 알아야 합니다: 구역 3을 유지보수 차단하려면 어떤 밸브를 잠가야 하며, 그로 인해 어떤 영역이 영향을 받는가? 이 정보는 모델 데이터에 명시적으로 포함되어야 합니다.

6. 오늘의 실천 과제
현재 작업 중이거나 샘플 소방 설비 모델을 열고, 모든 구역 밸브에 대한 Revit 스케줄을 실행하십시오. 각 밸브에 Mark, Zone, Valve Type, Location이 입력되어 있는지 확인하십시오. 비어 있는 필드가 있다면 인계 데이터 누락으로 기록하십시오.
