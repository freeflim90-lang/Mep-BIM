第87天 - 火灾保护参数检查清单

1. 为何重要
几何正确但参数为空或错误的火灾保护模型不是一个完整的BIM交付成果。参数是使模型可机器读取的关键——它们驱动报表、告知水力工程师、支持设施管理移交，并允许下游用户查询模型而无需打开图纸。正确填充参数与正确建模几何同样重要。

2. 核心概念
在LOD 350交付中火灾保护元素所需参数：

喷淋头：
| 参数 | 示例值 | 目的 |
|---|---|---|
| Mark | FP-H-03-0047 | 此喷淋头的唯一标识符 |
| System Name | FP-WET-ZONE-03 | 将喷淋头与其水力分区链接 |
| Head Type | Pendant Concealed | 用于报表和采购 |
| Coverage Area | 12.1 m² | 验证覆盖面积计算 |
| Operating Temperature | 68°C | 标准办公/商业评级 |
| Manufacturer / Model | [如指定] | 用于采购和审批 |

管道：
| 参数 | 示例值 | 目的 |
|---|---|---|
| System Name | FP-WET-ZONE-03 | 同它服务的喷淋头相同 |
| Nominal Diameter | DN50 | 必须与尺寸报表匹配 |
| Pipe Level | 支管线 | 识别层级 |
| Material / Specification | 黑色钢，SCH 10 | 用于采购 |
| RFI Reference | RFI-FP-047 | 如设计变更 |

阀件和特殊设备：
| 参数 | 示例值 | 目的 |
|---|---|---|
| Mark | FP-V-03-001 | 唯一标识符 |
| Valve Type | 区域控制阀 | 用于报表 |
| System Name | FP-WET-ZONE-03 | 分区链接 |
| Zone | 区域03 | 用于设施管理及调试 |

常见参数错误：
- 系统名称留空——元素在分区报表中未分配且不可见
- 覆盖面积未填充——覆盖面积计算报表无法运行
- 标记未指派——元素不能被引用在RFA或现场协调中
- 名义直径不匹配——报表显示DN40但模型为DN50

3. 实际项目视角
在发布第6级火灾保护模型前，BIM建模者通过Revit报表过滤系统名称="(none)"。十二个喷淋头和四个管道段落出现在未分配系统的列表中——它们是在协调重路由期间放置的，并且没有正确连接到系统。每个元素都进行了修正：将喷淋头重新连接到系统，为管道指派正确的系统名称。现在报表显示零个未分配的元素。

4. BIM检查
- 检查：每个喷淋头都有一个唯一的Mark、正确的System Name和填充的Coverage Area参数
- 检查：每根管道有与消防工程师尺寸报表匹配的System Name和Nominal Diameter
- 检查：通过过滤系统名称="(none)"运行Revit报表返回零个火灾保护元素——没有任何未分配

5. 常见错误
将参数视为建模结束后的行政任务。随着元素放置而逐步填充的参数总是比事后添加的更准确。两个月前放置的一个喷淋头难以追踪；一个在放置时标记有Mark和System Name的喷淋头则立即可识别。

6. 今天行动
运行当前或样本模型中所有喷淋头的Revit报表。过滤任何系统名称或覆盖面积为空的喷淋头。计算未完成记录的数量并估计完成它们所需的时间。设定一个期限，在下次模型发布前关闭所有空白项。
