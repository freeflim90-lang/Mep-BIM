# Work Boundary

## Owner Decision

Revit API and Navisworks API dependent build, runtime test and final validation will be done by the owner on a machine with Autodesk Revit/Navisworks installed.

## Codex Scope

Codex can continue working on:

- Autodesk Store submission materials
- Commercial release workspace management
- Pricing and product strategy
- Privacy/EULA/support documents
- QA checklists and evidence templates
- Upload package preparation scripts
- Non-Revit/Navisworks API dependent scripts
- Static source review
- Packaging manifests and release process documentation
- Customer support and sales operations templates

## Excluded From Codex Scope

Codex should not attempt:

- Revit API compile validation
- Navisworks API compile validation
- Running Revit or Navisworks
- Final installer execution validation
- Final add-in behavior validation inside Autodesk products
- Claiming version compatibility without owner QA evidence

## Handoff Pattern

1. Codex prepares documents, scripts and checklists.
2. Owner builds and tests on Autodesk-installed machine.
3. Owner places installer, screenshots and QA evidence in commercial workspace.
4. Codex prepares final Store upload package and submission checklist.
5. Owner manually uploads, previews and submits in Autodesk Publisher Center.
