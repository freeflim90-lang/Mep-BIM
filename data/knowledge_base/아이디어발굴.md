## BIM Add-in 아이디어 발굴 프레임워크 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: ideation,bim,pain-point

Pain Point 수집 채널:
1. Autodesk Community 포럼 — 반복 질문 패턴 분석
2. Revit API 개발자 그룹 (LinkedIn/Facebook) — 미해결 기능 요청
3. BIM 프로젝트 현장 인터뷰 — 수작업 반복 업무 목록화
4. 경쟁 제품 리뷰 — 낮은 평점 리뷰의 불만 사항
MVP 선정 기준: 반복 빈도(주 3회↑) × 절감 시간(30분↑/회) × Revit API 구현 가능성(L1~L2)
우선 제외: API 불가 기능, 기존 Revit 기본 기능과 95% 이상 중복.


## 최고지배자 일일 3대 아이디어 보고 프로세스 (2026-05-20)
- Source: LUA BIM LABS 운영 지침
- Tags: ideation,daily-report,monetization,local-only

최고지배자의 별도 요청이 없다면 아이디어발굴은 매일 내부 후보를 검토하고 3개만 선별 보고한다. 보고 후보는 LUA BIM LABS의 목적성인 BIM Add-in 상용화, Autodesk Store 판매 가능성, 유지보수 가능성, 고객 반복 고통 해결에 맞아야 한다.

보고서에는 다음 항목을 반드시 포함한다.
1. 해결하려는 고객 문제와 대상 사용자
2. MVP 기능 범위와 제외 범위
3. 예상 구현 토큰수, 개발 시간, 투입 비용
4. 예상 월 순매출, 회수 기간, 수익화 가능성
5. 협업 참여자와 다음 handoff 대상
