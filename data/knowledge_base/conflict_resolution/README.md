# AI 간 판단 충돌 해소 체계

> LUA BIM LABS 운영 기준 | 2026-06-05 수립
> 관련 프로토콜: [[조율차장]] "AI 간 판단 충돌 해소 프로토콜" 섹션

## 이 폴더의 역할

각 AI 담당자들이 같은 문제에 대해 서로 다른 판단을 내렸을 때,
그 충돌 과정·합의 결과·선례를 누적 기록한다.

기록된 선례는 동일 유형 충돌 재발 시 즉시 참조해 불필요한 중재를 줄인다.

AI 간 일반 협업, 추론 검증, 반론, 합의, 실행 인계는 `docs/internal_organization_documents/30_AI_TO_AI_COLLABORATION_SOP.md`를 기본 기준으로 적용한다.
월간 리허설과 상황별 테스트는 `COLLABORATION_TEST_PLAN.md`에 누적한다.

## KST 신뢰도 우선순위 (충돌 1단계 판단 기준)

| 태그 | 의미 | 충돌 시 우선순위 |
|------|------|----------------|
| KST01 공식확인 | 법령·공식 기준 출처 확정 | **최우선** |
| KST03 적용주의 | 프로젝트별 조건 확인 필요 | 중간 |
| KST04 자동수집 | AI 초안·자동수집 (미검증) | 최하위 |
| KST06 보안제한 | 외부 공개 금지 | 충돌 대상 제외 |

KST01 vs KST04 충돌 → KST01 즉시 채택, 로그 기록 후 종료.
같은 KST 등급 충돌 → 2단계 역할 경계 확인으로 진행.

## 에스컬레이션 경로

```
[충돌 발생]
    │
    ▼ 1단계: KST 신뢰도 비교 (즉시)
    │  → KST01 우선이면 즉시 채택
    │
    ▼ 2단계: 역할 경계 확인 (1시간)
    │  → 조율차장.md 8개 규칙 적용
    │
    ▼ 3단계: 당사자 간 직접 조율 (24시간)
    │  → 각자 근거 명시, CONFLICT_LOG에 기재
    │
    ▼ 4단계: 조율차장 중재 (48시간)
    │  → 중재안 작성 후 양측 통보
    │
    ▼ 5단계: 상위 결정권자 에스컬레이션 (72시간)
       기술·품질·납품 → COO
       계약·예산·법무 → CEO + 법무조항검토
       전략·제품 방향 → CSO
```

## 케이스 상태 코드

| 코드 | 의미 |
|------|------|
| OPEN | 충돌 발생, 해소 진행 중 |
| PENDING | 당사자 간 조율 대기 중 |
| MEDIATED | 조율차장 중재안 제시됨 |
| ESCALATED | 상위 결정권자로 에스컬레이션 |
| SETTLED | 최종 합의 완료 |
| PRECEDENT | 선례로 승격됨 |

---

## 선례 활용 규칙

1. 동일 유형(T1~T4) + 동일 당사자 조합의 충돌이 재발하면 PRECEDENT 케이스를 먼저 확인한다.
2. 선례와 현재 상황이 90% 이상 유사하면 선례 결과를 즉시 적용한다 (1단계 처리).
3. 선례와 다른 점이 있으면 CONFLICT_LOG에 "선례 참조 + 차이점"을 명시하고 3단계부터 진행한다.
4. KST04 지식이 충돌 원인이면 지식큐레이터에게 KST01 보강 요청 티켓을 생성한다.
