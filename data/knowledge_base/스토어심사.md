1. 제품 불안정 — 크래시, 심각한 지연, Revit 성능 저하

5. 개인정보 미고지 — 외부 통신·데이터 수집 미기재

7. 유사 기능 중복 — 기존 Revit 기본 기능과 사실상 동일

## in-app 리뷰 요청 UX 설계 가이드라인 (2026-05-24 19:21:00)
- Source: LUA BIM LABS internal product UX knowledge baseline
- Tags: store,qa,review-request,ux,in-app

in-app 리뷰 요청 UX 설계 원칙:

트리거 시점:
- 사용자가 성공적으로 작업을 완료한 직후 (첫 간섭 검토 완료, 첫 물량 산출 완료 등 긍정적 성과 경험 후)
- 최소 3회 이상 Add-in 실행 후 (신규 설치 직후 요청 금지)
- 구독 갱신 완료 직후
- NOT: 오류 발생 직후, 기능 실패 직후, 첫 실행 시

요청 방식 (Revit TaskDialog 기반):
- Title: BIM Command Center를 유용하게 사용하고 계신가요?
- Message: Autodesk App Store에 리뷰를 남겨주시면 제품 개선에 큰 도움이 됩니다. (30초면 충분합니다)
- Buttons: [지금 리뷰 작성] App Store 페이지 열기 / [나중에] 7일 후 재등장 / [더 이상 보지 않기] 영구 비표시

표시 빈도 제한:
- 최초 표시: 3회 이상 사용 후 처음 한 번
- 재표시: 나중에 선택 시 7일 후 1회만 재표시
- 더 이상 보지 않기 또는 리뷰 클릭 후: 완전 비활성화
- 총 최대 2회 (초회 + 재표시 1회)

로컬 상태 저장:
- 위치: %AppData%\LUA BIM LABS\BIMCommandCenter\user_prefs.json
- 필드: launch_count(int), review_requested(bool), review_snoozed_until(ISO datetime), review_dismissed(bool)

A/B 테스트 포인트:
- 버튼 텍스트: 리뷰 작성 vs 피드백 남기기 vs 별점 주기
- 트리거 타이밍: 3회 실행 vs 5회 실행
- 메시지 톤: 질문형 vs 감사형

목표 지표:
- 리뷰 전환율: 표시 횟수 대비 클릭율 >= 15%
- 평균 별점: >= 4.0
- 리뷰 수: 분기 누적 >= 10건

Autodesk App Store 심사 거절 사유 Top 5 실사례와 재제출 전략:

1. 설명과 기능 불일치: 제품 설명에 "자동 BIM 간섭 검토"를 명시했지만 실제 제품은 수동으로 결과를 확인해야 하는 구조였던 케이스. 심사팀이 설명대로 기능을 직접 테스트하며 불일치를 적발한다. 재제출 전략: 기능 설명을 실제 동작에 맞게 수정하고, 기능별 짧은 GIF 이미지나 비디오 링크를 추가해 투명성을 높인다.

2. 스크린샷 품질 미달: 1280×800px 미만 해상도 또는 개발자 환경의 디버그 UI가 포함된 스크린샷을 제출한 경우. Autodesk는 스크린샷이 실제 고객 사용 환경처럼 보일 것을 요구한다. 재제출 전략: 1920×1080px 이상의 실제 Revit 화면으로 교체하고, 제품 기능이 명확히 드러나는 컨텍스트(실 모델에서 동작)로 촬영한다.

3. 코드 서명 미적용: `.dll` 또는 MSI 설치 파일에 EV Code Signing이 없거나 인증서가 만료된 경우. 재제출 전략: DigiCert/Sectigo EV Certificate로 재서명 후 `signtool.exe /v`로 서명 유효성을 확인한 뒤 제출.

4. 미지원 Revit 버전 명시 누락: 제품 호환 버전 항목에 Revit 2024/2025를 추가하지 않아 자동 반려된 사례. 재제출 전략: 실제 빌드 매트릭스와 일치하는 지원 버전 목록을 호환성 항목에 모두 기재하고, 테스트 완료 체크리스트를 내부적으로 유지한다.

5. 무한 루프 버그: 심사 중 대형 모델에서 `FilteredElementCollector` 루프가 특정 조건에서 종료되지 않아 Revit이 응답 없음 상태가 된 경우. 재제출 전략: 루프 내 진행도 체크포인트와 `CancellationToken` 또는 최대 반복 횟수 상한을 추가하고, 대형 샘플 모델(요소 10만 개 이상)에서 사전 스트레스 테스트를 수행한다.
