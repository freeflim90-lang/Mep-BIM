**각 파이프라인 단계 기술 스펙:**
| 단계 | 스크립트 | 의존성 | 실패 시 처리 |

| RSS 수집 | `collect_signals.py` | aiohttp, feedparser | 재시도 3회, 슬랙 알림 |
| 신호 분류 | `classify_signals.py` | claude-haiku-4-5, SQLite | 분류 실패 시 raw 저장 |
| KB 업데이트 | `auto_enrich_knowledge_base.py` | claude-sonnet-4-6 | 파일 단위 롤백 |
| 품질 검증 | `mqa_obsidian_tools.py` | 자체 파서 | 경고 로그 저장 |
| 그래프 재빌드 | `build_global_obsidian_map.py` | d3.js (HTML 내장) | 이전 HTML 유지 |
| Git 동기화 | `git_sync.sh` | git | 충돌 시 stash 후 알림 |

**APScheduler 5.x 설정 예시:**
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger

scheduler = AsyncIOScheduler()
# 매일 자정 KB 업데이트
scheduler.add_job(run_kb_update, CronTrigger(hour=0, minute=0))
# 매시 신호 수집
scheduler.add_job(collect_signals, CronTrigger(minute=0))
scheduler.start()
```
