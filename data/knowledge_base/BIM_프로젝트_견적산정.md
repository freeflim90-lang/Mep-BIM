## 1. 견적 산정 흐름

```
① 프로젝트 규모 파악       → 연면적, 층수, 용도
② 수행 범위 확정           → 공종 수, BIM 단계, LOD
③ 기준 공수 산출           → 연면적 × 공종 × 단계별 계수
④ 가중치 반영             → 복잡도, 발주처 요구, 납기
⑤ AI/인력 비율 적용        → AI 처리 가능 범위 차감
⑥ 인력 단가 적용           → 연차별 단가 × 투입 공수
⑦ 직접 비용 합산           → 소프트웨어, 라이선스, 출장
⑧ 간접비·마진 반영         → 간접비율 + 목표 마진
⑨ 최종 제안가 결정         → 시장 경쟁가 검토 후 확정
```

## 2. 기준 공수 산출 (MM 기반)

> MM(Man-Month) = 1인이 1개월(22일, 176시간) 투입하는 공수

### 2.1 단계별 MM 계수 (연면적 1,000㎡ · 공종 1개 기준)

| BIM 단계 | LOD | MM 계수 | 비고 |
|---|---|---|---|
| 기획 BIM | 100~200 | 0.05~0.10 | 공간 구성·볼륨 위주 |
| 기본설계 BIM | 200~300 | 0.10~0.20 | 공종별 기본 모델 |
| 실시설계 BIM | 300~350 | 0.25~0.45 | 상세 모델 + 파라미터 |
| 시공 BIM | 350 | 0.15~0.30 | 시공 모델 갱신 |
| 준공 BIM | 400 | 0.08~0.15 | 준공 반영 + FM 속성 |

**계산식:**
```
기준 공수(MM) = 연면적(㎡) ÷ 1,000 × 공종 수 × 단계별 계수
```

**예시 — 연면적 20,000㎡, 건축+구조+MEP(3공종), 실시설계:**
```
20,000 ÷ 1,000 × 3공종 × 0.35(중간값) = 21 MM
```

## 4. AI vs 인력 처리 비율 차감

> AI(LUA BIM LABS 에이전트 + Add-in)로 처리 가능한 업무는 공수에서 차감한다.

| 업무 항목 | AI 처리율 | 비고 |
|---|---|---|
| 클래시 검출 및 보고서 생성 | 80% | Navisworks + LUA Add-in 자동화 |
| 파라미터 입력 검증 | 70% | Add-in 자동 체크 |
| 명명 규칙 검토 | 85% | Add-in 자동화 |
| 물량 산출 (표준 패밀리) | 60% | Revit 일람표 + 스크립트 |
| BEP 초안 작성 | 50% | AI 에이전트 초안 → 사람 검토 |
| 간섭 협의 및 조율 (대면) | 0% | 반드시 사람 |
| 발주처 보고 및 프레젠테이션 | 0% | 반드시 사람 |
| 복잡 구간 모델링 수정 | 10~20% | 대부분 사람 |
| EIR 충족 여부 최종 판단 | 40% | AI 검토 후 사람 확인 |

**AI 차감 적용 방식:**
```
실 투입 공수 = 기준 공수 × (1 - AI 처리율 가중 평균)
※ AI 처리율 가중 평균은 업무 구성에 따라 프로젝트별로 산출 (통상 20~35% 절감)
```

```
□ 발주처 EIR 엄격도 판단 → β 계수 결정
□ 복잡도 판단 → α 계수 결정
□ 납기 여유 판단 → γ 계수 결정
□ AI 처리 가능 업무 비율 산출
□ 간섭 검토 횟수 협의
□ 투입 인력 등급·기간 확정
□ 직접 비용(라이선스, 출장) 확인
□ 경쟁사 시장가 조사 후 최종가 조정
□ 견적 최종 CFO·CEO 검토
```
