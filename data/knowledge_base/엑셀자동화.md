## Qwen Coder 기반 엑셀 자동화 주 업무 기준 (2026-05-20)
- Source: LUA BIM LABS local development policy
- Tags: excel,qwen,automation,validation,plan

엑셀 자동화 요청은 Qwen_Coder_8B를 주 업무 구현 담당으로 배정한다. 단 모든 개발은 구현 전에 plan을 먼저 확정해야 한다. plan에는 목적, 입력 데이터, 출력 사용자, 시트/열 구조, 필수 필터, 대용량 행 수, 검증 샘플, 실패 시 복구 기준이 포함되어야 한다.

Revit/Navisworks에서 직접 데이터를 읽어야 하는 경우에는 Qwen이 원본 API 접근을 확정하지 않는다. 먼저 샘플 JSON/CSV 중간 포맷으로 엑셀 자동화가 가능한지 검증하고, 실제 Revit/Navisworks API 접근이 꼭 필요하다고 판단될 때만 최고지배자 실기 테스트 검증 게이트로 넘긴다.

## 엑셀 자동화 라이브러리 선택 기준 (2026-05-21)
- Source: LUA BIM LABS internal development standard
- Tags: excel,library,openpyxl,pandas,python

LUA BIM LABS의 엑셀 자동화 라이브러리 선택 우선순위는 다음과 같다.

1순위: Python 표준 csv + openpyxl
- 의존성이 가장 작고, 파일 손상 위험이 낮다.
- BIM 리포트(간섭, 물량, MEP 스케줄), 관리팀 반복 업무(근태, 정산, 비용)에 사용한다.
- .xlsx 생성, 셀 서식(색상·굵기·폰트), 필터 설정, 열 너비 조정 모두 지원한다.
- 대용량(5만 행 이상)은 write_only=True 모드로 메모리 절약한다.

2순위: pandas + openpyxl (엔진 지정)
- 데이터 집계(pivot_table, groupby)가 필요한 경우에 한해 허용한다.
- pandas 단독으로 .xlsx를 생성할 때는 반드시 engine='openpyxl'을 명시한다.
- pandas.ExcelWriter를 사용하면 여러 시트에 DataFrame을 쉽게 분배할 수 있다.

3순위: C# OpenXML SDK
- Revit Add-in 내부에서 직접 .xlsx를 생성해야 할 때만 사용한다.
- Python 환경이 없는 빌드 파이프라인 전용이다.

금지: VBA/매크로 방식
- 보안 리스크(악성코드 배포 경로), 크로스 플랫폼 비호환, 유지보수 불가로 전면 금지한다.

### 패턴 1 — 표준 필터 테이블 생성
용도: 간섭 리포트, 물량 산출, 근태/비용 집계
```python
from openpyxl import Workbook
from openpyxl.utils import get_column_letter
from openpyxl.worksheet.table import Table, TableStyleInfo

wb = Workbook()
ws = wb.active
ws.title = "리포트"
headers = ["공종", "레벨", "간섭수", "조치상태", "담당자", "우선순위"]
ws.append(headers)
for row in data_rows:
    ws.append(row)
# 테이블 범위 자동 계산
last_col = get_column_letter(len(headers))
last_row = ws.max_row
table = Table(displayName="리포트테이블", ref=f"A1:{last_col}{last_row}")
table.tableStyleInfo = TableStyleInfo(name="TableStyleMedium9", showRowStripes=True)
ws.add_table(table)
wb.save("output.xlsx")
```

### 패턴 2 — 조건부 서식 (우선순위 색상)
용도: 간섭 심각도, 보고서 상태 표시
```python
from openpyxl.styles import PatternFill
RED  = PatternFill(fill_type="solid", fgColor="FF4444")
YELLOW = PatternFill(fill_type="solid", fgColor="FFDD44")
GREEN  = PatternFill(fill_type="solid", fgColor="44CC66")
for row in ws.iter_rows(min_row=2):
    priority_cell = row[5]  # 우선순위 열
    if priority_cell.value == "긴급":
        for cell in row:
            cell.fill = RED
    elif priority_cell.value == "보통":
        for cell in row:
            cell.fill = YELLOW
```

### 패턴 3 — 다중 시트 분배 (공종별)
용도: BIM 공종별 간섭 리포트, 관리팀 월별 시트
```python
disciplines = {}
for row in data_rows:
    disc = row[0]  # 공종 필드
    disciplines.setdefault(disc, []).append(row)
for disc, rows in disciplines.items():
    ws = wb.create_sheet(title=disc[:31])  # 시트명 31자 제한
    ws.append(headers)
    for row in rows:
        ws.append(row)
```

### 패턴 4 — 인코딩 안전 CSV 입출력
용도: Revit/Navisworks 중간 포맷, 텔레그램 수신 파일 처리
```python
import csv, io
# 읽기: BOM 있는 CP949/UTF-8 모두 처리
def read_csv_safe(path):
    for enc in ("utf-8-sig", "cp949", "utf-8"):
        try:
            with open(path, newline="", encoding=enc) as f:
                return list(csv.DictReader(f))
        except (UnicodeDecodeError, LookupError):
            continue
    raise ValueError("인코딩 감지 실패")
```

### 패턴 5 — 대용량 write_only 모드
용도: 5만 행 이상 물량 산출, 전체 프로젝트 간섭 목록
```python
from openpyxl import Workbook
wb = Workbook(write_only=True)
ws = wb.create_sheet()
ws.append(headers)
for chunk in data_generator():   # 청크 단위 처리
    ws.append(chunk)
wb.save("large_output.xlsx")
# 주의: write_only 모드는 셀 서식 적용 불가 → 서식이 필요하면 일반 모드 사용
```

## openpyxl 심화 패턴 카탈로그 (2026-05-22)
- Source: LUA BIM LABS internal development standard
- Tags: excel,openpyxl,chart,conditional-formatting,validation,protection

### 차트 생성
- BarChart: 월별 간섭 건수, 항목별 비용 비교
- LineChart: KPI 추이 (조치율 누적)
- PieChart: 공종별 간섭 비율

### 데이터 유효성 검사
- type="list": 상태(신규/검토중/조치완료/보류/제외), 우선순위(긴급/보통/낮음) 드롭다운
- type="whole": 숫자 범위 제한 (0~9999 정수)

### 고급 조건부 서식
- ColorScaleRule: 3색 열지도 (간섭거리 분포)
- DataBarRule: 진행률 데이터 막대
- FormulaRule: 행 전체 강조 ($H2="긴급"), D-30 만료일 강조

### 셀 보호
- ws.protection.sheet = True / ws.protection.enable()
- 특정 열만 Protection(locked=False)로 편집 허용

### 명명된 범위 (DefinedName)
- 여러 시트에서 합계 공식 공유 시 활용
- wb.defined_names["간섭합계"] = DefinedName(...)

- Tags: excel,bim,navisworks,clash,xml,pipeline

Navisworks Clash Detective XML 내보내기 → ClashItem 파싱 → 간섭 리포트 Excel 생성 파이프라인.

핵심 처리 단계:
1. xml.etree.ElementTree로 clashresult 태그 반복 파싱
2. 거리 m → mm 변환, 우선순위 자동 분류 (>50mm=긴급, >10mm=보통, 나머지=낮음)
3. 상태값 영문 → 한국어 변환 (new→신규, approved→조치완료 등)
4. 공종별 요약 시트 + 전체 리포트 시트 다중 시트 구성
5. 긴급 행 조건부 서식 강조 (FormulaRule, fgColor="FF9999")
6. 상태/우선순위 드롭다운 유효성 검사 적용
7. 인쇄 설정: 가로 A4, fitToWidth=1, 머리글/바닥글

## 에러처리 핵심 패턴 (2026-05-22)
- Source: LUA BIM LABS internal development standard
- Tags: excel,error-handling,validation,encoding,memory

### 원자적 저장
tempfile.NamedTemporaryFile → wb.save(tmp) → shutil.move(tmp, target)
실패 시 원본 보존, 임시 파일 자동 삭제

### 백업 정책
save_with_backup(): 덮어쓰기 전 타임스탬프 백업, keep_backups=3 유지

### 인코딩 우선순위
utf-8-sig → cp949 → utf-8 → euc-kr → chardet 자동감지

### 메모리 안전
- 5만 행 이상: Workbook(write_only=True)
- 5MB 초과 우려: chunk_excel_by_rows() 파일 분할

### 입력 검증
ValidationError 데이터클래스로 (행, 필드, 값, 사유) 수집
validate_clash_data() → write_validation_report() 오류 리포트 생성

### 셀 값 타입 변환
safe_cell_value(value, expected_type): None 안전, 숫자/날짜 파싱 실패 시 기본값 반환


## pandas 집계 사용 기준 (2026-05-22)
- Source: LUA BIM LABS internal development standard
- Tags: excel,pandas,pivot,groupby,aggregation

pandas는 pivot_table/groupby 집계가 필요할 때만 사용한다. 단순 표 생성은 openpyxl 직접 사용.

### 간섭 데이터 집계
- pivot_table(index="공종조합", columns="상태", aggfunc="count", margins=True)
- groupby("공종조합", "우선순위").agg(건수, 평균거리, 최대거리)

### 월별 추이
- pd.to_datetime() → .dt.to_period("M") → groupby("월")

### ExcelWriter 멀티시트 + 서식 후처리
with pd.ExcelWriter(path, engine="openpyxl") as writer: ... 후 load_workbook()으로 서식 적용

### 주의사항
- engine='openpyxl' 반드시 명시
- NaN → df.fillna("") 후 저장
- 숫자형 문자: dtype=str 읽기 후 후처리 변환
- 10만 행 이상: chunksize 또는 write_only 전환
