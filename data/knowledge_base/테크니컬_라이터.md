- Docs-as-Code 파이프라인 구축: Markdown 문서를 Git으로 관리하고, Docusaurus v3(React 기반)를 정적 사이트 빌드 도구로 사용한다. GitHub Actions에서 `main` 브랜치 머지 시 자동 빌드 및 GitHub Pages 배포가 실행된다. 버전 관리(`versioned_docs/`)로 Revit 버전별 문서를 병렬 유지한다.
- 사용자 가이드 구조 표준(2025): ① 빠른 시작(5분 내 첫 실행) → ② 핵심 기능 레퍼런스 → ③ 실전 워크플로우(Use Case별) → ④ FAQ → ⑤ 문제 해결 가이드의 5섹션 구조를 표준으로 정의한다. 각 섹션에 스크린샷(Snagit 또는 CleanShot X로 제작) 최소 1장을 포함한다.
- AI 보조 문서 작성: Claude API를 활용하여 개발자 코드 주석과 PR 설명에서 사용자 문서 초안을 자동 생성한다. 생성된 초안은 테크니컬 라이터가 30분 내 검토·수정 후 배포한다. 한국어/영어 이중 언어 문서는 DeepL API로 초벌 번역 후 전문 검수를 거친다.

## 테크니컬라이터 마스터급 경험 지식 (2026-05-29)

### 비정형 BIM 기능 기술 문서 작성 특이사항

비정형 BIM 기능은 일반 사용자가 개념을 이해하기 어려움 → 문서 전략 차별화:

**비정형 기능 문서 구조 (권장):**
```
1. 개요 (2문장)
   "이 기능은 [곡면 지붕/비정형 입면/파라메트릭 패널]을
   모델링할 때 [공수 50% 절감/IFC 납품 실패 방지]를 위해 사용합니다."

2. 사전 요구사항 (체크리스트)
   - Revit 버전: 2022 이상
   - Dynamo: 2.17 이상
   - 해당 없음: 정형 건물만 모델링하는 경우

3. 단계별 사용법 (스크린샷 포함)
   Step 1: [정확한 메뉴/버튼 명칭]
   Step 2: ...

4. 결과 예시 (비포/애프터)
   - 적용 전: 수동 면적 계산 오차 10%
   - 적용 후: 자동 산출 오차 ±2%

5. 자주 발생하는 오류
   - 오류: "DirectShape 요소를 선택해야 합니다"
   - 원인: 일반 Revit 요소 선택 시 발생
   - 해결: 개념 매스 → DirectShape 변환 후 사용
```

### 기술 문서 실패 패턴 4가지

**1. 비정형 개념 미설명 → 사용자 혼란**
- "DirectShape를 선택하세요" → DirectShape가 뭔지 설명 없음
- 해결: 용어 설명 팝업 링크 또는 별도 용어집 파일

**2. 스크린샷 버전 불일치**
- Revit 2022 스크린샷 → 실제 2025 UI 다름
- 해결: 스크린샷에 버전 워터마크, 반기별 스크린샷 갱신

**3. 코드 예시 복붙 불가 (이미지로 삽입)**
- 해결: 코드 블록 텍스트 형식 강제, GitHub 링크 병행

**4. 긴 설명 → 사용자 스킵**
- 사용자의 90%는 문서를 안 읽고 바로 시도
- 해결: Quick Start (3단계, 30초 안에 결과) + 상세 가이드 분리
