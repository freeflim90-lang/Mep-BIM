**주요 MEP 계산식 (AI 즉시 답변용):**
```
냉방 부하 계산 (간이식):
냉방 용량 [kW] = 공조 면적 [㎡] × 단위 부하 [W/㎡] ÷ 1000
  - 사무소: 80~120 W/㎡
  - 서버실: 300~500 W/㎡
  - 일반 주거: 50~80 W/㎡

배관 유량 계산:
Q [L/min] = 냉방 용량 [kW] × 860 ÷ (ΔT × 1000/60)
  - ΔT = 냉수 온도차 (보통 5℃)

덕트 풍량 계산:
Q [CMH] = 냉방 용량 [kW] × 3600 ÷ (Cp × ρ × ΔT)
  - Cp ≈ 1.006 kJ/(kg·K), ρ ≈ 1.2 kg/m³
```

## 엔지니어링계산서 Claude Code 심화 업데이트 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: MEP계산서, 냉난방부하, 덕트사이징, BIM연동, KDS설비기준

MEP 엔지니어링 계산서는 KDS 31 00 00(건축기계설비 설계기준) 및 KDS 31 25 00(공기조화설비 설계기준)을 근거로 작성하며, Revit BIM 모델과 연동하여 설계 변경 시 자동으로 계산값을 갱신하는 체계를 구축한다.
- 냉난방 부하 계산: ASHRAE Handbook Fundamentals(2021)의 RTSM(Radiant Time Series Method) 방법론을 적용한다. 입력값: 외기 설계온도(서울 기준 하계 DB 34℃/WB 25℃, 동계 -12℃), 내부발열(인체 65W/인, 조명 10~15W/㎡, 기기 15~20W/㎡). Revit Space 파라미터에서 면적·높이·외피 구성 정보를 추출하여 Heating Load·Cooling Load를 자동 계산하는 Dynamo 스크립트를 활용한다.
- 덕트 사이징(KDS 31 25 10): 등압법(Equal Friction Method) 적용 시 주덕트 마찰손실 0.8~1.0 Pa/m 기준. Revit MEP 덕트 시스템에서 `DuctSystem.TotalFlow`와 `DuctSystem.StaticPressure`를 읽어 사이징 검증 자동화가 가능하다.
- 배관 사이징(KDS 31 30 00 급배수설비): 급수 배관 유속 1.5~2.5 m/s(지관 기준), 주관 1.0~2.0 m/s. Hazen-Williams 공식 적용, C=130(아연도강관)·C=150(동관).
- BIM 연동 계산 체계: Revit MEP 모델 → Dynamo로 기하·파라미터 추출 → Python(pandas) 계산 엔진 → 계산서 Excel 자동 작성 → 설계 변경 시 재계산 1클릭 실행.

## 엔지니어링 계산서 마스터급 경험 지식 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: 냉난방부하, RTSM, TransferFunction, 실측편차, Dynamo계산자동화, ASHRAE

**냉난방 부하 계산 결과 ±15% 이상 실측 편차 발생 시 원인 분석**: 준공 후 실측 냉난방 부하가 설계 계산값 대비 ±15% 이상 차이가 나는 경우, 원인 분석은 다음 4가지 항목을 순서대로 점검한다. ① 외피 열성능 입력 오류 — 설계 도면상 단열재 두께(예: 200mm)와 실제 시공된 두께(예: 150mm)가 다른 경우, U값 차이가 냉방 부하 계산에 직접 영향을 미친다. Revit Space 파라미터의 외피 구성을 시공 완료된 마감 기준으로 재검토. ② 내부발열 입력 오류 — 오피스 용도에서 사람 밀도(인/㎡), IT 기기 발열(W/㎡)이 실제 운영 패턴과 다를 경우 냉방 부하가 최대 20% 이상 차이. 실측 전력 로거로 실제 기기 발열량 측정 후 재계산. ③ 조명 제어 미반영 — DALI 디밍 제어가 설치된 경우 설계 조명 발열은 최대값(full load) 기준이지만 실제 운영은 40~60% 수준으로 냉방 부하가 과대 산정됨. ④ 환기 실외공기 조건 — 전열교환기 효율이 설계값(70%)보다 낮게 실측(55%)되면 외기 냉난방 부하가 증가.

**ASHRAE RTSM vs Transfer Function Method 선택 기준**: RTSM(Radiant Time Series Method)은 복사열 지연 효과를 시간대별 계수(Radiant Time Factor)로 처리하여 시간별 최대 부하 예측 정확도가 높다. 설계 초기 단계(기본설계)에서 냉방기 용량을 보수적으로 산정해야 할 때 권장. Transfer Function Method(TFM/CTF)는 건축물 열적 거동을 전달함수로 모델링하여 연간 에너지 시뮬레이션(EnergyPlus, eQUEST 연동)에 적합하지만 초기 계수 산정이 복잡하다. LUA BIM LABS 실무 기준: 장비 용량 선정(기본설계) → RTSM, 에너지 성능 인증(녹색건축인증, LEED) 제출 → TFM 기반 시뮬레이션 소프트웨어(OpenStudio, IDA ICE).

**Dynamo 자동화 계산서 vs 수기 검증 불일치 해소 방법**: Dynamo 스크립트로 Revit Space에서 자동 추출한 부하 계산값이 엔지니어의 수기 Excel 계산값과 불일치하는 경우가 발생한다. 원인 파악 순서: ① 면적 기준 차이 — Revit Space 면적(Computation Height 기준)과 엔지니어가 도면에서 측정한 면적이 다름. Computation Height를 1.2m로 통일하고 Space 배치 완결성(미배치 구역 0건)을 확인. ② 열관류율(U값) 기준 차이 — Revit 외피 레이어 구성의 열전도율 입력값과 ASHRAE Handbook 참조값이 다름. 재료 라이브러리를 표준화하고 KDS 31 기준 단열재 열전도율 값으로 통일. ③ 단위 혼동 — Revit 내부 단위(ft²·BTU/h)와 SI 단위(㎡·W) 변환 오류. Dynamo에서 `UnitUtils.ConvertFromInternalUnits(value, UnitTypeId.Watts)` 활용.
