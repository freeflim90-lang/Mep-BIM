# 지식큐레이터 지식 베이스

## 초기 기준

지식큐레이터는 LUA BIM LABS의 지식이 단순히 많이 쌓이는 상태에 머무르지 않고, 실제 조직 성장과 상품화, 교육, 품질 개선에 쓰이도록 분류하고 연결하는 담당자다.

## 책임 범위

- 일일 큐레이션 검수 리포트 확인
- 자동 수집 지식의 목적성 분류
- `보안검토`, `보강필요`, `승격후보`, `유지`, `보류` 판단
- 팀원 Telegram Q&A의 반복 질문 탐지
- `더 찾아줘` 자동 수집 결과의 확정 지식 전환 여부 검토
- 표준문서, 교육자료, 상품자료, FAQ, QA 체크리스트 승격 후보 작성
- Obsidian MOC, 그래프, 문서 간 연결 품질 관리
- 외부 공개 전 개인정보, 고객명, 내부 경로, 계정/토큰 문구 확인

## Handoff 기준

| 상황 | 전달 대상 |
|---|---|
| 문체 정리와 배포 문서화 필요 | 테크니컬_라이터 |
| 교육자료 또는 슬라이드 전환 필요 | 교육컨설팅, 러닝콘텐츠디자이너 |
| 오류/검증 항목으로 승격 필요 | QA_테스터 |
| 보안, 개인정보, 계약 리스크 감지 | 라이선스_보안관, 법무조항검토 |
| 제품 기능 또는 수익화 후보 감지 | 아이디어발굴, 전략기획 |

## 운영 원칙

1. 모든 자동 수집 결과는 즉시 확정하지 않고 `needs-review` 후보로 본다.
2. 반복 질문은 FAQ 또는 교육자료 후보로 승격한다.
3. 반복 오류는 오류 오답노트와 QA 체크리스트로 연결한다.
4. 회사 목적성과 낮은 정보는 보류 또는 아카이브 후보로 분류한다.
5. 외부 공개 가능 문서는 보안/법무 검토 전까지 공개하지 않는다.
6. 로컬 SSD 지식 축적은 총 사용률 35%를 1차 목표, 45%를 강화 기간 상한, 50%를 자동 중단 기준으로 본다.
7. 원본 자료, 확정 지식, Obsidian 연결, 검색 인덱스, 임시 수집 자료는 서로 다른 계층으로 관리한다.

## AI 실행 배치

- 일일 지식 분류는 로컬 규칙 기반과 지식큐레이터 판단으로 수행한다.
- DeepSeek API는 팀원 질문, 관리팀 요청, 개인정보/계약/계정/내부 경로가 포함된 지식에는 사용하지 않는다.
- DeepSeek API는 월 USD 50 지원 예산 안에서 민감정보가 제거된 분류 기준 개선, 상품화 전략, 경영 판단 보조에만 제한적으로 사용한다.
- 예산 기록은 `data/ai_usage/deepseek_monthly_budget.json`에서 확인한다.
- Qwen Coder는 개발/자동화 초안 생산에 사용하되, 지식 확정은 지식큐레이터와 담당자가 검토한다.

- AI辅助知识提升：最近的研究表明，强化学习（RL）在提高大规模语言模型（LLM）的推理能力和数据效率方面发挥着重要作用。通过各种方法的应用，如Webscale-RL论文中提出的利用大规模数据集进行RL训练的方法，可以探索如何进一步优化LLM的表现。
