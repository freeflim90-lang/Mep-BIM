국내 공공 BIM 프로젝트 BEP 필수 항목 (국토부 시행지침 기준):
1. 프로젝트 개요: 사업명·발주처·계약금액·BIM 적용 범위
2. 수행 조직: BIM 책임자·공종별 담당자·협력업체 역할
3. 소프트웨어 및 하드웨어: Revit 버전·Navisworks·IFC 설정
4. BIM 데이터 표준: 파일 명명규칙·폴더 구조·좌표계·단위
5. LOD 계획: 설계 단계별 LOD 목표 (기본설계 LOD300, 실시설계 LOD350)
6. CDE 운영: ACC/BIM 360 등 공통 데이터 환경 접근 권한·워크플로우
7. 납품 계획: 납품물 목록·형식·시기·검토 절차
8. 품질 관리: 간섭검토 계획·IFC Export 검증·ISO 19650 체크리스트
```

**EIR(발주자 정보 요구사항) vs BEP(수행 계획서) 차이:**
| 항목 | EIR | BEP |
|------|-----|-----|
| 작성 주체 | 발주처 | 수급자(설계사·시공사) |
| 시점 | 발주 전 | 계약 후 착수 전 |
| 내용 | "무엇을 요구하는가" | "어떻게 수행할 것인가" |
| 포함 내용 | LOD 수준·납품 형식·CDE 요건 | 구체적 BIM 계획·담당자·일정 |
| ISO 19650 기준 | PIR → OIR → AIR → EIR | BEP → Post-contract BEP |

**국내 주요 발주처별 BEP 특이 요건:**
| 발주처 | 특이 요건 |
|--------|---------|
| LH공사 | 세대 유형별 BIM 표준 패밀리 사용 의무 |
| SH공사 | 서울시 BIM 지침 준수, CDE 서울시 플랫폼 사용 |
| 조달청 | BIM 성과품 검토 프로세스 명시 |
| 한국도로공사 | WBS·OBS 체계 BEP에 명시 |
| 국가철도공단 | 철도 BIM 적용지침 준수, 신호·궤도 공종 추가 |

**BEP 작성 시 자주 누락되는 항목:**
- 좌표계 기준점 (프로젝트 북·측지 좌표 연계 여부)
- IFC Export 버전 명시 (IFC 2x3 vs IFC 4)
- 링크 모델 관리 방식 (Workset 구성·링크 경로 절대/상대)
- 파일 백업 및 복구 절차
- 발주처 검토 후 수정·재납품 횟수 제한

### 8.3 마스터 스케줄 기입 기준

```
산정 투입일 = 대상 연면적 ÷ 등급별 1일 처리 가능 면적 × 보정계수

## BEP 계약 분쟁 마스터급 경험 지식 (2026-05-28)

**발주처가 BEP 내용을 준수하지 않을 때 계약적 대응**: 발주처가 BEP에 합의한 검수 주기(주 1회 피드백)를 지키지 않거나, 승인된 모델을 3개월째 검토하지 않아 후속 작업이 중단되는 상황이 실무에서 자주 발생한다. 계약적 대응: ① BEP는 계약 첨부 문서로 편입시키고, "발주처 피드백 지연이 14영업일 초과 시 납기 자동 연장" 조항을 공사 계약서에 삽입한다. ② 발주처 미준수 사실을 ACC Issue 또는 공문으로 기록하여 분쟁 시 증거로 활용. ③ 발주처가 명확한 검수 기준 없이 반복 수정을 요구할 경우 "검수 기준서(Acceptance Criteria)" 작성을 요청하고, 기준서 없는 수정 요청은 변경 지시서(Change Order)로 처리.

**BEP 범위 외 추가 BIM 요구 시 변경 계약 협상**: 실시설계 납품 이후 발주처가 "COBie 스프레드시트도 뽑아달라", "에너지 시뮬레이션 결과도 BIM에 넣어달라"는 추가 요구를 하는 상황이 빈번하다. 대응 원칙: ① BEP Section 2.1 "BIM 활용 목표" 표에서 해당 항목의 "해당 여부"가 "-"(해당 없음)로 명시되었으면 계약 외 추가 업무임을 입증 가능하다. ② 추가 업무는 공수 산정(시간×단가) 후 변경 계약서를 발행하며, "무상으로 해주면 관계가 좋아진다"는 관행은 향후 수익 구조를 훼손한다. ③ 발주처 요구 단계에서 거절이 어려울 경우 "다음 프로젝트 계약 우선권"으로 협상하거나 부분 무상(핵심 기능)·부분 유상(추가 기능) 방식으로 분리 제안.

**중간에 BIM 소프트웨어 변경 요청 시 처리 방법**: "Revit 말고 ArchiCAD로 바꿔달라"거나 "BIM 360 대신 다른 CDE를 써달라"는 요구가 실시설계 중에 들어오는 경우가 있다. 처리 원칙: ① BEP Section 5(소프트웨어 환경)에 명시된 소프트웨어는 "프로젝트 기간 동안 고정"이며 변경 시 전 공종 동시 전환 필요. 소프트웨어 변경은 모든 링크 모델 재구성, 좌표계 재설정, 파라미터 재매핑, 기존 작업물 변환 비용을 수반한다. ② 변경에 필요한 추가 비용(소프트웨어 라이선스, 데이터 마이그레이션 공수)을 산정하여 변경 계약 체결 후 진행. ③ CDE 플랫폼 변경 시 ACC 등 기존 플랫폼에 축적된 이슈·버전 이력의 이전(Migration) 가능 여부를 사전에 확인하고 데이터 손실 리스크를 발주처에 서면 통보.
