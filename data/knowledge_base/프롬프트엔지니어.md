효과적인 프롬프트 구조:
1. 역할 명시: "당신은 [에이전트명] 전문가입니다"
2. 컨텍스트 주입: 관련 지식 베이스 섹션 (최대 2,000자)
3. 작업 명세: 입력 데이터 + 기대 출력 형식
4. 제약 조건: 제외 범위, 금지 표현, 출력 길이
5. 예시 (Few-shot): 유사 케이스 1~2개
토큰 절감: 지식 베이스는 관련도 상위 3섹션만 주입, 중복 제거, 표 형식 활용.

- Claude API 2025 프롬프트 최적화 기법: claude-sonnet-4-6 기준, 시스템 프롬프트에 역할·출력 형식·제약 조건을 명시하면 응답 일관성이 30% 향상된다. Extended Thinking 모드를 활성화(`"thinking": {"type": "enabled", "budget_tokens": 10000}`)하면 복잡한 BIM 분석 작업의 정확도가 유의미하게 개선된다.
- Chain-of-Thought(CoT) 설계 패턴: BIM 요구사항 분석 프롬프트에 "단계별로 생각하세요" 대신 구체적 단계를 명시한다. 예: "1) EIR 요구사항 추출 2) Revit 기능으로 매핑 3) 구현 난이도 평가 4) 우선순위 제안" 형식의 명시적 CoT가 암묵적 CoT보다 일관된 결과를 생성한다.
- 프롬프트 캐싱(Prompt Caching) 전략: `cache_control: {"type": "ephemeral"}` 설정으로 시스템 프롬프트와 긴 컨텍스트를 캐싱한다. LUA BIM LABS KB 전체를 시스템 프롬프트에 포함할 경우 첫 호출 이후 입력 토큰 비용의 최대 90%를 절감할 수 있다. 캐시 유효 시간은 5분이므로 반복 호출 시나리오에 효과적이다.
- RAG + 프롬프트 설계 통합: 검색된 KB 청크를 XML 태그(`<context>...</context>`)로 구조화하여 프롬프트에 삽입하면 Claude의 인용 정확도가 높아진다. "위 컨텍스트에서 답을 찾을 수 없으면 '정보 없음'이라고 답하세요" 형식의 Grounding 지시문을 포함하여 Hallucination을 방지한다.
- BIM 도메인 특화 프롬프트 라이브러리: Revit API 코드 생성, EIR 분석, 견적 산출, 법규 해석 등 업무별 시스템 프롬프트 템플릿을 Obsidian KB에 버전 관리하고, A/B 테스트(응답 품질 평가 5점 척도)를 통해 월 1회 갱신한다.

## BIM 도메인 프롬프트 라이브러리 심화 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: prompt-library,bim,revit,rag,few-shot,structured-output

**BIM 업무별 프롬프트 템플릿:**

**(1) Revit API 코드 생성:**
```
System: 당신은 Revit 2025 API(C# .NET 8) 전문가입니다.
요구사항을 받으면 다음 형식으로 답하세요:
1. 필요한 네임스페이스 목록
2. 완성된 코드 (주석 없음, 간결)
3. 예외 처리 포인트
4. Revit 버전별 주의사항 (2023/2024/2025)
```

**(2) EIR 요구사항 분석:**
```
System: 당신은 ISO 19650 기반 BIM 발주 전문가입니다.
<context>{{kb_eirbep}}</context>
입력된 EIR 문서에서: ①필수 파라미터 목록 ②LOD 단계별 요구사항
③납품 파일 형식 ④검수 기준 을 표 형식으로 추출하세요.
답을 찾을 수 없으면 "EIR 미기재"로 표시하세요.
```

**(3) BIM 견적 산출:**
```
System: 당신은 BIM 프로젝트 견적 전문가입니다.
<context>{{kb_estimate}}</context>
연면적: {{area}}㎡, 공종: {{discipline}, 등급: {{level}}
엔지니어링 대가기준 2023 기준으로 M/D를 산출하고
단가 테이블을 적용하여 총 견적을 계산하세요.
```

**프롬프트 품질 A/B 테스트 체계:**
- 평가 지표: 정확도(5점), 완결성(5점), 형식 준수(5점), 토큰 효율(5점)
- 평가 주기: 월 1회, 같은 10개 테스트 케이스로 버전 비교
- 버전 관리: Obsidian `prompts/` 폴더, Git 태그 `prompt-v1.x.x`
- 임계값: 평균 16점 미만 시 자동 fallback to 이전 버전

**Few-shot 예시 설계 원칙:**
- 예시 수: 2~3개 (4개 이상은 효과 미미, 토큰만 증가)
- 예시 선택: 경계 케이스(edge case) 포함 — 일반 케이스만 있으면 모델이 단순 패턴 매핑
- 예시 포맷: 입력/출력 동일 포맷, 출력 길이는 목표와 같게
