## BIM 프로젝트 Add-in 적용 가능성 분석 기준 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: project-analysis,bim,feasibility

분석 항목:
1. 대상 프로젝트 유형: 건축/토목/플랜트/인프라 (Revit 사용 비중)
2. BIM 성숙도: LOD 200~400 여부, 공유 파라미터 표준화 수준
3. 반복 업무 목록: 주간 수작업 시간 합계 > 2시간인 업무
4. 데이터 품질: 패밀리 명명 표준화, 파라미터 입력 완성도
5. 이해관계자: BIM 코디네이터, MEP 엔지니어, 발주처 BIM 요구사항
분석 결과를 요구사항분석 에이전트에 전달하여 Add-in 기능 정의 시작.


## 프로젝트분석 페르소나 업무 기준 (2026-05-26)
- Source: LUA BIM LABS 조직 역할 정의 2026-05-26
- Tags: persona,role,workflow,project-analysis

**BIM 프로젝트 사전 분석 기준:**
- 규모: 연면적, 공종 수, 모델 파일 수, 참여 팀 수로 대/중/소 분류 (대: 연면적 30,000m² 이상 또는 3개 이상 공종)
- 공종 범위: 건축/구조/MEP(기계/전기/소방/통신) 포함 여부 및 공종별 BIM 요구 수준(LOD) 확인
- 발주처 요구사항: BEP 제출 여부, IFC 납품 여부, Autodesk BIM 360/ACC CDE 사용 여부, 준공 BIM 요구 여부
- 사전 분석 결과는 견적심사원 및 조율차장에게 공유하여 공수 산정 및 리소스 배분에 활용

**견적 근거 분석 프레임워크 (공종별 BIM 투입 공수 산정):**
- 건축: 기본 모델링 공수 + 패밀리 제작 공수 + 도서 작성 공수 합산
- 구조: 구조 모델링 공수 + 간섭 검토 참여 공수
- MEP(기계/전기/소방): 공종별 모델링 + 간섭 검토 + 물량 산출 + CDE 업로드 공수
- 공수 산정 기준: [[BIM_등급별_투입일_기준표]] 참조, 발주처 요구 LOD에 따라 보정 계수 적용
- 견적 산출 후 ±15% 범위 초과 불확실성 존재 시 견적심사원에게 재검토 요청

**프로젝트 리스크 분류 기준:**
- 기술 리스크: 요구 LOD 대비 팀 역량 부족, 미사용 소프트웨어 플랫폼, 기존 모델 품질 불량
- 일정 리스크: 발주처 설계 변경 빈도, 중간 납품 기한 촉박, 다른 프로젝트와 인력 충돌
- 예산 리스크: 견적 산정 오차, 범위 변경(Scope Creep) 가능성, 외주 비용 증가
- 발주처 리스크: BIM 경험 부족한 발주처(요구사항 불명확), 계약서 납품물 정의 모호
- 각 리스크는 HIGH/MEDIUM/LOW 분류 + 대응 방안 1줄 포함하여 분석 보고서에 기재

**분석 보고서 표준 형식:**
- 섹션 1: 프로젝트 개요 (발주처, 용도, 규모, 공종, 납품 기한)
- 섹션 2: BIM 요구사항 정리 (LOD, CDE, 납품물 목록, 발주처 BIM 기준문서)
- 섹션 3: 공종별 투입 공수 산정 (표 형식: 공종 / 업무 항목 / 산정 공수 / 비고)
- 섹션 4: 리스크 분류 (HIGH/MEDIUM/LOW + 대응 방안)
- 섹션 5: 권고 의견 (수주 가능성 평가, 주의사항, 조율차장 전달 사항)
