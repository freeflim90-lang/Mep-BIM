## 보안/라이선스 기준 (2026-05-19 08:53:24)
- Source: LUA BIM LABS curated baseline, Autodesk official docs checked 2026-05-19
- Tags: security,license,privacy

토큰, API 키, 고객 모델 데이터, 라이선스 상태는 로그와 보고서에 노출하지 않는다. 외부 통신은 사용자 고지와 정책 문서가 필요하며, 모델 데이터 전송은 최소화한다.

## 상용 애드인 라이선스 보안 기준 (2026-05-19 09:37:53)
- Source: `docs/autodesk_store/PRIVACY_POLICY_DRAFT.md`, `260519 소스 폴더` 코드 감사
- Tags: license,privacy,security

상용 릴리스 DLL에는 활성화 비밀번호, HMAC 시크릿, API 토큰 같은 비밀값을 포함하지 않는다. 로컬 라이선스 파일만으로 판매 보호를 구현하면 우회와 유출 위험이 높으므로 서버 측 검증 또는 공식 entitlement 흐름을 우선 검토한다. 이름, 이메일, 전화번호, Machine ID, MAC/CPU 기반 식별값을 수집하거나 저장하면 개인정보 정책과 제품 설명에 반드시 고지한다. 외부 AI, Telegram, 로컬 서버, 클라우드 호출은 고객 모델 데이터가 외부로 나가는지 여부를 기준으로 별도 심사한다.


## 보안 취약점 체크리스트 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: security,checklist,addin

코드 보안 점검:
□ DLL에 하드코딩된 API 키·비밀번호 없음 (strings 명령어 또는 dotPeek로 확인)
□ 외부 HTTP 통신 시 HTTPS(TLS 1.2↑) 필수, 인증서 검증 우회 코드 없음
□ 로그 파일에 고객 모델 데이터·개인정보 미포함
□ 임시 파일은 %TEMP% 사용 후 종료 시 삭제
□ 레지스트리에 민감 정보 저장 금지
□ 라이선스 키·활성화 코드는 서버 측 검증 또는 Autodesk Entitlement API 사용
