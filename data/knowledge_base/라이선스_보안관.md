## 라이선스·보안 마스터급 경험 지식 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: license,security,VM-detection,crack,Named-User,migration,field-experience

하드웨어 ID 기반 라이선스에서 VM(가상머신) 환경 오인식 문제 해결: VM 내에서 HW-ID를 생성하면 물리 머신과 다른 MAC 주소, CPU ID, 디스크 시리얼이 반환되어 라이선스가 무효화되는 경우가 있다. 또 반대로 VM에서 물리 머신과 동일한 HW-ID가 생성되어 중복 인증이 허용되는 취약점도 존재한다. 해결 방법은 HW-ID 계산 시 VM 감지 플래그(WMI `Win32_ComputerSystem.Model`에서 "Virtual"·"VMware"·"VBOX" 포함 여부)를 확인하고, VM 환경에서는 라이선스 서버 토큰 방식으로 전환하거나 VM 사용을 라이선스 약관에 명시적으로 허용/금지 처리한다.

고객이 단일 라이선스로 여러 컴퓨터 사용 적발 시 처리 절차: ① Autodesk App Store 판매 기록과 실제 활성화 HW-ID 수를 대조하여 단일 라이선스에 복수 HW-ID가 등록된 경우를 탐지한다. ② 위반 고객에게 라이선스 약관 위반 사실을 개인 이메일로 고지한다. ③ 라이선스 서버에서 해당 라이선스 키를 일시 정지 처리하고 추가 라이선스 구매 안내를 함께 발송한다. ④ 선의의 위반(회사 PC 교체 등)인 경우 HW-ID 이전(Migration) 절차를 안내하여 갱등을 최소화한다. 법적 조치보다 구매 유도가 매출에 유리하다는 것을 원칙으로 삼는다.

해킹된 크랙 버전 유통 감지 방법: ① 배포된 정식 DLL의 SHA-256 해시값을 GitHub Releases에 공개 게시한다. ② 크랙 탐지 스크립트(Python 3.12)가 주요 Warez 사이트, Telegram 채널, GitHub 이슈에서 제품명+crack 키워드를 주 1회 스캔하고 발견 시 Slack 알림을 발송한다. ③ 크랙 파일과 정식 파일의 해시 비교로 변조 여부를 확인한다. ④ 바이너리 난독화(Obfuscator, ConfuserEx 또는 .NET Reactor)를 적용하면 크랙 제작 난이도가 높아진다.

Named User 전환 시 기존 Machine 라이선스 마이그레이션: Autodesk가 Named User 기반으로 전면 전환함에 따라 기존 HW-ID(Machine) 기반 라이선스 고객을 이관해야 하는 경우, ① 기존 라이선스 키 유효 기간 내에 APS(Autodesk Platform Services) OAuth 2.0 계정 연결 절차를 안내한다. ② Named User 라이선스는 최대 3대 디바이스 동시 사용을 허용하므로 고객이 HW-ID 제한보다 유연하게 사용 가능함을 홍보한다. ③ 이전 HW-ID 라이선스는 Named User 이관 후 자동 비활성화 처리하여 이중 사용을 방지한다.
