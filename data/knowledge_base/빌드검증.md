## 빌드 검증 Claude Code 심화 업데이트 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: build,verification,CI-CD,Revit-Addin,smoke-test,2025

- Revit 2022~2025 멀티버전 빌드 매트릭스: GitHub Actions 워크플로우에서 `matrix.revit_version: [2022, 2023, 2024, 2025]`를 정의하여 각 버전별 `RevitAPI.dll` 참조를 동적으로 교체한다. .NET 8(Revit 2025용)과 .NET Framework 4.8(Revit 2022~2024용)을 별도 타겟으로 멀티타게팅 빌드(`<TargetFrameworks>net8.0;net48</TargetFrameworks>`)를 구성한다.
- 스모크 테스트(Smoke Test) 자동화: Revit 2025 Journal File 재생 방식을 활용하여 Add-in 핵심 기능(실행, UI 로드, 데이터 추출, 파일 저장)을 자동 검증한다. RevitTestFramework(RTF) v3.x를 CI 파이프라인에 통합하여 헤드리스 테스트를 구현한다.
- 코드 서명(Code Signing) 필수화: Autodesk App Store 심사 통과 및 Windows SmartScreen 경고 방지를 위해 EV Code Signing Certificate(DigiCert 또는 Sectigo)로 `.dll` 및 MSI에 서명한다. `signtool.exe /fd SHA256 /tr http://timestamp.digicert.com` 명령을 빌드 파이프라인 마지막 단계에 포함한다.
- 빌드 아티팩트 관리: 각 빌드 결과물을 GitHub Releases 또는 Azure Blob Storage에 버전 태그와 함께 보관하며, SHA-256 해시를 생성하여 배포 무결성을 검증한다. 릴리스 브랜치(`release/v2.x`)에서만 공식 서명 빌드를 허용하는 브랜치 보호 규칙을 설정한다.
- 빌드 실패 알림 체계: GitHub Actions 실패 시 Slack Webhook 또는 이메일(SendGrid)로 빌드 로그 요약과 실패 원인을 자동 발송한다. 연속 3회 빌드 실패 시 릴리스 브랜치 머지를 자동 차단하는 Branch Protection Rule을 적용한다.

## 빌드검증 마스터급 경험 지식 (2026-05-28)
- Source: claude-code-enhanced 2026-05-28
- Tags: build,multi-version,assembly-conflict,RevitAPI,EV-signing,conditional-compile,field-experience

Revit 멀티버전 빌드 실패 케이스 1 — 공유 어셈블리 버전 충돌: 여러 Revit 버전을 단일 솔루션에서 빌드할 때 공통 유틸리티 프로젝트(Common.dll)의 AssemblyVersion이 각 Revit 버전용 타겟 프레임워크와 충돌하는 경우다. `net8.0` 타겟(Revit 2025)과 `net48` 타겟(Revit 2022~2024) 사이에서 같은 어셈블리의 다른 버전이 로드되어 `FileLoadException`이 발생한다. 해결책은 공유 프로젝트에서 `<TargetFrameworks>net8.0;net48</TargetFrameworks>`로 멀티타게팅 설정하고, 런타임 어셈블리 리다이렉션(`app.config` bindingRedirect)을 Revit 버전별로 분리한다.

Revit 멀티버전 빌드 실패 케이스 2 — RevitAPI.dll 버전 미스매치: GitHub Actions 환경에서 `Autodesk.Revit.DB.dll`을 NuGet 패키지가 아닌 로컬 SDK 경로로 참조하면 CI 환경에서 `Could not load file or assembly` 오류가 발생한다. 해결책은 `Revit.API.2025` 등 공식 커뮤니티 NuGet 패키지를 사용하거나, `.csproj`에서 `<HintPath>` 대신 `<PackageReference>`로 전환한다. 로컬에서는 동작하지만 CI에서만 실패하면 이 패턴이 가장 의심스럽다.

Revit 멀티버전 빌드 실패 케이스 3 — 조건부 컴파일 누락(`#if REVIT2025`): Revit API 버전별 deprecated 메서드나 새로운 오버로드를 처리할 때 `#if REVIT2025` 등 조건부 컴파일 심볼을 누락하면 특정 버전에서 컴파일 오류가 발생한다. `PropertyGroup` 안에 `<DefineConstants Condition="'$(RevitVersion)'=='2025'">REVIT2025</DefineConstants>`를 설정하고, GitHub Actions 매트릭스에서 `RevitVersion` 환경 변수를 주입한다.

EV Code Signing 인증서 만료 시 대응 절차: ① 만료 7일 전 인증서 갱신 신청(DigiCert, Sectigo 등 CA에 재발급 요청, 최소 영업일 2~3일 소요). ② 새 인증서로 대기 중인 빌드 산출물을 재서명(`signtool.exe /f new-cert.pfx`). ③ 기존 만료 인증서로 서명된 배포본은 타임스탬프가 유효하면 인증서 만료 후에도 검증이 유지되므로 이미 배포된 버전은 재서명 불필요. ④ Autodesk App Store에 재업로드가 필요한 경우 새 인증서로 서명 후 재제출.

- 모델 변경 명령에서 rollback 불가 또는 데이터 손상 가능성.

- DLL/MSI 서명 누락 또는 SHA-256 해시 미생성.
