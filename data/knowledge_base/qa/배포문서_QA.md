## 2026-06-09 배포문서 실전 Q&A
- Source: LUA BIM LABS 문서화 운영 기준, Autodesk App Store 문서 요건
- Tags: documentation,release-notes,install-guide,cs,qa,2026

**Q: 릴리스 노트에 꼭 들어가야 하는 항목은?**
A: Autodesk App Store 제출용 릴리스 노트 필수 항목: ① 버전 번호 (ex. v1.2.0) ② 릴리스 날짜 ③ 신규 기능 (New Features) ④ 버그 수정 (Bug Fixes) ⑤ 알려진 이슈 (Known Issues) ⑥ 지원 버전 (Revit 2023~2026). 고객에게 보이는 언어는 영어(스토어) + 한국어(내부 CS) 모두 준비한다.

**Q: 설치 가이드에 반드시 포함해야 하는 내용은?**
A: 필수 포함 항목: ① 시스템 요구사항 (Windows 버전, Revit 버전, .NET Framework) ② 다운로드 경로 ③ 설치 단계별 스크린샷 ④ .addin 파일 설치 경로 (버전별) ⑤ 설치 확인 방법 (Revit 재시작 후 메뉴 확인) ⑥ 제거 방법 ⑦ 자주 묻는 오류 코드 및 해결법 (1603, 로드 실패 등). 고객지원 CS와 내용 일치 여부를 상호 확인한다.

**Q: Autodesk App Store 제품 설명(Description)은 어떻게 작성하나요?**
A: 제품 설명 구조: ① 1행 요약 (핵심 가치, 예: "MEP 엔지니어를 위한 Revit 간섭 검토 자동화 Add-in") ② 주요 기능 3~5개 (bullet) ③ 지원 버전 ④ 라이선스 모델 (무료 체험 → 유료) ⑤ 지원 연락처. 자동 검수 결과를 "인증 보장" 문구로 표현하지 않는다. 브랜드마케팅 에이전트가 마케팅 언어를 검토한다.

**Q: 문서와 실제 기능이 불일치할 때 어떻게 처리하나요?**
A: 배포문서 에이전트는 불일치를 발견하면 ① 프로그램개발에 기능 확인 요청 ② 스토어심사에 제출 보류 요청 ③ 고객지원 CS에 임시 안내 문구 전달. 불일치 상태로 스토어에 제출하지 않는다. 심사 거절 원인이 되므로 배포 전 반드시 해소한다.

**Q: 개인정보 처리방침 문서는 누가 작성하나요?**
A: 초안은 법무조항검토 에이전트가 작성하고, 배포문서 에이전트가 스토어 포맷에 맞게 편집한다. 처리방침에 반드시 포함: ① 수집 데이터 항목(없으면 "수집 안 함" 명시) ② 저장 위치 ③ 제3자 전송 여부 ④ 삭제 요청 방법 ⑤ 연락처. 라이선스_보안관이 최종 검토 후 배포문서가 게시한다.
