# 빌드검증 Q&A 지식

## 2026-06-09 빌드검증 실전 Q&A
- Source: LUA BIM LABS 빌드/QA 운영 기준, Autodesk Add-in 배포 경험
- Tags: build,verification,smoke-test,release,qa,2026

**Q: Smoke test는 어떤 항목을 확인하나요?**
A: Revit Add-in smoke test 기본 항목: ① Add-in이 Revit에 정상 로드됨 ② 메뉴/리본 버튼이 표시됨 ③ 핵심 기능 1~2개가 에러 없이 실행됨 ④ Revit 종료 시 충돌 없음 ⑤ 라이선스 인증 UI가 표시됨. 각 지원 버전(2023/2024/2025/2026)별로 동일하게 수행한다.

**Q: P1 이슈와 P2 이슈의 차이는?**
A: P1(차단): 핵심 기능 실행 불가 / Revit 충돌 / 데이터 손실 / 라이선스 인증 실패. → P1 이슈가 있으면 출시 불가. COO에게 즉시 보고. P2(중요): 비핵심 기능 오동작 / UI 비정상 / 성능 저하 (기준치 3배 초과). → P2 이슈는 수용 가능 여부를 COO와 협의 후 결정.

**Q: 빌드 산출물 검증 기준은?**
A: 체크리스트: ① DLL 파일 버전 정보가 릴리스 노트 버전과 일치 ② .addin 매니페스트의 ModuleName 경로가 정확 ③ 서명(코드 서명 인증서) 적용 여부 확인 ④ 불필요한 디버그 파일(.pdb) 제외 ⑤ 사용자 데이터·API 키가 DLL에 하드코딩 안 됨. 빌드 검증 실패 시 릴리스 게이트 차단.

**Q: 회귀 테스트는 언제 수행하나요?**
A: ① 신규 기능 추가 시: 기존 핵심 기능 전체 smoke test 재수행 ② 버그 수정 시: 수정된 기능 + 관련 기능 재테스트 ③ 의존성(Revit 버전, .NET 버전) 변경 시: 전 버전 호환성 테스트 ④ 릴리스 전 최종 빌드: 전체 smoke test 의무. QA_테스터가 재현 절차를 기록하면 빌드검증이 회귀 항목에 추가한다.

**Q: 빌드 환경은 어떻게 관리하나요?**
A: CI 기준(권장): ① 빌드 서버에 Revit 2023~2026 설치 ② MSBuild를 통한 자동 빌드 ③ 빌드 번호를 AssemblyVersion에 자동 반영 ④ 빌드 성공 시 자동 smoke test 트리거. 현재 로컬 빌드 환경에서는 빌드검증 에이전트가 수동으로 위 항목을 체크하고 결과를 배포문서에 기록한다.
