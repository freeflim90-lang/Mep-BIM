# 성장전략그룹 지식 베이스

## 개요
- Source: LUA BIM LABS internal organization knowledge baseline
- Tags: #성장전략그룹 #전략기획 #시장선도 #아이디어발굴 #제품패키징 #글로벌유통 #BIM시장 #AX전략
- 업데이트: 2026-05-28

성장전략그룹은 시장 신호를 읽고, 회사의 방향성을 정하고, 아이디어를 제품·매출·브랜드 자산으로 전환하는 그룹이다. 핵심 질문은 “무엇을 만들고, 누구에게 팔고, 어떤 순서로 키울 것인가”다.

## 5. R&D 개발지원그룹으로 넘기는 기준

성장전략그룹은 “좋은 아이디어”가 아니라 “개발 가능한 문제정의”를 넘긴다.

R&D 전달 전 필수 조건:
- 대상 사용자와 반복 pain point가 명확해야 한다.
- MVP 범위와 제외 범위가 분리되어야 한다.
- 예상 입력 데이터와 출력 결과가 설명되어야 한다.
- Revit/Navisworks/IFC/웹/자동화 중 기술 경계가 지정되어야 한다.
- 수익화 가설 또는 내부 생산성 절감 효과가 있어야 한다.

전달 형식:
```yaml
growth_request:
  source_group: 성장전략그룹
  target_group: R&D_개발지원그룹
  problem: "반복되는 BIM 납품 오류를 자동 검출"
  target_user: "BIM PM / QA 담당"
  mvp_scope: "필수 파라미터 누락 검출 + Excel 보고서"
  excluded_scope: "AI 자동 수정, 전체 법규 판정"
  success_metric: "QA 검토 시간 30% 절감"
```

## 6. 실패 패턴 Top 5

1. 시장 신호가 아니라 내부 흥미만으로 개발 큐를 생성함.
2. 기능 범위가 넓어 MVP가 Store 제출 가능한 단일 문제로 좁혀지지 않음.
3. 가격·유통·지원 비용을 고려하지 않고 개발 착수.
4. 경쟁 제품의 UI·명칭·흐름을 벤치마크 이상으로 따라가 법무·브랜드 리스크 발생.
5. R&D에 넘길 때 성공 기준과 제외 범위가 없어 개발 중 재해석이 반복됨.

## 성장전략그룹 제품·시장 심화 기준 (2026-05-29)
- Source: claude-code-enhanced 2026-05-29
- Tags: growth-strategy,bim-market,autodesk-appstore,gtm,product-prioritization,revenue

**Autodesk App Store GTM 3단계 전략:**
- 1단계 (출시 0~6개월): 무료 버전 배포로 설치 수 500건 목표. 스토어 리뷰 20건+, 평균 4.5점 이상 확보 → Autodesk Technology Partner 지위 신청 기반
- 2단계 (6~12개월): 무료→유료 전환 게이트 — 설치 수 500+ 확보 후 Pro 기능 Paywall 도입. 월 $19 또는 연 $190 기준. 전환율 목표 5% (=설치 500건 × 5% = 25명 유료)
- 3단계 (12개월+): NRR(Net Revenue Retention) 110% 이상 목표. 기존 고객 업셀(Team 5-Pack) + Enterprise 계약 병행. CAC 대비 LTV 3:1 이상 유지

**BIM Add-in 제품 후보 평가 매트릭스:**
| 기준 | 가중치 | 판단 질문 |
|---|---|---|
| 반복 고통 빈도 | 30% | 같은 작업을 주 1회 이상 반복하는가? |
| Revit API 구현 가능성 | 25% | 공식 API로 가능하고 실기 검증이 1주 이내 가능한가? |
| Store 심사 리스크 | 20% | Autodesk 정책 위반, 성능 저하, 개인정보 이슈가 없는가? |
| 유료 전환 가능성 | 15% | 사용자가 이 기능에 월 $19를 지불할 의향이 있는가? |
| 시장 선점성 | 10% | 유사 기능 경쟁 제품이 App Store에 없거나 약한가? |

**K-BIM 의무화 타임라인과 제품 로드맵 연동:**
- 2025: 500억 이상 공공 프로젝트 BIM 의무화 → 납품 검증 자동화 수요 발생
- 2026~2027: LUA BIM LABS 최적 진입 창 — 공공 발주처 BIM 담당자 대상 무료 체험 프로그램 운영
- 2028: 200억 이상 의무화 → 중견 건설사로 수요 확산
- 2030: 전면 의무화 → 의무화 후 진입 시 기존 플레이어 레퍼런스 우위에 영업 사이클 6개월+ 지연 → 2026~2027에 선점 필수

**성장전략그룹 → R&D 전달 품질 기준:**
전달 문서에 다음 5가지가 없으면 R&D 큐 등록 거부:
1. "이 기능이 없어서 몇 분/몇 시간이 낭비되는가" — 정량 수치
2. Revit API 기반 구현 가능 여부 — "가능/불확실/불가" 중 하나
3. LOD 요구사항 — 사용자가 실제로 모델에서 필요로 하는 정보 수준
4. Store 배포 형태 — Free/Freemium/Pro의 기능 경계
5. 성공 지표 — "설치 100건" 또는 "QA 시간 X% 절감" 같은 측정 가능한 목표

## 성장전략그룹 마스터급 경험 지식 (2026-05-29)
- Source: claude-code-enhanced 2026-05-29
- Tags: growth-strategy,product-failure,pivot,competitive-intelligence,bim-market

**BIM Add-in 제품화 실패 패턴 5가지와 원인:**
1. **너무 넓은 MVP**: "모든 BIM 납품 검증"을 한 번에 만들려다 Store 제출 가능한 단일 기능을 완성하지 못함. 해결: 단일 문제(예: "스프링클러 헤드 위치 KCS 기준 자동 검증") 하나만 MVP로 정의
2. **Autodesk 내재화 리스크 무시**: Autodesk가 유사 기능을 기본 탑재하면 3개월 내 차별화 피벗 계획이 없으면 매출 0이 됨. 항상 "Autodesk가 이걸 만든다면 우리는 무엇으로 차별화할 것인가"를 제품 후보 단계에서 검토
3. **가격 설계 후행**: 개발 후 가격을 설정하면 기능 범위가 가격에 맞지 않는 경우 발생. 성장전략그룹은 개발 착수 전에 "이 기능은 월 $19 Freemium 상위 기능인가, Pro 전용인가"를 결정
4. **공공 시장 과의존**: 공공 발주처는 조달 사이클이 6~12개월이고 최저가 경쟁이 발생. BIM 스타트업은 초기에 민간 건설사(중견~대형) 타겟으로 CAC를 낮추고, 공공은 레퍼런스 확보 후 진입
5. **경쟁 UI 클론**: 유사 App Store 제품의 UI·명칭·워크플로를 벤치마크 이상으로 따라가면 Autodesk 심사에서 "중복 기능" 이슈 또는 법무 리스크 발생. 차별화 포인트를 UX와 한국 법규 대응에서 명확히 구분

**경쟁 제품 인텔리전스 수집 루틴:**
- 주간: Autodesk App Store 스프링클러/MEP/BIM QA 카테고리 신규 등록 제품 모니터링
- 월간: 상위 5개 경쟁 Add-in의 리뷰 신규 댓글에서 고객 불만 키워드 추출 → 아이디어발굴 큐에 등록
