# 제품패키징 지식 베이스


## Autodesk Store 제품 패키징 기준 (2026-05-19 08:53:24)
- Source: LUA BIM LABS curated baseline, Autodesk official docs checked 2026-05-19
- Tags: installer,msi,package

패키지는 고객이 개발자 도움 없이 설치/제거/업데이트할 수 있어야 한다. Autodesk Product Guidelines는 설치/제거/유지관리가 어려운 제품, 불완전한 제품, 최신 Autodesk 릴리스 미지원 제품을 거절 사유로 본다.

## BIM Command Center 패키징 실행 기준 (2026-05-19 09:37:53)
- Source: `docs/autodesk_store/STORE_SUBMISSION_CHECKLIST.md`
- Tags: installer,revit,release

Store 제출 패키지는 개발자 PC 경로 없이 설치되어야 하며 `.addin` 매니페스트의 Assembly 경로는 설치 결과와 일치해야 한다. 클린 Windows 계정에서 설치, Revit 실행, 명령 실행, Revit 종료, 제거까지 검증한다. WebView2 또는 .NET Desktop Runtime 같은 전제 조건은 설치 프로그램에서 포함하거나 사용자에게 명확히 안내한다. 첫 릴리스 지원 버전은 실제 테스트한 Revit 2024/2025/2026으로 제한하는 것이 안전하다.


## MSI 인스톨러 구성 기준 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: packaging,msi,installer

Revit Add-in MSI 구조:
- Product GUID: 버전마다 새 GUID 생성
- UpgradeCode: 제품 전체 라이프사이클 동안 고정
- 설치 폴더: Program Files\[회사명]\[제품명]
- Revit Addins 폴더 배포: CustomAction 또는 별도 Component로 각 버전 폴더에 복사
WiX Toolset 또는 Advanced Installer 사용 권장.
서명(Code Signing): EV 코드 서명 인증서로 DLL·MSI 서명 시 SmartScreen 경고 감소.


## 패키지 버전 관리 기준 (2026-05-19 17:26:40)
- Source: LUA BIM LABS domain knowledge baseline 2026-05-19
- Tags: packaging,versioning,release

버전 체계: Major.Minor.Patch.Build (예: 1.2.3.456)
Major: 호환성 변경, Minor: 기능 추가, Patch: 버그 수정, Build: CI 빌드 번호.
AssemblyVersion과 FileVersion을 일치시키고 InformationalVersion에 브랜치/커밋 정보 포함.
스토어 제출 버전은 MSI 버전과 AssemblyFileVersion이 반드시 일치해야 한다.
업그레이드 시 이전 버전 자동 제거(MajorUpgrade 설정) 또는 병렬 설치 명시적 지원 여부 결정.
