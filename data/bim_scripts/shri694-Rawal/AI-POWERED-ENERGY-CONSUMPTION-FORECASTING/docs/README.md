# Documentation Index

This folder contains the project documentation used to explain the design, implementation, and GitHub proofing process.

- `architecture.md` — System and data flow architecture for the forecasting pipeline.
- `github-proof-plan.md` — A structured plan for building, documenting, and publishing the repo.
- `implementation-phases.md` — Development phases and milestones for the project.
- `proof-checklist.md` — A checklist for ensuring the repository is polished and presentation-ready.

## Usage

Use this folder as a reference for how the project was designed and what artifacts are intended for GitHub presentation.

For a quick start, see the root `README.md` and `QUICKSTART.md`.
