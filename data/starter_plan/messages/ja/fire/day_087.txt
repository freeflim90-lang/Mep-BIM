Day 87 - Fire Protection Parameter Checklist

1. なぜこれが必要か
正確な幾何形状を持つ消防設備モデルが、パラメータが空欄や誤っている場合、完全なBIM成果物ではありません。パラメータはモデルを機械可読にします — スケジュールの作成、水力エンジニアへの情報提供、FM引き渡しの支援、およびモデルのクエリを行うための下流ユーザーへの支援です。パラメータを正しく埋めることが、幾何形状を正しくモデリングすることほど重要ではありません。

2. 核心概念
LOD 350配達時の消防設備要素の必須パラメータ：

スプリンクラーヘッド：
| パラメータ | 例の値 | 目的 |
|---|---|---|
| Mark | FP-H-03-0047 | このヘッドを一意に識別するための識別子 |
| System Name | FP-WET-ZONE-03 | ヘッドが属する水力ゾーンへのリンク |
| Head Type | Pendant Concealed | スケジュールや調達のため |
| Coverage Area | 12.1 m² | 被覆面積の計算を確認 |
| Operating Temperature | 68°C | 标準オフィス/商業用評価値 |
| Manufacturer / Model | [指定通り] | 調達と承認のために |

配管：
| パラメータ | 例の値 | 目的 |
|---|---|---|
| System Name | FP-WET-ZONE-03 | そのヘッドを提供する配管と同じ |
| Nominal Diameter | DN50 | サイズ表と一致するように |
| Pipe Level | 支管 | 層のレベルを識別 |
| Material / Specification | 黒鋼、SCH 10 | 調達のために |
| RFI Reference | RFI-FP-047 | 設計通りに変更された場合 |

バルブと特殊機器：
| パラメータ | 例の値 | 目的 |
|---|---|---|
| Mark | FP-V-03-001 | 一意の識別子 |
| Valve Type | 区域制御バルブ | スケジュールのために |
| System Name | FP-WET-ZONE-03 | 区域へのリンク |
| Zone | 区域 03 | FM利用と整備のために |

一般的なパラメータ誤り：
- System Nameが空白 — ゾーンスケジュールで要素が未割り当てかつ見えない
- Coverage Areaが埋め込まれていない — 被覆面積計算スケジュールが実行できない
- Markが割り当てられていない — RFIやフィールド調整で要素を参照できない
- Nominal Diameterが一致していない — スケジュールは DN40 だがモデルは DN50

3. 実際のプロジェクト視点
レベル6消防保護モデルを発行する前に、BIMモデラーはSystem Name = "(none)"でフィルタリングされたRevitスケジュールを実行します。12つのヘッドと4つの配管セグメントがシステム割り当てなしで表示され — これらは調整再ルーティング中に配置され、適切にシステムに接続されていません。各要素は修正されます：ヘッドがシステムに再接続され、配管が正しいSystem Nameに割り当てられます。スケジュールには未割り当ての要素がゼロとなっています。

4. BIMチェック
- 検証：すべてのスプリンクラーヘッドが一意なMark、正しく埋め込まれたSystem Name、およびCoverage Areaパラメータを持っていること
- 検証：すべての配管がSystem NameとNominal Diameterが消防エンジニアのサイズ表と一致していること
- 検証：System Name = "(none)"でフィルタリングされたRevitスケジュールがゼロの消防保護要素を返すこと — 何も未割り当てしていない

5. 常に起こるミス
パラメータをモデリング終了後に実施する管理業務と見なす。要素を配置しながら逐次的に埋めるパラメータは、後から追加されるパラメータよりも常に正確です。2ヶ月前に配置されたヘッドは追跡が困難ですが、置く時点でMarkとSystem Nameがタグ付けされているヘッドはすぐに識別できます。

6. 今日のアクション
現在またはサンプルモデル内のすべてのスプリンクラーヘッドのRevitスケジュールを実行します。System NameやCoverage Areaが空白であるヘッドでフィルタリングします。不完全なレコードの数をカウントし、それらを完了するのにどれだけ時間がかかるかを推定します。次のモデル発行前にすべての空白を閉じる期限を設定します。
